# DS 기출문제 학습 페이지 생성 프롬프트

## 📋 사용 방법

아래 **[입력 템플릿]** 에 해설지 내용을 복사해서 붙여넣고, 이 프롬프트 전체를 AI에게 전달하세요.

---

## 🎯 요청 사항

아래 해설지 내용을 바탕으로 **DS 기출문제 학습 페이지**를 만들어줘.

### 파일 생성 규칙:
1. **파일 위치**: `docs/ds/exam/{회차}-{교시}-{번호}-{영문주제}.md`
2. **기출문제 목록 링크 추가**: `docs/ds/exam/index.md` 에 해당 문제 링크 연결

### 페이지 구성 (2단계 수준):

#### 🎓 고딩 수준 설명
- 키워드 3개 (쉬운 설명)
- 등장배경 (이야기 형식, 문제→해결 구조)
- 개념 정의 (비유 포함)
- 기술요소 2그룹 (표 형식)
- 차별점 키워드 1개 (답안지 가산점)
- 6하원칙 요약

#### 🎯 기술사 수준 설명
- 키워드 3개 (기술 용어)
- 등장배경 2개 (핵심만)
- 개념 정의 (22자 이내)
- 구성요소 2그룹 × 4항목 (표 형식)
- **🧠 암기법**: 영어 약자 기반으로 (한글 X)
- 상세 비교표
- 차별점 키워드 1개 (CMAF 같은 심화 개념)
- 프로토콜/기술 간 관계도 (ASCII 다이어그램)
- 6하원칙 요약
- 상위 토픽 계층도
- 향후 전망 (기술/서비스/표준화)

### 암기법 규칙:
- **영어 약자 기반**으로 작성 (한글 번역 암기법 X)
- 풀네임(Full Name)과 약자의 관계가 명확하도록
- 진화/발전 순서가 있으면 순서대로 암기
- 예시: "RT-P → RT-CP → RT-SP → S-DP", "H → D → C → LL"

### Front Matter 형식:
```yaml
---

title: {회차}회-{교시}교시-{번호}번 {주제명}

grand_parent: DS (Digital Service)
nav_order: {순번}
permalink: /docs/ds/exam/{회차}-{교시}-{번호}-{영문주제}
---
```

## 📝 실제 사용 예시

```
[문제 정보]
- 회차: 137
- 교시: 1
- 번호: 1
- 문제: 멀티미디어 스트리밍 프로토콜의 종류
- 도메인: 디지털서비스
- 난이도: 상
- 키워드: RTP, RTCP, RTSP, HLS, MPEG-DASH, Smooth Streaming

[해설지 내용]
I. 멀티미디어 스트리밍 프로토콜의 개념
정의: 멀티미디어 데이터를 네트워크로 실시간·순차 전송하여 다운로드 완료 전에도 재생 가능하게 하는 통신 규약

특징:
- 지연 최소화: 실시간 전송과 동시에 재생 가능
- 순차적 데이터 전송: 전체 파일이 아니라 필요한 구간부터 전송
- 인터랙션 지원: 지터, 패킷 손실, 대역폭 변동에 대응

II. 멀티미디어 스트리밍 프로토콜의 종류
가. 실시간 전송 & 제어 프로토콜
- RTP: 오디오·비디오 데이터 전송, UDP 기반
- RTCP: 전송 품질 제어, 지연·손실 모니터링
- RTSP: 세션 제어, 재생·일시정지·중지

나. HTTP 기반 적응형 스트리밍 프로토콜
- HLS: 애플, .m3u8 분할, iOS/macOS 친화적
- MPEG-DASH: 국제 표준, 네트워크 상태 따라 화질 자동 조정
- Smooth Streaming: 윈도우 기반, 현재 HLS/DASH에 밀려 축소

III. 향후 전망
- HTTP 기반 스트리밍 확대 (HLS, MPEG-DASH 주류)
- 저지연 기술 발전 (LL-HLS, LL-DASH)
- MPEG-DASH 국제 표준 지위 강화
```
