# 기출문제 학습 페이지 생성 프롬프트

## 📋 사용 방법

아래 **[입력 템플릿]** 에 해설지 내용을 복사해서 붙여넣고, 이 프롬프트 전체를 AI에게 전달하세요.

---

## 🎯 요청 사항

아래 해설지 내용을 바탕으로 **기출문제 학습 페이지**를 만들어줘.

### 파일 생성 규칙:
1. **파일 위치**: `docs/{카테고리}/exam/{회차}-{교시}-{번호}-{영문주제}.md`
2. **기출문제 목록 링크 추가**: `docs/{카테고리}/exam/index.md` 에 해당 문제 링크 + **암기법 컬럼** 연결

### index.md 테이블 형식:
```markdown
| 정/컴 | 교시 | 번호 | 문제 | 관련토픽 | 암기법 |
|:------|:----:|:----:|:-----|:---------|:------|
| 관리 | 1 | 1 | [문제 제목](링크) | 토픽 | `A-B-C-D` `X-Y-Z` |
```

## 📝 문제 유형별 페이지 구성

### 🅰️ 1교시형 (단답형)

> 단일 주제에 대한 설명 문제

#### 목차 (2개만)
```markdown
## 📚 목차
- [🎓 고딩 수준 설명](#-고딩-수준-설명)
- [🎯 기술사 수준 설명](#-기술사-수준-설명)
```

#### 페이지 구성:

**🎓 고딩 수준 설명** ← `<details>` 태그로 접기/펼치기
- 키워드 3개 (쉬운 설명)
- 등장배경 (이야기 형식, 문제→해결 구조)
- 개념 정의 (비유 포함)
- 기술요소 2그룹 (표 형식)
- 차별점 키워드 1개
- 6하원칙 요약

```html
<details markdown="1">
<summary><h2 style="display:inline" id="-고딩-수준-설명">🎓 고딩 수준 설명 (클릭해서 펼치기)</h2></summary>
(내용...)
</details>
```

**🎯 기술사 수준 설명** (⭐ 되도록 한글로 작성)

1. **핵심 암기 (Quick Reference)** ← 하이라이트 박스 안에
```markdown
{: .highlight }
> **토픽명**: 핵심키워드1, 핵심키워드2, 핵심키워드3
> - (유형) `암기법1`
> - (도구) `암기법2`
> - (절차) `암기법3`
```

2. **개념 정의** (등장배경과 합쳐서 작성)
   - "~의 필요성으로 인해, ~하는 기술/개념"
   - 정의는 22자 이내로

3. **구성요소** (3단 표로 작성)
```markdown
| 항목 | 설명 | 예시 |
|:-----|:-----|:-----|
| **항**목1 | 설명1 | 예시1 |
```

4. **절차/프로세스** (있는 경우)
   - ASCII 다이어그램

5. **차별점 키워드** (가산점 포인트)

6. **상위 토픽 계층도**

7. **학습 체크리스트**

---

### 🅱️ 2교시형 (서술형)

> 가, 나, 다 등 세부 질문이 있는 문제

#### 목차 (2개만)
```markdown
## 📚 목차
- [🎓 고딩 수준 설명](#-고딩-수준-설명)
- [🎯 기술사 수준 설명](#-기술사-수준-설명)
```

#### 페이지 구성:

**🎓 Quick 개념잡기** (간략하게, 접기/펼치기)
- 한 줄 정의
- 쉬운 비유
- 왜 필요한가? (3줄 요약)

**🎯 기술사 수준 설명**

1. **핵심 암기 (Quick Reference)** ← 하이라이트 박스 안에
```markdown
{: .highlight }
> **토픽명**: 핵심키워드1, 핵심키워드2, 핵심키워드3
> - (가) `암기법1`
> - (나) `암기법2`
> - (다) `암기법3`
```

2. **가. 첫 번째 질문**
   - 개념 (등장배경 + 정의 통합)
   - 구성요소/특징 (3단 표)
   - 암기법

3. **나. 두 번째 질문**
   - 개념도 (ASCII 다이어그램)
   - 구성요소 상세 (3단 표)
   - 암기법

4. **다. 세 번째 질문**
   - 개념/정의
   - 비교표 (있다면)
   - 동작 원리/절차
   - 암기법

5. **차별점 키워드**

6. **상위 토픽 계층도**

7. **학습 체크리스트**

## 🧠 암기법 규칙

### 기본 규칙
- **한글 단어 → 한글 암기법** (예: 시응파메, 식우수무문)
- **영어 약자가 명확한 경우 → 영어 암기법** (예: E-V-W-C)
- 풀네임과 약자의 관계가 명확하도록
- 진화/발전 순서가 있으면 순서대로 암기

### 암기법 예시
| 유형 | 암기법 | 설명 |
|:----|:------|:----|
| **한글** | `시응파메` | 시스템, 응용프로그램, 파일, 메모리 |
| **한글** | `식우수무문` | 식별, 우선순위, 수집, 무결성, 문서화 |
| **영어** | `E-V-W-C` | EnCase, Volatility, Wireshark, Cellebrite |
| **영어** | `A-S-C-E` | API Server, Scheduler, Controller, Etcd |

### 📌 구성요소 제목 옆에 암기법 표시 (중요!)
- 스크롤 없이 한눈에 암기법 확인
- 구성요소/특징 섹션 제목 옆에 백틱으로 표시

```markdown
### 🏗️ 구성요소

#### 아트팩트 유형 `시응파메`

| 유형 | 설명 | 예시 |
|:-----|:-----|:-----|
| **시**스템 | ... | ... |
```
