# 기출문제 학습 페이지 생성 프롬프트

## 📋 사용 방법

아래 **[입력 템플릿]** 에 해설지 내용을 복사해서 붙여넣고, 이 프롬프트 전체를 AI에게 전달하세요.

---

## 🎯 요청 사항

아래 해설지 내용을 바탕으로 **기출문제 학습 페이지**를 만들어줘.

### 파일 생성 규칙:
1. **파일 위치**: `docs/{카테고리}/exam/{회차}-{교시}-{번호}-{영문주제}.md`
2. **기출문제 목록 링크 추가**: `docs/{카테고리}/exam/index.md` 에 해당 문제 링크 + **암기법 컬럼** 연결

### index.md 테이블 형식:
```markdown
| 정/컴 | 교시 | 번호 | 문제 | 관련토픽 | 암기법 |
|:------|:----:|:----:|:-----|:---------|:------|
| 관리 | 1 | 1 | [문제 제목](링크) | 토픽 | `(유형) 암기법1` `(도구) 암기법2` |
```

### 암기법 컬럼 작성 규칙 (중요!):
- **괄호로 카테고리 표시**: 각 암기법 앞에 `(카테고리)` 형식으로 표시
- 예시: `(유형) 시응파메` `(도구) E-V-W-C` `(절차) 식우수무문`
- 카테고리 예시: `(유형)`, `(도구)`, `(절차)`, `(공격기법)`, `(대응방안)`, `(구성요소)`, `(특징)` 등
- ⚠️ **이모지 사용 금지**: 암기법 컬럼에 이모지 넣지 말 것

## 📝 문제 유형별 페이지 구성

### 🅰️ 1교시형 (단답형)

> 단일 주제에 대한 설명 문제

#### 🔄 1교시형 비교형 (특수 케이스)

> 두 개념/기술을 비교하는 문제

**비교형 문제 작성 규칙:**
- **비교 구분자 최소 12개 이상** 포함 필수
- 비교 항목은 암기법으로 정리 (예: `목로그이사보자오통집분자유`)
- 상세 비교표에 모든 비교 항목을 포함
- 각 비교 항목은 명확하게 구분되어야 함

#### 목차 (2개만)
```markdown
## 📚 목차
- [🎓 고딩 수준 설명](#-고딩-수준-설명)
- [🎯 기술사 수준 설명](#-기술사-수준-설명)
```

#### 페이지 구성:

**🎓 고딩 수준 설명** ← `<details>` 태그로 접기/펼치기
- 키워드 3개 (쉬운 설명)
- 등장배경 (이야기 형식, 문제→해결 구조)
- 개념 정의 (비유 포함)
- 기술요소 2그룹 (표 형식)
- 차별점 키워드 1개
- 6하원칙 요약

```html
<details markdown="1">
<summary><h2 style="display:inline" id="-고딩-수준-설명">🎓 고딩 수준 설명 (클릭해서 펼치기)</h2></summary>
(내용...)
</details>
```

**🎯 기술사 수준 설명** (⭐ 되도록 한글로 작성)

1. **핵심 암기 (Quick Reference)** ← 하이라이트 박스 안에
```markdown
{: .highlight }
> **토픽명**: 핵심키워드1, 핵심키워드2, 핵심키워드3 → 개념 맺음말
> - (유형) `암기법1`
> - (도구) `암기법2`
> - (절차) `암기법3`
```

**📌 맺음말 작성 규칙:**
- 3개 키워드 뒤에 `→` 기호로 연결
- "~한 제도", "~한 기술", "~한 기법", "~한 모델" 등 개념의 본질을 한 마디로 정리
- 예시: `ISO/IEC 15408, CCRA, EAL → IT 제품 보안성 평가·인증 제도`
- 예시: `격리, 동적분석, 행위기반 → 악성코드 분석 기술`

2. **개념 정의** (등장배경과 합쳐서 작성)
   - "~의 필요성으로 인해, ~하는 기술/개념"
   - 정의는 22자 이내로

3. **구성요소** (3단 표로 작성)

4. **개념 정의 작성 시 개행 규칙**
   - 문장이 길 경우 콤마(,) 이후 또는 의미 단위로 개행
   - 가독성을 위해 한 줄에 너무 긴 문장 지양
```markdown
| 항목 | 설명 | 예시 |
|:-----|:-----|:-----|
| **항**목1 | 설명1 | 예시1 |
```

4. **절차/프로세스** (있는 경우)
   - ASCII 다이어그램

5. **차별점 키워드** (가산점 포인트)

6. **상위 토픽 계층도**

7. **학습 체크리스트**

### 🅱️ 2교시형 (서술형)

> 3단락 또는 4단락으로 구분되는 서술형 문제

#### 목차 (2개만)
```markdown
## 📚 목차
- [🎓 Quick 개념잡기](#-quick-개념잡기-클릭해서-펼치기)
- [🎯 기술사 수준 설명](#-기술사-수준-설명)
```

#### 페이지 구성:

**🎓 Quick 개념잡기** (간략하게, 접기/펼치기)
- 한 줄 정의
- 쉬운 비유
- 왜 필요한가? (3줄 요약)

**🎯 기술사 수준 설명**

1. **핵심 암기 (Quick Reference)** ← 하이라이트 박스 안에
```markdown
{: .highlight }
> **토픽명**: 핵심키워드1, 핵심키워드2, 핵심키워드3 → 개념 맺음말
> - (1) `암기법1` (첫 번째 단락)
> - (2) `암기법2` (두 번째 단락)
> - (3) `암기법3` (세 번째 단락)
> - (4) `암기법4` (네 번째 단락, 있는 경우)
```

**📌 맺음말 작성 규칙:**
- 3개 키워드 뒤에 `→` 기호로 연결
- "~한 제도", "~한 기술", "~한 기법", "~한 모델" 등 개념의 본질을 한 마디로 정리
- 예시: `정보주체 중심, 동의 기반, 안전한 제공·활용 → 공공 데이터 활용 서비스`

2. **1. 첫 번째 단락** (주제에 대한 개념/정의/구성요소 등)
   - **단락 요약** ← 하이라이트 박스로 각 단락 앞에 추가 (1교시형 핵심 암기 형식)
   ```markdown
   {: .highlight }
   > **단락명**: 키워드1, 키워드2, 키워드3
   > - (항목1) `암기법1`
   > - (항목2) `암기법2`
   > - (항목3) `암기법3`
   ```
   - 예시:
   ```markdown
   {: .highlight }
   > **공공 마이데이터**: 정보주체 중심, 동의 기반, 안전한 제공·활용
   > - (개념) `전자정부법 제43조의 2`
   > - (구성요소) `정보본제`
   > - (등장배경) `데분디개`
   ```
   - **가. 하위 항목 1**
   - 개념 (등장배경 + 정의 통합)
   - 구성요소/특징 (3단 표)
   - **나. 하위 항목 2**
   - 개념도 (ASCII 다이어그램)
   - 구성요소 상세 (3단 표)
   - **다. 하위 항목 3** (있는 경우)
   - 개념/정의
   - 비교표 (있다면)

3. **2. 두 번째 단락** (활용 방안, 기술 요소 등)
   - **단락 요약** ← 하이라이트 박스로 각 단락 앞에 추가 (1교시형 핵심 암기 형식)
   ```markdown
   {: .highlight }
   > **단락명**: 키워드1, 키워드2, 키워드3
   > - (항목1) `암기법1`
   > - (항목2) `암기법2`
   ```
   - 예시:
   ```markdown
   {: .highlight }
   > **행정 서비스 활용**: 맞춤형, 간소화, 통합
   > - (생애주기) `전청청장노`
   > - (활용방안) `맞행통취생`
   ```
   - **가. 하위 항목 1**
   - **나. 하위 항목 2**

4. **3. 세 번째 단락** (고려사항, 제언 등)
   - **단락 요약** ← 하이라이트 박스로 각 단락 앞에 추가 (1교시형 핵심 암기 형식)
   ```markdown
   {: .highlight }
   > **단락명**: 키워드1, 키워드2, 키워드3
   > - (항목1) `암기법1`
   > - (항목2) `암기법2`
   ```
   - 예시:
   ```markdown
   {: .highlight }
   > **민간 서비스 및 산업 활성화**: 금융, 헬스케어, 산업 활성화
   > - (민간서비스) `금취헬생`
   > - (산업활성화) `모스금유`
   ```
   - **가. 하위 항목 1**
   - **나. 하위 항목 2**
   - **다. 하위 항목 3**

5. **4. 네 번째 단락** (있는 경우)
   - **단락 요약** ← 하이라이트 박스로 각 단락 앞에 추가 (1교시형 핵심 암기 형식)
   ```markdown
   {: .highlight }
   > **단락명**: 키워드1, 키워드2, 키워드3
   > - (항목1) `암기법1`
   ```
   - 예시:
   ```markdown
   {: .highlight }
   > **고려사항**: 법제도, 기술보안, 이용자 중심, 정책운영
   > - (고려사항) `법기이정`
   ```
   - **가. 하위 항목 1**
   - **나. 하위 항목 2**

6. **차별점 키워드**

7. **상위 토픽 계층도**

8. **학습 체크리스트**

**중요 규칙:**
- 단락은 **1, 2, 3, 4**로 구분
- 하위 항목은 **가, 나, 다, 라**로 구분
- **각 단락(1, 2, 3, 4) 앞에 단락 요약을 하이라이트 박스로 추가**
- **단락 요약 형식은 1교시형 핵심 암기와 동일**: 키워드 3개 + (항목) `암기법` 형식
  - 평문 요약이 아닌, 키워드와 암기법으로 구성
  - 예시: `> **단락명**: 키워드1, 키워드2, 키워드3`
  - 예시: `> - (항목1) `암기법1``

## 🧠 암기법 규칙

### 기본 규칙
- **한글 단어 → 한글 암기법** (예: 시응파메, 식우수무문)
- **영어 약자가 명확한 경우 → 영어 암기법** (예: E-V-W-C)
- 풀네임과 약자의 관계가 명확하도록
- 진화/발전 순서가 있으면 순서대로 암기

### 암기법 예시
| 유형 | 암기법 | 설명 |
|:----|:------|:----|
| **한글** | `시응파메` | 시스템, 응용프로그램, 파일, 메모리 |
| **한글** | `식우수무문` | 식별, 우선순위, 수집, 무결성, 문서화 |
| **영어** | `E-V-W-C` | EnCase, Volatility, Wireshark, Cellebrite |
| **영어** | `A-S-C-E` | API Server, Scheduler, Controller, Etcd |

### 📌 구성요소 제목 옆에 암기법 표시 (중요!)
- 스크롤 없이 한눈에 암기법 확인
- 구성요소/특징 섹션 제목 옆에 백틱으로 표시

```markdown
### 🏗️ 구성요소

#### 아트팩트 유형 `시응파메`

| 유형 | 설명 | 예시 |
|:-----|:-----|:-----|
| **시**스템 | ... | ... |
```

    │   ├── {: .highlight } 단락명: 키워드1, 키워드2, 키워드3
    │   │   - (항목1) `암기법1`
    │   │   - (항목2) `암기법2`
    │   ├── 가. 하위 항목 1
    │   ├── 나. 하위 항목 2
    │   └── 다. 하위 항목 3
    ├── 4. 네 번째 단락 (있는 경우)
    │   ├── {: .highlight } 단락명: 키워드1, 키워드2, 키워드3
    │   │   - (항목1) `암기법1`

    │   └── 나. 하위 항목 2
    ├── ⭐ 차별점 키워드

**⚠️ 2교시형 단락 요약 작성 규칙:**
