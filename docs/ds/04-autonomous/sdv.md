# SDV (Software Defined Vehicle)
{: .fs-8 }

4.4 모빌리티
{: .label .label-purple }

---

## 정의

하드웨어 중심에서 **소프트웨어 중심**으로 전환된 자동차, OTA 업데이트로 **기능이 지속적으로 진화**하는 차량

---

## 기존 차량 vs SDV 비교

| 구분 | 기존 차량 | SDV |
|:-----|:---------|:----|
| **핵심 가치** | 하드웨어 성능 | 소프트웨어 기능 |
| **아키텍처** | 분산형 ECU | 중앙 집중형 컴퓨팅 |
| **업데이트** | 딜러 방문 | OTA (무선 업데이트) |
| **기능 추가** | 출고 시 고정 | 지속적 업그레이드 |
| **수익 모델** | 차량 판매 | 구독 서비스 |

---

## SDV 핵심 기술

| 분류 | 기술 | 설명 |
|:-----|:-----|:-----|
| **아키텍처** | 중앙 집중형 컴퓨팅 | 고성능 중앙 컴퓨터 |
| | 존(Zone) 아키텍처 | 영역별 ECU 통합 |
| **소프트웨어** | 차량용 OS | AUTOSAR, Android Automotive |
| | 미들웨어 | 통신, 데이터 관리 |
| **연결성** | OTA | Over-The-Air 업데이트 |
| | 클라우드 연동 | 차량-클라우드 통합 |

---

## SDV 가치 창출

| 영역 | 내용 |
|:-----|:-----|
| **안전 향상** | 실시간 보안 패치, 버그 수정 |
| **기능 확장** | 새로운 기능 추가 (자율주행 Level Up) |
| **개인화** | 사용자 맞춤 설정/서비스 |
| **수익 다변화** | 구독 서비스, 앱 스토어 |
