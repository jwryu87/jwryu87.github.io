# MaaS (Mobility as a Service)
{: .fs-8 }

4.4 모빌리티
{: .label .label-purple }

---

## 정의

다양한 교통수단을 **하나의 플랫폼**으로 통합하여 **최적 경로, 예약, 결제**를 제공하는 **통합 모빌리티 서비스**

---

## MaaS 통합 레벨

| Level | 명칭 | 설명 |
|:------|:-----|:-----|
| **Level 0** | 통합 없음 | 개별 교통수단 이용 |
| **Level 1** | 정보 통합 | 다양한 교통수단 정보 제공 |
| **Level 2** | 예약/결제 통합 | 통합 예약 및 결제 |
| **Level 3** | 서비스 통합 | 번들 서비스, 구독 모델 |
| **Level 4** | 완전 통합 | 사회적 목표 통합 (탄소 중립 등) |

---

## MaaS 구성요소

| 구성요소 | 설명 |
|:---------|:-----|
| **교통수단** | 버스, 지하철, 택시, 공유차, 킥보드 등 |
| **플랫폼** | 통합 앱/서비스 |
| **데이터** | 실시간 교통 정보, 사용자 데이터 |
| **결제** | 통합 결제 시스템 |
| **정책** | 규제, 인센티브 |

---

## MaaS 핵심 기술

| 기술 | 설명 |
|:-----|:-----|
| **경로 최적화** | AI 기반 최적 경로 추천 |
| **실시간 데이터** | GPS, IoT 기반 위치/상태 정보 |
| **통합 결제** | 간편 결제, 구독 모델 |
| **개인화** | 사용자 패턴 분석, 맞춤 추천 |

---

## MaaS 기대효과

| 측면 | 효과 |
|:-----|:-----|
| **사용자** | 편의성 향상, 비용 절감 |
| **교통** | 혼잡 감소, 효율성 향상 |
| **환경** | 탄소 배출 감소, 지속가능성 |
| **산업** | 새로운 비즈니스 모델 창출 |
