# ISO/SAE 21434
{: .fs-8 }

4.2 자율주행 표준
{: .label .label-purple }

---

## 정의/개념

차량 전기/전자 시스템 **사이버 보안 관리 국제 표준**

---

## 원칙

### ISO/SAE 21434의 절차 설명

```
자산식별 → "자산후보, 사이버보안 속성" → 위험처리결정 → "위험회피, 위험전가, 위험완화, 위험수용"
    ↓               ↓
  자산, 공격시나리오    위험가치
    ↓               ↓
위험시나리오 식별 → 타격평가 → 위험 결정
         ↓           ↓
      피해 시나리오   피해 시나리오 영향
              ↓
     위험시나리오& → 공격경로 → 공격가능성
     알려진 취약성    분석      평가
```

- 자동차 전기전자시스템 엔지니어링의 사이버보안 요소를 점검

### ISO/SAE 21434의 구성요소 설명

| 구분 | 핵심 | 설명 |
|:-----|:-----|:-----|
| **주요 원칙** | 사이클관리 | 자동차 전체 수명주기에 걸친 사이버 보안 활동과 프로세스 |
|  | 절차 기반 | 합리적으로 절차에 기반하여 안전한 자동차 시스템 개발 |
|  | 엔지니어링 중점 | 자동차 사이버 보안 엔지니어링에 주안점을 둠 |
|  | 최신기술 기반 | 사이버 보안의 최신기술에 기반해야 함 |
| **적용 범위** | 도로용 차량 | 도로용자동차(Road vehicles) |
|  | E/E 부품 | 전기전자시스템, 하드웨어와 소프트웨어부품, 인터페이스 |
|  | N/W 연결 | 외부장비/네트워크와의 연결(접속)되는 시스템 |
|  | 미포함 공통 적용 | 특정한 사이버 보안 기술과 해법 |
|  | 공통 요구 | 특정 통신 서비스 및 그 제공기관에 대한 요구사항 |

---

## 구성요소 (15개 Part)

| Part | 구성요소 | 설명 |
|:-----|:---------|:-----|
| 1 | 개요 | 발간 배경에 대한 설명과 목적에 대한 간략한 요약 |
| 2 | 참고 문헌 | 문서 작성에 인용된 참고 문헌 |
| 3 | 용어 및 약어 정의 | 문서 상에 등장하는 용어 및 약어에 대한 설명 |
| 4 | 일반적인 고려사항 | 해당 문서 이용에 대한 일반적인 고려사항 |
| 5 | 조직 사이버 보안 관리 | 사이버보안 정책, 사이버보안 문화, 정보 공유, 시스템 관리, 도구 관리, 정보 보안 관리, 조직 사이버문화 검사 |
| 6 | 프로젝트 사이버보안 관리 | 책임자 선정, 보안 계획 수립, 수정 작업, 재사용 작업, 불필요 요소, 사이버보안 사례, 사이버보안 평가, 관리 내역 공개 |
| 7 | 분산 사이버보안 활동 | 공급망 내 이해관계자 능력 평가, 보안 활동 업무 할당, 책임 소재 확립 |
| 8 | 지속적인 사이버보안 활동 | 사이버보안 모니터링, 이벤트 평가, 취약점 분석, 취약점 관리 |
| 9 | 제품 개념 설계 | 아이템 선정, 사이버보안 목표 설정, 사이버보안 개념 설계 |
| 10 | 제품 개발 | 제품 디자인, 사이버보안 목표 설정 |
| 11 | 사이버보안 검증 | 제품화 후의 사이버보안 검증 활동 실행, 사이버보안 목표 검증, 제품의 목표에 달성 여부, 잔여 위험의 수용 가능성 확인 |
| 12 | 제품 생산 | 제조 및 조립 과정에서의 사이버 보안 요구사항 설정 |
| 13 | 운영 및 유지 | 사이버보안 사고 대응, 업데이트 |
| 14 | 기술 지원 및 보조 종료 | 제품 및 부품의 보증이 종료될 때 안전한 방법으로 폐기될 것을 요구 |
| 15 | 위험 분석 및 위험 평가 방법 | 자산 정의 위험 시나리오 식별, 영향도 측정, 공격 경로 분석, 실현 가능성 측정, 위험도 측정, 위험 관리 방법 결정 |
