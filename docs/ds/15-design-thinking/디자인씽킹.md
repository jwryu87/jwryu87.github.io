# 디자인 싱킹 (Design Thinking)
{: .fs-8 }

15. 디자인씽킹
{: .label .label-purple }

> 암기: **공정아프테피** (공감-정의-아이디어-프로토타입-테스트-피드백)

---

## 핵심 키워드

`공감` `문제 정의` `아이디어 도출` `프로토타입` `테스트` `피드백`

---

## 정의/개념

공감, 문제 재해석, 시각적 아이디어 도출, 프로토타입 제작 **사용자 중심 문제 해결 방법론**

---

## 메커니즘

```
┌──────────────────────────────────────────────────────────────────────┐
│  1단계 Understanding          2단계 Create                          │
│  현상태 문제 정의하기         창의적인 통찰 사고로 더 좋은 상태로    │
│                               변화시키기                             │
│                                                                      │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐  │
│  │Empathize │→│ Define   │→│ Ideate   │→│Prototype │→│  Test    │  │
│  │(공감하기)│ │(문제정의)│ │(아이디어)│ │(시제품   │ │(검증하기)│  │
│  │          │ │          │ │          │ │ 만들기)  │ │          │  │
│  └──────────┘ └──────────┘ └──────────┘ └──────────┘ └──────────┘  │
│                              ↑                                       │
│                    3단계 Feedback                                    │
│                    다시 실행하기                                     │
└──────────────────────────────────────────────────────────────────────┘
```

---

## 핵심요소 (3가지 관점)

| 관점 | 설명 |
|:-----|:-----|
| **Desirability (바람직함)** | 사람들이 진정 원하는 것 |
| **Feasibility (실현가능성)** | 기술적 & 조직적으로 가능한 것 |
| **Viability (생존가능성)** | 사업 가치가 있는 것 |

---

## 프로세스

| 구분 | 프로세스 | 설명 | 기법 |
|:-----|:-----|:-----|:-----|
| **Inspiration (영감)** | 공감 (Empathize) | - 사용자 식별<br>- 사용자 관찰, 인터뷰 통한 공감<br>- 사용자 Needs 파악, 이슈에 대한 정보 수집 | 설문조사, 인터뷰, 관찰, Shadowing |
| | 문제 정의 (Define) | - 공감한 결과 바탕 문제점 파악 및 문제 우선순위화<br>- 요구사항 정의, 문제 인식 및 공유<br>- 페르소나 기반 Touch Point 식별, 이슈 파악 | 고객여정지도, 페르소나 |
| **Ideation (아이디어 도출)** | 아이디어 도출 (Ideate) | - 파악된 문제점 해결 위한 다양한 아이디어 발산<br>- 아이디어 우선순위화, 개선 방향 도출<br>- Divergence(확장적 사고), Convergence(집중적 사고) | 브레인스토밍, 아이디어 스케치, 시각화 |
| **Implementation (실행)** | 프로토타이핑 (Prototype) | - 도출된 아이디어의 시각화, 실체화<br>- 핵심 기능 구현 통한 시간, 비용 효율성 향상 | 스토리보드, MVP |
| | 테스트 (Test) | - 제작된 프로토타입 테스트<br>- 피드백 통한 아이디어 개선 및 회고 | 사용성 테스트, Role play |

---

## 주요 기법

| 기법 | 설명 | 적용 단계 |
|:-----|:-----|:-----|
| **설문조사** | 사용자 의견 수집 | 공감 |
| **인터뷰/관찰** | 직접 대화 및 행동 관찰 | 공감 |
| **Shadowing** | 사용자 따라다니며 관찰 | 공감 |
| **고객여정지도** | 사용자 경험 시각화 | 문제 정의 |
| **페르소나** | 가상의 사용자 프로필 | 문제 정의 |
| **브레인스토밍** | 자유로운 아이디어 발산 | 아이디어 도출 |
| **아이디어 스케치** | 시각적 아이디어 표현 | 아이디어 도출 |
| **스토리보드** | 시나리오 시각화 | 프로토타입 |
| **MVP** | 최소 기능 제품 | 프로토타입 |
| **사용성 테스트** | 사용자 검증 | 테스트 |
| **Role play** | 역할극을 통한 검증 | 테스트 |
