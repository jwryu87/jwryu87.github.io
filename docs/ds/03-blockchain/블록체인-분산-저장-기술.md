## 핵심 키워드

## 정의/개념

분산된 **P2P 망** 내 참여자들이 모든 거래 목록을 지속적 갱신하는 **디지털 원장 기술**

---

## 블록체인과 분산원장기술의 특징 비교

| 비교 | 분산원장기술(DLT) | 블록체인 |
|:-----|:-----------------|:---------|
| **개념** | 분산된 P2P 망 내 참여자들이 모든 거래 목록을 지속적 갱신하는 디지털 원장 기술 | 분산원장기술의 부분집합 개념으로 합의를 통한 공유 분산 데이터베이스 기술 |
| **용어 측면** | 기술에 초점을 맞춘 중성적인 용어로 주로 사용됨 | '토큰 이코노미' 레이어를 포괄하는 개념 |
| **제어 측면** | 목적에 따라 설계자가 구조와 기능 제어 가능 | 참여자들의 합의 과정이 필수적임. 모든 노드간 신뢰를 전제로 함 |
| **프로젝트** | - R3의 코다(Corda): 금융특화<br>- 리눅스재단의 하이퍼레저 패브릭: 범용 | - 이더리움<br>- IoTA: 사물인터넷과의 결합 |

## 기술

| 구분 | 주요기술 | 설명 |
|:-----|:---------|:-----|
| **분산 네트워크** | P2P NW | Peer To Peer Network |
|  | TOR NW | 클라이언트/서버 통신망과 달리 장치가 서버로서도 동작하고 클라이언트로서도 동작하면서 데이터를 송수신하는 NW. 여러 중계서버(라우터)를 통해 패킷을 전송, 트래픽 분석이나, IP주소 추적을 불가능하게 하여 프라이버시와 보안을 보장하는 익명성 네트워크 |
| **인증/암호화** | Merkle Tree | Hash 함수로 암호화된 Hash 값으로 구성된 Binary Hash Tree. 블록 용량을 효율적으로 활용 위해 사용되는 데이터 구조 |
|  | DAG | Directed Acyclic Graph. 블록 없에고 트랜잭션 간의 체인 형식만 유지하는 알고리즘. 사이클을 형성하지 않는 일방향성 비순환 그래프 |
|  | 전자서명 | 거래정보 무결성 보장 위한 ECDSA 등 디지털 서명 기술 |
|  | 암호알고리즘 | 대칭키(AES256), 비대칭키(RSA), 해시(SHA256) |
| **합의 알고리즘** | PoW | Proof of Work. 컴퓨팅 파워를 이용하는 합의방식 |
|  | PoS | Proof of Stake. 소유자가 보유 자산 증명방식 |
|  | DPoS | Delegated Proof of Stake. 모든 개인 채굴 가능하나 선출된 대표자만이 블록 생성하는 반중앙형 합의알고리즘 |
|  | PBFT | Practical Byzantine Fault Tolerance. 분산 시스템이 약속된 행동을 하지 않은 노드가 존재하는 비동기 시스템일 때 분산 시스템에 참여한 모든 노드가 성공적으로 합의를 이룰 수 있도록 개발된 합의 노드 |

## 분산저장 기술

### 샤딩 기술 (Sharding)

샤드 네트워크 구조로 나누어 여러 노드에 트랜잭션 정보를 저장하는 기술

| 특징 | 설명 |
|:-----|:-----|
| **분산저장** | 데이터를 여러 샤드에 분산 |
| **통신부담 감소** | 샤드 내 통신으로 부담 감소 |
| **병렬처리** | 동시 트랜잭션 처리 가능 |
| **한계점** | 보안취약 (샤드 내 51% 탈취 가능) |

### DHT 기반 기술 (Distributed Hash Table)

해시테이블을 이용해 네트워크에 분산 저장하는 기술

| 특징 | 설명 |
|:-----|:-----|
| **해시 기반 저장** | 참여 Host의 IP 기반 해시 = 공유파일 해시 연결 저장 |
| **인접 기반 저장** | 공유파일에 해시가 없다면 인접 노드에 저장 |
| **일괄 해시** | 효율적인 데이터 관리 |
| **빠른 탐색** | O(log N) 탐색 성능 |
| **비잔틴 장애 내성** | 악의적 노드 대응 |

### 이레이저 코드 기반 기술 (Erasure Code)

패리티 추가하여 일부 손실(Erasure)되는 상황에서도 데이터 복구 가능

| 특징 | 설명 |
|:-----|:-----|
| **블록 그룹화** | 블록을 그룹으로 묶어 이레이저 코드 생성 |
| **보안성 향상** | 데이터 가용성 증가 |
| **한계점** | 복구 비용이 많이 듦 |
