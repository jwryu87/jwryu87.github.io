# 퍼블릭, 프라이빗, 하이브리드 블록체인
{: .fs-8 }

3.1 블록체인 기본
{: .label .label-purple }

---

## 핵심 키워드

| 유형 | 키워드 |
|:-----|:-------|
| **Public** | 투명성, 공개성, 탈중앙화, 하드포크 가능 |
| **Private** | 개방성 제한, 효율성, 저비용 |
| **Hybrid** | 퍼블릭 + 프라이빗 |

---

## 정의/개념

- **Public**: 어떠한 조직, 기관 승인 절차 및 접근 제한 없이 **누구나 참여가능한 블록체인**
- **Private**: 미리 정해진 조직이나 개인들만 참여 가능한 **특정 권한 부여 비공개 블록체인**
- **Hybrid**: Public의 블록체인 **보안성**, 탈중앙화를 제공하면서 거래 내용 접근 및 **변경에 대해서 제한**하는 블록체인

---

## 아키텍처

| 구분 | 퍼블릭 블록체인 (Public) | 프라이빗 블록체인 (Private) | 하이브리드 블록체인 (Hybrid) |
|:-----|:------------------------|:---------------------------|:----------------------------|
| **개념도** | 완전 분산형 (모든 노드 연결) | 허가형 (제어 노드 사용) | P2P 네트워크와 분산형 앱 및 새로운 수익 모델을 활용하여 비용 절감 가능 |
| **특징** | 불량한 악의적 사용자로부터 보안을 요하지 않기 때문에, 접근 제어가 필요하지 않음 | 허가형 블록체인은 접근 제어를 사용하여 네트워크 접근자를 관리한다 | - |

---

## 비교

| 구분 | 퍼블릭 블록체인 | 프라이빗 블록체인 | 하이브리드 블록체인 |
|:-----|:---------------|:-----------------|:-------------------|
| **장점** | - 투명성<br>- 익명성<br>- 신뢰성 | - 효율성<br>- 공정성 | - 공정성 |
| **단점 (문제점)** | - 느린 처리 속도<br>- 확장성 제한<br>- 의도적 합의 조작 | - 개방성, 탈 중앙화 제한<br>- 신뢰성 낮음 | - 개방성, 탈 중앙화 제한<br>- 신뢰성 낮음 |
| **투명성** | 숨기는 정보 없이 정보가에 공개 | 접속 권한이 있는 사용자에게만 정보가 공개 | 소유자가 규칙을 설정한 방식에 따라 상이 |
| **하드포크** | 하드포크 가능 | 하드포크 불가능 | 하드포크 불가능 |
| **속도** | 7~20 TPS | 1000TPS 이상 고성능 | 1000TPS 이상 고성능 |
| **탈 중앙성** | 탈 중앙성 높음 | 탈 중앙성 낮음 | 탈 중앙성 낮음 |
| **합의 알고리즘** | PoW, PoS, DPos 등 | PBFT, PoA | 퍼블릭과 프라이빗 이용 |
| **사례** | 비트코인, 이더리움 | 리플, 코다 | 하이브리드, R3 |
| **활용** | - | 공공 프로젝트나 사업을 작업프로토콜의 원밸적 제어를 공급망, 은행, 금융, 사물인터넷 등 위해, 조직 블록체인 구현 |  |
| **거래 비용** | 비용이 많이 소요 | 적은 비용 소요 | 적은 비용 소요 |
| **KYC 필요** | KYC 필요 없음 | KYC 필요 | KYC 필요 |
| **인센티브** | 네트워크 성장에 대한 참여자들에게 인센티브 | 제한적 접근만 가능하기 때문에 인센티브가 없다 | 원하는 경우 사용자에게 인센티브를 제공 가능 |

---

## 블록체인 유형에 따라

접근성 외의 성능, 합의 알고리즘 등의 차별화 된 특징으로, 필요한 분야에 따라 선택적 활용이 가능함
