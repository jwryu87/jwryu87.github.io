# 머클트리 (Merkle Tree)
{: .fs-8 }

3.2 블록체인 암호 기술
{: .label .label-purple }

---

## 특징

| 특징 | 설명 |
|:-----|:-----|
| **위변조 검증** | 해시값 비교를 통한 데이터 무결성 검증 |
| **트리구조** | 이진 트리(Binary Tree) 형태의 해시 구조 |
| **일방향 암호화** | SHA256 등 해시 함수 사용 |

---

## 활용

| 활용 | 설명 |
|:-----|:-----|
| **트랜잭션 변경 확인** | 특정 트랜잭션의 변경 여부 빠르게 확인 |
| **트랜잭션 유효성 보장** | 전체 데이터 검증 없이 유효성 확인 |
| **머클 경로(Path) 제공** | 특정 데이터까지의 검증 경로 |
| **Light Node** | 전체 블록체인 없이 검증 가능 |

---

## 구조

```
              Root Hash
             /         \
        Hash 0-1       Hash 2-3
        /     \        /      \
    Hash 0  Hash 1  Hash 2  Hash 3
       |       |       |       |
    Tx 0    Tx 1    Tx 2    Tx 3
```

---

## 활용 동향

| 시스템 | 활용 |
|:-------|:-----|
| **비트코인** | 암호통화 시스템 및 비트토렌트 프로토콜 |
| **ZFS 파일 시스템** | 파일 시스템 물리적 변경 없이 동적 분배 |
| **구글웨이브 프로토콜** | 인스턴스 메시징과 이메일 통합 프로토콜 |
| **GIT** | 버전관리 및 배포 시 무결성 검증 |
