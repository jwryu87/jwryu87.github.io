# 중앙은행디지털화폐 (CBDC)
{: .fs-8 }

3.2 블록체인 활용
{: .label .label-green }

---

## 핵심 키워드

`실물 명목 화폐` `중앙은행` `시중은행` `블록 체인`

---

## 정의/개념

실물 명목화폐(Fiat Money)를 대체하거나 보완하기 위해서 **중앙은행**이 직접 전자적 형태로 발행한 **중앙은행 화폐**

---

## 필요성

| 배경 | 설명 |
|:-----|:-----|
| **현금 이용 감소** | 디지털 결제 증가 |
| **디지털 가속** | 비대면 거래 확대 |
| **빅테크 시장 지배력** | 민간 결제 플랫폼 영향력 증가 |
| **글로벌 스테이블코인 등장** | 달러 연동 코인 확산 |

---

## 유형 (직간하)

| 유형 | 설명 |
|:-----|:-----|
| **직접형** | 중앙은행이 직접 개인에게 발행 |
| **간접형** | 중개기관(시중은행) 통해 발행 |
| **하이브리드형** | 직접형 + 간접형 혼합 |

---

## 구현방식 (단분)

### 단일원장 방식

| 항목 | 설명 |
|:-----|:-----|
| **관리 주체** | 중앙은행 |
| **절차** | 개인간 거래 발생 → 중앙은행에 전달 → 단일 원장에 기록 |

### 분산원장 방식

| 항목 | 설명 |
|:-----|:-----|
| **관리 주체** | 분산저장 + 중앙은행 |
| **절차** | 개인간 거래 발생 → 여러 참여 주체에 전달 → 분산 원장에 기록 |

### 방식 비교

| 구분 | 단일원장 | 분산원장 |
|:-----|:---------|:---------|
| **속도** | 빠름 | 느림 |
| **비용** | 절감 | 증가 |
| **복원력** | 전체 장애 위험 | 분산 장애 |
| **보안** | 고도 기술 필요 | 분산 보안 |
| **혁신** | 스마트계약 안됨 | 스마트계약 가능 |

---

## 가상화폐 비교

| 비교 항목 | 중앙은행 디지털화폐 (CBDC) | 가상화폐 (Virtual Currency) |
|:---------|:-------------------------|:---------------------------|
| **개념** | 중앙은행내 지준예치금이나 결제성 예금과는 별도로 중앙은행이 전자적 형태로 발행하는 새로운 화폐 | 금전적 가치가 전자적 형태로 저장·이전·거래 가능 민간 통화를 의미하며 가상환경의 통용되는 실체 없는 화폐 |
| **사용목적** | - 소액/거액 지급결제<br>- 초국경 지급결제 | - 특정 사용도 없음 |
| **데이터 기록** | - 블록체인, 별도의 데이터베이스에 기록 | - 블록체인 상에서 분산되어 기록 |
| **검증 및 원장관리** | - 중앙은행이나 정부 등 정해진 기관 관리 | - 블록체인 참여자가 모두 관리 |
| **발행주체** | - 중앙은행에서 발행 | - 민간에서 발행 |
| **발행규모** | - 고정되지 않은 전체 공급량<br>- 중앙은행의 정책에 따른 공급량 | - 고정된 전체 공급량(2100만개 등)<br>- 지속적인 공급량의 감소 |
| **가치** | - 정부가 보장<br>- 액면가 고정 | - 블록체인 참여자의 신뢰에 비례<br>- 수요공급에 의한 결정 |
| **사례** | - 중국 인민은행 디지털화폐 시험운영 | - 비트코인, 이더리움 |

## CBDC 장단점

### 장점
- 신용 리스크 경감
- 효율성 제고
- 비용 절감
- 지하 경제 양성화
- 통화 정책 수단 증가

### 단점
- 사회 금융 중개 기능 위축
- CBDC로 자금 이동 우려
- 시중은행 수익성 약화
- 금융자원 배분 효율성 저하
- 신용 경로 약화

---

## CBDC 설계원칙

| 원칙 | 설명 |
|:-----|:-----|
| **복원력** | 시스템 장애 복구 능력 |
| **확장성** | 사용량 증가에 대응 |
| **보안성** | 해킹/위변조 방지 |
| **익명성(기밀성)** | 거래 프라이버시 보호 |
| **호환성** | 기존 시스템과 연계 |
| **이자지급가능성** | 금리 정책 반영 가능 |
