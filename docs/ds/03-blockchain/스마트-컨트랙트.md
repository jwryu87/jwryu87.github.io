# 스마트 계약 (Smart Contract)
{: .fs-8 }

3.2 블록체인 활용
{: .label .label-green }

---

## 핵심 키워드

`개발환경(Byte Code, ABI)` `엔진(EVM)` `애플리케이션(Dapp, Solidity)`

---

## 정의/개념

블록체인 기반 금융거래, 부동산 거래 등 **Solidity**, **Dapp**, **EVM** 기술 활용 **이더리움 플랫폼에서 자동 체결 기술**

---

## 특징

| 측면 | 특징 | 설명 |
|:-----|:-----|:-----|
| **계약 측면** | 자동 계약 | 조건 충족 시 자동 실행 |
| | 프로세스 간소화 | 중개자 없이 직접 계약 |
| **기술 측면** | 탈중앙화 | P2P 기반 실행 |
| | 코드 실행 | 프로그래밍 가능한 계약 |
| | 튜링 완전성 | 모든 계산 가능 |

---

## 원칙 (강검관사)

| 원칙 | 설명 |
|:-----|:-----|
| **강제 가능성** | 계약 조건 강제 실행 가능 |
| **검증 가능성** | 계약 실행 결과 검증 가능 |
| **관측 가능성** | 계약 상태 모니터링 가능 |
| **사생활 보호** | 개인정보 보호 |

---

## 메커니즘

```
Smart Contract 개발환경 → BlockChain Engine
                            
Contract Code → Byte Code → Contract Tx
             ↓             ↓
     Function Signature → Parameter → Message Tx
             ↓                            ↓
            ABI → Application(Dapp) → Block Chain Network
             ↓
    JavaScript | Solidity
    RPC/IPC   | Java
```

---

## 기술요소

| 구분 | 핵심 기술 | 설명 |
|:-----|:---------|:-----|
| **Development Environment** | Byte Code | Message TX, Call 통해 EVM 위에서 실행 |
|  | Function Signature | Contract Address, Method Signature |
|  | ABI | Application Binary Interface, 다중타입 함수 구현 |
| **Blockchain Engine** | EVM | Ethereum Virtual Machine, Byte Code 실행 플랫폼 |
|  | Ethereum | Public Blockchain 2.0, PoW |
| **Application** | Dapp | Decentralized Application, Web3.js, HTML/CSS/JS |
|  | Solidity, LLC, Serpent | EVM 목표로 설계된 프로그래밍 언어 |

- 스마트 계약은 **Off-Chain(현실세계)**과 **On-Chain(블록체인)** 데이터 연계시 **오라클 문제**가 존재

---

## 오라클 문제

| 유형 | 문제점 | 해결방안 |
|:-----|:-------|:---------|
| 하드웨어 오라클 | 데이터 위변조 | 중간자 |
| 소프트웨어 오라클 | 데이터 무결성 | 투표 |
| 아웃바운드 오라클 | 데이터 수집 | 중앙값 |
