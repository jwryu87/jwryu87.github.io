## 핵심 키워드

`확장성(TPS)` `보안성(트랜잭션 승인)` `탈중앙성(Mining Node, Staking Node)`

---

## 정의/개념

블록체인에서 **확장성**, **탈중앙화**, **보안성** 3가지 주요 목표 중 어느 하나를 희생해야 다른 두 가지 목표를 달성할 수 있다는 **이론**

---

## 배경

| 기술측면 | 항목 | 설명 |
|:---------|:-----|:-----|
| **기술측면** | 확장성 미고려 | 초기 비트코인은 보안과 탈중앙에 초점을 두고 프로젝트 진행 |
|  | Trade-off | 높은 TPS를 위해 탈중앙성이나 합의 과정을 변경하는 Trade Off 필요 |
|  | Killer App 등장 | 지정학적 요인과 블록체인 기술의 기대감으로 다양한 서비스의 등장 |
| **서비스측면** | 가상자산 활성화 | 다양한 거래소(CEX, DEX 등)의 등장으로 가상자산 거래 활성화 |

가상자산의 공개방식의 다양화로 다수의 프로젝트 들이 동장하게 되고 일부 프로젝트들이 성공하면서 투자 및 자산으로서 가치를 인정 받음

---

## 구성

| 구성 요소 | 설명 | 상세 요소 |
|:----------|:-----|:----------|
| **확장성 (Scalability)** | - 사용자 수의 증대에 유연하게 대응할 수 있는지 여부<br>- 블록체인에서는 사용자수의 증가에 따라 거래건수가 늘어나더라도 무리 없이 전송 처리용량을 증대시킬 수 있는 능력을 의미 | - user performance<br>- transaction throughput |
| **탈중앙화 (Decentralization)** | - 중앙집중화를 벗어나 분산된 소규모 단위로 자율적으로 운영 되는 정도<br>- 기존 서버-클라이언트 형태가 아닌 개별 노드들의 자발적인 연결에 의한 P2P 방식으로 작동 | - user controlled<br>- open access, anonymity of actors<br>- no censorship of information |
| **보안성 (Security)** | - 보안이란 블록체인 내의 데이터나 프로그램을 권한이 없는 이용자가 사용할 수 없도록 하는 것을 의미 | - resistance against malicious attacks<br>- majority decisions<br>- ledger immutability |

## 대안 기술

| 대안기술 | 기술요소 | 설명 |
|:---------|:---------|:-----|
| **알고랜드** | 보상없는 탈중앙화 | 특정한 작업(채굴, 위임, 스테이킹)을 위한 보상이 없고 지갑 소유자 모두에게 이자로 보상 제공 |
|  | 키 재조정 | 하나의 퍼블릭 키에 여러 프라이빗 키를 사용 가능, 제공하여 우수한 보안 체계 제공 |
|  | 온체인 NFT | 별도의 레이어 없이 온체인에서 ASA 통해 안전성과 빠른 속도 모두를 성취하면서 NFT를 제공 |
| **라이트닝** | 온체인 | 오프체인에서 발생 거래를 종합하여 기록하는 체인 |
| **네트워크** | 오프체인 | 일반 거래들을 별도의 지불채널에서 수행하는 체인 |
| **연관기술** | BPU | Blockchain Processing Unit, 블록체인 전용 프로세서 |
| **고도화** | BSN | Blockchain Service Network, 블록체인 기반 어플리케이션 개발 위한 플랫폼 |

- 블록체인 트릴레마의 대응을 위한 다양한 기술 및 기법이 연구 중
