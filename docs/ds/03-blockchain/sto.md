# STO (Security Token Offering)
{: .fs-8 }

3.2 블록체인 활용
{: .label .label-green }

---

## 핵심 키워드

`증권형 토큰` `증권거래법` `자본시장법` `자산의 소유권`

---

## 정의/개념

해당 암호화폐를 발행한 회사의 자산에 대한 소유권을 가진 **토큰 형태로 발행하는 증권**

---

## 특징

| 특징 | 설명 |
|:-----|:-----|
| **소액 투자** | 적은 금액으로 투자 가능 |
| **비용 절감** | 중개 비용 감소 |
| **안정성 확보** | 증권법 적용 |
| **거래 투명성** | 블록체인 기반 투명 거래 |
| **신뢰도 제고** | 규제 준수 |
| **투자자 법적 보호** | 증권법상 투자자 보호 |
| **자산 유동성 증가** | 24/7 거래 가능 |

---

## 유형

| 유형 | 설명 |
|:-----|:-----|
| **증권 발행형** | 주식/채권 등 증권 토큰화 |
| **자산 유동화형** | 부동산/미술품 등 자산 토큰화 |

---

## STO 절차 (발청중 유증결은)

### 발행시장

| 참여자 | 역할 |
|:-------|:-----|
| **청약자** | 크라우드펀딩 청약, STO 토큰 매도 |
| **중개사** | 위탁계약 등록, 발행 심사 |

### 유통시장

| 참여자 | 역할 |
|:-------|:-----|
| **증권사** | 토큰 거래 중개, 캐시 토큰 요청 |

### 결제시장

| 참여자 | 역할 |
|:-------|:-----|
| **은행** | 토큰 교환 요청 수용, 캐시 토큰 교환 |

---

## DABS 플랫폼

일반 수익형 증권을 블록체인 기술을 사용하여 **디지털 증권으로 전환**시켜 발행 및 유통이 가능한 플랫폼

| 구분 | 기술 |
|:-----|:-----|
| **플랫폼 기술** | 디지털 증서 관리, API 관리, 에스크로(Escrow) 서비스 |
| **블록체인 기술** | DABS 발행, 합의 알고리즘, 노드 운영 |

---

## STO 발행절차

```
요건을 갖춘 발행인 → 직접 → 토큰 생성/이전 (법적관리정부)
                              ↓
                        KSD (한국예탁결제원)
                        등록심사 / 총량관리 / 결제
                              ↓
요건을 갖추지 못한 발행인 → 계좌관리기관 (증권사/은행 등) → 장외거래 중개업자 → 장외시장 (IT모제한)
                                                              ↓
                                                   증권사 (거래 중재) → KRX 디지털 증권시장
                                                                      ↓
                                                              장내시장 상장심사 / 매매체결 / 시장감시
```

---

## STO 특징 및 적용기술

| 구분 | 요소 | 설명 |
|:-----|:-----|:-----|
| **특징** | 혁신성 | 다양한 권리와 비정형적 승인을 디지털 통해 거래 |
|  | 안정성 | 투자자 보호를 위한 안정적인 토큰 가격 유지 |
|  | 유동성 | 증권 시장에서 상장과 자산 분할이 가능 쉽게 거래 |
|  | 효율성 | 블록체인 기술 이용한 조각거래/거래시간 및 비용 감소 |
| **적용 기술** | 블록체인 | 블록체인 네트워크에서 발행 및 거래가 이루어짐 |
|  | 스마트 컨트랙트 | 거래와 유효성 검증을 위한 스마트 컨트랙트 기술 적용 |
|  | KYC | 증권 시장 규제 적용 따른 투자자 신원 인증 위한 기술 |
|  | AML | 돈 세탁 방지를 위한 AML 절차 수행 |
