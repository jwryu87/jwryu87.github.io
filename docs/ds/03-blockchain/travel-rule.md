# 트레블 룰 (Travel Rule)
{: .fs-8 }

3.7 블록체인 생태계
{: .label .label-purple }

> 암기: **절차** (출정송저)

---

## 정의

가상자산 거래 시 **송금인과 수취인의 정보를 함께 전송**해야 하는 자금세탁방지(AML) 규정

---

## 특징

| 특징 | 설명 |
|:-----|:-----|
| **암호화값 대조** | 송수신 정보 암호화 검증 |
| **블록체인 활용** | 분산원장 기반 정보 전달 |

---

## 절차 (출정송저)

| 단계 | 주체 | 내용 |
|:-----|:-----|:-----|
| **출금요청** | 송금사업자 | 지갑 확인 |
| **정보송신** | 송금사업자 → 수취사업자 | 송금인/수취인 정보 전송, 확인 |
| **송금** | 송금사업자 | 정보 검증 후 출금 → 정상 입금 |
| **데이터저장** | 양측 | VA(가상자산) 거래 양사 저장 |

---

## 전송 정보

| 구분 | 정보 |
|:-----|:-----|
| **송금인 정보** | 이름, 지갑주소, 식별번호 |
| **수취인 정보** | 이름, 지갑주소 |
| **거래 정보** | 거래금액, 거래일시 |

---

## 관련 규정

| 규정 | 설명 |
|:-----|:-----|
| **FATF 권고** | 국제자금세탁방지기구 권고사항 |
| **특금법** | 한국 특정금융정보법 |
| **자금세탁방지법** | AML/CFT 규정 |
