# 블록체인 오라클
{: .fs-8 }

3.2 블록체인 활용
{: .label .label-green }

---

## 핵심 키워드

`On-Chain` `Off-Chain` `하드웨어/소프트웨어/아웃바운드 오라클` `중간자` `투표` `중앙값`

---

## 정의/개념

블록체인 밖(**Off-Chain**)의 데이터를 블록체인 안(**On-Chain**)으로 가져올 때 원 데이터가 오류가 있거나 가져오는 과정에서 위변조가 발생하여 **블록체인의 신뢰성이 훼손되는 현상**

---

## 오라클 문제

| 문제 | 설명 |
|:-----|:-----|
| **데이터 수집** | 외부 데이터 획득의 어려움 |
| **데이터 위변조** | 데이터 조작 가능성 |
| **데이터 무결성** | 데이터 정확성 보장 필요 |

---

## 블록체인 오라클 유형 분류

### 오라클 형태

| 유형 | 개념도 | 설명 |
|:-----|:-------|:-----|
| **하드웨어 오라클** | 현실세계 → 센서 → 블록체인 | 센서(sensor)를 통해 온도, 특정 물품의 파손율, 온실가스 배출량 등, 현실세계의 다양한 정보를 데이터화해서 블록체인 네트워크에 제공하는 기술 |
| **소프트웨어 오라클** | 환율, 상품 가격 등 이벤트 → 블록체인 | 온라인상의 신뢰할 수 있는 채널(기상청, 중앙은행, 항공사 홈페이지)로 부터 데이터를 가져오는 기술 |

### 정보 이동 방향성

| 유형 | 설명 |
|:-----|:-----|
| **내향성 오라클 (Inbound)** | 외부 → 블록체인 (데이터 유입) |
| **외향성 오라클 (Outbound)** | 블록체인 → 외부 (데이터 전송) |

### 작동 메커니즘

| 유형 | 설명 |
|:-----|:-----|
| **중앙화 오라클** | 단일 데이터 제공자 |
| **탈중앙화 오라클** | 다수 데이터 제공자 합의 |
| **연방 오라클** | 중앙화 + 탈중앙화 혼합 |

---

## 블록체인 오라클 패러독스

**복잡성과 리스크 반비례**: 탈중앙화할수록 복잡성 증가, 중앙화할수록 신뢰 리스크 증가

### 해결방안

| 구분 | 방안 |
|:-----|:-----|
| **오라클 측면** | 통합 프로토콜, 탈중앙화 오라클, 오라클 행동 분석 |
| **외부 검증 측면** | 평판 시스템, 다중 데이터 제공, 신뢰도 판단 |

---

## 블록체인 오라클 해결방안

| 해결방안 | 설명 | 유형 |
|:---------|:-----|:-----|
| **중간자** | - Dapp이 자체적으로 오라클 모델을 만들지 않아도 중립 외부 데이터를 가져올 수 있게 다업체로부터 솔루션을 제공 받는 것(데이터 API 제공) | - 오라클라이즈(Oraclize)<br>- 체인링크(Chainlink) |
| **투표** | - Dapp의 일정 지분을 보유한 노드 들로부터 외부 데이터를 블록체인에 보고, 보고된 데이터 중 가장 많은 지분이 걸린 데이터를 참으로 인정하는 방식 | - 아토믹 스왑(Atomic swap)<br>- 어거(Augur) |
| **중앙값** | - 여러 개의 데이터를 크기 순서대로 늘어놓았을 때 한가운데 있는 값(Median)을 선택하여 이용하는 방식 | - 메이커 토큰(Maker token) |
