## 블록체인 특징

| 구분 | 특징 | 내용 |
|:-----|:-----|:-----|
| **기술/구조적 측면** | 탈 중앙 (Decentralization) | 거래내역을 중앙 집중된 곳에서 관리하지 않고 P2P N/W에서 참여자간 공유함으로 중앙 서버에 의한 변조, 유용 불가 |
|  | 분산 네트워크 (Distributed N/W) | 분산된 네트워크 상의 노드(node/peer)들은 거래를 검증한 다음 자신의 장부에 거래를 추가하여 네트워크 간 연계를 구성 |
|  | 탈 중개성(P2P-based) | 공인된 제3자의 공증 없이 개인간 거래가 가능하여 중개비용 등에 따른 비효율성이 감소되고 불필요한 수수료가 경감 |
|  | 타임 스탬핑 (Timestamping) | 전자기록물의 진본성 및 유효성을 입증하기 위해 거래시점의 확인정보를 관리하여 무결성과 신뢰성 보장 |
|  | 합의 알고리즘 (Consensus Algorithm) | 대규모 노드들 사이에서 각 노드에 분산저장된 장부의 데이터를 항상 최신 상태로 유지하고, 네트워크 참여자들의 동의/합의 기술에 의해 신규 블록 검증 |
| **업무/융합성 측면** | 경제성(Economy) | - 특허가 없는 오픈 소프트웨어 사용에 따른 비용 절감<br>- 시스템 개발/유지보수, DB 운영/관리, 보안에 따른 비용 절감<br>- 송금/결제 등에 발생하는 중개수수료 비용 절감 |
|  | 보안성(Secure) | 정보를 다수가 공동 분산 소유하여 해킹 불가 |
|  | 신속성 (Instantaneous) | 거래의 승인 및 기록은 다수의 참여에 의해 주기적으로 자동 실행하여 신속성이 극대화 |
|  | 확장성(Scalable) | 공개 소스에 의해 구축, 연결, 확장이 용이 |
|  | 투명성(Transparent) | 각 블록 내에 포함된 모든 기록 장보에 대한 조회가 가능하여 거래 양상화 및 규제비용 절감 |

---

## 기술요소 비교

| 구분 | 세부 기술요소 | 설명 |
|:-----|:-------------|:-----|
| **트랜잭션 기록 기술요소** | 분산 원장 | 탈중앙화를 가능하게 하며, 참여자들 간 합의에 의해 복제 및 공유되며, 블록이라는 저장소에 동기화된 정보를 기록 |
|  | 공개키 기반 구조 | PKI(공개키 기반 구조)기반의 디지털 서명을 통한 신원 인증과 부인 방지 |
|  | 암호화 해시 | 암호화 해시를 통한 데이터 무결성 유지와 분산원장 간 연결성 부여 |
| **플랫폼 기술요소** | P2P 네트워크 | 모든 노드가 서버이자 동시에 클라이언트 역할 수행 |
|  | 합의 알고리즘 | 모든 참여자들이 데이터의 적합성을 판단하고 동의하는 과정<br>- PoW(작업증명), PoS(지분증명), PBFT 등 |
|  | 스마트 컨트랙트 | 특정 계약 조건을 자동적으로 실행하기 위한 전자상거래를 위한 기술 |
