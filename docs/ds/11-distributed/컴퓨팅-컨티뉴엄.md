# 컴퓨팅 컨티뉴엄 (Computing Continuum)
{: .fs-8 }

11. 분산 컴퓨팅
{: .label .label-purple }

---

## 핵심 키워드

`GigaMEC` `CompactEdge` `EdgeCPS` `임베디드-엣지-클라우드` `디지털 트윈`

---

## 정의/개념

3가지 컴퓨팅 환경 최적 응답시간 및 QoE 제공 위해, **임베디드-엣지-클라우드 다계층 컴퓨팅 연속체 통합 동적 제공 아키텍처**

- 임베디드, 엣지, 클라우드까지 이어지는 연속성 있는 컴퓨팅 환경 제공
- 연속성 있는 컴퓨팅 환경은 디바이스 영역이 클라우드와 유기적으로 연계되어 완전한 형태의 CPS(Cyber Physical System)과 디지털 트윈(Digital Twin) 구현 가능

## 구성요소

| 구분 | 구성요소 | 핵심 |
|:-----|:-----|:-----|
| **클라우드 영역** | GigaMEC | MEC, SDK, MANO 연계 MEC 관리 수행 |
| | CompactEdge | 경량화된 엣지 서버 시스템 자원 관리·제어 |
| **디바이스 영역** | EdgeCPS | 디바이스리스 컴퓨팅, 디바이스 트윈 제공 |

---

## 컴퓨팅 계층별 특징

| 계층 | 특징 | 역할 |
|:-----|:-----|:-----|
| **임베디드/엣지** | 저지연, 단말이동성, 레거시연동, 민감데이터 보호 | 탈중앙화, 현실 세계 지능화 |
| **클라우드** | 고성능, 관리용이성, 규모의 경제, 거버넌스 | 중앙 집중화, 가상 세계 동기화 |

---

## 컴퓨팅 컨티뉴엄 핵심 기술

| 기술 | 설명 |
|:-----|:-----|
| **GigaMEC** | 대규모 MEC 환경 관리, SDK 및 MANO 연계 |
| **CompactEdge** | 경량화 엣지 서버, 자원 효율적 관리 |
| **EdgeCPS** | 디바이스와 사이버 세계 연결, 디바이스 트윈 |
| **Digital Twin** | 물리적 객체의 가상 복제본 |
