# 서버리스 컴퓨팅 (Serverless Computing)
{: .fs-8 }

11. 분산 컴퓨팅
{: .label .label-purple }

---

## 핵심 키워드

`FaaS` `BaaS` `Event Router` `API Gateway` `SPA`

---

## 정의/개념

특정 이벤트의 반응하는 함수가 실행되는 **FaaS와 BaaS 기반**의 마치 물리적 서버가 없는 것처럼 동작하여 기존의 방식보다 연결 및 처리속도를 개선한 **컴퓨팅 기술**

---

## 아키텍처

```
                         < Event Router >
┌─────────┐           ┌───────────────────────────────────────────┐
│   Web   │           │                                           │
├─────────┤           │  ┌──────────────┐    ┌──────────────┐    │
│         │    ┌──────┴──┤Authentication│    │   Database   │    │
│ Mobile  │───→│  API    ├──────────────┤    ├──────────────┤    │
│         │    │Gateway  │   Purchase   │    │   Message    │    │
├─────────┤    └──────┬──┤──────────────┤    ├──────────────┤    │
│   IoT   │           │  │  Management  │    │  File System │    │
└─────────┘           │  └──────────────┘    └──────────────┘    │
                      └───────────────────────────────────────────┘
   Client          Front End     Function Service      Back End
```

---

## 기술요소

| 구분 | 핵심 기술 | 설명 |
|:-----|:-----|:-----|
| **Client** | SPA | 단일 페이지의 UI/UX 구조 |
| | Web App | Web App 기반의 단말 (Mobile, IoT, PC) |
| **Front End** | API Gateway | Event Trigger, Load Balancing |
| | RESTful API | REST 아키텍처 구현 웹서비스 |
| **Function Service** | FaaS (Function as a Service) | 어플리케이션 조합에 필요한 패키지, 패턴, 참조 아키텍처 |
| | Event Router | 이벤트 전달, 기능 서비스, 트리거 기반 Function 실행 |
| **Back End** | BaaS (Backend as a Service) | 인증, 구매, DB, 보안 등 Back End 기능의 서비스 제공 |
| | Container | Docker 기반 신속 배포/릴리즈 |

---

## FaaS vs BaaS 비교

| 구분 | FaaS | BaaS |
|:-----|:-----|:-----|
| **전체 명칭** | Function as a Service | Backend as a Service |
| **역할** | 함수 단위 실행 | 백엔드 기능 서비스 제공 |
| **특징** | 이벤트 기반 트리거 | 인증, DB, 스토리지 등 제공 |
| **과금** | 실행 시간 기준 | 사용량 기준 |
| **예시** | AWS Lambda, Azure Functions | Firebase, AWS Amplify |

---

## 서버리스 컴퓨팅 장단점

| 구분 | 내용 |
|:-----|:-----|
| **장점** | - 서버 관리 불필요<br>- 자동 확장(Auto Scaling)<br>- 비용 효율성 (사용한 만큼 과금)<br>- 빠른 배포 및 개발 |
| **단점** | - Cold Start 지연<br>- 벤더 종속성(Lock-in)<br>- 디버깅 어려움<br>- 실행 시간 제한 |

---

## 서버리스 플랫폼 예시

| 벤더 | 서비스 |
|:-----|:-----|
| **AWS** | Lambda, API Gateway, DynamoDB |
| **Azure** | Azure Functions, Logic Apps |
| **Google** | Cloud Functions, Firebase |
| **기타** | Cloudflare Workers, Vercel |
