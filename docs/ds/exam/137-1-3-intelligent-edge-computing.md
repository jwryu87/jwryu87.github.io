## 📖 등장배경 (왜 필요해?)

### 🤔 문제 상황
```
😰 "자율주행차가 클라우드에 물어보고 결정하면?"
   → 0.1초 늦으면 사고! 실시간 판단 필요

😰 "모든 CCTV 영상을 클라우드로 보내면?"
   → 네트워크 터지고, 개인정보 유출 위험

😰 "공장에서 인터넷 끊기면?"
   → 클라우드 의존하면 생산라인 멈춤!
```

### 💡 해결
```
✅ 현장(엣지)에 AI 두뇌를 심자!
   - 자율주행차: 차량 내 AI가 직접 판단
   - CCTV: 현장에서 분석하고 결과만 전송
   - 공장: 네트워크 끊겨도 자율 동작
```

---

## 📝 개념 정의

{: .highlight }
> **지능형 엣지 컴퓨팅이란?**
> 
> 네트워크 말단(엣지)에서 **AI가 직접 분석·판단**하여 실시간 의사결정하는 컴퓨팅
> 
> 쉽게 말해: **"스마트한 현장 두뇌"** 🧠
> 
> - 클라우드에 물어보지 않고 현장에서 즉시 판단
> - 학습은 클라우드, 추론은 엣지에서!

---

## 🏗️ 기술요소 (2그룹)

### 그룹 1: 유형 (어디에 설치?) `디게서네클`

| 유형 | 위치 | 예시 |
|:----|:----|:-----|
| **디바이스 엣지** | 스마트폰, IoT 센서 | 스마트워치가 심박수 이상 감지 |
| **게이트웨이** | 공장/빌딩 현장 | 여러 센서 데이터 모아서 처리 |
| **엣지 서버** | 기업 내, 기지국 근처 | GPU로 고성능 AI 분석 |
| **네트워크 엣지** | 통신사 기지국 | 밀리초 단위 대규모 처리 |
| **클라우드 연계형** | 클라우드+엣지 협업 | 학습↔추론 분리 |

### 그룹 2: 장점 (왜 좋아?) `실분보자`

| 장점 | 설명 |
|:----|:-----|
| **실시간성 (Real-time)** | 지연(latency) 최소화, 즉시 처리 |
| **분산 지능 (Distributed)** | 중앙집중 → 분산형 전환 |
| **보안 강화 (Security)** | 민감 데이터 로컬 처리, 전송 최소화 |
| **자율성 (Autonomy)** | 네트워크 단절에도 독립 동작 |

---

## ⭐ 차별점 키워드 (답안지 가산점!)

{: .highlight }
> **하이브리드 AI 아키텍처**
> 
> - **학습(Training)** → 클라우드 (빅데이터, GPU 클러스터)
> - **추론(Inference)** → 엣지 (실시간, 경량 모델)
> 
> 💡 "Train on Cloud, Infer on Edge" 전략!

---

| 배경 | 설명 |
|:----|:-----|

| **데이터 폭증** | 모든 데이터를 클라우드로 전송 불가, 현장 처리 필수 |

## 📝 개념 (22자)

> **AI탑재 엣지에서 실시간 분석·판단하는 컴퓨팅**

### 상세 개념

| 구분 | 내용 |
|:----|:-----|
| **정의** | 클라우드 중심 연산을 네트워크 말단에서 수행, AI 분석자동화 기능을 탑재하여 실시간 의사결정과 서비스 제공을 가능하게 하는 컴퓨팅 패러다임 |
| **핵심** | 단순 데이터 중계가 아닌 엣지에서 스스로 학습·판단하는 'AI+Edge 융합' |
| **구조** | 학습(Training)은 클라우드, 추론(Inference)은 엣지 수행 |

---

## 🏗️ 구성요소 (2그룹 × 5항목) `디게서네클` `디전지클앱`

### 그룹 1: 유형 (Deployment Type) `디게서네클`

| 유형 | 위치 | 특징 |
|:----|:----|:-----|
| **Device Edge** | IoT 센서, 스마트폰, 웨어러블 | 초경량 AI 모델 내장, 현장 데이터 즉시 분석·추론 |
| **Gateway/Node** | 공장/빌딩/차량 현장 | 여러 디바이스 데이터 집계 처리 후 클라우드 전송 |
| **Server Edge** | 기업 내, 기지국 근처 | GPU/NPU 기반 고성능 AI 분석 처리 |
| **Network Edge** | 통신사업자 기지국, ISP | 밀리초 단위 데이터 처리, 대규모 사용자 대응 |
| **Cloud-Integrated** | 클라우드+엣지 협업 구조 | Training은 Cloud, Inference는 Edge |

---

### 그룹 2: 계층 (Layer Architecture) `디전지클앱`

| 계층 | 주요 구성요소 | 핵심 기능 |
|:----|:------------|:---------|
| **Device/Sensor** | 센서, CCTV, 로봇·차량 ECU, 웨어러블 | 데이터 수집 |
| **Edge Device** | 게이트웨이, 산업용 PC(IPC), 엣지 서버 | 전처리·필터링, 프로토콜 변환, 경량화 |
| **Edge Intelligence** | NPU/Edge TPU, 컨테이너(K8s at Edge), AI 모델 서빙 | 실시간 AI 분석·의사결정, 자율제어, 오프라인 동작 |
| **Cloud & Data Center** | 클라우드/데이터레이크, MLOps, 관제 | 대규모 학습 배포, 정책·보안 모니터링, 통합 관리 |
| **Application** | 스마트팩토리, 스마트시티, 모빌리티, 헬스케어 | 서비스 제공·대시보드·API |

> | **네** | **네**트워크 엣지 | 통신사 기지국 |
> | **클** | **클**라우드 연계형 | 클라우드+엣지 협업 |

{: .note }

>
> ```
> 디 → 전 → 지 → 클 → 앱 (데이터 흐름 순서)
> ```
>
> | 암기 | 계층 | 설명 |
> |:-----|:----|:------|
> | **디** | **디**바이스/센서 | 데이터 수집 |
> | **전** | 엣지 디바이스 (**전**처리) | 필터링, 프로토콜 변환 |
> | **지** | 엣지 **지**능 | AI 분석·의사결정 |
> | **클** | **클**라우드 | 대규모 학습, 관제 |
> | **앱** | **앱**플리케이션 | 서비스 제공 |

## 📊 클라우드 vs 엣지 vs 지능형 엣지 비교

| 구분 | 클라우드 컴퓨팅 | 엣지 컴퓨팅 | 지능형 엣지 컴퓨팅 |
|:----|:--------------|:-----------|:-----------------|
| **처리 위치** | 데이터센터 | 네트워크 말단 | 네트워크 말단 |
| **지연** | 높음 (100ms+) | 낮음 (10~50ms) | 매우 낮음 (<10ms) |
| **AI 기능** | 학습+추론 | 단순 필터링 | **실시간 추론** |
| **자율성** | 없음 | 제한적 | **완전 자율 동작** |
| **대표 기술** | AWS, Azure | CDN, MEC | **Edge AI, K8s at Edge** |

## ⭐ 차별점 키워드

{: .highlight }
> **TinyML / Federated Learning**
> 
> | 기술 | 설명 |
> |:----|:-----|
> | **TinyML** | 마이크로컨트롤러급 초경량 ML 모델 (mW 수준 전력) |
> | **Federated Learning** | 데이터를 중앙에 모으지 않고 분산 학습, 프라이버시 보장 |

> 💡 "데이터는 현장에, AI는 분산으로"
