### 📖 등장배경

> **문제**: 블록체인을 어디에 어떻게 쓸까?
> - 🌍 비트코인처럼 전 세계 누구나 참여? → 느리고 비효율
> - 🏢 기업 내부에서만 사용? → 탈중앙화 장점 사라짐
>
> **해결**: 용도에 맞게 3가지 타입 등장!
> - 퍼블릭: 완전 공개 (비트코인, 이더리움)
> - 프라이빗: 기업 전용
> - 컨소시엄: 기업 연합

## 📝 I. 블록체인의 개요 (1단락)

### 정의 (22자)

> **P2P 분산원장 기반 체인형 데이터 저장 기술로 거래 투명성과 무결성 보장**

### 블록체인 핵심 특징 `탈분탈타합` `경보신확투`

#### 기술/구조적 측면 `탈분탈타합`

| 특징 | 내용 |
|:----|:----|
| **탈**중앙 (Decentralization) | P2P N/W에서 참여자간 공유, 중앙 서버 변조 불가 |
| **분**산 네트워크 (Distributed N/W) | 노드들이 거래 검증 후 자신의 장부에 추가 |
| **탈** 중개성 (P2P-based) | 공인된 제3자 없이 개인간 거래, 수수료 경감 |
| **타**임스탬핑 (Timestamping) | 거래시점 확인정보로 무결성과 신뢰성 제공 |
| **합**의 알고리즘 (Consensus) | 네트워크 참여자들의 동의/합의로 신규 블록 검증 |

#### 업무 효율성 측면 `경보신확투`

| 특징 | 내용 |
|:----|:----|
| **경**제성 (Economy) | 오픈소스 사용, 중개수수료 절감 |
| **보**안성 (Secure) | 다수가 공동 분산 소유, 해킹 불가 |
| **신**속성 (Instantaneous) | 다수의 참여로 주기적 자동 실행 |
| **확**장성 (Scalable) | 공개 소스로 구축/연결/확장 용이 |
| **투**명성 (Transparent) | 모든 거래 정보 조회 가능, 규제비용 절감 |

## 📝 II. 블록체인 네트워크 종류 `퍼프컨`

### 그룹: 3대 블록체인 네트워크 유형 `퍼프컨`

| 유형 | 개념 | 목적 |
|:----|:----|:----|
| **퍼**블릭 (Public) | 탈중앙화, 누구나 참여 가능한 공개형 | 에코시스템 형성, 가상화폐 기반 |
| **프**라이빗 (Private) | 중앙기관 관리, 허가된 사용자만 참여 | 비즈니스 효율화, 고성능/보안성 |
| **컨**소시엄 (Consortium) | 다자간 연합, 퍼블릭+프라이빗 혼합형 | 단점 보완, 장점 극대화 |

## 📝 III. 블록체인 네트워크 종류별 차이점

### 상세 비교표

| 구분 | 퍼블릭 블록체인 | 프라이빗 블록체인 | 컨소시엄 블록체인 |
|:----|:--------------|:----------------|:----------------|
| **관리자** | 모든 거래 참여자 | 한 중앙기관 | 컨소시엄 소속 참여자 |
| **참여 방식** | 별도 허가 없이 누구나 | 허가된 대상만 | 컨소시엄 멤버만 |
| **거버넌스** | 법칙 변경 매우 어려움 | 중앙기관 결정으로 용이 | 참여자 합의로 변경 가능 |
| **거래 속도** | 느림 (확장 어려움) | 빠름 (확장 쉬움) | 빠름 (확장 쉬움) |
| **데이터 접근** | 누구나 접근 가능 | 허가 받은 사용자만 | 허가 받은 사용자만 |
| **식별성** | 익명성 | 식별 가능 | 식별 가능 |
| **거래 증명** | PoW, PoS 등 알고리즘 | 중앙기관에 의해 | 합의된 규칙에 따라 |
| **활용 사례** | 비트코인, 이더리움 | 나스닥 링크 | R3 CEV |

```
  컨 : 컨소시엄 (Consortium) - 연합형, 반중앙, R3

🧠 관참거속익 = "네트워크 유형별 5대 비교 포인트"

  관 : 관리자 (탈중앙/중앙/반중앙)
  참 : 참여방식 (개방/폐쇄/반개방)
  거 : 거버넌스 (어려움/쉬움/합의)
  속 : 속도 (느림/빠름/빠름)
  익 : 익명성 (익명/실명/실명)
```

## ⭐ 차별점 키워드

### 🏆 하이브리드 블록체인 (Hybrid Blockchain)
> 퍼블릭과 프라이빗 블록체인의 **단점을 보완**하고 **장점을 극대화**한 형태
> - **퍼블릭의 장점**: 투명성, 탈중앙화, 신뢰성
> - **프라이빗의 장점**: 속도, 확장성, 규제 준수
> - **현재 트렌드**: 많은 기업들이 하이브리드/컨소시엄 채택 중

---

### 📋 6하원칙 요약

| 구분 | 내용 |
|:----|:----|
| **누가 (Who)** | 개인(퍼블릭), 기업(프라이빗), 연합(컨소시엄) |
| **무엇을 (What)** | 용도에 맞는 블록체인 네트워크 유형 선택 |
| **언제 (When)** | 비트코인 이후 기업 적용 수요 증가 시점 |
| **어디서 (Where)** | 금융, 물류, 의료, 공공, 제조 등 |
| **왜 (Why)** | 퍼블릭의 속도/확장성 한계 극복, 규제 준수 |
| **어떻게 (How)** | 퍼블릭/프라이빗/컨소시엄 중 요구사항에 맞게 선택 |
