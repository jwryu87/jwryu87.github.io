### 📖 등장배경

> **문제**: 회사마다 여러 클라우드를 쓰는데...
> - ☁️ AWS도 쓰고, Azure도 쓰고, GCP도 쓰고...
> - 😵 각각 따로 관리하면 너무 복잡해!
> - 💸 비용도 얼마 나가는지 파악이 어려워
>
> **해결**: CMP 등장!
> - 🎛️ 한 화면에서 모든 클라우드 관리
> - 📊 비용, 자원, 보안 통합 관리

---

### 📝 개념 정의

> 🎮 **게임 컨트롤러**에 비유하면...
>
> | 구분 | 비유 |
> |:----|:----|
> | **개별 관리** | 각 게임기마다 다른 컨트롤러 사용 |
> | **CMP** | 모든 게임기를 하나의 만능 컨트롤러로 조작 |

---

### 🏗️ CMP 주요 기능

| 기능 | 설명 |
|:----|:----|
| **모니터링** | 모든 클라우드 자원 상태 확인 |
| **비용 관리** | 사용량/과금 추적 |
| **자동화** | 서버 생성/배포 자동화 |
| **보안** | 통합 보안 정책 관리 |

---

### ⭐ 차별점 키워드

**IaC (Infrastructure as Code)** 💻
> 인프라를 **코드로 관리**하는 기술
> - 서버 설정을 코드로 작성
> - 버전 관리, 자동화 가능
> - 예: Terraform, Ansible

## 📝 I. 클라우드 관리 플랫폼의 정의 및 필요성

### 가. 정의 (22자)

> **기업이 클라우드 컴퓨팅 리소스를 모니터링하고 제어하는 통합 소프트웨어 도구 모음 플랫폼**

### 나. 필요성 `시기멀IT`

#### 시장 측면 `시기`

| 필요성 | 설명 |
|:------|:----|
| **시**장 수요 증가 | 시간과 장소 구애 없이 효율적 비용으로 제공, 클라우드 수요 매년 상승 |
| **기**술 전환 가속화 | 코로나19로 디지털 중심 서비스 확산, 매니지드 서비스 수요 증대 |

#### 기술 측면 `멀IT`

| 필요성 | 설명 |
|:------|:----|
| **멀**티클라우드 통합 관리 | 멀티클라우드 환경 전환으로 통합 관리 플랫폼 중요성 부각 |
| **IT** 기술 전문성 부족 | 클라우드 플랫폼 구축, 시스템 이전, 자원 관리 전문 인력 부족 |
| **IT** 기술 인력 수급 | CSP 전문 인력 구인 어려움 증가 |

## 📝 II. 클라우드 관리 플랫폼의 필수 기능 `인운인클자인` `표자거워지`

### 가. 운영자 관점의 필수 기능 `인운인클자인`

| 필수 기능 | 설명 |
|:--------|:----|
| **인**프라 가시성 확보 | 다수의 클라우드, H/W, S/W 등 전반적인 가시성 확보 |
| **운**영 최적화 | CAPEX/OPEX 고려 최소 비용으로 최적의 비즈니스 자원 제공 |
| **인**적자원의 효과적인 운영 | 반복적인 작업들을 정책 설정과 자동화로 효과적 운영 |
| **클**라우드 자원 라이프사이클 관리 | IaC 이용 라이프사이클 관리 자동화, 유지보수 기능 제공 |
| **자**원 최적화 | AI 기술로 사용 패턴 분석, 성능과 비용 최적화 권고 |
| **인**터페이스 구조 | 온프레미스, 관리/모니터링 툴, ERP 등과 연동 인터페이스 |

### 나. 사용자 관점의 필수 기능 `표자거워지`

| 필수 기능 | 설명 |
|:--------|:----|
| **표**준화 | 비즈니스 목표, 정책, 보안, 거버넌스 충족하는 서비스 표준화 |
| **자**동화 | CAPEX/OPEX 고려 최적의 비즈니스 자원 제공 방안 제시 |
| **거**버넌스 & 보안 | 계정/권한 관리, 데이터 및 통신 암호화 기능 |
| **워**크플로 자동화 & 자가 치유 | 비즈니스 정책 기반 클라우드 환경 자동 생성/관리/모니터링 |
| **지**원 도구 (Utilization tools) | 사용 편리성을 위한 자동화 도구 및 사용 지원 도구 제공 |

---

### 🧠 암기법 상세 (필수 기능)

```
🧠 인운인클자인 = "운영자 관점 6대 필수 기능"

  인 : 인프라 가시성 확보
  운 : 운영 최적화 (CAPEX/OPEX)
  인 : 인적자원 효과적 운영
  클 : 클라우드 자원 라이프사이클 관리
  자 : 자원 최적화 (AI 분석)
  인 : 인터페이스 구조

🧠 표자거워지 = "사용자 관점 5대 필수 기능"

  표 : 표준화
  자 : 자동화
  거 : 거버넌스 & 보안
  워 : 워크플로 자동화 & 자가치유
  지 : 지원 도구
```

## 📝 III. 클라우드 관리 플랫폼의 선정 기준 `플중IaC클편사유`

| 선정 기준 | 관련 필수 기능 | 설명 |
|:--------|:-------------|:----|
| **플**랫폼 대상 | 인터페이스 구조 | 어떤 클라우드 플랫폼 대상으로 개발되었는지 확인 |
| **중**점 기능 | 지원 도구, 자동화 | 어떤 기능에 강한지, 기업 내 필수 기능 여부 |
| **IaC** 지원 | 자원 라이프사이클 관리, 자원 최적화 | 서비스 카탈로그형 제공으로 자원 최적화 용이성 |
| **클**라우드 자원 관리 | 라이프사이클 관리, 자원 최적화 | 라이프사이클 관리 자동화 여부 |
| **편**의성과 확장성 | 라이프사이클 관리, 인프라 가시성 | 로깅/모니터링 기능의 편의성, 성능, 확장성 |
| **사**용 요금의 자동화 | 동적 사용량 미터링/과금 | 미터링 및 과금 자동 처리, 안정성 제공 |
| **유**지보수성 | 워크플로 자동화 & 자가 치유 | 기능 추가/변경 등 작업의 용이성 |

## 📝 IV. 클라우드 관리 플랫폼의 기대효과 `IT복전비IT`

| 기대 효과 | 설명 |
|:--------|:----|
| **IT** 서비스 향상 | 인프라 자동화/효율화로 서비스 배포 가속화, 오류 발생 감소 |
| **복**잡한 비즈니스 과제 해결 | 복잡한 비즈니스 로직 자동화/단순화로 일반 문제 해결 |
| **전**체 시스템 효율성 향상 | 전체 클라우드 기능 통합과 관리, 워크로드 자동화 |
| **비**즈니스 커뮤니케이션 강화 | 단위 비즈니스간 커뮤니케이션 강화 |
| **IT** 환경 제어 유지 | 전체 클라우드 리소스 가시성 확보로 IT 환경 제어 |
| **IT** 비용 절감 | 동적 사용량 미터링과 과금 가시성 확보로 비용 절감 |

## 📊 종합 정리: CMP 4대 영역 `정필선기`

| 영역 | 핵심 내용 | 암기법 |
|:----|:---------|:------|
| **정**의 및 필요성 | 통합 관리 플랫폼, 멀티클라우드/인력부족 | `시기멀IT` |
| **필**수 기능 | 운영자 관점 + 사용자 관점 | `인운인클자인` `표자거워지` |
| **선**정 기준 | 플랫폼 대상, IaC, 편의성, 비용 | `플중IaC클편사유` |
| **기**대효과 | IT서비스향상, 비용절감, 효율성 | `IT복전비IT` |

---

### 🧠 전체 암기법 요약

```
🧠 정필선기 = "CMP 답안 4단락 구성"

  정 : 정의 및 필요성
  필 : 필수 기능
  선 : 선정 기준
  기 : 기대효과

🧠 시기멀IT = "필요성 4가지"

  시 : 시장 수요 증가
  기 : 기술 전환 가속화 (코로나)
  멀 : 멀티클라우드 통합 관리
  IT : IT 기술 전문성 부족

🧠 IT복전비IT = "기대효과 6가지"

  IT : IT 서비스 향상
  복 : 복잡한 비즈니스 과제 해결
  전 : 전체 시스템 효율성 향상
  비 : 비즈니스 커뮤니케이션 강화
  IT : IT 환경 제어 유지 + IT 비용 절감
```

## ⭐ 차별점 키워드

### 🏆 IaC (Infrastructure as Code)
> 인프라를 **코드로 정의하고 관리**하는 기술
> - **버전 관리**: 인프라 변경 이력 추적
> - **자동화**: 서버 생성/배포 자동화
> - **일관성**: 환경별 동일한 인프라 구성
> - **도구**: Terraform, Ansible, CloudFormation
