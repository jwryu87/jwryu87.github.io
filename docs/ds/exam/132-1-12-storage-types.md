---

### 📖 등장배경

> **문제**: 데이터가 폭발적으로 늘어나면서...
> - 📁 파일 수백만 개를 어떻게 관리하지?
> - 🗄️ 데이터베이스는 빠른 I/O가 필요한데...
> - 🌐 클라우드에서 사진/동영상을 어떻게 저장하지?
>
> **해결**: 용도에 맞는 3가지 스토리지 등장!
> - **블록**: DB, VM 디스크 (빠른 속도)
> - **파일**: 문서 공유, NAS (익숙한 구조)
> - **오브젝트**: 클라우드, 빅데이터 (무한 확장)

---

### 📝 개념 정의

> 📦 **택배 보관**에 비유하면...
>
> | 스토리지 | 택배 비유 |
> |:--------|:---------|
> | **블록** | 택배를 잘라서 여러 사물함에 분산 보관 (주소로 찾음) |
> | **파일** | 택배를 캐비닛 서랍에 폴더별로 정리 (경로로 찾음) |
> | **오브젝트** | 택배에 태그 붙여서 큰 창고에 저장 (ID로 찾음) |
>
> **핵심**: **접근방식**이 다름! (주소 vs 경로 vs ID)

## 📊 I. 블록 스토리지 `FC-iSCSI-SAN`

### 1. 개념

> **일정한 크기의 블록**으로 나누어진 스토리지의 논리 볼륨을 **블록 단위**로 액세스할 수 있는 스토리지

### 2. 접근방식

| 특징 | 설명 |
|:----|:----|
| **분할 저장** | 단일 스토리지 볼륨을 '블록'이라는 개별 단위로 분할하여 저장 |
| **고유 주소** | 각 블록은 저장된 위치에 대한 고유주소 보유 |
| **재구성** | 서버에서 파일 요청 시 블록들을 재구성하여 하나의 데이터로 전달 |
| **SAN 사용** | SAN(Storage Area Network) 또는 가상 머신의 디스크로 사용 |

### 3. 장단점

| 장점 | 단점 |
|:----|:----|
| 탐색이 유연하고 신속 (단일경로 비종속) | 상대적으로 **고비용** 소모 |
| 대규모 트랜잭션, 대용량 DB 운영에 유리 | **메타데이터 처리 기능 제한적** |
| 컨테이너 기술과 호환 가능 (가상화 용이) | 효율적인 데이터 정리 난해 |

---

## 📊 II. 파일 스토리지 `SMB-CIFS-NFS-NAS`

### 1. 개념

> **SMB, CIFS, NFS** 등의 프로토콜을 사용하여 **파일 기준**으로 읽고 쓸 수 있으며 공유 가능한 스토리지

### 2. 접근방식

| 특징 | 설명 |
|:----|:----|
| **계층적 트리구조** | 데이터를 저장하고 디렉토리에 접근하는 방식 |
| **친숙한 구조** | 윈도 탐색기, 맥 OS 파인더와 동일한 형태 |
| **폴더/캐비닛 방식** | 종이 파일 및 폴더가 캐비닛에 정리되는 방식 모방 |
| **NAS 사용** | 일반적으로 NAS(Network Attached Storage)에서 사용 |

### 3. 장단점

| 장점 | 단점 |
|:----|:----|
| 논리적이고 **직관적인 계층구조** | 파일수량/디렉토리 복잡 시 **성능저하** |
| 다양한 매체 저장 가능 | 데이터 접근이 **단일 경로**로만 이루어짐 |
| 트리구조 변경만으로 확장 가능 | **운영체제간 호환성** 낮음 |

---

## 📊 III. 오브젝트 스토리지 `HTTP-REST-S3`

### 1. 개념

> **데이터와 메타데이터**를 하나의 오브젝트로 묶어 **고유 ID**로 접근하는 스토리지

### 2. 접근방식

| 특징 | 설명 |
|:----|:----|
| **Flat 구조** | 계층 없이 평면적으로 저장 (Object ID로 접근) |
| **메타데이터** | 데이터 자체에 풍부한 메타데이터 포함 |
| **HTTP/REST** | 웹 API로 접근 (S3 API 등) |
| **무제한 확장** | 수평적 확장으로 페타바이트급 저장 가능 |

### 3. 장단점

| 장점 | 단점 |
|:----|:----|
| **무제한 확장성** (수평 확장) | 상대적으로 **느린 속도** (고지연) |
| 풍부한 **메타데이터** 지원 | 트랜잭션 처리에 부적합 |
| **저비용** 대용량 저장 | 파일 수정 시 전체 재업로드 필요 |
| 지리적 분산 저장 용이 | 기존 애플리케이션 호환 어려움 |
