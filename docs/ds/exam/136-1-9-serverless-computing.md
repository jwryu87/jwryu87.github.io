> 🍕 **피자 배달**과 비슷해요!
>
> | 방식 | 비유 |
> |:----|:----|
> | **기존 서버** | 집에 피자 화덕 설치 (비싸고 관리 필요) |

### 🏗️ 기술요소 (2그룹)

#### 그룹 1: 앞단 (사용자가 보는 부분)

| 요소 | 역할 |
|:----|:----|
| **SPA** | 한 페이지에서 모든 게 동작 (앱처럼) |
| **API Gateway** | 요청을 받아서 적절한 곳으로 전달 |

#### 그룹 2: 뒷단 (실제 처리하는 부분)

| 요소 | 역할 |
|:----|:----|
| **FaaS** | 함수(코드 조각)를 실행 (AWS Lambda 등) |
| **BaaS** | 인증, 결제 등 기본 기능 제공 |

---

### ⭐ 차별점 키워드

**Cold Start (콜드 스타트)** 🧊
> 처음 요청이 오면 잠자던 함수를 깨워야 해서 느림 (1~2초 지연)
> → 이걸 줄이는 게 서버리스의 핵심 과제!

## 🏗️ 구성요소 (2그룹 × 4항목) `S-W-A-R` `B-C-F-E`

### 그룹 1: Client / Front End `S-W-A-R`

| 구성요소 | 설명 |
|:--------|:-----|
| **SPA** | 단일 페이지 UI/UX 구조로 사용자 편의성 확보 |
| **Web APP** | Web 기반 환경으로 Page 접속하여 관리 가능 |
| **API Gateway** | Event Trigger 기반 부하분산 및 트래픽 관리 |
| **RESTful API** | 아키텍처 구현 웹서비스 데이터 제공/처리 구현 |

### 그룹 2: Back End / Service `B-C-F-E`

| 구성요소 | 설명 |
|:--------|:-----|
| **BaaS** | 인증, 구매, 보안 기능 서비스 제공 (Backend as a Service) |
| **Container** | Docker 기반 유연한 서비스 제공 및 배포 기능 지원 |
| **FaaS** | 패키지, 패턴, 참조 아키텍처를 통한 Function 제공 |
| **Event Router** | 이벤트 처리 위한 서비스 라우팅 구현 제공 |

---

### 🧠 암기법 상세

```
🧠 S-W-A-R = "SPA → Web APP → API Gateway → RESTful"
  
  S : SPA (Single Page Application)
  W : Web APP (웹 애플리케이션)
  A : API Gateway (API 게이트웨이)
  R : RESTful API (REST 기반 API)

🧠 B-C-F-E = "BaaS → Container → FaaS → Event Router"
  
  B : BaaS (Backend as a Service)
  C : Container (Docker 컨테이너)
  F : FaaS (Function as a Service)
  E : Event Router (이벤트 라우터)
```

### 📊 FaaS vs BaaS 비교

| 구분 | FaaS | BaaS |
|:----|:----|:----|
| **정의** | 함수 단위 실행 서비스 | 백엔드 기능 제공 서비스 |
| **예시** | AWS Lambda, Azure Functions | Firebase, Auth0 |
| **개발자 역할** | 함수 로직 작성 | API 호출만 |
| **확장성** | 자동 확장 | 자동 확장 |
| **과금** | 실행 횟수/시간 | 사용량 기반 |
| **적용** | 이벤트 처리, 배치 작업 | 인증, DB, 스토리지 |

---

### 📊 서버리스 vs 컨테이너 비교

| 구분 | 서버리스 | 컨테이너 (K8s) |
|:----|:--------|:--------------|
| **관리 수준** | 인프라 관리 불필요 | 오케스트레이션 필요 |
| **확장** | 자동 확장 | 설정 필요 (HPA) |
| **실행 시간** | 짧은 실행 (15분 제한) | 제한 없음 |
| **Cold Start** | 있음 (1~2초) | 없음 |
| **비용** | 종량제 | 상시 비용 발생 |
| **적합 워크로드** | 이벤트 기반, 간헐적 | 지속적, 복잡한 처리 |

---

### ⭐ 차별점 키워드

### 🏆 Cold Start Optimization (콜드 스타트 최적화)
> 서버리스 함수가 **유휴 상태에서 처음 호출될 때 발생하는 지연 시간**을 최소화하는 기술
> - **Provisioned Concurrency**: 미리 인스턴스 예열
> - **SnapStart**: 스냅샷 기반 빠른 시작 (AWS Lambda)
> - **Keep-alive 호출**: 주기적 호출로 유휴 방지
