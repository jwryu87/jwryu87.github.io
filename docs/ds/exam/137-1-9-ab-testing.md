## 🔑 핵심 키워드 3개

| 키워드 | 설명 |
|:------|:-----|
| **두 가지 버전 비교** | A안과 B안 중 뭐가 더 좋은지 실험으로 확인 |
| **정량적 평가** | "느낌"이 아니라 숫자(클릭률, 전환율)로 판단 |
| **무작위 분배** | 공정한 비교를 위해 사용자를 랜덤으로 나눔 |

## 📖 등장배경

```
🛒 쇼핑몰에서 "구매하기" 버튼 색상을 바꾸고 싶음

💡 문제: "파란색 vs 빨간색, 뭐가 더 많이 클릭될까?"
   → 팀장: "파란색이 좋아 보여" (주관적 의견)
   → 디자이너: "빨간색이 눈에 띄어" (주관적 의견)
   → 누구 말이 맞지? 🤔

✅ 해결: A/B 테스트!
   → 사용자 절반에게 파란색(A), 절반에게 빨간색(B) 보여줌
   → 클릭률 측정: 파란색 17% vs 빨간색 24%
   → 데이터로 증명! 빨간색 버튼 채택! 🎉
```

## 📝 개념 정의

> **A/B 테스팅**이란?
> 
> 🔬 **두 가지 버전(A와 B)**을 만들어서  
> 👥 **실제 사용자에게 랜덤으로 보여주고**  
> 📊 **어떤 게 더 효과적인지 숫자로 비교**하는 실험 방법

### 쉬운 비유

| 일상 상황 | A/B 테스트 |
|:---------|:----------|
| 🍕 피자 맛 테스트: A매장 vs B매장 | 웹사이트 A버전 vs B버전 |
| 친구 10명에게 먹여보고 투표 | 사용자 1000명에게 보여주고 클릭률 비교 |
| "A가 7:3으로 더 맛있어!" | "B가 전환율 24%로 더 높아!" |

---

## 🏗️ 기술요소 (2가지 그룹)

### 그룹 1: 실험 구성 (어떻게 나누나?)

| 구성요소 | 설명 | 예시 |
|:--------|:----|:----|
| **A 그룹 (Control)** | 기존 버전, 변경 없음 | 현재 파란색 버튼 |
| **B 그룹 (Variant)** | 새로운 버전, 변경됨 | 새로운 빨간색 버튼 |
| **무작위 분배** | 사용자를 랜덤으로 A/B에 배정 | 50:50 분배 |

### 그룹 2: 측정 지표 (뭘 비교하나?)

| 지표 | 영어 | 의미 | 계산법 |
|:----|:----|:----|:------|
| **클릭률** | CTR | 얼마나 클릭했나 | 클릭 수 ÷ 노출 수 × 100 |
| **전환율** | CVR | 목표 행동(구매, 가입) 완료율 | 전환 수 ÷ 방문자 수 × 100 |
| **체류 시간** | Duration | 얼마나 오래 머물렀나 | 평균 방문 시간 |

---

## ⭐ 차별점 키워드 (답안지 가산점!)

{: .highlight }
> **p-value (유의확률)**
> 
> "이 결과가 우연히 나왔을 확률"
> 
> - p-value < 0.05 → **통계적으로 유의미!** (B가 진짜 더 좋음)
> - p-value > 0.05 → 우연일 수 있음 (더 실험 필요)
> - 단순히 "B가 높네~"가 아니라 **과학적으로 검증**

## 🏗️ 구성요소 (2그룹 × 4항목) `C-V-R-S` `C-C-A-R`

### 그룹 1: 실험 구성 (Experiment Design) `C-V-R-S`

| 구성요소 | 영문 | 설명 | 특징 |
|:--------|:----|:----|:-----|
| **Control Group** | A Group | 기존 버전 (현재 디자인/기능) | 변경 없음, 기준선 |
| **Variant Group** | B Group | 실험 대상 버전 (새 디자인/기능) | 단일 변수 변경 |
| **Random Assignment** | - | 사용자 무작위 분배 | 편향 제거, 동질성 확보 |
| **Sample Size** | n | 실험 참여자 수 | 통계적 검정력 확보 |

### 그룹 2: 측정 지표 (Metrics) `C-C-A-R`

| 구성요소 | 영문 | 계산식 | 활용 |
|:--------|:----|:------|:----|
| **CTR** | Click-Through Rate | Clicks / Impressions × 100 | 광고, 버튼 효과 |
| **CVR** | Conversion Rate | Conversions / Visitors × 100 | 구매, 가입률 |
| **ARPU** | Avg Revenue Per User | Revenue / Users | 수익성 분석 |
| **Retention** | Retention Rate | 재방문자 / 전체 사용자 × 100 | 고객 유지율 |

---

## 🧠 암기법 (Mnemonic)

{: .note }
> ### 그룹 1: **"C-V-R-S"** (Control → Variant → Random → Sample)
> 
> | 암기 | 구성요소 | 의미 |
> |:-----|:--------|:-----|
> | **C** | **C**ontrol Group | 기존 버전 (A) |
> | **V** | **V**ariant Group | 실험 버전 (B) |
> | **R** | **R**andom Assignment | 무작위 분배 |
> | **S** | **S**ample Size | 충분한 표본 수 |
>
> 💡 **"CVRS"** — Control/Variant 나누고, Random하게 Sample 모은다

{: .note }
> ### 그룹 2: **"C-C-A-R"** (Click → Conversion → ARPU → Retention)
> 
> | 암기 | 지표 | 측정 |
> |:-----|:----|:-----|
> | **C** | **C**TR | 클릭률 |
> | **C** | **C**VR | 전환율 |
> | **A** | **A**RPU | 인당 수익 |
> | **R** | **R**etention | 잔존율 |
>
> 💡 **"CCAR"** — Click부터 Retention까지 깔때기(Funnel) 순서

## 📊 A/B 테스트 수행 절차

| 절차 | 영문 | 설명 | 산출물 |
|:----|:----|:----|:------|
| **목표 설정** | Goal Setting | 개선 지표 정의 (예: CTR 5% 증가) | KPI 정의서 |
| **가설 수립** | Hypothesis | "B가 A보다 전환율이 높을 것이다" | 가설 문서 |
| **실험 설계** | Design | A/B 버전 구성, 기간/대상 수 설정 | 실험 설계서 |
| **실험 수행** | Execution | 무작위 분배, 데이터 수집 | 로그 데이터 |
| **통계 분석** | Analysis | p-value, 신뢰구간, 검정력 분석 | 분석 리포트 |
| **결론 도출** | Conclusion | 채택/기각/재실험 결정 | 의사결정 |

---

## ⭐ 차별점 키워드 (답안 가산점!)

{: .important }
> **MDE (Minimum Detectable Effect)**
> 
> - 실험에서 **탐지 가능한 최소 효과 크기**
> - 표본 크기 결정의 핵심 요소
> - MDE = f(α, β, σ, n)
>   - α: 유의수준 (보통 0.05)
>   - β: 검정력 (보통 0.8)
>   - σ: 분산
>   - n: 표본 크기
> - **"5% 전환율 개선을 탐지하려면 몇 명이 필요한가?"**

## 🔗 A/B 테스트 vs 다변량 테스트

```
┌─────────────────────────────────────────────────────────────┐
│                   실험 설계 방법론                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌─────────────────┐              ┌─────────────────┐       │
│  │   A/B Test      │              │ Multivariate    │       │
│  │   (단일 변수)    │              │ Test (다변수)    │       │
│  └────────┬────────┘              └────────┬────────┘       │
│           │                                │                │
│           ▼                                ▼                │
│   • 버튼 색상만 변경          • 버튼 색상 + 문구 + 위치       │
│   • A vs B (2개)              • 2×3×2 = 12개 조합           │
│   • 빠른 결과                  • 상호작용 효과 분석           │
│   • 적은 트래픽 필요           • 많은 트래픽 필요             │
│                                                             │
│           │                                │                │
│           ▼                                ▼                │
│  ┌─────────────────────────────────────────────────┐        │
│  │              A/B/n Test (다중 변형)              │        │
│  │         A vs B vs C vs D ... (여러 변형)         │        │
│  └─────────────────────────────────────────────────┘        │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```
