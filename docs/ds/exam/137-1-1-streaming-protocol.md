## 📝 개념 정의

> **멀티미디어 스트리밍 프로토콜**이란?
> 
> 🎥 영상·음악 데이터를 **실시간으로 순차 전송**하여  
> 📱 **다운로드 완료 전에도 재생 가능**하게 하는  
> 📡 통신 규약(약속)

### 쉬운 비유

| 기존 방식 (다운로드) | 스트리밍 방식 |
|:------------------|:------------|
| 🍕 피자 전체가 올 때까지 대기 | 🍕 피자 한 조각씩 먹으면서 다음 조각 기다림 |
| 파일 100% → 재생 시작 | 버퍼 10% → 재생 시작! |

---

## 🏗️ 기술요소 (2가지 그룹)

### 그룹 1: 실시간 전송 프로토콜 (전통적 방식)

| 프로토콜 | 역할 | 쉬운 설명 |
|:--------|:----|:---------|
| **RTP** | 오디오·비디오 데이터 전송 | 📦 실제 영상 데이터 배달부 |
| **RTCP** | 전송 품질 모니터링 | 📊 "잘 받고 있어요" 확인 담당 |
| **RTSP** | 재생·일시정지·중지 제어 | 🎮 리모컨 역할 |

**활용**: 화상회의(Zoom), VoIP, CCTV, 실시간 방송

### 그룹 2: HTTP 기반 적응형 스트리밍 (현대적 방식)

| 프로토콜 | 특징 | 사용처 |
|:--------|:----|:------|
| **HLS** | 애플이 만듦, .m3u8 파일 사용 | 유튜브, 애플TV, 아프리카TV |
| **MPEG-DASH** | 국제 표준, 오픈 표준 | 넷플릭스, 구글 무비 |
| **Smooth Streaming** | 마이크로소프트 방식 | Xbox Video (현재는 거의 안 씀) |

**활용**: 유튜브, 넷플릭스, 왓챠 등 OTT 서비스

---

## ⭐ 차별점 키워드 (답안지 가산점!)

{: .highlight }
> **ABR (Adaptive Bitrate Streaming)**
> 
> 네트워크 상태를 실시간 감지하여 **화질을 자동으로 조절**하는 기술
> 
> - WiFi 연결 → 1080p/4K 고화질
> - 지하철 이동 중 → 480p 저화질로 자동 전환
> - 끊김 없이 시청 경험 유지!

---

## 🔗 RTP vs HLS 관계

```
┌─────────────────────────────────────────────────────┐
│                  스트리밍 프로토콜                    │
├──────────────────────┬──────────────────────────────┤
│   실시간 전송 방식    │      HTTP 적응형 방식         │
│   (RTP/RTSP 계열)    │    (HLS/DASH 계열)           │
├──────────────────────┼──────────────────────────────┤
│ ✓ 초저지연 (ms 단위)  │ ✓ 지연 있음 (10~30초)       │
│ ✓ 양방향 통신        │ ✓ 단방향 (서버→클라이언트)   │
│ ✓ 전용 포트 필요     │ ✓ HTTP 80/443 사용          │
│ ✓ 방화벽 통과 어려움  │ ✓ 방화벽 통과 쉬움          │
├──────────────────────┼──────────────────────────────┤
│ 화상회의, 게임, CCTV  │ 유튜브, 넷플릭스, OTT       │
└──────────────────────┴──────────────────────────────┘
```

## 🔑 핵심 키워드 3개

| 키워드 | 기술 의미 |
|:------|:---------|
| **ABR (Adaptive Bitrate)** | QoE 최적화를 위한 적응형 비트레이트 스트리밍 |
| **Segment-based Delivery** | 청크 단위 분할 전송, CDN 캐싱 최적화 |
| **Low Latency Streaming** | LL-HLS, LL-DASH 기반 초저지연 스트리밍 |

| **저지연·고품질 요구** | 실시간 스트리밍, 라이브 커머스 등 QoE 향상 요구 증가 |

## 🏗️ 구성요소 (2그룹 × 4항목)

### 그룹 1: 실시간 전송 & 제어 프로토콜

| 구성요소 | 계층 | 역할 | 특징 |
|:--------|:----|:----|:-----|
| **RTP** | Transport | 오디오·비디오 페이로드 전송 | UDP 기반, 타임스탬프·시퀀스번호 |
| **RTCP** | Transport | QoS 피드백, 동기화 | SR/RR 리포트, 지연·손실 모니터링 |
| **RTSP** | Application | 세션 제어 (PLAY/PAUSE/TEARDOWN) | TCP 기반, SDP 활용 |
| **SDP** | Session | 미디어 속성 기술 | 코덱, 비트레이트, 포맷 정보 |

### 그룹 2: HTTP 기반 적응형 스트리밍 프로토콜

| 구성요소 | 개발사 | 특징 | 확장자 |
|:--------|:------|:-----|:------|
| **HLS** | Apple | iOS/macOS 네이티브, TS 세그먼트 | .m3u8, .ts |
| **MPEG-DASH** | ISO/IEC | 국제표준(ISO/IEC 23009), 코덱 무관 | .mpd, .m4s |
| **CMAF** | Apple+MS | HLS/DASH 통합 포맷, fMP4 기반 | .cmfv, .cmfa |
| **LL-HLS/LL-DASH** | 각사 | 저지연 확장, Partial Segment 전송 | 동일 |

---

## 🧠 암기법 (Mnemonic)

{: .note }
> ### 그룹 1: **"전품제속"** (전송-품질-제어-속성)
> 
> | 암기 | 프로토콜 | 역할 |
> |:-----|:--------|:----|
> | **전**송 | RTP | 실제 데이터 전송 |
> | **품**질 | RTCP | 품질 모니터링 |
> | **제**어 | RTSP | 재생/정지 제어 |
> | **속**성 | SDP | 미디어 속성 기술 |
>
> 💡 **"영상 전송할 때 품질 제어하며 속성 맞춘다"**

{: .note }
> ### 그룹 2: **"애국통저"** (애플-국제-통합-저지연)
> 
> | 암기 | 프로토콜 | 의미 |
> |:-----|:--------|:----|
> | **애**(플) | HLS | Apple이 만든 표준 |
> | **국**(제) | MPEG-DASH | ISO 국제 표준 |
> | **통**(합) | CMAF | HLS+DASH 통합 포맷 |
> | **저**(지연) | LL-HLS/LL-DASH | Low Latency 확장 |
>
> 💡 **"애플이 시작해서 국제표준 되고, 통합되어 저지연까지 진화"**

## 📊 상세 비교표

| 구분 | RTP/RTSP | HLS | MPEG-DASH |
|:----|:---------|:----|:----------|
| **전송 프로토콜** | UDP/TCP | HTTP | HTTP |
| **지연시간** | ~100ms | 10~30초 | 2~10초 |
| **방화벽 통과** | 어려움 | 용이 | 용이 |
| **CDN 호환** | 제한적 | 완벽 | 완벽 |
| **DRM 지원** | 제한적 | FairPlay | Widevine, PlayReady |
| **표준화** | IETF RFC | Apple 독자 | ISO/IEC 23009 |
| **주요 활용** | 화상회의, CCTV | iOS, Safari | OTT, 범용 플레이어 |

---

## ⭐ 차별점 키워드 (답안 가산점!)

{: .important }
> **CMAF (Common Media Application Format)**
> 
> - Apple + Microsoft 공동 개발
> - HLS와 DASH의 **통합 미디어 포맷**
> - fMP4(fragmented MP4) 기반 단일 인코딩
> - **비용 절감**: 하나의 인코딩으로 HLS/DASH 동시 지원
> - **저지연 지원**: Chunked Transfer Encoding 활용

## 🔮 향후 전망

### 기술적 전망

| 전망 | 설명 |
|:----|:-----|
| **HTTP 기반 확대** | RTSP/RTP 대비 CDN 친화적인 HLS/DASH 주류 유지 |
| **저지연 기술 발전** | LL-HLS, LL-DASH, CMAF 기반 저지연 확산 |
| **AI/ML 활용** | 네트워크 상태 예측, QoE 최적화 ABR 알고리즘 고도화 |

### 서비스/산업 전망

| 전망 | 설명 |
|:----|:-----|
| **OTT 경쟁 심화** | MPEG-DASH 중심 표준화 지속 |
| **실시간 스트리밍 증가** | e스포츠, 라이브 커머스 등 초저지연 프로토콜 수요 |
| **5G/6G 융합** | VR/AR, 메타버스용 최적화 프로토콜 수요 증가 |

### 표준화 전망

| 전망 | 설명 |
|:----|:-----|
| **MPEG-DASH 강화** | 유일한 국제표준 HTTP 적응형 스트리밍 (CMAF, LL-DASH 확장) |
| **HLS 병행** | Apple 생태계(iOS/macOS)에서 HLS 유지, MPEG-DASH 호환성 확대 |
| **폐쇄형 축소** | Smooth Streaming, HDS 등 폐쇄형 프로토콜 퇴출 |
