3. **Best-of-Breed**: 각 클라우드의 최고 서비스만 조합

# 가. 멀티클라우드의 개념 및 필요성 `단벤페서`

## 1. 개념 (22자)
> **둘 이상의 클라우드를 연계하여 상호 운용하는 구축 모델**

## 2. 상세 개념

| 구분 | 설명 |
|:----|:-----|
| **정의** | 둘 이상의 동일 퍼블릭 또는 프라이빗 클라우드 공급업체의 서비스 및 컴퓨팅, 스토리지 서비스를 사용하여 클라우드 간 연계와 상호 운용, 탄력성을 보장하는 클라우드 구축모델 |
| **특징** | 다양한 요구사항을 반영하여 기업에 적절한 환경을 구성해야 함 |

## 3. 필요성 `단벤페서`

### 사용자 측면

| 필요성 | 설명 |
|:------|:-----|
| **단일 클라우드의 한계** | 하나의 클라우드로 모든 기능을 구현할 수 없음 |
| **벤더 종속성(Lock-in) 탈피** | 특정 클라우드 벤더에 종속되지 않고, 각 서비스의 특성을 고려한 최고의 조합으로 인프라 구성 |
| **Fail-over** | 높은 확장성을 기반 데이터, 워크플로우, 시스템을 백업 가능 |
| **서비스 탄력성 확보** | 서비스의 니즈에 가장 적합한 컴퓨팅 자원을 선정, 활용함으로써 서비스 품질의 최적화 필요 |
| 서비스 근접성 | 고객의 지역 근처에 있는 클라우드 제공 업체를 통해 워크로드를 호스팅 (해외 진출서비스) |
| Shadow IT 대응 | IT팀에서 배포하는 HW, SW와 연계하여 멀티클라우드 구축 |
| 조직 다양한 특성 고려 | MSA, Polyglot, Cross Platform, NoSQL등 서비스, 조직 특성에 따라 멀티클라우드 인프라 환경 구성 |

### 공급자 측면

| 필요성 | 설명 |
|:------|:-----|
| **국내 CSP 기회** | 글로벌(AWS, Azure)의 국내 진출로 국내사업자 입지가 좁아짐 |
| **그룹사 IT자회사의 기회** | 그룹 계열사의 IT서비스가 외부의 클라우드 서비스로 전환시 그룹사 IT자회사의 역할이 모호 해짐 |
| **통신&IDC사업자의 기회** | 기존 통신 & IDC업자는 글로벌 클라우드 사업자와 제휴를 통해 클라우드 허브로서 역할 기대 |
| **Cloud MSP의 기회** | MSP는 복수의 클라우드의 매니지드 서비스의 새로운 기회 |

## 4. 암기법 상세

```
🧠 단벤페서 = "단일 한계 → 벤더 탈피 → Fail-over → 서비스 탄력성"

  단 : 단일 클라우드의 한계
  벤 : 벤더 종속성(Lock-in) 탈피
  페 : Fail-over (장애 대응)
  서 : 서비스 탄력성 확보
```

| 주요 기능 | 설명 |
|:---------|:-----|

| 인프라 연동 **드**라이버 | 개별 클라우드의 API로 개발된 단일 API를 준수 |

|:---------|:-----|
| **에**이전트(Agent) | 표준화된 대규모 모니터링 데이터 수집/처리를 위한 모니터링 구조 및 방식 |
| **푸**쉬(PUSH) | 안정적인 모니터링 위해 에이전트가 정해진 주기에 따라 푸쉬(PUSH) 방식으로 모니터링 서버에 전송 |
| **풀**(PULL) | 상태 모니터링 및 실시간 성능 모니터링을 풀(PULL) 방식의 모니터링 데이터 수집 |
| **서**비스간 모니터링 | 지역적으로 분산된 복수의 서비스간의 관계(연결성, 보안성, 부하분산 등)에 대한 모니터링 정보 |

## 📊 멀티클라우드 vs 하이브리드 클라우드 비교

| 구분 | 멀티클라우드 | 하이브리드 클라우드 |
|:----|:-----------|:-----------------|
| **정의** | 2개 이상 퍼블릭 클라우드 조합 | 퍼블릭 + 프라이빗 클라우드 조합 |

| **비용** | CSP별 최적화 가능 | 프라이빗 인프라 비용 발생 |
| **적합 대상** | 글로벌 서비스, 대기업 | 금융, 의료 등 규제 산업 |

## ⭐ 차별점 키워드

### 🏆 Cloud Brokerage (클라우드 브로커리지)
> 멀티클라우드 환경에서 **중개자 역할**을 수행하며, 사용자의 요구사항에 맞는 **최적의 클라우드 서비스를 선정/조합/관리**하는 서비스 모델
> - **통합(Aggregation)**: 여러 CSP 서비스를 단일 창구로 제공
> - **중개(Intermediation)**: 부가가치 서비스 추가 (보안, 모니터링)
> - **차익거래(Arbitrage)**: 비용 최적화를 위한 CSP 선택
