> 영상 데이터의 **중복/불필요 정보를 제거**하여 크기를 줄이는 기술

### 쉬운 비유 🎒
> **여행 가방 싸기**와 같아요!
> - **무손실** = 옷을 잘 접어서 넣기 (꺼내면 원래 그대로)
> - **손실** = 안 입을 옷은 빼고 가기 (조금 불편해도 가방이 가벼움)
> - **혼합** = 안 입을 옷 빼고 + 나머지는 잘 접기 (최적!)

### 왜 필요한가? (3줄 요약)
1. **공간 절감**: 64KB → 16KB로 저장 공간 1/4
2. **속도 개선**: 전송 속도 향상 (6MB → 1.5MB)
3. **대역폭 확대**: HDTV 1.4GB → 30MB (약 47배 절감)

## I. 영상압축기법 개요 `개요는 1단락!`

### 1. 개념 (22자)
> **중복·불필요 정보 제거로 데이터 크기를 축소하는 기술**

### 2. 필요성

| 필요이유 | 설명 | 압축 효과 |
|:--------|:-----|:---------|
| **공간 절감** | 저장 공간 점유율 절약 | 64KB → 16KB |
| **속도 개선** | 전송 속도 향상 | 6MB → 1.5MB |
| **대역폭 확대** | 단위시간당 전송량 증가 | 1.4GB → 30MB (HDTV) |

### 3. 압축기법 분류 개념도

```
┌─────────────────────────────────────────────────────────────────┐
│                     영상압축기법                                  │
├─────────────────┬─────────────────┬─────────────────────────────┤

│  (Lossless)     │   (Lossy)       │        (Hybrid)            │
├─────────────────┼─────────────────┼─────────────────────────────┤
│ • 원본 100% 복원 │ • 일부 정보 제거  │ • 손실 + 무손실 결합        │
│ • 낮은 압축률    │ • 높은 압축률     │ • 최적 효율               │
├─────────────────┼─────────────────┼─────────────────────────────┤
│ RLE, Huffman    │ DCT, Wavelet    │ JPEG, MPEG, H.264         │
│ Arithmetic, LZW │ Inter-frame,VQ  │ JPEG2000, H.265           │
└─────────────────┴─────────────────┴─────────────────────────────┘
```

# 가. 무손실 압축기법의 특징과 종류 `R-H-A-L`

## 1. 무손실 압축기법의 개념
> **압축 후 복원 시 원본 데이터와 100% 동일**하게 복원되는 압축 방식

## 2. 특징

| 특징 | 설명 |
|:----|:-----|
| **데이터 보존성** | 압축 후 복원 시 원본 데이터와 100% 동일 |
| **적용 분야** | 의료 영상, 법적 증거 자료 등 손실 불가 영역 |
| **압축률** | 손실 압축보다 상대적으로 낮음 (2:1 ~ 8:1) |
| **알고리즘** | 통계적 중복성(Statistical Redundancy) 제거 |

## 3. 종류 `R-H-A-L`

| 기법 | 영문명 | 설명 | 활용 예시 |
|:----|:------|:-----|:---------|
| **RLE** | Run-Length Encoding | 반복 데이터를 (값, 반복횟수)로 표현 | 단순 패턴, 흑백 이미지 |
| **허프만** | Huffman Coding | 빈도 높은 심볼 → 짧은 코드 할당 | JPEG, PNG |
| **산술** | Arithmetic Coding | 데이터를 0~1 구간에 확률적 매핑 | 영상/오디오 압축 |
| **LZW** | Lempel-Ziv-Welch | 사전(Dictionary) 기반 압축 | GIF, TIFF |

## 4. 암기법 상세

```
🧠 R-H-A-L = "Run → Huffman → Arithmetic → LZW"

  R : RLE (Run-Length Encoding) - 반복 압축
  H : Huffman (허프만 부호화) - 빈도 기반
  A : Arithmetic (산술 부호화) - 확률 매핑
  L : LZW (사전 기반) - 패턴 사전화
```

# 나. 손실 압축기법의 특징과 종류 `D-I-V-W`

## 1. 손실 압축기법의 개념
> 압축 후 복원 시 **원본과 완전히 동일하지 않지만**, 인간이 인지하지 못하는 정보를 제거하여 **높은 압축률** 달성

## 2. 특징

| 특징 | 설명 |
|:----|:-----|
| **데이터 손실** | 압축 후 복원 시 일부 정보 제거됨 |
| **인간 감각 기반** | 눈·귀가 구분 못하는 세부 정보 과감히 삭제 |
| **압축률** | 무손실보다 훨씬 높음 (10:1 ~ 100:1) |
| **비가역적** | 원본 복원 불가 (Irreversible) |

## 3. 종류 `D-I-V-W`

| 기법 | 영문명 | 설명 | 활용 예시 |
|:----|:------|:-----|:---------|
| **DCT** | Discrete Cosine Transform | 주파수 영역 변환 후 중요 정보만 저장 | JPEG 이미지 |
| **Inter-frame** | Inter-frame Coding | 연속 프레임 간 중복 제거 | MPEG, H.264, H.265 |
| **VQ** | Vector Quantization | 코드북(Codebook)으로 양자화 | 음성 압축, 영상 코덱 |
| **Wavelet** | Wavelet Transform | 다중 해상도 분석 | JPEG2000 |

## 4. 암기법 상세

```
🧠 D-I-V-W = "DCT → Inter → Vector → Wavelet"

  D : DCT (이산 코사인 변환) - 주파수 영역
  I : Inter-frame (프레임 간 예측) - 시간 중복 제거
  V : VQ (벡터 양자화) - 코드북 근사
  W : Wavelet (웨이블릿 변환) - 다해상도 분석
```

# 다. 혼합 압축기법의 특징과 종류 `J-J-M-H`

## 1. 혼합 압축기법의 개념
> **손실 압축으로 불필요 정보 제거** 후, **무손실 압축으로 중복 제거**하여 **고효율과 품질 동시 확보**

## 2. 특징

| 특징 | 설명 |
|:----|:-----|
| **혼합 방식** | 손실 압축 + 무손실 압축을 함께 사용 |
| **효율성** | 손실로 압축률↑, 무손실로 정확성 보장 |
| **균형성** | 화질/음질 유지 + 저장 공간 절감 |
| **적용 분야** | JPEG, MPEG, H.264 등 표준 포맷 |

## 3. 종류 `J-J-M-H`

| 기법 | 손실 부분 | 무손실 부분 | 활용 예시 |
|:----|:---------|:-----------|:---------|
| **JPEG** | DCT 변환, 양자화 | 허프만 부호화 | 디지털 이미지 |
| **JPEG2000** | 웨이블릿 변환 | 산술 부호화 | 고해상도 이미지 |
| **MPEG** | 프레임 간 예측, 양자화 | 허프만, VLC | 디지털 동영상 |
| **H.264/H.265** | 블록 예측, 변환, 양자화 | CABAC, CAVLC | 고화질 방송, 스트리밍 |

## 4. 혼합 압축 동작 흐름

```
┌─────────────────────────────────────────────────────────────────┐
│                    혼합 압축 파이프라인                           │
├─────────────────────────────────────────────────────────────────┤
│  원본 영상                                                      │
│     │                                                          │
│     ▼                                                          │
│  ┌──────────────────────────────────────────────────────┐      │
│  │         [손실 압축 단계]                               │      │
│  │  DCT/Wavelet 변환 → 양자화 → 프레임 간 예측            │      │
│  └──────────────────────────────────────────────────────┘      │
│     │                                                          │
│     ▼                                                          │
│  ┌──────────────────────────────────────────────────────┐      │
│  │         [무손실 압축 단계]                             │      │
│  │  Huffman/Arithmetic → 엔트로피 코딩 → 비트스트림       │      │
│  └──────────────────────────────────────────────────────┘      │
│     │                                                          │
│     ▼                                                          │
│  압축 파일 (JPEG, MPEG, H.264...)                             │
└─────────────────────────────────────────────────────────────────┘
```

## 5. 암기법 상세

```
🧠 J-J-M-H = "JPEG → JPEG2000 → MPEG → H.264"

  J : JPEG (DCT + Huffman)
  J : JPEG2000 (Wavelet + Arithmetic)  
  M : MPEG (Inter-frame + VLC)
  H : H.264/H.265 (Block 예측 + CABAC)
```

## 📊 3가지 압축기법 비교표

| 구분 | 무손실 | 손실 | 혼합 |
|:----|:------|:----|:----|
| **복원** | 100% 동일 | 일부 손실 | 최적 균형 |
| **압축률** | 2:1 ~ 8:1 | 10:1 ~ 100:1 | 20:1 ~ 50:1 |
| **용도** | 의료, 법적 증거 | 스트리밍, 방송 | 범용 멀티미디어 |
| **대표 기술** | RLE, Huffman, LZW | DCT, Wavelet | JPEG, MPEG, H.264 |
| **특징** | 정확성 중시 | 효율성 중시 | 정확성 + 효율성 |

## ⭐ 차별점 키워드

### 🏆 Perceptual Coding (지각적 부호화)
> 인간의 시각·청각 특성을 활용하여 **인지 불가능한 정보를 선택적으로 제거**하는 기법
> - HVS (Human Visual System) 기반 DCT 양자화
> - 마스킹 효과 (Masking Effect) 활용
