> 클라우드 = 남의 컴퓨터 빌려쓰기! 얼마나 빌리느냐의 차이

---

### 한 줄 정의

| 서비스 | 쉬운 설명 | 비유 |
|:------|:---------|:-----|
| **IaaS** | 컴퓨터 빌려쓰기 | 🏠 빈 땅 임대 (집은 내가 지음) |
| **PaaS** | 개발환경 빌려쓰기 | 🏗️ 반건조 주택 (인테리어만 하면 됨) |
| **SaaS** | 완제품 소프트웨어 사용 | 🏨 호텔 (그냥 들어가서 살면 됨) |
| **FaaS** | 함수만 올려서 실행 | 🍽️ 배달 주문 (필요할 때만 시킴) |

---

### 🤔 왜 필요해?

```
😰 "서버 직접 사면?"
   → 💸 비쌈, 🔧 관리 힘듦, 📈 갑자기 사용자 늘면 터짐

✅ "클라우드 쓰면?"
   → 💰 쓴 만큼만 내고 (pay-per-use)
   → 📈 필요하면 늘리고, 줄이면 줄이고 (확장성)
   → 🔧 관리는 클라우드 업체가!
```

---

### 📊 누가 뭘 관리해?

```
┌─────────────────────────────────────────────────────────────┐
│         On-Premise    IaaS      PaaS      FaaS      SaaS    │
├─────────────────────────────────────────────────────────────┤
│ Functions    [내가]    [내가]    [내가]    [내가]    [업체]   │
│ Application  [내가]    [내가]    [내가]    [업체]    [업체]   │
│ Runtime      [내가]    [내가]    [업체]    [업체]    [업체]   │
│ OS           [내가]    [내가]    [업체]    [업체]    [업체]   │
│ Virtualization [내가]  [업체]    [업체]    [업체]    [업체]   │
│ Networking   [내가]    [업체]    [업체]    [업체]    [업체]   │
│ Storage      [내가]    [업체]    [업체]    [업체]    [업체]   │

└─────────────────────────────────────────────────────────────┘
         ↑ 위로 갈수록 "내가 해야 할 것"
         ← SaaS로 갈수록 "업체가 더 많이 해줌"
```

# 가. IaaS(Infrastructure as a Service) `물확비유`
{: .fs-7 }

---

## 1. 개념 (22자)

> **Infra 자원을 가상화하여 서비스로 제공하는 모델**

### 상세 개념

| 구분 | 내용 |
|:----|:-----|
| **정의** | 서비스 제공자가 컴퓨팅 리소스를 네트워크를 통해 서비스로 제공하는 모델 |
| **제공 범위** | Hardware, Storage, Networking, Virtualization |
| **사용자 책임** | OS, Runtime, Application, Data |
| **대표 서비스** | AWS EC2, Azure VM, GCP Compute Engine |

---

## 2. 특징 `물확비유`

| 특징 | 설명 |
|:----|:-----|
| **물**리자원 가상화 | Infra를 구성하는 물리적인 자원을 가상화하여 서비스 제공 |
| **확**장성 | 추가 자원 필요시 서버를 통해 리소스 확보 가능 |
| **비**용효율화 | 고정비가 들지 않아 비용적인 측면에서 효율화 |
| **유**지관리 용이성 | 소프트웨어의 유지 및 관리가 용이 |

---

## 3. 아키텍처 `SDN-SDC-SDS`

```
┌──────────────────────────────────────────────────────────────────┐
│                      IaaS 서비스 아키텍처                          │
├──────────────────────────────────────────────────────────────────┤
│  ┌─────────────────────────────────────────────────────────────┐ │
│  │  서비스 운영체계: 워크플로우 | 클라우드관리시스템 | 사용자포털   │ │
│  │                Controller Node (Server, Storage, Network)    │ │
│  └─────────────────────────────────────────────────────────────┘ │
│                              │                                    │
│  ┌───────────────────────────┼───────────────────────────────┐   │
│  │  SDN (Software Defined Networking)                         │   │
│  │    Frontend      Backend       Fabric & Overlay            │   │
│  │    VxLAN Overlay VxLAN Overlay                             │   │
│  └───────────────────────────────────────────────────────────┘   │
│                              │                                    │
│  ┌───────────────────────────┼───────────────────────────────┐   │
│  │  SDC (Software Defined Compute)                            │   │
│  │    Bin/Lib | Guest OS | Hypervisor | Container Engine      │   │
│  │    → X86기반 컴퓨팅 (Scale out, 무중단, Live 마이그레이션)    │   │
│  └───────────────────────────────────────────────────────────┘   │
│                              │                                    │
│  ┌───────────────────────────┼───────────────────────────────┐   │
│  │  SDS (Software Defined Storage)                            │   │
│  │    SAN Storage | File Storage | Block Storage | Object     │   │
│  │    → X86기반 분산 스토리지 (필요시 확장, 유연성 확보)          │   │
│  └───────────────────────────────────────────────────────────┘   │
└──────────────────────────────────────────────────────────────────┘
```

> ### 🧠 암기법: **"물확비유"** (IaaS 특징)
>
> | 암기 | 특징 | 핵심 |
> |:-----|:----|:-----|
> | **물** | **물**리자원 가상화 | Infra 가상화 제공 |
> | **확** | **확**장성 | 리소스 확보 용이 |
> | **비** | **비**용효율화 | 고정비 無 |
> | **유** | **유**지관리 용이성 | SW 관리 용이 |

# 나. PaaS(Platform as a Service) `개다소`
{: .fs-7 }

---

## 1. 개념 (22자)

> **개발환경과 실행환경을 서비스로 제공하는 모델**

### 상세 개념

| 구분 | 내용 |
|:----|:-----|
| **정의** | 기업의 애플리케이션 실행 환경 및 개발 환경을 서비스로 제공하는 모델 |
| **제공 범위** | IaaS + OS, Middleware, Runtime |
| **사용자 책임** | Application, Data |
| **대표 서비스** | Heroku, Google App Engine, Azure App Service |

---

## 2. 특징 `개다소`

| 특징 | 설명 |
|:----|:-----|
| **개**발 및 배포 효율성 | 필요 플랫폼만 사용하여 개발 및 배포 프로세스의 빠른 확보 |
| **다**중 액세스 | 수많은 사용자가 동일 개발 응용프로그램에 접근 가능 |
| **소**프트웨어 최소설치 | 소프트웨어 설치 시 물리적 자원이 필요하지 않음 |

---

## 3. 주요 기능 `접런개격`

```
            ┌─────────────────┐
            │      접근       │
            │  - DNS         │
            │  - 부하분산     │
            └────────┬────────┘
                     │
    ┌────────────────┼────────────────┐
    │                │                │
┌───┴───┐      ┌─────┴─────┐    ┌─────┴─────┐
│ 런타임 │      │   개발     │    │   격리    │
│       │      │   도구     │    │          │
│공통 Lib│      │CI(Conti- │    │ 가상머신  │
│미들웨어│      │nuous     │    │ 컨테이너  │
│서비스  │      │Integra-  │    │ (Docker) │
│       │      │tion) 도구 │    │          │
└───────┘      └───────────┘    └───────────┘
```

| 기능 | 설명 |
|:----|:-----|
| **런타임** | 응용 실행에 필요한 공통된 라이브러리, 미들웨어 서비스 |
| **격리** | 런타임에서 실행하는 Application이 독립된 공간에서 실행 (컨테이너) |
| **접근** | 개발된 Application 외부 접근을 위해 DNS 동적 연동 기술 |
| **개발도구** | 다양한 CI(Continuous Integration)도구 등이 PaaS에 포함 |

---

{: .note }
> ### 🧠 암기법: **"개다소"** (PaaS 특징)
>
> | 암기 | 특징 | 핵심 |
> |:-----|:----|:-----|
> | **개** | **개**발 및 배포 효율성 | 빠른 배포 |
> | **다** | **다**중 액세스 | 동시 접근 |
> | **소** | **소**프트웨어 최소설치 | 물리 자원 불필요 |

# 다. SaaS(Software as a Service) `타유`
{: .fs-7 }

---

## 1. 개념 (22자)

> **완성된 소프트웨어를 서비스로 제공하는 모델**

### 상세 개념

| 구분 | 내용 |
|:----|:-----|
| **정의** | 서비스 제공자가 사용자에게 소프트웨어(애플리케이션) 서비스를 제공하는 모델 |
| **제공 범위** | IaaS + PaaS + Application |
| **사용자 책임** | Data (설정, 사용만) |
| **대표 서비스** | Gmail, Office 365, Salesforce, Dropbox |

---

## 2. 특징 `타유`

| 특징 | 설명 |
|:----|:-----|
| **타**임 투 마켓 (Time to market) | 소프트웨어의 즉시 사용 가능 |
| **유**지보수 간소화 | 클라우드 제공업체를 통한 기술적 문제 지원 |

---

# 라. FaaS(Function as a Service) `이서자`
{: .fs-7 }

---

## 1. 개념 (22자)

> **함수 단위 코드를 이벤트 기반으로 실행하는 모델**

### 상세 개념

| 구분 | 내용 |
|:----|:-----|
| **정의** | 애플리케이션을 구성하는 여러 함수(function)들을 클라우드에서 직접 제공하는 모델 |
| **제공 범위** | IaaS + PaaS + Runtime (함수 실행 환경) |
| **사용자 책임** | Function 코드 (비즈니스 로직)만 |
| **대표 서비스** | AWS Lambda, Azure Functions, Google Cloud Functions |

---

## 2. 특징 `이서자`

| 특징 | 설명 |
|:----|:-----|
| **이**벤트 기반 실행 | 애플리케이션 전체가 아니라 특정 함수(기능) 단위로 실행형 |
| **서**버리스(Serverless) 아키텍처 | 서버 관리 불필요 → 개발자는 코드(비즈니스 로직)에만 집중 |
| **자**동 확장성 | 요청 발생 시 자동으로 확장(스케일 아웃) |

---

## 3. 아키텍처

```
┌─────────────────────────────────────────────────────────────────┐
│                        FaaS 아키텍처                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│   ┌──────────┐      ┌───────────────────────────────────────┐   │
│   │          │      │       Serverless Cloud Provider        │   │
│   │  Client  │─────▶│  ┌─────────────┐                       │   │
│   │          │      │  │ API Gateway │                       │   │
│   └──────────┘      │  └──────┬──────┘                       │   │
│                     │         │                               │   │
│   ┌──────────┐      │         ▼                               │   │
│   │          │      │  ┌─────────────┐    ┌───────────────┐  │   │
│   │  Event   │─────▶│  │    FaaS     │───▶│     BaaS      │  │   │
│   │          │      │  │  Fn(X)      │    │  DB, Files    │  │   │
│   └──────────┘      │  │  Fn(Y)      │    │  Servers      │  │   │
│                     │  │  Fn(Z)      │    │  Message      │  │   │
│                     │  └─────────────┘    └───────────────┘  │   │
│                     └───────────────────────────────────────────┘   │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

| 구성요소 | 설명 |
|:--------|:-----|
| **이벤트 소스** | 함수를 트리거(trigger)하는 원인 |
| **함수(Function/코드)** | 실제 실행되는 비즈니스 로직, 특정 이벤트 발생 시 실행되는 코드 블록 |
| **API Gateway** | 외부 서비스나 클라이언트와 연결하기 위한 진입점 |

> 💡 **FaaS 흐름**: 이벤트 소스 → 함수(코드) → 런타임 실행 → 외부 리소스와 연동으로 동작

---

> |:-----|:----|:-----|

# ⭐ 차별점 키워드
{: .fs-7 }

---

{: .highlight }
> ### 구축 시 고려사항 (SLA 기반)
>
> | 구분 | 고려항목 | 설명 |
> |:----|:--------|:-----|
> | **성능** | 가용률 | 정해진 서비스 운영시간 대비 클라우드 컴퓨팅 서비스에 접속이 가능한 시간의 율 |
> | | 응답시간 | 이용자의 조회 또는 요구 시점부터 처리가 완료될 때까지 걸리는 시간 |
> | **신뢰성** | 서비스 회복 시간 | 서비스 중단시점부터 정상 상태로 회복까지 소요시간 |
> | | 백업 주기 | 정기적으로 수행하는 데이터 백업 주기 |
> | **고객 대응** | 고객 대응 체계 | 고객 의견을 수렴하기 위한 다양한 방법 제공 및 운영 여부 |
> | | 고객 불만 처리 체계 | 고객 불만을 신속하고 정확하게 수용 및 처리하는 능력 |
>
> 💡 **사전 클라우드 SLA를 수립 후 서비스 구축 진행!**

# 📊 IaaS vs PaaS vs SaaS vs FaaS 비교
{: .fs-7 }

---

| 구분 | IaaS | PaaS | SaaS | FaaS |
|:----|:-----|:-----|:-----|:-----|
| **제공 범위** | Infra | Infra + Platform | 완성 SW | 함수 실행환경 |
| **사용자 책임** | OS~App | App, Data | Data만 | 함수 코드만 |
| **자유도** | 높음 | 중간 | 낮음 | 매우 낮음 |
| **관리 부담** | 높음 | 중간 | 낮음 | 매우 낮음 |
| **과금 방식** | 시간/용량 | 시간/용량 | 구독 | 실행 횟수 |
| **대표 서비스** | EC2 | Heroku | Gmail | Lambda |
| **암기법** | `물확비유` | `개다소` | `타유` | `이서자` |
