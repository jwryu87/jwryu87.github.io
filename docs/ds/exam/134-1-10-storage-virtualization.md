```
   → 사용자는 복잡한 구조 몰라도 OK
```

## 📝 개념 정의

> **스토리지 가상화**란?
> 
> 💿 여러 개의 **물리적 저장장치**를  
> 🔗 하나의 **논리적 저장장치**처럼 묶어서  
> 📦 사용자가 편하게 쓸 수 있게 만드는 기술

### 쉬운 비유

| 가상화 전 | 가상화 후 |
|:---------|:---------|
| 🗄️ 서랍장 3개를 각각 관리 | 🗄️ 서랍장 3개를 하나의 큰 옷장처럼 사용 |
| 어디에 뭐 넣었는지 일일이 기억 | 그냥 옷장에 넣으면 알아서 정리 |

## 🏗️ 기술요소 (3가지 유형)

### 유형 1: 블록 스토리지 🧱

| 항목 | 내용 |
|:----|:-----|
| **뭔가요?** | 데이터를 일정 크기 블록으로 쪼개서 저장 |
| **어디서 쓰나요?** | 데이터베이스, 가상머신 (빠른 속도 필요할 때) |
| **비유** | 🧱 레고 블록처럼 조각조각 저장 |
| **프로토콜** | FC, iSCSI |

### 유형 2: 파일 스토리지 📁

| 항목 | 내용 |
|:----|:-----|
| **뭔가요?** | 우리가 아는 폴더/파일 구조로 저장 |
| **어디서 쓰나요?** | 회사 공유 폴더, NAS (협업할 때) |
| **비유** | 📁 컴퓨터 탐색기처럼 폴더/파일로 정리 |
| **프로토콜** | SMB, CIFS, NFS |

### 유형 3: 오브젝트 스토리지 📦

| 항목 | 내용 |
|:----|:-----|
| **뭔가요?** | 파일마다 고유 ID를 붙여서 저장 |
| **어디서 쓰나요?** | 유튜브 영상, 인스타 사진 (대용량 저장) |
| **비유** | 📦 택배 송장번호처럼 ID로 찾음 |
| **프로토콜** | HTTP/HTTPS (REST API) |

## ⭐ 차별점 키워드 (답안지 가산점!)

{: .highlight }
> **SDS (Software-Defined Storage)**
> 
> 소프트웨어로 스토리지를 제어하는 기술
> 
> - 하드웨어에 종속되지 않음
> - 어떤 스토리지든 소프트웨어로 통합 관리
> - 클라우드 환경에서 필수 기술!

## 🏗️ 구성요소 (3유형) `블파오` `F-S-H`

### 유형 1: 블록 스토리지 (Block Storage) `고블공S효`

| 특징 | 설명 |
|:----|:-----|
| **고유주소** | 각 블록은 저장된 위치에 대한 고유 주소를 가짐 |
| **블록단위** | 데이터를 고정된 크기의 블록 단위로 나누어 저장 |
| **공유주소** | 서버에서 파일 요청 시 블록을 재구성하여 하나의 데이터로 전달 |
| **SAN 환경** | SAN(Storage Area Network)이나 가상 머신의 디스크로 사용 |
| **효율적 저장** | 데이터 블록은 운영체제와 무관하게 효율적으로 저장 가능 |

| 항목 | 내용 |
|:----|:-----|
| **프로토콜** | FC (Fibre Channel), iSCSI |
| **송수신 단위** | 블록 단위 |
| **용도** | 데이터베이스, 가상 머신, 트랜잭션 처리 |
| **장점** | 낮은 레이턴시, 빠른 데이터 전송, 오버헤드 적음 |

### 유형 2: 파일 스토리지 (File Storage) `파네디접쉬`

| 특징 | 설명 |
|:----|:-----|
| **파일시스템** | 데이터를 파일 단위로 저장하며 파일 시스템을 필요로 함 |
| **네트워크 공유** | 네트워크를 통해 파일을 쉽게 공유할 수 있음 |
| **디렉터리 구조** | 사용자가 친숙한 디렉터리와 파일 구조를 제공 |
| **접근 제어** | 파일 및 폴더에 대한 접근 권한을 세분화하여 관리 가능 |
| **쉬운 협업** | 여러 사용자가 쉽게 파일에 접근하고 협업할 수 있음 |

| 항목 | 내용 |
|:----|:-----|
| **프로토콜** | SMB, CIFS, NFS |
| **송수신 단위** | 파일 단위 |
| **용도** | 파일 서버, NAS, 협업 환경 |
| **장점** | 접근 제어 편리, 속성 정보 관리 편리, 사용자 친화적 |

### 유형 3: 오브젝트 스토리지 (Object Storage) `객메R무비`

| 특징 | 설명 |
|:----|:-----|
| **객체ID** | 각 객체는 고유한 객체 ID(URI)를 가지며, 이를 통해 데이터에 접근 |
| **메타데이터** | 객체와 함께 저장되는 메타데이터를 통해 데이터 검색과 관리가 용이 |
| **RESTful API** | 주로 RESTful API를 통해 데이터에 접속 |
| **무한 확장성** | 스토리지 용량을 무한정 확장할 수 있는 구조 |
| **비용 효율성** | 대용량 데이터 저장에 적합하며, 비용 효율적임 |

| 항목 | 내용 |
|:----|:-----|
| **프로토콜** | HTTP/HTTPS (RESTful API) |
| **송수신 단위** | 오브젝트 단위 |
| **용도** | 비정형 대용량 데이터, 미디어 파일, 백업/아카이브 |
| **장점** | OS 파일 시스템 의존성 없음, 무한 확장, 비용 효율적 |

## 🧠 암기법 (Mnemonic)

{: .note }
> ### 3가지 유형: **`블파오`** (블록-파일-오브젝트)
> 
> | 암기 | 유형 | 프로토콜 | 용도 |
> |:-----|:-----|:--------|:----|
> | **블** | 블록 스토리지 | FC, iSCSI | DB, VM |
> | **파** | 파일 스토리지 | SMB, NFS | 파일서버 |
> | **오** | 오브젝트 스토리지 | HTTP REST | 대용량 저장 |
>
> 💡 **"블파오"** — "블록으로 파일을 오브젝트처럼" 진화 순서!

{: .note }
> ### 프로토콜: **`F-S-H`** (FC→SMB→HTTP)
> 
> | 암기 | 프로토콜 | 유형 |
> |:-----|:--------|:----|
> | **F** | FC, iSCSI | 블록 |
> | **S** | SMB, CIFS, NFS | 파일 |
> | **H** | HTTP/HTTPS | 오브젝트 |
>
> 💡 **"F-S-H"** — 속도 순서대로 "Fast → Shared → HTTP"

{: .note }
> ### 블록 스토리지 특징: **`고블공S효`**
> 
> 고유주소 → 블록단위 → 공유주소 → SAN → 효율적

{: .note }
> ### 파일 스토리지 특징: **`파네디접쉬`**
> 
> 파일시스템 → 네트워크공유 → 디렉터리 → 접근제어 → 쉬운협업

{: .note }
> ### 오브젝트 스토리지 특징: **`객메R무비`**
> 
> 객체ID → 메타데이터 → RESTful API → 무한확장 → 비용효율

## 📊 상세 비교표

| 구분 | 블록 스토리지 | 파일 스토리지 | 오브젝트 스토리지 |
|:----|:------------|:------------|:----------------|
| **프로토콜** | FC, iSCSI | SMB, CIFS, NFS | HTTP/HTTPS |
| **송수신 단위** | 블록 단위 | 파일 단위 | 오브젝트 단위 |
| **데이터 구조** | 블록 주소 | 계층적 디렉터리 | Flat 구조 + 메타데이터 |
| **특징** | 오버헤드 적음<br>빠른 전송 | 접근제어 편리<br>속성관리 편리 | 고유 ID(URI) 부여<br>OS 의존성 없음 |
| **레이턴시** | 낮음 (Low) | 중간 (Medium) | 높음 (High) |
| **확장성** | 제한적 | 중간 | 무한 확장 |
| **용도** | DB, VM<br>트랜잭션 | 파일서버<br>협업 환경 | 대용량 미디어<br>백업/아카이브 |
| **대표 서비스** | AWS EBS | AWS EFS, NFS | AWS S3, Azure Blob |

## ⭐ 차별점 키워드 (답안 가산점!)

{: .important }
> **SDS (Software-Defined Storage)**
> 
> - 스토리지 하드웨어와 소프트웨어를 분리
> - **소프트웨어 계층**으로 이기종 스토리지 통합 관리
> - 벤더 종속성 탈피, 유연한 확장성
> - **클라우드 네이티브 환경**의 핵심 기술
> - CSI (Container Storage Interface) 표준 활용

```
    │       ├── 네트워크 가상화 (SDN)
    │       │
    │       └── 🎯 스토리지 가상화 (SDS)
    │               ├── 블록 스토리지
    │               │   ├── SAN (Storage Area Network)
    │               │   └── FC, iSCSI 프로토콜
    │               ├── 파일 스토리지
    │               │   ├── NAS (Network Attached Storage)
    │               │   └── SMB, NFS 프로토콜
    │               └── 오브젝트 스토리지
    │                   ├── HTTP REST API
    │                   └── AWS S3, Azure Blob
    │
    └── 클라우드 컴퓨팅
            ├── IaaS (Infrastructure as a Service)
            └── 클라우드 스토리지 서비스
```
