---

### 📖 등장배경

> **문제**: 인증할 때 개인정보가 노출돼요...
> - 🔐 비밀번호를 서버에 보내면 해킹 위험
> - 🎂 성인인증 하려면 생년월일 공개?
> - 💰 블록체인 거래 내역이 다 보여...
>
> **해결**: 영지식 증명!
> - 🤫 비밀을 말하지 않고도 "나 이거 알아"를 증명
> - ✅ 검증자는 사실 여부만 알고, 내용은 모름
> - 🔒 프라이버시 완벽 보호!

---

### 📝 개념 정의

> 🔐 **동굴의 비밀 통로**에 비유하면...
>
> | 상황 | 설명 |
> |:----|:----|
> | **비밀** | 동굴 안에 비밀 통로가 있음 (비밀번호 알아야 통과) |
> | **증명자** | "나 비밀번호 알아!" (통로로 원하는 출구로 나옴) |
> | **검증자** | "정말? 그럼 A로 나와봐" (요청한 출구로 나오는지 확인) |
> | **결과** | 검증자는 "이 사람이 알긴 아는구나!" 확신하지만, 비밀번호 자체는 모름 |
>
> **핵심**: 비밀을 **공개하지 않고도** 알고 있음을 **증명**!

---

### 🏗️ 기술요소 (2그룹)

#### 그룹 1: 3대 속성

| 속성 | 의미 |
|:----|:----|
| **완전성** | 참이면 검증자가 납득할 수 있음 |
| **건전성** | 거짓이면 속일 수 없음 |
| **영지식성** | 비밀 정보는 전혀 노출되지 않음 |

#### 그룹 2: 종류

| 종류 | 특징 |
|:----|:----|
| **대화형 (Interactive)** | 증명자-검증자 여러 번 대화 |
| **비대화형 (Non-Interactive)** | 한 번에 증명 (zk-SNARK 등) |

---

### ⭐ 차별점 키워드

**zk-SNARK** 🚀
> **Z**ero-knowledge **S**uccinct **N**on-interactive **AR**gument of **K**nowledge
> - **간결함**: 증명이 매우 짧음
> - **비대화형**: 한 번에 증명 완료
> - 블록체인에서 가장 많이 사용

| 구분 | 내용 |
|:----|:----|
| **누가 (Who)** | 증명자(Prover), 검증자(Verifier) |
| **무엇을 (What)** | 비밀을 공개하지 않고 사실을 증명 |
| **언제 (When)** | 프라이버시 보호가 필요한 인증/검증 시 |
| **어디서 (Where)** | 블록체인, 인증 시스템, 전자투표 |
| **왜 (Why)** | 개인정보 보호, 프라이버시 강화 |
| **어떻게 (How)** | 완전성, 건전성, 영지식성 만족 |

## 🎯 기술사 수준 설명

### 🔑 핵심 키워드 3개

| 키워드 | 설명 |
|:------|:-----|
| **Completeness** | 완전성 - 참인 명제는 검증자가 납득 가능 |
| **Soundness** | 건전성 - 거짓 명제로는 검증자를 속일 수 없음 |
| **Zero-Knowledge** | 영지식성 - 명제의 참/거짓 외 정보 누출 없음 |

---

### 📖 등장배경 (2개)

| 배경 | 설명 |
|:----|:----|
| **프라이버시 요구** | 개인정보 노출 없이 인증/검증 필요성 증가 |
| **블록체인 확장** | 트랜잭션 프라이버시와 확장성 문제 해결 필요 |

---

### 📝 개념 정의 (22자)

> **증명자가 검증자에게 비밀 정보를 노출하지 않고 해당 정보의 유효성을 증명하는 암호학적 프로토콜**

## 📊 영지식 증명 개념도

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Zero Knowledge Proof                              │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│    ┌──────────────┐                        ┌──────────────┐         │
│    │   증명자      │                        │   검증자      │         │
│    │  (Prover)    │                        │  (Verifier)  │         │
│    │              │                        │              │         │
│    │  비밀: x     │                        │  비밀 모름   │         │
│    │  (Secret)    │                        │              │         │
│    └──────┬───────┘                        └──────┬───────┘         │
│           │                                       │                 │
│           │    증명 (Proof)                      │                 │
│           │  ─────────────────────────────────▶  │                 │
│           │    "나는 x를 알고 있다"              │                 │
│           │                                       │                 │
│           │    도전 (Challenge)                  │                 │
│           │  ◀─────────────────────────────────  │                 │
│           │    "이걸 증명해봐"                   │                 │
│           │                                       │                 │
│           │    응답 (Response)                   │                 │
│           │  ─────────────────────────────────▶  │                 │
│           │    (비밀 x 없이 증명)                │                 │
│           │                                       │                 │
│           │                              ┌───────┴───────┐         │
│           │                              │ Accept/Reject │         │
│           │                              │ (x 모른 채로) │         │
│           │                              └───────────────┘         │
│                                                                      │
│   결과: 검증자는 "증명자가 x를 안다"는 것만 확신                     │
│         x 자체는 전혀 알 수 없음 → 영지식성(Zero-Knowledge)         │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

---

## 🏗️ 영지식 증명 3대 속성 `완건영`

| 속성 | 영문 | 설명 |
|:----|:----|:----|
| **완**전성 | Completeness | 명제가 참이면, 정직한 증명자는 정직한 검증자를 **항상 납득**시킬 수 있음 |
| **건**전성 | Soundness | 명제가 거짓이면, 부정한 증명자는 검증자를 **속일 수 없음** (확률적으로 무시 가능) |
| **영**지식성 | Zero-Knowledge | 검증자는 명제의 **참/거짓 외에 어떤 정보도 얻지 못함** |

---

### 🧠 암기법 상세

```
🧠 완건영 = "영지식 증명 3대 속성"

  완 : 완전성 (Completeness) - 참이면 납득
  건 : 건전성 (Soundness) - 거짓이면 속일 수 없음
  영 : 영지식성 (Zero-Knowledge) - 비밀 노출 없음
```

## 📊 영지식 증명 종류 `대비`

### 그룹 1: 대화형 vs 비대화형

| 구분 | Interactive ZKP | Non-Interactive ZKP |
|:----|:---------------|:-------------------|
| **상호작용** | 증명자-검증자 여러 라운드 대화 | 한 번에 증명 전송 |
| **효율성** | 낮음 (다회 통신) | 높음 (단일 증명) |
| **적용** | 초기 연구 | 블록체인 실제 적용 |
| **예시** | Schnorr Protocol | zk-SNARK, zk-STARK |

### 그룹 2: 주요 프로토콜 비교 `SNARK-STARK`

| 구분 | zk-SNARK | zk-STARK |
|:----|:---------|:---------|
| **풀네임** | Succinct Non-interactive ARgument of Knowledge | Scalable Transparent ARgument of Knowledge |
| **신뢰 설정** | 필요 (Trusted Setup) | **불필요** |
| **증명 크기** | **매우 작음** (~200B) | 큼 (~100KB) |
| **검증 속도** | **빠름** | 느림 |
| **양자 저항** | 취약 | **저항성 있음** |
| **적용** | Zcash, zkSync | StarkWare |

## 📊 대표적 응용 사례

| 분야 | 응용 | 설명 |
|:----|:----|:----|
| **블록체인** | Zcash, zkSync | 프라이버시 코인, L2 확장 솔루션 |
| **인증** | 성인인증, 자격증명 | 생년월일 노출 없이 성인 여부만 증명 |
| **전자투표** | 비밀투표 검증 | 투표 내용 노출 없이 유효성 검증 |
| **금융** | 규정 준수 증명 | 잔액 공개 없이 지급 능력 증명 |
| **의료** | 건강 상태 증명 | 상세 진료 기록 없이 건강 상태 인증 |

## ⭐ 차별점 키워드

### 🏆 zk-Rollup (Zero-Knowledge Rollup)
> 블록체인 L2 확장 솔루션의 핵심
> - 수천 개의 트랜잭션을 **하나의 증명**으로 압축
> - **영지식 증명**으로 유효성 검증
> - Ethereum 확장성 문제 해결 (zkSync, StarkNet)
> - 가스비 절감, 처리 속도 향상
