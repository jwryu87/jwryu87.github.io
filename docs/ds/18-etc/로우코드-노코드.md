# 로우코드/노코드 (Low-Code/No-Code)
{: .fs-8 }

18. DS 기타
{: .label .label-purple }

---

## 핵심 키워드

`노코드` `시민개발플랫폼` `IDE` `Drag & Drop UI` `템플릿 모델링` `WYSWYG Editor` `RestAPI` `Angular.js` `3rd Party API` `데이터모델링`

---

## 정의/개념

**Drag&Drop UI와 Workflow모델링**을 활용하여 복잡한 코딩 과정을 단순화해 소프트웨어를 빠르게 개발하고 배포할 수 있게 지원하는 **SW개발 기술**

## 로우코드 기술요소

### 환경

| 구분 | 기술 | 설명 |
|:-----|:-----|:-----|
| **환경** | IDE | Polyglot 프로그래밍 가능한 비주얼 개발도구 |
| | Drag & Drop UI | 시각적 프로그래밍 모델이 가능한 인터페이스 |
| | 템플릿 모델링 | 재사용의 효율을 극대화하는 코드 템플릿 |
| | WYSWYG Editor | 문서 편집 화면과 출력물이 동일하게 나오는 방식 |

### 개발

| 구분 | 기술 | 설명 |
|:-----|:-----|:-----|
| **개발** | RestAPI | 웹의 모든 자원에 대한 고유한 URI를 부여한 경량형 API |
| | Angular.js | 구글에서 만든 자바스크립트 프레임워크 |
| | 3rd Party API | 다양한 서드파티 API를 호출하여 재사용 및 통합 |
| | 데이터모델링 | 서비스 워크플로우, SQL 쿼리 자동화 |

---

## 장점

| 구분 | 항목 | 설명 |
|:-----|:-----|:-----|
| **장점** | 개발속도 향상 | 핸드코딩 최소화를 통한 개발 속도 향상 |
| | 생산성 향상 | 별도 개발된 컴포넌트 및 샘플 프로그램 이용가능 |
| | 저비용 관리가능 | 개발기간 단축 통한 비용감소 모바일 앱/PC IDE 지원 |
| | 라이프사이클 관리 | 프로세스 및 Workflow 모델링 지원 가능 App Marketplace 지원(템플릿 및 기능 다운로드 지원) |

---

## 단점

| 구분 | 항목 | 설명 |
|:-----|:-----|:-----|
| **단점** | 제한된 커스터마이징 | 사용자 커스터마이징이 제한됨 |
| | 기업용 시스템 통합 어려움 | 기업용 솔루션등과 연결되는 플랫폼이 없어 통합 어려움 |
| | 국내 레퍼런스 부족 | 국내 도입 사례 부족으로 운영 편의성 낮음 |
| | 해외 솔루션 기술지원 한계 | 해외 개발된 솔루션 도입시 세부 기술지원의 한계점 존재 |
| | 보안 이슈 존재 | 솔루션내 제공되는 소스에 대한 보안 이슈 |
| | 벤더 의존성 높음 | 개발된 어플리케이션의 벤더 의존성이 높음 |

## 로우코드 vs 노코드 비교

| 구분 | 로우코드 (Low-Code) | 노코드 (No-Code) |
|:-----|:-----|:-----|
| **대상 사용자** | 개발자 + 비개발자 | 비개발자 (시민 개발자) |
| **코딩 필요성** | 최소한의 코딩 필요 | 코딩 불필요 |
| **커스터마이징** | 높음 | 제한적 |
| **복잡도** | 중복잡 애플리케이션 | 단순 애플리케이션 |
| **유연성** | 높음 | 낮음 |
