# 데이터 메시 (Data Mesh)
{: .fs-8 }

18. DS 기타
{: .label .label-purple }

---

## 핵심 키워드

`Data Product` `Domain` `Data Platform` `Federated Governance` `Domain Ownership`

---

## 정의/개념

빅데이터 분석 시스템의 **아키텍처 스타일**로, 마이크로 서비스 아키텍처 컨셉과 유사하게 데이터 분석 시스템을 각각의 분산된 서비스 형태로 개발 관리하는 **아키텍처 모델**

---

```
└─────────────────────────────────────────────────────────┘
```

## 데이터 메시 4대 원칙

| 원칙 | 설명 |
|:-----|:-----|
| **Domain Ownership** | 분석데이터, 메타 데이터에 대한 소유권을 비즈니스 도메인을 기준으로 나누고 도메인 별 운영 API와 하나 이상의 분석 데이터를 제공 |
| **Data As A Product** | 분산환경에서의 데이터 사일로, 데이터 품질 문제를 해결하기 위해 분석데이터를 Product로 취급하여 도메인에서 제공 |
| **Self-Serve Data Platform** | 도메인 별로 Data Product를 개발, 운영하기 위해 수명주기를 프로비저닝 및 관리 |
| **Federated Computational Governance** | Data Product의 상호 운영을 위해 플랫폼에 의한 의사 결정을 자동화 위한 거버넌스 모델 |

---

## 데이터 메시 구성요소

### Architecture

| 구분 | 구성요소 | 설명 |
|:-----|:-----|:-----|
| **Architecture** | Domaina | 비즈니스 도메인 별로 나누어진 도메인 팀으로 데이터에 대한 오너쉽을 갖으며 Data Product를 생성 및 공유 |
| | Self-Serve Data Platform | 셀프 서비스로 데이터를 분석할 수 있는 기능과 Data Product를 생성할 수 있는 기능을 제공하는 플랫폼 |
| | Federated Governance | 상호운용성, 문서표준화, 보안, 프라이버시 및 컴플라이언스 등 글로벌 정책 |

### Domain

| 구분 | 구성요소 | 설명 |
|:-----|:-----|:-----|
| **Domain** | Operational Data | 비즈니스 도메인에서 사용되는 운영 데이터 |
| | Analytical Data | 운영데이터를 ingest, cleaning 하고 Analytics를 이용하여 분석한 데이터 |
| | Data Product | 다른 도메인에서 API 등을 통해 접근할 수 있도록 게시된 데이터 셋 |

---

## 기존 아키텍처 vs 데이터 메시

| 구분 | 기존 (Data Lake/Warehouse) | 데이터 메시 |
|:-----|:-----|:-----|
| **데이터 소유권** | 중앙 데이터 팀 | 도메인 팀 |
| **거버넌스** | 중앙 집중식 | 연합형 (Federated) |
| **확장성** | 병목현상 발생 가능 | 도메인별 독립 확장 |
| **데이터 접근** | 중앙 플랫폼 통해 | 데이터 프로덕트 API |
| **품질 책임** | 데이터 팀 | 도메인 팀 |

---

## 데이터 메시 도입 효과

| 효과 | 설명 |
|:-----|:-----|
| **확장성** | 도메인별 독립적 확장 가능 |
| **품질 향상** | 도메인 전문가가 데이터 품질 관리 |
| **민첩성** | 도메인별 독립적 개발/배포 |
| **거버넌스** | 연합형 거버넌스로 유연성 확보 |
