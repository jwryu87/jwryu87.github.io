## 정의/개념

**Open API, 핀테크 기업, 오픈소스**, 확장성 **소프트웨어 플랫폼**

- 외부 개발자들이 API를 통해 서비스를 활용하고 확장할 수 있는 플랫폼
- 핀테크, 금융, 공공 등 다양한 분야에서 활용

## 구성요소

| 구분 | 설명 |
|:-----|:-----|
| **API Gateway** | - 다양한 API를 통합하여 하나의 API를 통한 제공 및 모니터링과 트래픽 제어<br>- 3rd Party에 API 서버 은닉 및 이용자 식별을 위한 인증처리 |
| **API Portal** | - Open API 관련 제휴사의 정보 관리<br>- 대상자들에게 키 발급 및 API 사용 관리 |
| **API Support & Analysis** | - API 이용 로그 데이터 추출<br>- 통계데이터 생성 및 API 대시 보드 개발 |
| **API Engine** | - 레거시 서비스에 대해 API/인터페이스 생성만으로 오픈 API 제공 기술<br>- Open API 호출에 대해 JSON 형태의 입력 전문을 전환해 MCI/EAI와 연계하여 호출하고 JSON 형태로 응답을 해 사용자에게 정보를 전달 |
| **OAS** | - API의 이름, 파라미터, 응답 값 등의 구문, 인증방법, 데이터 전달형식 등 필요한 정보를 일관된 형식으로 기술한 문서<br>- OpenAPI Specification |
| **Authentication** | - API에 접근에 대한 인증 정보 검증 |
| **API Developer Portal** | - API 문서화, 샘플코드 및 어플리케이션 제공<br>- API 검색을 위한 다양한 검색 방법을 제공 |

---

## OAS (OpenAPI Specification)

| 항목 | 설명 |
|:-----|:-----|
| **정의** | API를 기술하기 위한 표준 명세 |
| **포함 정보** | API 이름, 파라미터, 응답값, 인증방법, 데이터 형식 등 |
| **형식** | YAML 또는 JSON |
| **버전** | OpenAPI 3.0, OpenAPI 3.1 등 |

## 오픈 플랫폼 활용 분야

| 분야 | 활용 예시 |
|:-----|:-----|
| **금융/핀테크** | 오픈뱅킹, 마이데이터, 결제 API |
| **공공** | 공공데이터포털, 전자정부 API |
| **커머스** | 쇼핑몰 연동, 배송 추적 API |
| **소셜** | SNS 로그인, 공유 API |
