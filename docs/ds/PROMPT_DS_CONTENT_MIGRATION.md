# DS 토픽 콘텐츠 마이그레이션 프롬프트

## 📋 작업 개요

CSV 파일(`기술사_기본필수노트_DS - DS의 사본.csv`)의 학습 내용을 Jekyll 블로그의 DS 하위 페이지들에 추가/병합하는 작업

### 작업 규칙

1. **빈 페이지(껍데기)**: CSV 내용을 그대로 추가
2. **기존 내용 있는 페이지**: 기존 내용과 병합 (중복 제거, 구조화)
3. **마크다운 형식**: 일관된 헤딩, 리스트 스타일 유지

## 📝 콘텐츠 변환 규칙

### 1. 마크다운 헤딩 구조

```markdown
---
layout: default
title: [토픽명]
parent: [상위 카테고리명]
nav_order: [순서]
---

# [토픽명]

[한 줄 정의/설명]

---

## 개요
[기본 개념 설명]

## 특징
- 특징1
- 특징2

## 구성요소
| 구성요소 | 설명 |
|:---------|:-----|
| 요소1 | 설명1 |
| 요소2 | 설명2 |

## 기술요소
- **기술1**: 설명
- **기술2**: 설명

## 프로세스/절차
1. 단계1
2. 단계2
3. 단계3

## 유형/분류
- 유형1
- 유형2

## 관련 키워드
`키워드1`, `키워드2`, `키워드3`
```

### 2. CSV 내용 파싱 규칙

- `[특징]`, `[구성요소]` 등 대괄호 → `## 특징`, `## 구성요소` 헤딩으로
- `+항목` → `- **항목**` 볼드 리스트로
- `항목 = 설명` → `- **항목**: 설명` 형태로
- `>항목` → 들여쓰기 리스트로
- 암기팁 (예: `서대가C분병`) → `> 📌 암기팁: 서대가C분병` blockquote로

### 3. 병합 우선순위

1. 기존 내용 유지 (삭제하지 않음)
2. CSV 내용 중 새로운 정보 추가
3. 중복 내용은 더 상세한 것 선택
4. 구조화되지 않은 내용은 적절한 섹션에 배치

## 🔧 실행 방법

### Step 1: 페이지 상태 확인
각 대상 페이지를 읽어서 빈 페이지인지, 내용이 있는지 확인

### Step 2: 콘텐츠 변환
CSV 내용을 마크다운 형식으로 변환

### Step 3: 페이지 업데이트
- 빈 페이지: 변환된 내용 추가
- 내용 있는 페이지: 기존 내용과 병합

### Step 4: 검증
- 마크다운 문법 오류 확인
- 링크 정상 동작 확인

---

## ⚠️ 주의사항

1. **Front Matter 보존**: 기존 파일의 `---` 사이 메타데이터 유지
2. **nav_order 유지**: 기존 순서 변경하지 않음
3. **신규 파일 생성 시**: 적절한 nav_order 부여
4. **이미지/링크**: 상대경로 사용 (`{{ site.baseurl }}`)
