# 클라우드 컴퓨팅 보안
{: .fs-8 }

1.3 클라우드 보안
{: .label .label-purple }

---

## 핵심 키워드

`ISO 27017` `ISO 27018` `가상화 보안` `데이터 암호화` `접근제어`

---

## 정의/개념

클라우드 컴퓨팅 환경에서 **데이터, 애플리케이션, 인프라를 보호**하기 위한 정책, 기술, 통제 방안의 집합

---

## 보안 표준 (클보1718)

> 📌 암기팁: 클라우드 보안은 **27017**, **27018**

| 표준 | 설명 | 주요 내용 |
|:-----|:-----|:----------|
| **ISO/IEC 27017** | 클라우드 서비스 정보보안 통제 | 클라우드 특화 보안 통제 |
| **ISO/IEC 27018** | 클라우드 개인정보 보호 | 개인정보처리자 지침 |

---

## 보안위협

| 위협 유형 | 설명 |
|:----------|:-----|
| **가상화 취약점 상승** | 하이퍼바이저 취약점, VM 탈출 |
| **정보위탁에 따른 정보유출** | 제3자 관리로 인한 유출 위험 |
| **사용 단말의 다양성과 분실** | 모바일 기기 분실로 인한 정보유출 |
| **자원 공유 및 집중화에 따른 서비스 장애** | 단일 장애점(SPOF) |
| **분산 처리에 따른 보안 적용의 어려움** | 여러 지역에 분산된 보안 관리 |
| **법규 및 규제의 문제** | 국가별 상이한 법규 준수 |

---

## 보안요소기술 (기무가보 원인내공)

> 📌 암기팁: **기**밀성 **무**결성 **가**용성 **보**안정책 / **원**격확인 **인**증 **내**부 **공**격

| 영역 | 기술 | 설명 |
|:-----|:-----|:-----|
| **기밀성** | 데이터 암호화 | 저장/전송 데이터 암호화 |
| **무결성** | 데이터 무결성 | 해시, 전자서명 |
| **가용성** | 가용성 및 복구 | 백업, DR, HA 구성 |
| **보안정책** | 보안정책 관리 및 비용분석 | 보안 거버넌스 |
| **원격확인** | 원격확인 및 가상머신 보호 | VM 보안, 원격 검증 |
| **인증** | 사용자 인증과 접근제어 | IAM, MFA |
| **네트워크** | 네트워크 보안 및 웹 보안 | 방화벽, WAF, IDS/IPS |
| **공격대응** | 공격모델 및 시뮬레이션 | 침투테스트, 위협모델링 |

---

## ISO/IEC 27017 주요 통제 (정조자인 암물운통 개공사연준)

> 📌 암기팁: 정조자인 암물운통 개공사연준

| 통제 영역 | 내용 |
|:----------|:-----|
| **정보보안 정책** | 클라우드 보안 정책 수립 |
| **조직** | 보안 조직 및 역할 정의 |
| **자산관리** | 자산 식별 및 분류 |
| **인적보안** | 인력 보안 관리 |
| **암호화** | 데이터 암호화 통제 |
| **물리적 보안** | 데이터센터 물리 보안 |
| **운영보안** | 운영 절차, 변경관리 |
| **통신보안** | 네트워크 보안 통제 |
| **개발보안** | 시스템 개발 보안 |
| **공급망보안** | 외부 공급자 관리 |
| **사고관리** | 보안사고 대응 |
| **연속성** | 비즈니스 연속성 관리 |
| **준거성** | 법규 및 계약 준수 |

---

## ISO/IEC 27018 주요 원칙 (동합수최 공정개투 참책보규)

> 📌 암기팁: 동합수최 공정개투 참책보규

| 원칙 | 내용 |
|:-----|:-----|
| **동의** | 개인정보 수집/이용 동의 |
| **합법성** | 적법한 처리 |
| **수집제한** | 최소 수집 원칙 |
| **최소화** | 목적 달성 최소 정보 |
| **공개** | 처리 목적 공개 |
| **정확성** | 정보 정확성 유지 |
| **개인참여** | 정보주체 권리 보장 |
| **투명성** | 처리 내역 투명 공개 |
| **참여** | 정보주체 참여권 |
| **책임성** | 처리자 책임 |
| **보안** | 개인정보 보호조치 |
| **규제준수** | 법규 준수 |

## 클라우드 보안 아키텍처

```
┌─────────────────────────────────────────────────────────┐
│                    Application Layer                     │
│                  (WAF, API Security)                     │
├─────────────────────────────────────────────────────────┤
│                     Data Layer                           │
│              (암호화, DLP, Data Masking)                 │
├─────────────────────────────────────────────────────────┤
│                   Platform Layer                         │
│              (IAM, MFA, Container Security)              │
├─────────────────────────────────────────────────────────┤
│                 Infrastructure Layer                     │
│              (Firewall, IDS/IPS, VPN)                    │
├─────────────────────────────────────────────────────────┤
│                   Physical Layer                         │
│              (데이터센터, 물리적 접근통제)                │
└─────────────────────────────────────────────────────────┘
```
