## 클라우드 네이티브 기술요소

### 1. 조직/문화 측면

| 기술요소 | 설명 |
|:---------|:-----|
| DevOps | 개발과 운영조직의 Silo를 제거하며 Build와 구현 협업 문화 |
| Agile | UserStory기반 반복적 개발, 요구 적극적 수용 Adaptive 개발방법론 |
| MSA | Micro Service를 이용한 복잡한 애플리케이션을 구축 및 API를 통한 타 시스템과의 연결성 제공 |

### 2. 아키텍처 측면

| 기술요소 | 설명 |
|:---------|:-----|
| API | REST, Gateway 통한 서비스 인터페이스 호출 통신 |
| 12 Factor App | 방대한 앱의 개발, 운영, 확장을 위한 클라우드 개발 방법론 |
| 컨테이너 | 다른 환경으로 신뢰성있는 포팅지원 |
| 쿠버네티스 | 컨테이너화 된 애플리케이션의 자동화된 배포, 확장 관리용 플랫폼 |

### 3. 인프라 측면

| 기술요소 | 설명 |
|:---------|:-----|
| Multi-tenancy | 단일 소프트웨어 인스턴스가 여러 Client 서비스를 제공<br>Application, DB 등 조직 내 공유와 동시에 독립성 확보 가능 |

### 4. 통합/배포 측면

| 기술요소 | 설명 |
|:---------|:-----|
| CI/CD | 애플리케이션 코드 변경에 대한 정기적 빌드, 테스트 병합 수행, 자동 릴리즈 등 수행 |

## 클라우드 네이티브 성숙도 (CNMM) - 온인옵네

> 📌 암기팁: **온**프레미스 **인**에이블드 **옵**티마이즈드 **네**이티브

| 레벨 | 단계 | 설명 |
|:-----|:-----|:-----|
| **CNMM 0** | On-Premise | 기존의 애플리케이션들은 조직 내부와 별도의 데이터센터 등의 장소에 보관되어 사용되는 모델 |
| **CNMM 1** | Cloud-Enabled | 조직의 애플리케이션을 단순하게 클라우드 서비스를 지원하는 수준에서 사용되는 단순한 형태의 애플리케이션 |
| **CNMM 2** | Cloud-Optimized | 조직의 애플리케이션을 클라우드 네이티브 애플리케이션의 형태로 다시 개발하지 않고, 더 많은 클라우드의 기능과 서비스를 사용할 수 있도록 전환시킨 애플리케이션 |
| **CNMM 3** | Cloud-Native | 클라우드 네이티브의 모든 기능을 충분히 이용하는 환경에서 애플리케이션, 장치, 플랫폼, 서비스 제공자들 간 동작 일관성 확보 |

---

## 클라우드 네이티브 아키텍처 (인프런컨애)

> 📌 암기팁: **인**프라 프로비저닝 **런**타임 **컨**테이너 **애**플리케이션

| 계층 | 구성요소 | 설명 |
|:-----|:---------|:-----|
| **애플리케이션** | 애플리케이션 구현 | 비즈니스 로직 구현 |
| **컨테이너 배포** | 컨테이너 오케스트레이션 | 통합/관리 (Kubernetes) |
| **런타임** | 네트워크/저장소 | 런타임 환경 구성 |
| **환경 구성** | 프로비저닝 | 리소스 자동 할당 |
| **인프라** | Bare Metal, Cloud | 기반 인프라 |
