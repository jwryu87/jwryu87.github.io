# 클라우드 전환 (Cloud Migration)
{: .fs-8 }

1.2 클라우드 아키텍처/전환
{: .label .label-purple }

---

## 핵심 키워드

`마이그레이션` `Rehost` `Replatform` `Refactor` `6R`

---

## 정의/개념

기존 온프레미스 환경의 애플리케이션, 데이터, 인프라를 **클라우드 환경으로 이전**하는 과정

---

## 전환 대상 선정기준

| 기준 | 설명 |
|:-----|:-----|
| 유지관리 부하 감소 | 운영 복잡도가 높은 시스템 |
| 대용량 처리 요구 | 확장성이 필요한 워크로드 |
| 효율성 증대 | 비용 절감 효과가 큰 시스템 |
| 개발 및 테스트 효율화 | 개발/테스트 환경 |
| 부정기적인 이벤트 대응 | 트래픽 변동이 큰 시스템 |

---

## 마이그레이션 5단계 (분설구검이)

> 📌 암기팁: **분**석 **설**계 **구**축 **검**증 **이**관

| 단계 | 활동 | 주요 내용 |
|:-----|:-----|:----------|
| **1. 분석** | 시스템 현황 분석 | AS-IS 분석, 워크로드 분석, 의존성 분석 |
| **2. 설계** | 아키텍처/마이그레이션 설계 | TO-BE 설계, 전환 전략 수립, 일정 계획 |
| **3. 구축** | 인프라 구축, 마이그레이션 수행 | 클라우드 환경 구축, 데이터/앱 이전 |
| **4. 검증** | 시스템/서비스 검증 | 기능 테스트, 성능 테스트, 보안 검증 |
| **5. 이관** | 이관, 안정화 운영 | Cut-over, 모니터링, 최적화 |

---

## 마이그레이션 6R 전략

| 전략 | 설명 | 복잡도 | 비용 |
|:-----|:-----|:------:|:----:|
| **Rehost** | Lift & Shift, 그대로 이전 | 낮음 | 낮음 |
| **Replatform** | Lift & Reshape, 일부 최적화 | 중간 | 중간 |
| **Repurchase** | SaaS로 전환 | 낮음 | 중간 |
| **Refactor** | 클라우드 네이티브로 재설계 | 높음 | 높음 |
| **Retire** | 폐기 | - | - |
| **Retain** | 유지 (전환 보류) | - | - |

---

## 공공기관 클라우드 전환 전략 (기센방법)

> 📌 암기팁: **기**본원칙 **센**터지정 **방**안수립 **법**제도개선

| 단계 | 전략 | 내용 |
|:-----|:-----|:-----|
| **기본원칙 수립** | 이용대상 선정 | - 공공 클라우드 이용대상 선정<br>- 민간 클라우드 이용대상 선정 |
| **공공 클라우드 센터 지정** | 기관별 배치 | - 기관별 클라우드 센터 배치 계획 |
| **전환 및 통합 실행 방안** | 대상/시기 선정 | - 전환 대상 및 시기 선정<br>- 우선순위 결정 |
| **협력체계 및 법/제도 개선** | 협의체 구성 | - 협의체 구성 운영<br>- 통합기준 개정 |

---

## 마이그레이션 고려사항

| 관점 | 고려사항 |
|:-----|:---------|
| **기술적** | 데이터 이전 방안, 네트워크 대역폭, 호환성 |
| **보안** | 데이터 암호화, 접근통제, 컴플라이언스 |
| **운영** | 다운타임 최소화, 롤백 계획, 모니터링 |
| **비용** | TCO 분석, 라이선스 비용, 전환 비용 |
