# 민간 클라우드 이용 절차
{: .fs-8 }

1.2 클라우드 아키텍처/전환
{: .label .label-purple }

---

## 핵심 키워드

`공공기관` `민간 클라우드` `보안성 검토` `SLA` `계약 해지`

---

## 정의/개념

공공기관이 **민간 클라우드 서비스를 도입**하기 위한 **사전 검토부터 계약 종료까지의 전체 절차**

---

## 이용절차 (검계품종)

> 📌 암기팁: **검**토 **계**약 **품**질 **종**료

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 1.사전   │───▶│ 2.발주   │───▶│ 3.품질   │───▶│ 4.계약   │
│   검토   │    │   및     │    │   및     │    │   해지   │
│          │    │   계약   │    │  보안관리 │    │  종료    │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
```

---

## 단계별 상세 절차

### 1단계: 사전 검토

| 항목 | 내용 |
|:-----|:-----|
| **효과성 검토** | 클라우드 도입 효과 분석 |
| **이용 대상 정보 시스템** | 전환 대상 시스템 선정 |
| **이용 가능한 민간 클라우드** | 적합한 클라우드 서비스 검토 |
| **보안성 검토** | 보안 요구사항 충족 여부 확인 |

### 2단계: 발주 및 계약

| 항목 | 내용 |
|:-----|:-----|
| **발주 방식** | 조달 방식 결정 |
| **서비스 제공자 선정** | CSP 선정 기준 및 평가 |
| **계약서 작성** | 표준 계약서 기반 계약 |
| **서비스 수준 협약 (SLA)** | 품질 기준 및 위약 조건 명시 |

### 3단계: 품질 및 보안관리

| 항목 | 내용 |
|:-----|:-----|
| **서비스 연속성 확보** | 가용성, 백업, 재해복구 체계 |
| **정보의 보호와 관리** | 데이터 암호화, 접근통제 |
| **보안 관제** | 침입탐지, 로그 모니터링 |
| **개인정보처리의 위탁** | 개인정보 처리 위탁 계약 |
| **침해사고 및 장애 통지** | 사고 발생 시 통지 체계 |

### 4단계: 계약 해지/서비스 종료

| 항목 | 내용 |
|:-----|:-----|
| **계약 해제 및 해지** | 계약 종료 절차 |
| **종료 통보** | 사전 통보 및 협의 |
| **이용 정보의 반환 및 파기** | 데이터 반환 후 완전 파기 |

---

## 전환 전략 (시점별)

### 검토 및 계약 시점

| 단계 | 전략 |
|:-----|:-----|
| **사전 검토** | - 전환 대상 선정 및 우선순위 결정<br>- 전담 조직 구성 및 전문 인력 배치 |
| **발주 및 계약** | - 전문 컨설팅 수행<br>- 표준 계약서 기반 계약 검토 |

### 운영 및 계약 해지 시점

| 단계 | 전략 |
|:-----|:-----|
| **운영(품질/보안)** | - 안정적 마이그레이션 진행 방안 수립<br>- 관제 및 사고 대응체계 수립 |
| **계약 해지** | - 데이터 파기 절차 및 확인 방안 수립 |

---

## SLA 주요 항목

| 항목 | 내용 |
|:-----|:-----|
| 가용성 | 서비스 가동률 (예: 99.9%) |
| 응답시간 | 서비스 응답 속도 |
| 복구시간 | 장애 발생 시 복구 시간 (RTO) |
| 데이터 보호 | 백업 주기, 보관 기간 |
| 보안 | 보안 인증, 감사 로그 |
