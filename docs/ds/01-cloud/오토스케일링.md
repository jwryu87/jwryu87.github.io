# 오토스케일링 (Auto Scaling)
{: .fs-8 }

1.2 클라우드 아키텍처/전환
{: .label .label-green }

---

## 핵심 키워드

`Metrics` `Scale out` `L/B` `Health Check`

---

## 정의/개념

시스템 자원 **메트릭 값 모니터링** **서버 사이즈 자동 조절 기술**

---

## 동작방식

```
                    ④ Metrics
                        │
    ┌───────────────────▼───────────────────┐
    │         AutoScaling 그룹              │
    │  ┌─────────────────────────────────┐  │     모니터링
    │  │  ┌─────┐  ┌─────┐  ┌─────┐     │  │
    │  │  │신규 │  │ 신규│  │     │     │  │        ②
    │  │  │서버 │  │서버 │  │     │     │◄─┼─── Alert
    │  │  │인스 │  │인스 │  │ ... │     │  │     발송
    │  │  │턴스 │  │턴스 │  │     │     │  │
    │  │  └─────┘  └─────┘  └─────┘     │  │
    │  └─────────────────────────────────┘  │
    │        ▲              │               │
    │   ⑥ 트래픽      ③ Scale Out         │
    │        │              ▼               │
    │  ┌─────┴─────┐   ┌──────────┐        │
    │  │   Load    │   │ 스케일링  │        │
    │  │ Balancer  │   │   정책    │        │
    │  └───────────┘   └──────────┘        │
    │        │                              │
    │   ⑤ Health check                     │
    │      (Ping)                           │
    └───────────────────────────────────────┘
              │
              ▼
    ┌─────────────────┐
    │   서버         │    서버        ③ Scale Out
    │   생성    →    │   프로비저닝  →  서버
    │                │                  이미지
    └─────────────────┘
```

---

## 동작 절차

| 구분 | 동작 절차 | 설명 |
|:-----|:---------|:-----|
| **모니터링** | 1. Metrics | CPU 사용량, NW Packet, Disk I/O 등의 시스템 자원 정보 수집 |
|  | 2. Alert 발송 | - 사전 정의된 임계치 초과시 오토스케일링 Alert 발송<br>- 각 지표기준에 부합될 시 정책 트리거 수행(부하기반, 스케줄기반) |
| **서버 생성** | 3. Scale Out | 스케일링 정책에 의해 고정/비율/증분으로 서버 추가 |
|  | 4. 서버 프로비저닝 | - scale out 대상 서버와 동일한 머신 이미지 생성<br>- 사전 커스텀 이미지 생성 및 AutoScaling 그룹에 마운트(Mount) |
| **서비스 연결** | 5. 로드밸런싱 Health check | - 신규 생성된 서버의 서비스 상태 확인을 위한 HTTP Request<br>- 정상적인 ACK 수신으로 신규 생성된 서버의 정상 작동 확인 |
|  | 6. 트래픽 처리 | - 신규 생성된 서버를 로드밸런서와 연결<br>- 기존 운영중인 서버와 동일하게 트래픽 처리 |

> 클라우드 워크로드의 요구사항 변화에 즉각적인 대응을 지원하는 오토스케일링 기술

---

## 메트릭 지표

| 지표 | 설명 |
|:-----|:-----|
| **CPU 사용률** | used (%), 서버 그룹 내 서비스별 CPU들의 사용률 메트릭 지표 |
| **NW Packets** | 네트워크 in/out packet 및 bytes 메트릭 지표 |
| **Disk I/O** | 디스크의 read/write bytes와 Access 횟수 메트릭 지표 |
