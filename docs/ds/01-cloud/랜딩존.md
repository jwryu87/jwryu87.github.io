# 랜딩존 (Landing Zone)
{: .fs-8 }

1.2 클라우드 아키텍처/전환
{: .label .label-purple }

---

## 핵심 키워드

`클라우드 환경 설정` `Best Practice` `계정관리` `가드레일` `멀티계정`

---

## 정의/개념

클라우드 환경에서 **워크로드 배포 전 사전 구성된 안전하고 확장 가능한 환경**으로, 계정 구조, 네트워크, 보안 정책 등을 표준화한 클라우드 기반 환경

---

## 등장배경

| 문제점 | 설명 |
|:-------|:-----|
| 정책 적용 어려움 | 다중 계정 환경에서 일관된 정책 적용 어려움 |
| Best Practice 부족 | 클라우드 환경 구성 표준 가이드 부재 |
| 보안 취약점 | 계정별 상이한 보안 수준 |

## 기술요소 (계보설)

> 📌 암기팁: **계**정 **보**안 **설**계

### 1. 계정 관리

| 기술 | 설명 |
|:-----|:-----|
| **SSO** | Single Sign-On, 통합 인증 |
| **IAM** | Identity And Access Management, 신원/접근 관리 |

### 2. 보안

| 기술 | 설명 |
|:-----|:-----|
| **제로트러스트** | 신뢰하지 않고 항상 검증 |
| **가드레일** | 정책 위배 체크, 자동 교정 |
| **API 및 모니터링** | API 보안, 실시간 모니터링 |

### 3. 설계

| 기술 | 설명 |
|:-----|:-----|
| **리소스 프로비저닝** | 신규 계정 자동 환경 구성 |
| **구조적 조직화** | 다중 계정 구조 설계 |

---

## 고려요소

| 관점 | 고려사항 |
|:-----|:---------|
| **관리적 관점** | - 정책 복잡도 증가 대응<br>- 전사 공감대 형성 |
| **기술적 관점** | - 클라우드 Lock-in 방지<br>- 전문가 확보 |

---

## CSP별 랜딩존 서비스

| CSP | 서비스명 | 특징 |
|:----|:---------|:-----|
| AWS | Control Tower | 다중 계정 환경 설정, 가드레일 |
| Azure | Landing Zone | Blueprint 기반 환경 구성 |
| GCP | Cloud Foundation Toolkit | 인프라 코드화 |

> 💡 최근 CSP에서는 랜딩존 구성에 대해 관리적 편의성을 결합하여 별도의 서비스로 제공

---

## 구성요소

| 구성요소 | 설명 |
|:---------|:-----|
| 계정 구조 | OU(조직 단위), 계정 분리 전략 |
| 네트워크 | VPC, 서브넷, 라우팅 설계 |
| 보안 | IAM 정책, 암호화, 로깅 |
| 거버넌스 | 태깅 정책, 비용 관리 |
