# Public, Private, Hybrid Cloud
{: .fs-8 }

1.1 클라우드 개념
{: .label .label-purple }

---

## 핵심 키워드

| 유형 | 키워드 |
|:-----|:-------|
| Public | `일반 사용자` `pay-per-use` |
| Private | `기업` `통제권` |
| Hybrid | `Public+Private` |

---

## 정의/개념

| 유형 | 정의 |
|:-----|:-----|
| **Public** | **누구나 이용 가능**, 외부 서버, 높은 확장성 클라우드 |
| **Private** | **특정 조직 전용**, 내부 서버, 높은 보안성, 높은 비용, 낮은 유연성 클라우드 |
| **Hybrid** | **Public과 Private 결합**, 유연성, 보안 강화 클라우드 |

## 활용구조 (4가지 유형)

| 구분 | 설명 |
|:-----|:-----|
| **퍼블릭** | - 일반 대중이 이용하고 비즈니스와 외부 조직 또는 결합된 조직에 의해 관리<br>- 일반적인 서버에서 여러 클라이언트의 데이터를 저장하고 동시에 접근<br>- 보안 조치를 사용하여 인증된 사용자만 데이터에 액세스 |
| **프라이빗** | - 기업 소속 고객이 이용하고 조직이 관리<br>- **단일 조직에만 서비스를 제공** |
| **커뮤니티** | - 조직에서 같은 우려를 가진 고객 커뮤니티가 사용하도록 프로비저닝<br>- 커뮤니티에 속한 여러 조직, 외부 공급업체 또는 커뮤니티와 외부 업체가 결합된 조직이 소유하고 관리 |
| **하이브리드** | - 고유한 서비스 엔터티인 **둘 이상의 클라우드 인프라**(프라이빗, 커뮤니티 또는 퍼블릭)로 구성<br>- 데이터 이동을 지원하는 표준 기술로 제한 |

---

## 비교표

| 구분 | Public | Private | Hybrid |
|:-----|:-------|:--------|:-------|
| 이용자 | 일반 대중 | 특정 조직 | 혼합 |
| 서버 위치 | 외부 | 내부 | 내부+외부 |
| 확장성 | 높음 | 낮음 | 중간 |
| 보안성 | 낮음 | 높음 | 높음 |
| 비용 | 저렴 (종량제) | 높음 | 중간 |
| 유연성 | 높음 | 낮음 | 높음 |
| 통제권 | 낮음 | 높음 | 중간 |
