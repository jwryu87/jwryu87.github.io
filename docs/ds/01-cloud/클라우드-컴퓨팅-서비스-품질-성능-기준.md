## 평가기준 7가지 (가응확신/지지고)

> 암기: **가응확신**/**지지고**

| No | 기준 | 세부기준 | 주요 내용 |
|:--:|:-----|:---------|:----------|
| ① | 가용성 | 가용률 | - 가용률 보장을 위한 관리체계 및 운영정책<br>- 가용성 모니터링, 운영상태 로그 기능, 장애 알림 기능<br>- 가용률 측정 절차, 대상 및 구간, 방법 및 도구, 산정방식 |
| ② | 응답성 | 응답시간 | - 응답시간 수준제시, 조직 및 책임 설정, 응답시간 분석<br>- 응답시간 측정 절차, 대상 및 구간, 방법 및 도구, 산정방식 |
| ③ | 확장성 | 확장성 | - 확장성 정책수립, 수준제시, 시스템 도입 및 관리<br>- 임계치 설정 및 알람 기능, 수동/자동 리소스 확장 기능<br>- 관리 콘솔, 리소스 관리 API, Timestamp 활용 |
| ④ | 신뢰성 | 서비스 회복시간 | - 평균복구시간(MTTR), 최대 서비스 복구시간(MTTSR) |
|    |        | 백업 주기 | - 백업시스템 확보 및 관리, 증분/전체 백업, 백업 스케줄링 |
|    |        | 백업 준수율 | - 백업시행 및 복구 테스트 |
|    |        | 백업 데이터 보관 기간 | - 데이터 반환 미 파기 |
| ⑤ | 서비스 지속성 | 서비스 제공 능력 | - 서비스 지속역량 평가, 서비스 제공자의 재무상태<br>- 비전 및 중장기 목표 수립, 서비스 지원 조직 및 인력<br>- 서비스 개선 계획 및 정책 |
| ⑥ | 서비스 지원 | 서비스 지원력 | - 이용자 서비스 정책 수립, 서비스 제공방식 다양성<br>- 서비스 이전 정책 및 프로세스, 이용자 교육 |
| ⑦ | 고객대응 | 고객 대응체계 | - 고객의견 수렴채널 구축, 이용자 알림 프로세스 수립<br>- 내부 처리 절차, 모니터링 지원 시스템 제공 |
|    |         | 고객불만 처리체계 | - 고객불만 수집 및 처리절차<br>- 사용자 만족도 등급 기준 수립 |

## 평가지표

### MTTR (Mean Time To Repair) - 평균복구시간

| 구분 | 내용 |
|:-----|:-----|
| 개념 | 지정된 기간의 서비스 지속상태를 측정한 후 측정된 총 고장횟수와 총 장애시간을 통해 산출된 클라우드 서비스 평균복구시간 |
| 계산식 | **MTTR = 총 고장시간 / 고장횟수**<br>(예시) 총 고장시간 = 6시간, 고장횟수 = 3회 → MTTR = 6/3 = 2 |
| 고려사항 | - 측정 실패가 지속적으로 발생될 경우 총 1회의 고장 횟수 계산<br>- 복합적인 장애 요인으로 발생될 경우 원인에 따라 고장 횟수 산정 필요 |

### MTTSR (Maximum Time To Service Recovery) - 최대 서비스 복구시간

| 구분 | 내용 |
|:-----|:-----|
| 개념 | 지정된 기간의 서비스 지속상태를 측정한 후 장애 식별 및 조치 후 서비스를 정상적으로 가동하기까지 소요되는 최대 서비스 복구 시간 |
| 계산식 | **MTTSR = MAX(복구시간)**<br>(예시) H/W Fault: 5HR, N/W Down: 1HR, S/W Bug: 2HR → MTTSR = 5 |
| 고려사항 | - 측정 실패가 지속적으로 발생될 경우 연속된 복구시간으로 산정<br>- 복합적인 장애 요인으로 발생될 경우 원인에 따라 복구시간 산정 필요<br>- 그 외 정량적 지표로 MTBF(Mean Time Between Failure), MTTF(Mean Time to Failure) 등의 지표도 사용 |
