# 멀티 클라우드 (Multi Cloud)
{: .fs-8 }

1.1 클라우드 개념
{: .label .label-purple }

---

## 핵심 키워드

`고가용성` `유연성` `비용 효율성` `Fail over` `벤더 종속성 탈피`

---

## 정의/개념

**2개 이상의 CSP(Cloud Service Provider)**가 제공하는 Public or Private 구성된 **클라우드**

---

## 필요성

### 사용자 측면

| 필요성 | 설명 |
|:-------|:-----|
| 단일 클라우드의 한계 | 특정 CSP 서비스 한계 극복 |
| **벤더 종속성 탈피** | Lock-in 방지 |
| 서비스 근접성 | 지역별 최적 서비스 선택 |
| Shadow IT 대응 | 비공식 클라우드 사용 관리 |
| **Fail over** | 장애 시 대체 클라우드 활용 |
| 서비스 탄력성 확보 | 유연한 리소스 확장 |
| 조직 특성의 다양성 | 부서별 요구사항 대응 |

### 공급자 측면

| 필요성 | 설명 |
|:-------|:-----|
| 국내 클라우드 사업자의 위기 | 글로벌 CSP 대응 |
| 그룹사 IT자회사의 위기 | 내부 IT 역할 변화 |
| 통신&IDC사업자의 새로운 기회 | 멀티클라우드 연동 서비스 |
| Cloud MSP의 새로운 기회 | 관리 서비스 수요 증가 |

---

## 멀티클라우드 구성요소

| 구성요소 | 설명 |
|:---------|:-----|
| Public Cloud | 다양한 클라우드 환경에서 변화하는 가상 장치가 조직과 연결 |
| Private Cloud | 서비스 오케스트레이션과 함께 작동하는 분석, SDN 및 운영 관리 |
| End-to-End Connectivity | sd-wan 라우팅 및 sd-branch 보안으로 액세스 필요<br>안전한 이더넷 코어 스위치 및 무선 액세스 |
| Data Center | Leaf, Spine, Core 및 DCI 라우팅, 스위칭, 보안 및 관리를 기반으로 구축 |
| Campus and branch | 데이터 센터, 공공/사설 클라우드 라우팅 및 보안 정책과 원활하게 연계되는 sd-wan 라우팅 및 sd-branch 보안으로 액세스 안전한 이더넷 코어 스위칭 및 무선 액세스 필요 |

## 주요 기술 (연인애모개)

> 📌 암기팁: **연**동 **인**프라 **애**플리케이션 **모**니터링 **개**방형

| 기술 영역 | 세부 기술 | 설명 |
|:----------|:----------|:-----|
| **인프라 연동** | 단일 API, 인프라 연동 드라이버, 자원 라이프사이클 관리, Scale out | 여러 클라우드 인프라 통합 연동 |
| **인프라 서비스 통합 운용/관리** | 최적 배치 스케줄링, 통합 서비스 관리, 실행환경 연계 | 클라우드 간 서비스 통합 관리 |
| **애플리케이션 통합 운용/관리** | App 생성 및 배치, App 운용 및 관리 | 애플리케이션 라이프사이클 관리 |
| **통합 모니터링** | 에이전트(Agent), 푸쉬(PUSH), 풀(PULL), 서비스간 모니터링 | 전체 클라우드 모니터링 |
| **개방형 인터페이스** | 개방형 API, CLI(Command Line Interface), 웹도구 | 표준화된 관리 인터페이스 |

## 보안 취약점 및 대응방안

| 구분 | 내용 |
|:-----|:-----|
| **취약점** | CSP별 다른 보안 정책, 복잡성 증대, 가시성 부족 |
| **대응방안** | - 표준화된 보안 정책 수립<br>- CSAP 적용<br>- ISO 27017/27018 준수<br>- 통합 보안 관리 도구 도입 |

## 멀티클라우드 vs 하이브리드 클라우드 비교

| 구분 | 멀티 클라우드 | 하이브리드 클라우드 |
|:-----|:-------------|:-------------------|
| 개념 | 레거시 데이터센터와 2개 이상 클라우드 서비스 업체의 조합 | 기존의 레거시 데이터센터와 클라우드에서 소비하는 일부 서비스의 결합을 의미 |
| 접근방식 | 동일한 유형의 클라우드를 2개 이상 배포 | 여러 배포 유형(퍼블릭, 프라이빗) 사이에 통합이나, 오케스트레이션이 특정 방식으로 이뤄어짐 |
| 장점 | 여러 클라우드에서 제공하는 다양한 서비스를 제공 | 퍼블릭 클라우드, 프라이빗 클라우드 두 환경 간에 워크로드, 리소스, 플랫폼, 애플리케이션을 마이그레이션할 수 있는 공유 코어 소프트웨어 서비스를 제공 |
