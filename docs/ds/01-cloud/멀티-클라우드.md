---

## 정의/개념

**2개 이상의 CSP(Cloud Service Provider)**가 제공하는 Public or Private 구성된 **클라우드**

---

## 멀티클라우드 구성요소

| 구성요소 | 설명 |
|:---------|:-----|
| Public Cloud | 다양한 클라우드 환경에서 변화하는 가상 장치가 조직과 연결 |
| Private Cloud | 서비스 오케스트레이션과 함께 작동하는 분석, SDN 및 운영 관리 |
| End-to-End Connectivity | sd-wan 라우팅 및 sd-branch 보안으로 액세스 필요<br>안전한 이더넷 코어 스위치 및 무선 액세스 |
| Data Center | Leaf, Spine, Core 및 DCI 라우팅, 스위칭, 보안 및 관리를 기반으로 구축 |
| Campus and branch | 데이터 센터, 공공/사설 클라우드 라우팅 및 보안 정책과 원활하게 연계되는 sd-wan 라우팅 및 sd-branch 보안으로 액세스 안전한 이더넷 코어 스위칭 및 무선 액세스 필요 |

## 멀티 클라우드 유형

```
                    멀티 퍼블릭 클라우드
                          │
    ┌─────────────────────┼─────────────────────┐
    │                     │                     │
┌───▼───┐           ┌─────▼─────┐         ┌─────▼─────┐
│Public │           │ Public    │         │ Private   │
│Cloud  │           │ Cloud     │         │ Cloud     │
│(AWS)  │           │ (Azure)   │         │ (KT, LG)  │
└───────┘           └───────────┘         └───────────┘
    │                     │                     │
    └─────────────────────┴─────────────────────┘
                          │
              Traditional Infrastructure
                          │
                  하이브리드 클라우드
```

## 멀티클라우드 vs 하이브리드 클라우드 비교

| 구분 | 멀티 클라우드 | 하이브리드 클라우드 |
|:-----|:-------------|:-------------------|
| 개념 | 레거시 데이터센터와 2개 이상 클라우드 서비스 업체의 조합 | 기존의 레거시 데이터센터와 클라우드에서 소비하는 일부 서비스의 결합을 의미 |
| 접근방식 | 동일한 유형의 클라우드를 2개 이상 배포 | 여러 배포 유형(퍼블릭, 프라이빗) 사이에 통합이나, 오케스트레이션이 특정 방식으로 이뤄어짐 |
| 장점 | 여러 클라우드에서 제공하는 다양한 서비스를 제공 | 퍼블릭 클라우드, 프라이빗 클라우드 두 환경 간에 워크로드, 리소스, 플랫폼, 애플리케이션을 마이그레이션할 수 있는 공유 코어 소프트웨어 서비스를 제공 |
