# 서버리스 컴퓨팅 (Serverless Computing)
{: .fs-8 }

1.1 클라우드 개념
{: .label .label-purple }

---

## 핵심 키워드

`FaaS` `BaaS` `이벤트 기반` `Auto Scaling` `Pay-per-execution`

---

## 정의/개념

서버 관리 없이 **이벤트 기반**으로 애플리케이션 코드를 실행하고, **실행 시간만큼만 비용을 지불**하는 클라우드 컴퓨팅 실행 모델

---

## 특징

| 특징 | 설명 |
|:-----|:-----|
| **서버관리 불필요** | 인프라 관리를 CSP에 위임 |
| **유연한 규모조정** | 트래픽에 따른 자동 스케일링 |
| **자동화된 고가용성** | CSP가 가용성 보장 |
| **비용 효율** | 실행 시간/횟수 기반 과금 |

---

## 개념도 (클게파디)

> 📌 암기팁: **클**라이언트 API **게**이트웨이 **Fa**aS **D**B

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│클라이언트 │───▶│ API GW   │───▶│  FaaS    │───▶│    DB    │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
                     │                │
              ┌──────▼──────┐  ┌──────▼──────┐
              │ 서버리스     │  │ 서버리스     │
              │ 프레임워크   │  │ 플랫폼       │
              └─────────────┘  └─────────────┘
```

---

## 기술요소 (애환바자)

> 📌 암기팁: **애**플리케이션 **환**경 **바**인딩 **자**원

### 1. API 단위 응용

| 기술 | 설명 |
|:-----|:-----|
| API 단위 응용 조합 | 마이크로서비스 조합 |
| 미터링 및 로깅 | 사용량 측정, 로그 관리 |
| 로드 밸런싱 | 부하 분산 |
| 라우팅 | 요청 라우팅 |

### 2. 개발/운용 환경

| 기술 | 설명 |
|:-----|:-----|
| MSA | 마이크로서비스 아키텍처 |
| BaaS | Backend as a Service |
| FaaS | Function as a Service |
| Admin/User Portal | 관리 포털 |
| 개발 환경 | IDE, CLI 도구 |

### 3. 서비스 바인딩

| 기술 | 설명 |
|:-----|:-----|
| 서비스 연결 및 연동 | 외부 서비스 통합 |
| API 추상화 | API 레이어 추상화 |

### 4. 자원 자동 관리

| 기술 | 설명 |
|:-----|:-----|
| 자원 할당 및 관리 | 동적 리소스 할당 |
| Auto Scaling | 자동 확장/축소 |
| 실시간 모니터링 | 리소스 모니터링 |
| 자원 프로비저닝 | 자동 환경 구성 |

---

## 서버리스 유형

| 유형 | 설명 | 예시 |
|:-----|:-----|:-----|
| **FaaS** | Function as a Service<br>함수 단위 실행 | AWS Lambda, Azure Functions |
| **BaaS** | Backend as a Service<br>백엔드 기능 제공 | Firebase, AWS Amplify |

---

## 장단점

| 구분 | 내용 |
|:-----|:-----|
| **장점** | - 서버 관리 불필요<br>- 비용 효율성 (사용량 기반)<br>- 빠른 배포<br>- 자동 확장 |
| **단점** | - Cold Start 지연<br>- 벤더 종속성<br>- 디버깅 어려움<br>- 실행시간 제한 |
