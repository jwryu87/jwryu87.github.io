# 컨테이너 오케스트레이션 (Container Orchestration)
{: .fs-8 }

7.2 컨테이너
{: .label .label-green }

---

## 핵심 키워드

`서비스 디스커버리` `로드밸런싱` `클러스터링`

---

## 정의/개념

컨테이너 배포, 탐색, 스케일링, 관리, **동적 배치 기술**

---

## 기능

| 구분 | 기능 | 설명 |
|:-----|:-----|:-----|
| **탐색** | 서비스 디스커버리 (Service Discovery) | - 서비스 탐색 기능<br>- 클라우드 환경에서 컨테이너의 생성과 배치 이동 여부 알 수 없기에 IP, PORT 정보 업데이트 및 관리 통해 서비스 지원 |
| **스케일링** | 로드 밸런싱 (Load Balancing) | - 생성된 컨테이너의 컴퓨팅 자원 사용량의 설정 및 자동배분 |
|  | 스케줄링 (Scheduling) | - 증가된 컨테이너를 적합한 서버에 분배/배포<br>- 서버 다운 경우 실행 중인 컨테이너를 다른 서버에서 구동 |
|  | 클러스터링 (Clustering) | - 여러 개의 서버를 묶어 하나의 서버처럼 사용<br>- 가상 네트워크 이용하여 서버 연결 |
| **관리** | 로깅/모니터링 (Logging/Monitoring) | - 컨테이너 상태와 클러스터 내 호스트를 추적하고 감시<br>- 여러 개의 서버 동시 관리시 모니터링 및 로그 관리 |

- 사용자가 필요한 기능을 추가하여 다양한 형태의 **컨테이너 오케스트레이션** 기능을 사용가능
- 대표적인 오픈소스 플랫폼으로 **쿠버네티스**, **도커**, **아파치** 등이 있음

---

## 플랫폼 비교

| 구분 | 구글 Kubernetes | 도커 Swarm | 아파치 Mesos |
|:-----|:---------------|:-----------|:------------|
| **특징** | 다양한 테스트를 만족하는 안정적인 솔루션 | 사용이 용이한 솔루션 | UI수준이 높고 기능이 풍부하나 설치 및 관리가 어려운 솔루션 |
| **핵심요소** | 안정적 | 위울 | 화려함 |
| **기술지원** | CNCF와 긴밀히 협조, 기술자료 풍부 | 개념/기능의 긴결합, 기술자료 풍부 | MS와 Mesosphere가 적극 드라이브, 기술자료 부족 |
| **운영가능 호스트 수** | 1,000 nodes | 1,000 nodes | 10,000 nodes |
| **라이선스** | 아파치 | 아파치 | 아파치 |

- 이외에도 CoreOS fleet, AWS EC2, Redhat Cockpit 등 다양한 플랫폼 존재
- 쿠버네티스가 다른 플랫폼에 비해 규모가 큰 엔터프라이즈급의 컨테이너 관리에 좀 더 안정적인 환경을 제공하며, 오픈소스 기반의 클라우드 친화적인 기술 자료가 풍부
