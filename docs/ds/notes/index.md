### 💬 M멘토님 피드백

#### 일반 조언
- DS는 다양한 해설지로 공부해야 한다.
- 다양한 키워드를 사용해야 한다.
- 좀 더 답안지스럽게 쓰시면 좋겠다.
- **아는거 쓰는게 아니라 물어본거를 토픽스럽게 써야 함** ⭐ 이게 중요

#### 개별 문제 피드백

| 토픽 | 피드백 |
|:-----|:-------|
| **MODBUS** | 프로토콜의 구조에 집중해야 한다. 유형이 메인 컨텐츠가 될 수 없다. (역시 물어본 것에 집중하기. 해설지에 집중하지 않기. 생각하면서 공부하기. 암기랑은 구분할 것) |
| **A/B 테스팅** | A/B가 뭔지를 정의해야 한다. 테스팅에 집중해야 한다. 테스팅 → 절방산관기도 |
| **멀티미디어 스트리밍 프로토콜** | 종류만 쓰면 된다. 프로토콜은... |
| **엣지 컴퓨팅** | 1홉 |
| **서버리스** | cold start (오랜만에 수행 시에 시작하느라고 늦어지는 상황) |
| **CBDC** | 국가적 관점 |
| **영상압축기법** | 기법만 물어본게 아님. 원리, 핵심 구성요소도 써야 한다. (이것도 해설지가 답이 아니다. 차라리 AI에서 물어봐라). 한번 꼬아서 낸 문제. 상위 토픽을 생각해야 함 |
| **Open API** | 스펙 |
| **영지식증명** | 상위 토픽은 개인정보보호기술 |
| **쿠버네티스** | HPA, VPA 비교하는게 1교시형으로 나올 수 있다. |

#### XaaS 15점 받으려면
- 정의에 대해서 작성
- 클라우드 SLA에 대해서 써야 함
- 구축과정을 써야 함
- 클라우드 품질 지표
- 기반기술까지
- 해설지 봐라

#### 블록체인
- 해설지 봐라
- 절방산관기도 등과 다르게 물어보는 서비스에 맞게 비교항목자를 추가해서 작성해야 한다. **해당 토픽 키워드에 맞는 비교구분자** ⭐ 강조

#### 지속가능 스마트시티
- 스마트시티 한 판 정리할 것. City스럽게 써야 한다! (스마트 + 도시)
- **DS는 그 토픽 도메인스럽게 쓰는게 포인트이다. 최근 키워드들로 가야 함** ⭐
- 데이터 거버넌스 (해설지) 구분자에 앞에 '스마트 시티'라고 쓴다.

#### 기타
- SaaS 가이드라인도 정리하면 좋을 듯

### 💬 Y멘토님 피드백

#### 시험 분야별 특징
- 보안 문제가 훨씬 많다.
- **네트워크**: OSI 7 레이어. 각 계층의 기능. 와이파이, 5G/6G
- **DS의 서비스는 기술의 집약체!**

#### 보안 학습 포인트
- **기무가부인** - 1교시형으로 하나씩 쓸 수 있어야 함 (최상위 토픽들이므로)
- 공격(취약점) / 방어(취약점 제거, 탐지, 들어왔을때 방어) - 이걸 예방, 대응 측면에서 보안 프레임워크를 가지고 이어야 한다.
- **항상 마지막은 보안 프레임워크로 전체 프로세스 대비한다고 해야함**
- 프레임워크가 머리속에 그려져 있어야 함
- 암호화 알고리즘, 양자 암호화(최근 기사)

#### 보안 관련 인사이트
- 제로트러스트만 되면 되냐. 아니 쿠팡처럼 관리의 문제.
- 보안 왈. **착실한 관리자가 운영하는 시스템은 뚫기 어렵다.**
- GPT 자동화 등장 이후 보안 사고 엄청 많음
- ISMS-P 무용론. 인증, 심사는 그 시점에만 검사함.
- 품질은 100%가 없다. 미니멈.

#### 주요 토픽 키워드
- BPFdoor, 샌드박스/화이트박스, 제로데이
- SOAR
- 개인정보보호법
- PET 문제 다양하게 나오고 있음
- 보안 공시?
- 포렌식
- 클라우드 보안
- OPC UA - 스마트 팩토리와 연결
- GAN 적대적 공격. 인공지능 윤리, 보안 윤리

#### 답안 작성 팁
- **2교시형은 몰라도 앞과 뒤를 맞춰야지 된다. 채점자가 해설지를 옆에 놔두고 채점하는건 아니다.**
