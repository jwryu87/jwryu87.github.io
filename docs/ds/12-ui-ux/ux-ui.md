# UX/UI
{: .fs-8 }

12. UI/UX
{: .label .label-purple }

> 암기: UX/UI 조건 - **유사감** (유용성, 사용성, 감성)

---

## 정의

| 구분 | 정의 |
|:-----|:-----|
| **UX** | 사용자의 인지와 반응, 사용상 기대하는 것 |
| **UI** | UX를 기술로 처리한 것 |

---

## UX 기술요소

| 분류 | 기술 | 설명 |
|:-----|:-----|:-----|
| **SW 기술** | RIA | Rich Internet Application |
| | Search 2.0 | 차세대 검색 기술 |
| **HCI** | 멀티모달 | 다중 입력 방식 |
| | 햅틱 인터페이스 | 촉각 피드백 |
| **비즈니스 기술** | CEM | Customer Experience Management |
| | 웹 마이닝 | 웹 데이터 분석 |
| | QoE | Quality of Experience |

---

## UX/UI 조건 (유사감)

| 조건 | 설명 |
|:-----|:-----|
| **유용성** | 목적 달성에 도움이 되는지 |
| **사용성** | 쉽게 사용할 수 있는지 |
| **감성** | 감정적으로 만족스러운지 |

---

## UX/UI 개발 절차

| 단계 | 활동 |
|:-----|:-----|
| **목표 정의** | UX/UI 개발 목표 범위 |
| **프로젝트 계획** | UX 전략 수립 |
| **요구사항 정의** | 사용자 요구 → UX 컨셉 → 검증 |
| **설계 및 구현** | UI 상세설계 → GUI 화면 디자인 → UI/GUI 화면 검증 |
| **테스트** | 사용성 테스트 계획 → 실행 → 결과 분석 |
| **배포 및 관리** | 사용자 매뉴얼 → UX 피드백 |
