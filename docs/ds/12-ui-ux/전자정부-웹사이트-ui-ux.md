## 정의/개념

전자정부 웹사이트 **사용자 경험 제공 설계 및 구현, 진단 및 개선 가이드라인**

---

## 7가지 설계기준

| 설계기준 | 주요 활동 | 핵심 질문 |
|:-----|:-----|:-----|
| **1. 사용자에게 필요한 기능과 정보를 제공** | - 웹사이트를 이용자 정의<br>- 웹사이트 목적 정의<br>- 기능, 콘텐츠, 디자인 등 결정 | - 사용자가 원하는 정보와 기능을 제공<br>- 사용자가 목적을 달성할 수 있도록 충분한 양의 정보와 기능을 제공<br>- 중요한 정보와 기능을 효과적이며 이해하기 쉽게 표현 |
| **2. 작업에 소요되는 시간과 단계를 최소화** | - 사용자 여정 지도 작성 및 개선<br>- 요청작업의 시간과 절차의 측정 지표 정의<br>- 가장 낮은 수준의 디지털 환경과 사용자 기술 역량을 기준 측정 | - 웹사이트는 사용자가 요청한 작업에 대해 적절한 속도로 반응<br>- 사용자가 원하는 정보 및 기능에 빠르게 접근<br>- 하나의 과업을 완료하기 위해, 동일한 작업을 여러 번 반복하지 않아야함<br>- 작업 수행에 불필요한 단계가 포함 유무 |
| **3. 직관적이고 일관성 있게 개발** | - 설계 가이드 등을 프로젝트 초기에 배포 및 설명서 기준 검수 시행<br>- UI를 사용하는 방법에 대한 설명이 필요하다면, 설명을 추가하는 것보다 UI를 수정<br>- 이용 경험이 없는 사용자도 원하는 목적을 달성하였는지 테스트 | - 사용자가 이해하기 어려운 단어와 문장을 사용<br>- 용어, 스타일, 레이아웃, 기능의 작동 방식 등을 전체 페이지에서 일관성 있게 적용<br>- 정보의 표현과 기능의 작동 방식은 일반적인 사용자의 기대에 부합<br>- 사용자의 행동에 도움이 되는 단서 및 정보를 적절한 시기와 위치에 제공 |
| **4. 사용자가 원하는 방식으로 이용할 수 있게 개발** | - 사용자가 실제로 어떻게 사용하고 있는지 조사 및 관찰<br>- 빈번하게 사용하는 과업에 대해 다양한 방법과 절차를 조사하고 점검 | - 사용자의 행동과 웹사이트의 반응 사이에 불필요한 요소 제거<br>- 사용자는 원하는 입출력 도구를 자유롭게 사용<br>- 사용자가 선호에 따라 웹사이트의 상태 및 설정을 변경 가능<br>- 사용자에게 다양한 맞춤형 정보를 제공 |
| **5. 사용자가 실수하지 않게 개발** | - 사용자 관찰을 통해, 도움이 필요한 상황을 정의<br>- 다양한 피드백 수집 및 체크리스트 관리<br>- 오류를 가장 높게 우선순위 선정 및 신속한 개선 | - 사용자의 실수를 유발하는 요소 유무<br>- 심각한 결과를 유발하는 기능의 실행여부를 충분에게 확인<br>- 사용자가 오류를 빠르게 인지할 수 있도록 표현<br>- 동작을 취소하거나 이전 상태로 돌아갈 수 있는 수단이 제공 |
| **6. 모든 사용자가 이용할 수 있게 개발** | - 보조 기술 검토 및 사용법 이해<br>- 정기적 접근성 테스트 실시 | - 저사용방식에 대한 주요 장애, 제약 유형 정의<br>- 장애가 있는 사용자에게 비장애와 동등한 수준의 정보와 기능을 제공<br>- 다양한 웹 브라우저에서 모든 기능을 동등하게 이용<br>- 초보 사용자와 숙련된 사용자가 웹사이트에서 제공하는 모든 기능을 올바르게 이용 |
| **7. 원하는 서비스와 정보를 쉽게 찾을 수 있게 개발** | - 사용대역터 기반 사용자의 목적과 이용패턴 세분화<br>- 빈번하게 검색하는 항목 식별<br>- 정보를 더 쉽게 찾을 수 있는 방법을 먼저 고려 | - 사용자가 원하는 페이지나 위치로 이동할 수 있는 여러 가지 수단을 제공<br>- 일반적으로 자료는 내비게이션 모형을 적절하게 구현<br>- 중요한 정보와 다른 정보 간 위계를 명확하게 구분 |

---

## 설계기준별 핵심 포인트

| 번호 | 설계기준 | 핵심 포인트 |
|:-----|:-----|:-----|
| 1 | 필요한 기능과 정보 제공 | 사용자 목적 달성을 위한 충분한 정보와 기능 |
| 2 | 시간과 단계 최소화 | 적절한 반응 속도, 빠른 접근, 반복 최소화 |
| 3 | 직관적/일관성 | 용어/스타일/레이아웃 일관성, 사용자 기대 부합 |
| 4 | 원하는 방식 이용 | 다양한 입출력 도구, 맞춤형 설정 |
| 5 | 실수 방지 | 오류 유발 요소 제거, 취소/복구 기능 |
| 6 | 모든 사용자 이용 | 접근성, 보조 기술 지원 |
| 7 | 쉬운 검색/탐색 | 다양한 탐색 수단, 정보 위계 구분 |
