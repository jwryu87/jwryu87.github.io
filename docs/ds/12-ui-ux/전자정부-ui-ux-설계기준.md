# 전자정부 UI/UX 설계기준
{: .fs-8 }

12. UI/UX
{: .label .label-purple }

> 암기: 설계기준 7가지 - **최원일 실모찾기** / 품질요인 4가지 - **호접개최**

---

## 정의

전자정부 웹사이트의 사용자경험 특성에 대한 설명과 설계 시 점검 및 수행해야 할 사항을 정의한 **가이드라인**

- 표준화된 사용자 경험 가이드 제공
- 전자정부서비스를 효율적으로 제공할 수 있는 사용자경험 설계 지원
- 사용자들에게 최적의 사용경험 제공

---

## 설계기준 7가지 (최원일 실모찾기)

| 기준 | 설명 |
|:-----|:-----|
| **최소화** | 작업에 소요되는 시간과 단계 최소화 |
| **원하는 방식** | 사용자가 원하는 방식으로 이용 가능하게 개발 |
| **일관성** | 직관적이고 일관성 있게 개발 |
| **실수 방지** | 사용자가 실수하지 않게 개발 |
| **모든 유형** | 모든 유형의 사용자가 이용할 수 있게 개발 |
| **찾기 쉬움** | 원하는 서비스와 정보를 쉽게 찾을 수 있게 개발 |
| **기능/정보** | 필요 기능과 정보 제공 |

---

## 4개 품질요인 진단지표 (호접개최)

| 품질요인 | 세부 지표 | 설명 |
|:---------|:---------|:-----|
| **웹호환성** | 문호비 | 웹 표준 문법 준수, 웹 호환성 확보, 비표준 기술 제거 |
| **웹접근성** | 인운이견 | 인식 용이성, 운용 용이성, 이해성, 견고성(내구성) |
| **웹개방성** | 접로수U | 특정 검색엔진 접근 차단, 검색 로봇 접근 차단, 페이지 정보 수집 거부, 페이지별 URL 차단 |
| **웹최적화** | 응크 | 웹사이트 응답 속도 최소화, 웹사이트 링크 관리 |

---

## 웹접근성 4대 원칙 (인운이견)

| 원칙 | 영문 | 설명 |
|:-----|:-----|:-----|
| **인식 용이성** | Perceivable | 모든 콘텐츠를 인식할 수 있도록 |
| **운용 용이성** | Operable | 모든 기능을 운용할 수 있도록 |
| **이해성** | Understandable | 콘텐츠를 이해할 수 있도록 |
| **견고성** | Robust | 다양한 기술에서 동작하도록 |
