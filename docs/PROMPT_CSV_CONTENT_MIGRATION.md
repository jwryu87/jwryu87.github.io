# 📋 CSV 기반 콘텐츠 마이그레이션 프롬프트

## 🎯 목적
CSV 파일의 토픽 데이터를 Jekyll 블로그의 Markdown 페이지에 자동으로 추가/병합하는 작업

---

- CSV 파일을 다운로드 폴더에 저장
- 예: `/Users/jaewoo.ryu/Downloads/기술사_기본필수노트_{토픽명}.csv`

### 2. 프롬프트 실행
아래 프롬프트를 복사하여 사용하세요:

## 🤖 프롬프트 템플릿

```
@{CSV_파일_경로} 이 CSV 파일의 내용을 {블로그_폴더_경로} 폴더의 각 해당 페이지에 추가해줘.

작업 규칙:
1. CSV의 각 행을 해당하는 마크다운 파일에 매핑
2. 빈 페이지는 CSV 내용으로 새로 작성
3. 기존 내용이 있는 페이지는 CSV 내용을 병합 (중복 방지)
4. 암기팁(두음)이 있으면 상단에 > 블록으로 표시
5. 표 형식으로 정리하고, 구조화된 마크다운 사용
6. 작업 완료 후 커밋

카테고리별로 순차적으로 진행하고, 완료되면 알려줘.
```

## 📊 CSV 파일 형식

### 필수 컬럼
| 컬럼명 | 설명 | 예시 |
|:-------|:-----|:-----|
| `NO` | 항목 번호 | 1, 2, 3... |
| `토픽` | 대분류 | 클라우드, 블록체인 |
| `중토픽` | 소분류/파일명 매핑 | 오픈스택, CASB |
| `기술요소` | 실제 콘텐츠 | 정의, 구성요소, 특징 등 |

### 콘텐츠 작성 규칙 (CSV 내)
```
[섹션명] 암기팁
내용1 = 설명1
내용2 = 설명2
```

예시:
```
[구성요소] 우제사
우주부 = GPS 위성, 원자 시계
지상제어부 = 주관제소, 지상안테나
사용자부 = 비행기, 선박, 자동차
```

## 🔄 상세 작업 프로세스

### Step 1: 폴더 구조 확인
```
대상 폴더의 파일 목록 확인
├── index.md
├── 토픽1.md
├── 토픽2.md
└── ...
```

### Step 2: CSV 내용 검색
```python
# CSV에서 관련 키워드로 검색
keywords = ['관련 키워드1', '관련 키워드2', ...]
```

### Step 3: 파일 상태 확인
| 상태 | 처리 방법 |
|:-----|:---------|
| **빈 페이지** | CSV 내용으로 전체 작성 |
| **내용 있음** | CSV 내용 중 없는 섹션만 병합 |

### Step 4: 마크다운 변환 규칙

#### 암기팁 표시
```markdown
> 암기: {섹션명} - **{암기팁}** ({풀이})
```

#### 정의
```markdown
## 정의

{정의 내용}
```

#### 표 형식
```markdown
| 구분 | 설명 |
|:-----|:-----|
| **항목1** | 설명1 |
| **항목2** | 설명2 |
```

#### 프로세스/절차
```markdown
```
단계1 → 단계2 → 단계3 → 단계4
```
```

## 🔧 트러블슈팅

### CSV 파일을 찾을 수 없는 경우
- 파일 경로 다시 확인
- 파일명에 특수문자나 공백 확인

### 파일명 매핑이 안 되는 경우
- CSV의 `중토픽`과 마크다운 파일명 비교
- 한글/영문 변환 확인

### 내용이 중복 추가되는 경우
- 기존 파일 내용을 먼저 확인하도록 요청
- "병합" 키워드 명시
