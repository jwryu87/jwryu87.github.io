# 파라미터, 하이퍼파라미터
{: .fs-8 }

머신러닝 프로세스
{: .label .label-blue }

> 암기 (튜닝방법): **맥그리배** (Manual, Grid, Random, Bayesian)

---

## 핵심 키워드

- **파라미터**: 최적화 모델, 서포트 벡터, 선형회귀 결정계수
- **하이퍼파라미터**: 최적화 파라미터, 경사하강법, 활성화 함수

---

## 정의/개념

- **파라미터**: 주어진 데이터로부터 학습을 통해 모델 내부에서 결정되는 변수
- **하이퍼파라미터**: 최적 모델링 위해 사용자가 직접 설정해주는 변수

---

## 파라미터 vs 하이퍼파라미터

| 비교 | 파라미터 | 하이퍼파라미터 |
|:-----|:--------|:-------------|
| **목적** | 최적화 모델 구현 | 최적화 파라미터 값 도출 |
| **생성주체** | 데이터 학습한 모델이 생성 | 사용자 판단 통해 생성 |
| **유형** | - 인공신경망 가중치<br>- SVM 서포트 벡터<br>- 선형회귀 결정계수 | - 학습률<br>- 경사하강법 반복 횟수<br>- 활성화 함수 |
| **조정** | 파라미터 임의 조정 불가 | 하이퍼파라미터 조정 가능 |

---

## 하이퍼파라미터 튜닝방법

```
    ┌─────────────────┐
    │   탐색           │        ┌─────────────────┐
    │   체계성 부여    │───────▶│  Manual Search   │    - 휴리스틱 기반 방식
    └────────┬────────┘        └─────────────────┘    - 탐색의 단순성
             │
             ▼
    ┌─────────────────┐        ┌─────────────────┐
    │   불필요         │───────▶│   Grid Search    │    - 모든 조합 탐색
    │   반복횟수 개선   │        └─────────────────┘    - 균등 탐색
    └────────┬────────┘
             │
             ▼
    ┌─────────────────┐        ┌─────────────────┐
    │   추출 범위 개선  │───────▶│  Random Search   │    - 랜덤샘플링
    └────────┬────────┘        └─────────────────┘    - 최대값-최소값 부여
             │
             ▼
    ┌─────────────────┐        ┌─────────────────┐
    │                 │───────▶│    Bayesian      │    - 기존 추출 기반 탐색
    └─────────────────┘        │   Optimization   │    - Bayesian theory
                               └─────────────────┘
```

> 학습의 규모가 커질수록 탐색 시간 기준 Bayesian Optimization 이 가장 뛰어나지만 생성하려는 모델 수준 등을 고려할 때 상황에 맞는 튜닝방법을 선택하는 것이 핵심

---

## 튜닝방법 상세

| 방법 | 설명 | 장단점 |
|:-----|:-----|:-------|
| **Manual Search** | 휴리스틱 기반 수동 탐색 | 간단하지만 비효율적 |
| **Grid Search** | 모든 조합을 균등 간격으로 탐색 | 철저하지만 시간 소모 |
| **Random Search** | 무작위 조합 탐색 | 효율적, 최적해 보장 안됨 |
| **Bayesian Optimization** | 이전 결과 기반 확률적 탐색 | 가장 효율적, 복잡 |
