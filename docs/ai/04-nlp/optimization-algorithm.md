## 알고리즘 유형별 특징

| 구분 | 알고리즘 | 특징 | 장단점 |
|:-----|:--------|:-----|:-------|
| **모든 상황 미고려** | Greedy | 매 순간 최적의 선택 | 빠름, 전역 최적해 보장 안됨 |
| | Beam Search | 상위 K개 후보 유지 | 탐색 공간 제한, NLP 디코딩에 활용 |
| **모든 상황 고려** | Dynamic Programming | 부분 문제 해결 후 결합 | 최적해 보장, 메모리 사용 많음 |
| | Divide and Conquer | 분할 정복 | 병렬 처리 가능, 재귀 오버헤드 |

---

## 알고리즘별 상세 설명

### 1. Greedy (탐욕 알고리즘)

| 항목 | 설명 |
|:-----|:-----|
| **원리** | 각 단계에서 현재 상황에서 가장 좋은 선택 |
| **특징** | 지역 최적해 선택, 전역 최적해 보장 안됨 |
| **활용** | 허프만 코딩, 최소 신장 트리 |

### 2. Beam Search

| 항목 | 설명 |
|:-----|:-----|
| **원리** | 각 단계에서 상위 K개(beam width) 후보만 유지 |
| **특징** | 탐색 공간 제한으로 효율적 |
| **활용** | 기계 번역, 음성 인식, 텍스트 생성 |

### 3. Dynamic Programming (동적 계획법)

| 항목 | 설명 |
|:-----|:-----|
| **원리** | 큰 문제를 작은 부분 문제로 나누어 해결 |
| **특징** | 중복 계산 방지(Memoization), 최적 부분 구조 |
| **활용** | 최단 경로, 배낭 문제, 문자열 비교 |

### 4. Divide and Conquer (분할 정복)

| 항목 | 설명 |
|:-----|:-----|
| **원리** | 문제를 분할 → 정복 → 결합 |
| **특징** | 재귀적 해결, 병렬 처리 용이 |
| **활용** | 병합 정렬, 퀵 정렬, FFT |

---

## NLP에서의 활용

| 알고리즘 | NLP 활용 사례 |
|:---------|:-------------|
| **Greedy** | 문장 생성 시 가장 높은 확률의 단어 선택 |
| **Beam Search** | 번역, 요약에서 다양한 후보 문장 탐색 |
| **DP** | 형태소 분석, 품사 태깅(비터비 알고리즘) |
| **Divide and Conquer** | 병렬 텍스트 처리 |
