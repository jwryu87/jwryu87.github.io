# 인공지능(AI) 개인정보보호 자율점검표

인공지능 윤리
{: .label .label-red }

---

## 핵심 키워드

`개인정보보호법` `Privacy by Design` `인공지능 윤리`

---

## 구성 체계

AI 개인정보보호는 **법규**, **설계**, **윤리** 세 가지 관점에서 자율점검을 수행합니다.

---

## 법규 (개인정보보호법)

| # | 개인정보 보호법 원칙 |
|:--|:-------------------|
| 1 | 수집 제한 원칙 |
| 2 | 정보 정확성 원칙 |
| 3 | 목적 명확화 원칙 |
| 4 | 이용제한의 원칙 |
| 5 | 안전성 확보 원칙 |
| 6 | 공개 원칙 |
| 7 | 개인 참가 원칙 |
| 8 | 책임 원칙 |

---

## 설계 (Privacy by Design)

| # | 개인정보 중심 설계(PbD) 원칙 |
|:--|:--------------------------|
| 1 | 사후 조치 아닌 사전 예방 |
| 2 | 초기 설정부터 프라이버시 보호 조치 |
| 3 | 프라이버시 보호 내재한 설계 |
| 4 | 프라이버시 보호와 사업기능 균형 |
| 5 | 개인정보 생애주기 전체에 대한 보호 |
| 6 | 개인정보 처리과정 투명성 유지 |
| 7 | 이용자 프라이버시 존중 |

---

---

## 자율점검 체크리스트

### 데이터 수집 단계

| 점검 항목 | 점검 내용 |
|:---------|:---------|
| 수집 동의 | 개인정보 수집 시 명시적 동의 획득 |
| 목적 명시 | 수집 목적의 명확한 고지 |
| 최소 수집 | 필요 최소한의 정보만 수집 |

### 데이터 활용 단계

| 점검 항목 | 점검 내용 |
|:---------|:---------|
| 목적 외 사용 금지 | 수집 목적 외 활용 제한 |
| 익명화/가명화 | 개인 식별 불가 처리 |
| 접근 통제 | 권한 기반 데이터 접근 관리 |

### AI 모델 개발 단계

| 점검 항목 | 점검 내용 |
|:---------|:---------|
| 편향성 검토 | 학습 데이터의 편향 여부 점검 |
| 설명가능성 | AI 의사결정 과정의 설명 가능 여부 |
| 보안 적용 | 모델 및 데이터 보안 조치 |
