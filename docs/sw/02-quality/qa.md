# 품질보증(QA), 품질통제(QC)
{: .fs-8 }

2.1 소프트웨어 품질 일반
{: .label .label-purple }

---

## 핵심 키워드

**품질보증(QA)**: `생산성 향상` `품질 문제 조기 발견` `Review` `Inspection` `Validation` `IEEE 610`

**품질통제(QC)**: `인도물 중심 평가` `무결성` `정확성` `완전성 부합 검증` `파레토` `체크시트` `히스토그램`

---

## 정의/개념

| 구분 | 정의 |
|:-----|:-----|
| **품질보증(QA)** | SW 제품이나 아이템이 정해진 요구에 적합하다는 것을 보장하는데 필요한 계획적이고 체계적인 활동 |
| **품질통제(QC)** | 품질 허용 범위 내에 들어오도록 산출물 또는 제품 대상으로 지표를 관리하고 통제하는 활동 |

---

## 품질보증(QA : Quality Assurance)

| 구분 | 설명 |
|:-----|:-----|
| **개념** | SW 제품이나 아이템이 정해진 요구에 적합하다는 것을 보장하는데 필요한 계획적이고 체계적인 활동 (IEEE 610, 1991) |
| **ITTO** | Inputs → Tools & Techniques → Outputs |
| **Inputs** | - 프로젝트 관리 계획<br>- 품질지표<br>- 작업성과 정보<br>- 품질통제 측정치 |
| **Tools & Techniques** | - 품질감사<br>- 프로세스 분석<br>- 품질계획 수립 및 품질통제 수행 도구 및 기법 |
| **Outputs** | - 변경요청<br>- OPA 갱신<br>- 프로젝트 계획 갱신<br>- 프로젝트 문서 갱신 |
| **필요성** | 생산성 향상, 제품의 확장성 지원<br>품질 문제점의 조기 발견 및 제거, 재사용성 증가 |
| **주요 도구** | Management Review, Technical Review, Inspection, Walk-Through<br>Validation & Verification, 시험평가, 표준화 검사 |

- 품질에 영향을 주는 작업 규정이 준수되며 QC 활동이 적절히 수행되는지 감시하는 프로세스

---

## 품질통제(QC : Quality Control)

| 구분 | 설명 |
|:-----|:-----|
| **개념** | 품질 허용 범위 내에 들어오도록 산출물 또는 제품 대상으로 지표를 관리하고 통제하는 활동, 산출물 중심의 검사를 중심으로 수행 |
| **ITTO** | Inputs → Tools & Techniques → Outputs |
| **Inputs** | - 프로젝트 관리 계획<br>- 품질지표<br>- 작업성과 측정치<br>- 승인된 변경요청<br>- 인도물<br>- 조직 프로세스 자산 |
| **Tools & Techniques** | - 인과관계도<br>- 관리도<br>- 히스토그램<br>- 파레토차트<br>- 산점도<br>- 승인된 변경요청 검토 |
| **Outputs** | - 품질 통제 측정치<br>- 확인된 변경요청<br>- 확인된 인도물<br>- OPA 갱신<br>- 프로젝트 관리계획 갱신<br>- 프로젝트 문서 갱신 |
| **필요성** | 오류 및 누락의 확인 및 설명<br>자료의 무결성, 정확성, 완전성을 보장 |

### QC 주요 도구 (7가지 기본도구)

| 번호 | 도구 | 설명 |
|:-----|:-----|:-----|
| 1 | 특성요인도 | 문제의 원인 전체 정리 |
| 2 | 파레토 다이어그램 | 문제의 중점화, 우선순위 매김 |
| 3 | 체크시트 | 데이터 수집 및 문제 분석을 효과적으로 실시 |
| 4 | 산점도 | 요인간의 관계를 파악 |
| 5 | 히스토그램 | 데이터 산포 상태 파악 |
| 6 | 층별 | 문제의 발생 원인 구분 |
| 7 | 그래프 | 데이터의 정리 및 정보 파악을 위해 사용 |

- 제작된 인도물이 인수 기준에 부합하는지 검증하는 과정
