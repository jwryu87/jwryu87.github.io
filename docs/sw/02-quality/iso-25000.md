## 핵심 키워드

`품질 관리` `품질 모델` `품질 측정` `품질 요구사항` `품질 평가` `확장`

---

## 정의/개념

소프트웨어 개발 공정 각 단계, 산출 제품, 사용자 요구 만족, 품질 측정, 평가 통합 국제표준

- **SQuaRE (S/W Quality and Requirement Evaluation)**
- 소프트웨어 개발 공정 각 단계에서 산출되는 제품이 사용자 요구를 만족하는지 검증하기 위해 품질 측정과 평가를 위한 모델, 측정기법, 평가 방안에 대한 통합한 국제표준

## 구성요소

### ISO/IEC 25000의 Division 구성요소

| 구분 | 설명 |
|:-----|:-----|
| **Quality Management** | - 소프트웨어 품질 관리 지침 정의<br>- 2500n: 구 표준 14598-2 대응 |
| **Quality Model** | - 소프트웨어 품질 모델 제시<br>- 25010: 구 표준 9126-1 해당<br>- 25012: 신규 추가 표준 |
| **Quality Measurement** | - 소프트웨어 품질 측정 Metrics 정의<br>- 2502n: 구 표준 9126-2/3/4 대응 |
| **Quality Requirement** | - 소프트웨어 품질 요구사항 설정 프로세스<br>- 25030: ISO15288(System Life Cycle Process)를 참조한 신규 표준 |
| **Quality Evaluation** | - 소프트웨어 품질 평가 정의 및 지침<br>- 2504n: 구 표준 14598 대응 |
| **Extension** | - 25051: 제품으로서 소프트웨어(COTS: Commercial Off-The-Shelf) 품질 요구 사항 및 테스트 요건의 정의, 구 표준 12119 대응 |

- ISO 25000의 적절한 활용을 통해 S/W 품질 평가의 편리성, 신뢰성, 명확성을 제고하고 국제적 경쟁력 확보
