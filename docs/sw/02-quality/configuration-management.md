## 정의/개념

SW 개발과정의 형상 항목을 식별하고 기록과 변경 제어를 하고 요구 사항에 부합하는지 검증하는 활동

---

## 메커니즘

```
형상관리기법
    │
┌───┴───────────────────────────────────────────────────────────────┐
│                                                                   │
│   형상식별 ──► 형상대상 ──► 형상항목 ──► CCB ──► 변경승인        │
│      │           │            │                                   │
│      ▼           ▼            ▼                                   │
│   형상통제 ──► 등재기준 ──► Baseline ──► 변경확인 ──► 변경허용   │
│      │                                               │            │
│      ▼                                               ▼            │
│   형상감사 ──────────────► Configuration ◄──────── 변경실시       │
│      │                     Management                             │
│      ▼                     System                                 │
│   형상기록 ──────────────►    │                                   │
│                              ▼                                    │
│                          Repository                               │
│                          (SVN, Git)                               │
└───────────────────────────────────────────────────────────────────┘
```

### 절차 및 세부절차

| 절차 | 세부절차 |
|:-----|:--------|
| **형상관리 준비** | - 수행 계획 정의 활동<br>- 형상 관리 표준, 절차 기술 |
| **형상 식별** | - 형상 항목의 정의 및 선정<br>- 기준선, 참조 등 세부 사항 식별 |
| **형상 통제** | - 변경 요청에 따른 변경 심사 및 실시, 확인 |
| **형상 감사** | - 체크리스트 기반 감사<br>- 결과 문서화 |
| **형상 기록** | - Repository 기록 |

## 기준선

| SDLC | 기준선 | 형상형목 |
|:-----|:-------|:--------|
| **계획 단계** | 기능적 기준선 | 프로젝트 계획서, 개발 표준 및 프로세스 |
| **요구분석 단계** | 분배적 기준선 | 요구사항 정의서, 기능분해도, 작업 흐름도, 자료흐름도 |
| **설계 단계** | 설계 기준선 | - 기본 설계: 화면 보고서, 명세서<br>- 상세 설계: ERD, 아키텍처 설계서, 프로그램 설계서 |
| **구현 단계** | 시험 기준선 | 원시 코드, 목적 코드, 실행 코드, 단위 시험 보고서 |
| **시스템 통합 및 시험 단계** | 제품 기준선 | - 통합 시험: 통합 시험 계획서, 케이스<br>- 시스템 시험: 계획서, 케이스 및 보고서 |
| **설치 및 운영 단계** | 운용 기준선 | 운영자 지침서, 사용자 지침서, 이관 소스 |
