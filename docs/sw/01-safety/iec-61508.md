# IEC 61508
{: .fs-8 }

1.2 안전 표준 & 기법
{: .label .label-yellow }

> 암기: **기일전소/정안지기**

---

## 핵심 키워드

`안전 요구사항` `기능안전성` `SIL` `전기/전자/프로그램 가능 전자시스템`

---

## 정의/개념

전기/전자/프로그램 가능한 전자 안전 관리 시스템의 기능 안전으로 산업에 적용되는 규칙 국제표준

---

## 구성요소

### IEC 61508의 구성

| 구분 | 핵심 | 설명 |
|:-----|:-----|:-----|
| **Part-0** | 기능안전성과 IEC61508 | 기능안전의 개념과 파트별 개요 |
| **Part-1** | 일반 요구사항 | 안전 수명주기에 따른 목적, 적용범위, 입력 및 산출물로 전체 프레임워크를 정의 |
| **Part-2** | 전기/전자/프로그램 가능한 전자장치 안전 관련 시스템의 요구사항 | 안전 요구사항을 결정하기 위한 다양한 기법의 적용 및 SIL의 등급화 기술 |
| **Part-3** | 소프트웨어 요구사항 | 소프트웨어에 적용되는 안전기능과 SIL에 대하여 명시 |
| **Part-4** | 정의와 약어 | 사용되는 용어에 대한 정의 및 설명 |
| **Part-5** | 안전무결성 수준 결정 방법 예시 | 리스크와 안전무결성의 개념 및 관계를 명시하고, 안전무결성 확보 위한 방법론들에 관하여 예시를 들어 소개 |
| **Part-6** | IEC 61508의 Part 2와 Part 3의 적용 지침 | 하드웨어 고장 확률 평가 기법, 진단 범위 및 안전 고장비율계산, 소프트웨어 안전무결성에 관하여 명시 |
| **Part-7** | 기법과 수단의 개요 | Part 2 및 Part 3과 관련된 다양한 안전기법에 관한 설명 제공 |

---

## 요구사항 도출과정

### IEC 61508의 안전기능 요구사항 도출 과정 및 상세설명

| 단계 | 절차 | 설명 |
|:-----|:-----|:-----|
| **정의** | 1. 개념 | 시스템 및 기능에 대한 개념 정의 |
| | 2. 범위정의 | 전체 범위 정의 |
| **분석** | 3. 위험원 및 위험 분석 | 위험요인(Hazard)과 이에 따른 위험성(Risk) 분석 |
| | 4. 안전 요구사항 명세 | 전반적 안전 요구사항 도출 |
| **도출** | 5. 안전 요구사항 할당 | 안전 기능의 할당, 목표장 기준과 무결성 수준 할당 |
| | 다른 methods 탐색 | 정의된 methods 리스트 내 다른 방법 탐색 |
| **확정** | 6. 안전 기능별 요구사항 명세 | 안전기능 요구사항들의 명세화 |

- 요구사항 도출은 Part-1에 정의 되어 있고 총 6단계로 수행
- 산업 특성에 맞는 기능안전 등급, 분석 절차 필요성으로 표준 세분화
