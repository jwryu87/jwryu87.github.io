# 연동 계획 (Rolling Wave Planning)

3.1 일정/범위/위험 관리
{: .label .label-purple }

> 암기: **시계관산단종**

---

## 핵심 키워드

`점진적 구체화` `장기적` `대략적` `일정관리 활동정의` `100% Rule 원칙`

---

## 정의/개념

가까운 미래에 완료할 작업은 상세하게 계획하고, 과업이 불분명하거나 장기적인 작업은 높은 수준에서 대략적으로 계획하는 프로젝트 관리 기법

---

## 프로세스

### 연동 계획 (Rolling Wave Planning) 절차

| 절차 | 세부특징 | 설명 |
|:-----|:---------|:-----|
| **프로젝트 시작** | - 프로젝트 목표 파악 및 문서화<br>- 의사결정 진행 | - 신규 개발의 미션, 기능, 문제 분석 설명<br>- 의사결정에 필요한 정보 수집 및 제공 |
| **계획 & 기획** | - 요구사항 수집 및 문서화<br>- 백로그, 단위 목표 수립 | - 기능, 비기능, 제약사항 수집 및 문서화<br>- 기능간에 Activity 관계 및 연결 |
| **프로젝트 관리 & 제어** | - 프로세스 관리 및 제어<br>- 자원 투입 및 관리 | - 단위 수행 별로 상세 관리<br>- 칸반보드 활용한 진척사항 관리 |
| **산출물 제공 & 관리** | - 개발 활동 관리 및 제어<br>- 산출물 점검, 회고 | - 관리 프로세스에 따른 활동 제어<br>- 지속적 산출물 관리 및 제어 |
| **단계 마무리** | - 다음 단계 고도화<br>- 추가, 변경 사항 관리 | - 활동 간 자원, 일정 조정<br>- 변경 사항 문서화 및 협의 |
| **프로젝트 종료** | - 프로젝트 리뷰 및 보고<br>- 최종 작업 종료 | - 프로젝트 지속성 여부 판단<br>- 종료 작업 보고서 작성 및 보고 |

- 연동기획은 Progressive Elaboration을 통해 프로젝트 구체화
- Rolling Wave Planning 작성 시 100%Rule 원칙을 적용하여, 모든 작업이 포함되도록 수행

---

## 작성 원칙

### 연동 계획 (Rolling Wave Planning) 작성 원칙

| 원칙 | 설명 |
|:-----|:-----|
| **관리 가능한 크기** | - 자원/일정 산정 및 할당 가능 크기 단위<br>- 내/외부 의사소통 가능한 단위로 작업을 분할 |
| **단위 별 레벨 구분** | - 통상적 분할은 적정수준 3~5 단계 권장 |
| **Work Package 적정한 크기** | - 전체 작업의 1~10%정도의 크기<br>- 80M/H 또는 10일(2주) 단위 크기로 스프린트 규모 |
| **100% Rule 적용** | - 레벨 별 작업의 총합은 100%<br>- WBS 작성 Rule과 스크럼 방법론을 적용한 형태로 변화 발전 |
