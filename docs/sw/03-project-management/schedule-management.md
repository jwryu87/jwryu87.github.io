## 핵심 키워드

**일정 추정 기법**: `경험(유사산정)` `통계(3점산정)` `협의(미팅)`

**일정 단축 기법**: `시점(Crashing, Fast Tracking)` `제약(Critical Chain)` `식별(What-If)`

---

## 정의/개념

프로젝트를 성공적으로 완수하고 납기를 준수할 수 있도록 개발 기간을 단계별로 철저히 계획하여 관리하는 프로젝트 관리 영역

---

## 프로세스

### 프로젝트 일정관리 프로세스의 상세 설명 (계정순산계통)

| 프로세스 | 설명 | 산출물 |
|:---------|:-----|:-------|
| **일정관리 계획수립** | 정책, 절차 문서화, 기준수립 | 일정관리 계획서 |
| **활동정의** | 인도물 생산을 위한 수행활동 문서화 | 활동목록, 활동속성, 마일스톤, 목록 |
| **활동순서 배열** | 프로젝트 활동사이 관계 식별 (FS관계) | 프로젝트 일정 네트워크 |
| **활동기간 산정** | 산정된 자원의 개별 활동 완료하는데 필요한 총 작업기간 단위수를 산정 | 활동기간 산정치 |
| **일정 개발** | 활동순서, 기간, 자원 요구사항 및 일정 제약을 분석하여 프로젝트 일정 모델을 생성 | 일정기준선, 프로젝트 일정 |
| **일정 통제** | 계획 달성위한 활동 감시, 진행을 갱신하고 일정기준선 일정예측, 변경요청, 프로젝트문서 갱<br>변경/관리 |

## 일정 추정과 일정 단축 기법 설명

### 일정 추정 기법

| 구분 | 핵심 기술 | 설명 |
|:-----|:---------|:-----|
| **경험 기반** | 전문가 판단<br>(Expert Judgment) | 과거 유사 프로젝트 수행 정보를 활용하는 전문가를 통해 활동 기간을 산정하는 방법 |
| | 유사 산정<br>(Analogous Estimating) | 프로젝트의 기간, 규모, 복잡성 같은 특성을 고려하여 과 유사한 프로젝트의 실제 기간을 참조하여 산정하는 방법 |
| **산술 통계 기반** | 모수 산정<br>(Parametric Estimating) | 수집 또는 축적된 과거 경력 데이터를 기반으로 수학적 입수를 정하여 산출하는 방법 |
| | 3점 산정<br>(Three Point Estimating) | 낙관치(Optimistic), 비관치(Pessimistic), 평균치(Most likely) 산정 건의 편균을 계산현재 기간을 산정하는 방법<br>- 삼각분포 Triangular Distribution : tE = (tO + tM + tP) / 3<br>- 베타분포 Beta Distribution : tE = (tO + 4tM + tP) / 6<br>* t : Time |
| | 상향식 산정<br>(Bottom-Up Estimating) | WBS 분류 체계(WBS)의 활동별로 기간을 산정한 뒤, 포괄 산정치를 집계하는 방법 |
| | 데이터 분석<br>(Data Analysis) | 대안 분석 : 자원, 수용 구입, 지재 개발, 워크 개발 등 여러가지 도구를 비교하여 대안 분석을 통해 프로젝트 활동기간 결정<br>예비 분석 : 기간 산정의 불확실성을 대비해 위험을 고려 하여 우발사태를 대비하는 기간(버퍼)를 전체 일정에 포함 하는 방법 |
| **협의 기반** | 의사결정<br>(Decision Making) | 만장일치, 과반수, 다수결, 단독 결정 등의 기법 사용<br>투표 방법의 예로 손가락 기수법(Fast to Five) 이 있음<br>(엄지면대 : 주먹, 강한찬성 : 손가락5개를 펼) |
| | 미팅<br>(Meeting) | 팀원들이 참여하는 미팅을 통해 활동 기간을 산정<br>애자일 프로세스에서 활용하고 있는 협의 기반의 일정산정 기법인 플래닝 포커 등의 기법이 있음 |

### 일정 단축 기법

| 구분 | 핵심 기술 | 설명 |
|:-----|:---------|:-----|
| **직접 일정 단축** | Crashing (자원추가) | 자원을(Critical Path 상의 활동(activity)에 추가 투입하여 프로젝트기간 단축 |
| | Fast Tracking (병행진행) | 작업 간 관계를 조정하여 병행추진하여 기간을 단축 |
| **자원 제약 해결** | Resource Leveling | 제한된 자원을 보다 중요한 활동(activity)에 먼저 투입 |
| | Critical Chain<br>(세가지 버퍼활용) | 자원 제약 의존관계를 고려해서 프로젝트 일정관리<br>(프로젝트 / 피딩/ 자원버퍼) |
| **추가 위험사항에<br>대한 대응** | What-If 분석<br>(몬테카를로분석) | 일정에 영향을 미치는 요인(Factor)를 기준으로 분석 및 대안 시뮬레이션 진행 |
