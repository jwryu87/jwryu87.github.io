# 소프트웨어 비용 산정방법

---

## 핵심 키워드

`상향식` `하향식` `수학적`

---

## 정의/개념

소프트웨어 양적/질적 수준을 측정하여 **소요자원 및 기간산정 등을 계획, 수립하는 활동**

---

## 산정방법

| 구분 | 비용산정 방법 | 설명 |
|:-----|:------------|:-----|
| **상향식 산정** | 비용 산정 방법 | - 프로젝트의 세부 작업 단위 별로 비용을 산정한 후 전체 비용을 합산하여 산정하는 소프트웨어 비용 산정 방법 |
| | 원시 코드 라인수 기법<br>(LOC, Line Of Code) | - 각 기능의 원시 코드 라인 수의 비관치, 낙관치, 기대치를 측정하여 예측치를 구해 비용을 산정 |
| | 개발 단계별 노력(M/M) 기법 | - 기능을 구현하는 데 필요한 M/M(인/월수)을 소프트웨어 개발 생명주기의 각 단계에 적용하여 단계별로 산정하는 기법 |
| **하향식 산정** | | - 과거의 유사 경험을 바탕으로 전문지식이 많은 개발자들이 참여한 회의를 통해 비용을 산정하는 비과학적인 방법 |
| | 전문가 판단 기법 | - 조직 내에 있는 경험이 많은 두 명 이상의 전문가에게 비용 산정을 의뢰하는 방법 |
| | 델파이 기법 | - 전문가 판단 기법의 주관적인 편견을 보완하기 위해 많은 전문가의 의견을 종합하여 산정하는 기법 |
| **수학적 산정** | | - 경험적 추정모형, 실험적 추정 모형이라고 하며, 개발 비용 산정의 자동화를 목표로 하며, 각 모형에서 지정된 공식을 사용하여 비용을 산정 |
| | 기능점수(FP) 모형 | - 기능 점수(Function Point)를 구한 후 비용을 산정<br>- 사용자 관점의 요구 기능을 정량적으로 산정 |
| | COCOMO | - 개발할 소프트웨어의 규모(LOC)를 예측한 후 소프트웨어의 종류에 따라 각 비용 산정 공식에 대입하여 비용 산정 |
| | Putnam 기법 | - 소프트웨어 생명주기의 전 과정에 사용될 노력의 분포를 가정해 주는 모형 |

---

## 비용 산정방법 분류

| 분류 | 방법 | 특징 |
|:-----|:-----|:-----|
| **상향식** | LOC, 개발단계별 노력 | 상세 작업 단위 → 전체 합산 |
| **하향식** | 전문가 판단, 델파이 | 경험 기반 전문가 의견 |
| **수학적** | FP, COCOMO, Putnam | 공식 기반 자동화 산정 |

---

## 주요 비용 산정 모형

### 기능점수 (Function Point)

| 항목 | 설명 |
|:-----|:-----|
| **정의** | 사용자 관점의 기능을 정량화 |
| **측정 요소** | ILF, EIF, EI, EO, EQ |
| **장점** | 사용자 요구사항 기반, 개발 언어 독립적 |

### COCOMO

| 유형 | 설명 |
|:-----|:-----|
| **Basic** | 소프트웨어 크기(LOC) 기반 |
| **Intermediate** | 비용 승수(Cost Drivers) 적용 |
| **Detailed** | 단계별 상세 산정 |
