## 프로세스

### 범위관리 수립 프로세스

```
요구사항 기반, WBS 작성을 통한 범위 통제
         │
         ▼
┌─────────────────────────────────────────────────────────┐
│ 범위관리계획 ──► 요구사항 수집 ──► 범위정의 ──► WBS작성 ──► 범위확인 ──► 범위통제 │
│    수립                                                  │
├─────────────────────────────────────────────────────────┤
│ 범위통제   요구사항    상세범위    작업세분화   범위승인     │
│ 계획수립   식별        정의기술    인도물 구분              │
└─────────────────────────────────────────────────────────┘

- WBS 중심으로 범위와 일정, 원가의 기준선을 작성하여 프로젝트 수행 통제
```

---

## 구성요소

### 범위관리의 구성요소

| 구분 | 구성요소 | 설명(산출물) |
|:-----|:---------|:------------|
| **Planning Phase** | 범위관리계획수립<br>(Plan Scope Mgmt) | 범위관리에 대한 절차와 방법 계획 수립<br>(범위관리계획서) |
| | 요구사항 수집<br>(Collect Requirements) | 이해관계자의 요구사항 문서화/관리화<br>(과업대비표, 요구사항 추적매트릭스) |
| **Execution Phase** | 범위정의<br>(Define Scope) | 주요 액티비티/일정/산출물/제약조건 정의<br>(범위기술서) |
| | WBS작성<br>(Create WBS) | 산출물을 Breakdown하여 관리 가능한 수준까지 세분화한 후, 비용/일정/자원을 추정하여 기준선 설정 및 책임 할당(WBS)<br><작업세분화 및 인도물 설정> |
| **Validation & Control Phase** | 범위확인<br>(Validate Scope) | 프로젝트 범위에 대한 고객의 승인 획득<br>(승인서 등 수용 산출물) |
| | 범위통제<br>(Control Scope) | 범위변경이 프로젝트에 미치는 영향을 판단하고 변경에 대한 결정과 관리 수행(권고된 시정조치) |

- 과업대비표, 요구사항추적매트릭스, 범위기술서, WBS 등 각 단계별 수행에 따른 산출물 도출
