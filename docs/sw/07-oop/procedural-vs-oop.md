# 절차지향 프로그래밍, 객체지향 프로그래밍
{: .fs-8 }

7.1 객체지향 기본
{: .label .label-purple }

---

## 핵심 키워드

- **절차지향**: 프로시저 호출, 프로세스 지향 설계
- **객체지향**: Entity, Attribute, Method 결합

---

## 정의/개념

- **절차지향**: 프로시저 호출, 프로세스 지향 설계 **구조적 프로그래밍**
- **객체지향**: Entity, Attribute, Method 결합, **Object 모델링 프로그래밍**

---

## 설계 방법 비교

| 비교 항목 | 절차지향 프로그래밍 | 객체지향 프로그래밍 |
|:---------|:-------------------|:-------------------|
| **설계 중심** | - 프로세스 지향 설계(Process Oriented Design) | - 객체 지향 설계(Object Oriented Design) |
| **설계 방법** | - 업무 처리 절차를 중심으로 설계의 구성 요소들을 구분<br>- 어떤 절차를 거쳐서 작업을 수행하는가? 어떤 입출력 자료를 생성하는가에 중점 | - 시스템의 실제 객체 요소를 중심으로 설계<br>- 추상화, 캡슐화, 다형성, 정보은닉, 상속<br>- 자료구조와 그에 대한 연산을 묶어서 구성되는 객체들을 정의하고 이들이 상호 작용의 기본이 되도록 설계 |
| **설계 접근 방식** | - Top-Down 방식 접근<br>- 업무 처리 절차를 최상위에서 분할하면서 하위로 접근 | - Bottom-Up 방식 접근<br>- 하위의 객체가 서로 협력하는 관계를 설계하면서 상위로 접근 |
| **모델 구조** | 자료 → 함수 → 함수 → 자료<br>(순차적 데이터 흐름) | 객체들이 메시지를 통해 상호 연결된 구조 |
| **프로그램 모듈 단위** | - 함수 또는 프로시저 중심으로 분할<br>- 프로그램은 자료와 함수로 구성 | - 속성과 메소드(Method)를 중심으로 구성한 객체로 분할<br>- 객체는 자료와 함수로 구성 |

---

## 구현 방법 비교

| 비교 항목 | 절차지향 프로그래밍 | 객체지향 프로그래밍 |
|:---------|:-------------------|:-------------------|
| **사용 언어** | - FORTRAN, BASIC, COBOL, PASCAL, C 등 | - Object Pascal, C++, Java, C# 등 |
| **파라미터 전달 방식** | - 프로시저 호출 통한 파라미터 전달 | - 객체를 통해 파라미터 전달 |
| **데이터 접근 방식** | 함수 → 함수 → 공유데이터 ← 함수<br>(모든 함수가 접근 가능) | METHOD → DATA ← METHOD<br>Message 기반 통신 |
| **재사용성** | - 자주 사용되는 작업을 함수로 정의하고 이를 반복해서 호출<br>- 작업과 작업에 필요한 변수의 구분이 모호 | - 객체간 멤버함수로만 공유하며 객체 Method를 통해서 요청<br>- 자주 사용하는 작업의 객체를 선언하고 이를 반복해서 호출<br>- 작업은 public 섹션에 위치하고 변수는 private 섹션에 위치하여 하나의 클래스로 구성하여 변수와 작업 간의 관계가 명확 |
| **변수와 함수의 관계** | - extern과 같은 기능을 사용할 경우 변수와 함수의 관계는 더더욱 구분 난해 | |

## 모델 구조 비교

### 절차지향 모델 구조

```
         자료          자료
          │            │
          ▼            ▼
       ┌─────┐      ┌─────┐
       │ 함수 │──────│ 함수 │
       └─────┘      └─────┘
          │            │
          ▼            ▼
         자료          자료
```

### 객체지향 모델 구조

```
                    ┌─────────┐
                    │ METHOD  │ ◄───── METHOD
                    │         │
                    │  DATA   │
                    └─────────┘
                        │
                    Message
                        │
                        ▼
                    ┌─────────┐
                    │ METHOD  │
                    │         │
                    │  DATA   │
                    └─────────┘
```
