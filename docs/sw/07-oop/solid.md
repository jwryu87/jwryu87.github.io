## SOLID 원칙 상세

### SRP (Single Responsibility Principle) - 단일 책임 원칙

**클래스와 메소드는 하나의 역할 수행**

```
          ┌─────────────────┐
          │     Person      │
          ├─────────────────┤
          │ 성명()          │
          │ 사택()          │     DB 관련 책임
          │ 졸료()          │─────────────────▶
          │ 비즈니스 로직()  │     비즈니스 로직 관련 책임
          │ 급여 계산()      │─────────────────▶
          │ 마일리지()       │
          │ 연봉이자()       │
          └─────────────────┘
          
                 ↓ 적용 후
                 
    ┌───────────┐     ┌───────────────┐
    │  Person   │     │    Person     │
    ├───────────┤     ├───────────────┤
    │ 성명()    │     │ 비즈니스 로직()│
    │ 사택()    │     │ 급여 계산()    │
    │ 졸료()    │     │ 마일리지()    │
    │           │     │ 연봉이자()    │
    └───────────┘     └───────────────┘
      DB 관련          비즈니스 관련
```

---

### OCP (Open Closed Principle) - 개방 폐쇄 원칙

**확장에는 열려있고, 수정에는 닫혀있어야 하는 설계 원칙**

```
          적용 전                          적용 후
                                    ┌───────────────┐
                                    │  <<interface>>│
    ┌──────────┐                    │      도형     │
    │   도형   │                    └───────┬───────┘
    └────┬─────┘                            │
         │                    ┌─────────────┼─────────────┐
    ┌────┴─────┐              │             │             │
    │  Client  │              ▼             ▼             ▼
    └──────────┘         ┌─────────┐  ┌─────────┐  ┌─────────┐
                         │  삼각형  │  │   원    │  │ 사각형  │
                         └─────────┘  └─────────┘  └─────────┘
                         
                         새로운 도형 추가 시 기존 코드 수정 없이 확장 가능
```

---

### LSP (Liskov Substitution Principle) - 리스코프 치환 원칙

**부모 클래스의 객체(타입과 메소드의 집합)들이 자식 클래스 사용되는 곳에 대체될 수 있어야 한다는 설계 원칙**

```
          ┌─────────────────────┐
          │     Introduce       │
          ├─────────────────────┤
          │ -name : String      │
          │ +Display() : void   │
          └──────────┬──────────┘
                     │
                     │ 상속
                     ▼
          ┌─────────────────────┐
          │  DetailIntroduce    │
          ├─────────────────────┤
          │ -money : Int        │
          │ +Display() : void   │
          └─────────────────────┘
          
    부모 타입으로 선언된 변수에 자식 객체를 할당해도 정상 동작
```

---

### ISP (Interface Segregation Principle) - 인터페이스 분리 원칙

**인터페이스는 하나의 역할 수행**

```
          적용 전                              적용 후
          
    ┌────────┐  ┌────────┐  ┌────────┐     ┌─────────┐  ┌─────────┐
    │ clientA│  │ clientB│  │ clientC│     │ clientA │  │ clientB │
    └────┬───┘  └────┬───┘  └────┬───┘     └────┬────┘  └────┬────┘
         │           │           │               │            │
         └───────────┴───────────┘               ▼            ▼
                     │                     ┌──────────┐ ┌──────────┐
                     ▼                     │interface │ │interface │
              ┌───────────┐                │interfaceA│ │interfaceB│
              │  server   │                └────┬─────┘ └────┬─────┘
              ├───────────┤                     │            │
              │ methodA() │                     └─────┬──────┘
              │ methodB() │                           │
              │ methodC() │                           ▼
              └───────────┘                    ┌───────────┐
                                               │  server   │
                                               └───────────┘
```

---

### DIP (Dependency Inversion Principle) - 의존성 역전 원칙

**고차원 모듈은 저차원 모듈에 의존 금지**

```
          적용 전 (구체 클래스에 의존하는 형식 구조)
          
                     ┌───────────┐
                     │   상위층   │
                     └─────┬─────┘
                           │
              ┌────────────┼────────────┐
              ▼            ▼            ▼
         ┌─────────┐ ┌─────────┐ ┌─────────┐
         │ 정상적인 │ │지각상태별│ │이름별상태│
         │ area1   │ │ area2   │ │ area3   │
         └─────────┘ └─────────┘ └─────────┘

          적용 후 (추상 클래스에 의존하는 형식 구조)
          
                          ┌────────────┐
                          │<<interface>>│
                          │    도형     │
                          └──────┬─────┘
                                 │
                ┌────────────────┼────────────────┐
                ▼                ▼                ▼
           ┌─────────┐     ┌─────────┐     ┌─────────┐
           │ 정상적인 │     │지각상태별│     │이름별상태│
           │ area1   │     │ area2   │     │ area3   │
           └─────────┘     └─────────┘     └─────────┘
```

## SOLID 원칙 요약

| 원칙 | 영문명 | 설명 |
|:-----|:-------|:-----|
| **S** | Single Responsibility Principle | 클래스는 하나의 책임만 가짐 |
| **O** | Open Closed Principle | 확장에 열림, 수정에 닫힘 |
| **L** | Liskov Substitution Principle | 자식은 부모를 대체 가능 |
| **I** | Interface Segregation Principle | 인터페이스는 하나의 역할 |
| **D** | Dependency Inversion Principle | 추상화에 의존 |
