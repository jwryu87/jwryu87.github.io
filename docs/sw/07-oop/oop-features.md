## 캡슐화 (Encapsulation)

```
┌───────────────┐     정보접근     ┌───────────────┐
│    Class A    │◄────────────────►│    Class B    │
├───────────────┤                  ├───────────────┤
│  Attribute    │                  │  Attribute    │
├───────────────┤                  ├───────────────┤
│  Method ◄─────┼──────────────────┼─►  Method     │
└───────────────┘                  └───────────────┘
```

| 항목 | 내용 |
|:-----|:-----|
| **개념도** | 관련 메소드(Method)와 데이터를 하나의 객체로 구성하고 객체 내부 로직과 데이터를 감추고 기능은 인터페이스 통해 외부에 공개하는 매커니즘 |
| **개념 역할** | - 추상화 및 유지보수 용이, 객체 독립성 보장, 재사용성 증가 |

---

## 추상화 (Abstraction)

```
           추상화                    구체화
                    ┌─────────┐  ┌─────────┐  ┌─────────┐
┌─────────┐        │ Class A1│  │ Class A2│  │ Class A3│
│ Class A │        ├─────────┤  ├─────────┤  ├─────────┤
├─────────┤ ──────►│ Method A│  │ Method A│  │ Method A│
│ Method A│        │ Method A1│  │ Method A2│  │ Method A1│
└─────────┘        └─────────┘  └─────────┘  └─────────┘
```

| 항목 | 내용 |
|:-----|:-----|
| **개념 역할** | - 하위 클래스로부터 공통의 성질을 추출해 상위 클래스로 구성하는 매커니즘<br>- 기능 추상화(메소드), 데이터 추상화(속성), 제어 추상화(연산)<br>- 변하지 않는 부분을 추상화, 변하는 부분을 구체화 |

---

## 다형성 (Polymorphism)

```
                    ┌─────────────────────┐
                    │       Class P       │    Overriding
                    ├─────────────────────┤    - 재정의
                    │ Method1(Parent, Param2)│    - 동적
                    └──────────┬──────────┘    - 수직적
                               │
               ┌───────────────┴───────────────┐
               ▼                               ▼
    ┌─────────────────────┐        ┌─────────────────────┐
    │       Class C       │        │       Class C       │  Overloading
    ├─────────────────────┤        ├─────────────────────┤  - 중정의
    │Method1(Parent, Param2)│       │Method2(Parent, Param1)│  - 수평적
    │Method2(Parent)       │        │Method2(Parent, Param2, Param3)│
    └─────────────────────┘        └─────────────────────┘
```

| 항목 | 내용 |
|:-----|:-----|
| **개념** | - 동일한 이름의 메소드(Method)를 여러 형태의 다른 동작방식으로 재정의하는 특성 |
| **개념 역할** | - **Overloading**: 매개변수의 수/타입을 달리하여 동작 방식을 다중 정의<br>- **Overriding**: 상위 클래스의 메소드를 상속받아 동작 방식을 재정의 |

---

## 정보은닉 (Information Hiding)

```
    ┌───────────────┐      ╳      ┌───────────────┐
    │    Class A    │◄───────────►│    Class B    │
    ├───────────────┤   접근불가    ├───────────────┤
    │ - Attribute A1│             │ - Attribute B1│
    │ + Method A1   │◄───────────►│ - Attribute B2│
    └───────────────┘   접근가능    │ + Method B1   │
                                  └───────────────┘
```

| 항목 | 내용 |
|:-----|:-----|
| **개념 역할** | - 캡슐화된 정보를 외부로부터 은닉하는 매커니즘<br>- 객체의 메소드(Method)를 통해서만 정보에 대한 접근 가능 |

---

## 상속 (Inheritance)

```
                    ┌─────────────────┐
                    │  Parent Class   │
                    ├─────────────────┤
                    │ Method1         │
                    │ Method2         │   상속        상속
                    └────────┬────────┘  ◄────        ────►
                             │
                    ┌────────┴────────┐
                    │   Child Class   │
                    ├─────────────────┤
                    │ Method1         │
                    │ Method2         │
                    │ Method3         │
                    └─────────────────┘
```

| 항목 | 내용 |
|:-----|:-----|
| **개념도** | 상위 클래스의 속성과 메소드를 하위 클래스가 상속받아 사용하는 매커니즘 |
| **개념 역할** | - 상위 클래스는 추상적(일반화), 하위 클래스는 구체적(특수화) |

---

## OOP 특징 요약

| 특징 | 설명 |
|:-----|:-----|
| **캡슐화** | 데이터와 메소드를 하나로 묶고 외부에서 접근 제한 |
| **추상화** | 공통 특성 추출하여 상위 클래스 구성 |
| **다형성** | 동일 메소드명, 다른 동작 (Overloading, Overriding) |
| **정보은닉** | 내부 데이터를 외부에서 직접 접근 불가 |
| **상속** | 부모 클래스의 속성/메소드를 자식이 물려받음 |
