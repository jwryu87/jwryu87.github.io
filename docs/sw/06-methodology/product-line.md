## 구성요소 상세

| 구분 | 항목 | 내용 |
|:-----|:-----|:-----|
| **도메인공학** | 도메인 공학 | 특정 도메인에 포함된 제품들의 공통점과 차이점을 분석하여 제품라인 자산 (Product line Asset) 생성 |
| | 아키텍처 설계 | 넓고 광범위한 의미의 설계 결정 수행 |
| | 컴포넌트 설계 | 재사용성과 조립성을 고려하여 아키텍처 컴포넌트 정제 |
| **애플리케이션 공학** | 애플리케이션 | 핵심자산을 각 어플리케이션의 요구사항에 맞게 인스턴스화하여 어플리케이션을 효과적으로 개발하는 과정 |
| **기획** | 스코핑(Scoping) | 포트폴리오 결정, 도메인 범위결정, 자산 범위결정 |
| | 개발 계획 수립 | - Make / Buy / Mining / Commission 분석, 기술 예측 |
| **관리** | 기술 관리 | 형상관리, 프로세스 개선, |
| | 조직 관리 | 조직 구성 및 자원 할당 |

{: .note }
> 제품군을 정의하고 제품을 개량하는 다섯 가지 영역의 기술 적용 프로세스가 수행됨

## SSPL 주요 활동

| 영역 | 활동 | 설명 |
|:-----|:-----|:-----|
| **도메인 공학** | 요구사항 분석 | 제품군 공통/가변 요구사항 분석 |
| | 아키텍처 설계 | 공통 아키텍처 설계 |
| | 핵심자산 개발 | Core Asset 개발 |
| **애플리케이션 공학** | 요구사항 분석 | 개별 제품 요구사항 분석 |
| | 아키텍처 적용 | 공통 아키텍처 적용/변형 |
| | 제품 개발 | 핵심자산 활용 제품 개발 |

---

## Core Asset

| 구성요소 | 설명 |
|:---------|:-----|
| **요구사항 자산** | 공통 요구사항, 가변 요구사항 |
| **아키텍처 자산** | 참조 아키텍처, 설계 패턴 |
| **컴포넌트 자산** | 재사용 컴포넌트, 라이브러리 |
| **테스트 자산** | 테스트 케이스, 테스트 데이터 |
| **문서 자산** | 개발 문서, 사용자 문서 |
