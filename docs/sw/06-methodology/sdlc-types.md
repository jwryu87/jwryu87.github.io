## SDLC 모델 분류

### 순차적 모델

| 모델 | 특징 | 장점 | 단점 |
|:-----|:-----|:-----|:-----|
| **폭포수** | 단계별 순차 진행 | 관리 용이, 문서화 | 요구사항 변경 어려움 |

### 반복적/증분적 모델

| 모델 | 특징 | 장점 | 단점 |
|:-----|:-----|:-----|:-----|
| **프로토타이핑** | 시제품 개발 후 수정 | 요구사항 명확화 | 시간/비용 증가 |
| **나선형** | 위험 분석 기반 반복 | 위험 최소화 | 복잡한 관리 |
| **반복적** | 기능 단위 반복 개발 | 유연성 | 전체 범위 파악 어려움 |
| **증분형** | 기능 단위 순차 추가 | 조기 배포 가능 | 아키텍처 변경 어려움 |
| **진화형** | 초기 버전 발전 | 요구사항 변경 대응 | 구조 복잡 |

### 고속 개발 모델

| 모델 | 특징 | 장점 | 단점 |
|:-----|:-----|:-----|:-----|
| **RAD** | 신속한 개발 | 개발 기간 단축 | 대규모 부적합 |

---

## SDLC 모델 선택 기준

| 기준 | 폭포수 | 반복적 | 나선형 | RAD |
|:-----|:-------|:-------|:-------|:----|
| 요구사항 명확성 | 명확 | 불명확 | 불명확 | 명확 |
| 위험 수준 | 낮음 | 중간 | 높음 | 낮음 |
| 프로젝트 규모 | 소~중 | 중~대 | 대 | 소 |
| 사용자 참여 | 적음 | 많음 | 많음 | 많음 |
