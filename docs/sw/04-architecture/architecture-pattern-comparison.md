**디자인 패턴**: `모듈` `인터페이스` `참조 패턴`

---

## 정의/개념

| 구분 | 정의 |
|:-----|:-----|
| **SW 아키텍처 스타일** | 시스템 전체의 구조를 정의하는 고수준의 설계 패턴 |
| **디자인 패턴** | 특정 문제에 대한 재사용 가능한 해결책을 제공하는 설계 패턴 |

- SW 아키텍처는 전체 시스템에서의 구조적 관점 고려, Design 패턴은 단위 컴포넌트에서의 설계 관점 고려
- 소프트웨어 아키텍처와 디자인 패턴은 소프트웨어 설계에 있어 적용 시점 및 대상에 있어 차이점이 있음

---

## 상세비교

| 구분 | 소프트웨어 아키텍처 스타일 | 디자인 패턴 |
|:-----|:------------------------|:----------|
| **적용수준** | 시스템 레벨 (Order System, Payable System, Receivable System) | 클래스 레벨 (Order System 내 Customer, Product, RentalUI 등) |
| **적용시점** | 소프트웨어 기본 설계 단계 | 소프트웨어 상세 설계 단계 |
| **관점** | 조직적 구성 관점<br>경험 기반의 아키텍처 구성 | 개발자의 구현 관점<br>경험 기반의 모델링 |
| **특징** | 아키텍처의 품질요소를 개선<br>기능적 컴포넌트를 변경하고 확장시키는 데 초점<br>동일한 기능을 수행하는 또다른 아키텍처 구성 패턴 사용의 제약 | 단순화, 이해 용이성<br>분석과 설계에 대한 추상적인 관점 제공<br>개발자가 이해하여 적용할 수 있는 형태로 제공<br>확장성, 재사용성, 이해용이성 등 제공 |
| **주요 유형** | 분산스타일<br>파이프와 필터 스타일<br>데이터 중심 스타일<br>규칙기반 스타일<br>블랙보드 스타일<br>레이어 스타일 | Adaptor Pattern<br>Factory Pattern<br>Command Pattern<br>Interpreter Pattern<br>Façade Pattern<br>Builder Pattern |
| **문제해결 측면** | 접근 방법에 집중<br>경량화 된 가이드라인 | 특정 컨텍스트에서의 검증된 구조적 해결 방안 제시 |
