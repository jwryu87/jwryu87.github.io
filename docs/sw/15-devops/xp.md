|:-----|:-----|:---------|

| | 단순성 | 정복 제거, 공통 모듈 |

## XP의 실천방법

### 기술 실천방법

| 구분 | 실천방법 | 특징 |
|:-----|:---------|:-----|
| **기술** | 테스트 수행 | 반복적 테스트 수행 |
| | 리팩토링 | 재설계 가능, 재사용성 |
| | 단순 설계 | 모듈화, 스몰 사이즈 |
| | 짝 프로그래밍 | Drive/Partner 고려 |

### 팀 실천방법

| 구분 | 실천방법 | 특징 |
|:-----|:---------|:-----|
| **팀** | 지속가능 통합 | 수시 점검, 짧은 수행 |
| | 공동 소유 | 지식 공유, 전환 |
| | 속도 | 체크 40시간 배포 약속 금지 |
| | 메타포 | 공통 생각, 전략적 공유 |
| | 전체팀 | 전체 팀 같이 진행 |

### 비즈니스 실천방법

| 구분 | 실천방법 | 특징 |
|:-----|:---------|:-----|
| **비즈니스** | 작은 릴리즈 | 필수적 기능 릴리즈 |
| | 인수 테스트 | 필수적 인수 테스트 |
| | 계획 게임 | 점진적, 점진적 실행 |

---

## XP 12가지 실천방법 요약

| 번호 | 실천방법 | 설명 |
|:-----|:---------|:-----|
| 1 | **계획 게임 (Planning Game)** | 고객과 함께 반복적 계획 수립 |
| 2 | **작은 릴리즈 (Small Release)** | 짧은 주기로 실행 가능한 SW 배포 |
| 3 | **메타포 (Metaphor)** | 시스템 공통 비전 공유 |
| 4 | **단순 설계 (Simple Design)** | 현재 필요한 것만 설계 |
| 5 | **테스트 (Testing)** | TDD 기반 테스트 |
| 6 | **리팩토링 (Refactoring)** | 코드 품질 지속 개선 |
| 7 | **짝 프로그래밍 (Pair Programming)** | 두 명이 한 컴퓨터로 개발 |
| 8 | **공동 소유 (Collective Ownership)** | 코드는 팀 전체 소유 |
| 9 | **지속적 통합 (Continuous Integration)** | 코드 수시 통합 |
| 10 | **40시간 작업 (40-Hour Week)** | 주 40시간 근무, 지속 가능한 페이스 |
| 11 | **현장 고객 (On-site Customer)** | 고객이 팀에 상주 |
| 12 | **코딩 표준 (Coding Standards)** | 팀 코딩 규칙 준수 |
