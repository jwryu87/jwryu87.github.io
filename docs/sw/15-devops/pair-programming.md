## 핵심 키워드

- **페어 프로그래밍**: Driver, Partner
- **핑퐁 프로그래밍**: 페어 프로그래밍, TDD

---

## 정의/개념

- **페어 프로그래밍**: 두 사람이 한 컴퓨터를 사용해서 Pair Work로 프로그래밍 기법
- **핑퐁 프로그래밍**: 팀 내 지식 공유 용도 사용 위해, 페어 프로그래밍과 TDD 방법 결합한 프로그래밍 기법

---

## 페어 프로그래밍

### 개념

- 두 사람이 한 컴퓨터를 사용해서 **Pair work**로 프로그래밍 하는 방법
- 애자일 개발 방법론 중의 하나로 하나의 컴퓨터에 두 사람의 프로그래머가 작업하는 방법

### 개념도

```
                    ┌─────────────────────┐
                    │      프로그래밍      │
                    │      If a==b        │
                    └──────────┬──────────┘
                               │
            ┌──────────────────┴──────────────────┐
            │                                     │
            ▼                                     ▼
     ┌─────────────┐                      ┌─────────────┐
     │             │  코딩전행            │             │  확인, 점검
     │   Driver    │◄────────────────────►│   Partner   │
     │             │                      │ (navigator) │
     └─────────────┘                      └─────────────┘
```

### 구성요소

| 역할 | 설명 |
|:-----|:-----|
| **Driver** | - 실제 코딩 표준에 따라 코드를 작성하는 프로그래머 |
| **Partner** | - Driver에게 전략과의 일치 여부 등 모든 것을 상기시켜주는 역할을하는 프로그래머 |

{: .note }
> - 한명은 직접 코딩을 하고 다른 한명은 옆에서 지켜보며 질문을 하고, 논의 하는 방식으로 진행. Driver와 Partner의 역할을 바꿔가며 수행하면 코드와 시스템에 대한 이해가 높아짐

## 핑퐁 프로그래밍

### 개념

- **팀 내에서 지식 공유의 용도로 사용**하기 위해 페어 프로그래밍과 TDD 방법의 결합한 프로그래밍 방법

### 개념도

```
                    ┌─────────────────────┐
                    │      Fail Test      │
                    └──────────┬──────────┘
                        코딩진행 │
                    ┌─────┬─────┐
                    │  A  │  B  │
                    └─────┴─────┘
                           │
                           ▼
            ┌─────────────────────────────────┐
            │  Pass Test 수정, Fail Test 생성  │
            └──────────────┬──────────────────┘
                     코딩진행 │
                    ┌─────┬─────┐
                    │  A  │  B  │
                    └─────┴─────┘
                           │
                           ▼
                    ┌─────────────────────┐
                    │    Pass Test 완료    │
                    └──────────────────────┘
                        코딩완행
                    ┌─────┬─────┐
                    │  A  │  B  │
                    └─────┴─────┘
```

### 절차

| 단계 | 설명 |
|:-----|:-----|
| 1) Ping | A가 실패하는 새 테스트를 만들고, 키보드를 B에게 넘긴다 |
| 2) | B가 테스트를 통과하도록 코드를 작성한다 |
| 3) Pong | B가 실패하는 새 테스트를 만들고, 키보드를 A에게 넘긴다 |
| 4) | A가 테스트를 통과하도록 코드를 작성한다 |

{: .highlight }
> 피드백 주기가 짧아 두 프로그래머가 지속적으로 집중할 수 있도록 하는 장점이 있음

---

## 페어 프로그래밍 vs 핑퐁 프로그래밍

| 구분 | 페어 프로그래밍 | 핑퐁 프로그래밍 |
|:-----|:---------------|:---------------|
| **TDD 결합** | × | ○ |
| **역할 전환** | 주기적 | 테스트 작성마다 |
| **피드백 주기** | 상대적 김 | 매우 짧음 |
| **집중도** | 보통 | 높음 |
