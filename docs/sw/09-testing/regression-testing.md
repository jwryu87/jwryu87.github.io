# 회귀 테스트 (Regression Test)
{: .fs-8 }

9.3 테스트 종류
{: .label .label-green }

---

## 핵심 키워드

`Side Effect` `Ripple Effect` `Reset All` `Selective` `Priority`

---

## 정의/개념

소프트웨어 결함수정 또는 기능변경 발생 시 해당 부분의 테스트 이외에도 기존 기능까지 테스트하여 기존의 정상적인 기능에 부정적인 영향을 미치지 않았음을 확인하는 테스트

---

## 메커니즘

```
                    ┌──────────────────┐
                    │   Ripple effect  │
                    │    (파급효과)     │
                    │                  │
      SW 기능        │    변경모듈에    │
      변경요청  ──────►    의해 영향     ├────►  모듈 B
                    │    예상되는 모듈  │
                    │                  │
                    └──────────────────┘
                           │
          ┌────────────────┴────────────────┐
          │                                 │
          ▼                                 ▼
    ┌──────────┐                      ┌──────────┐
    │  Side    │                      │  Side    │
    │ effect   │                      │ effect   │
    │ (부작용) │                      │ (부작용) │
    └──────────┘                      └──────────┘
          │                                 │
          │           기존기능              │
          │            영향도              │
          │             확인              │
          └────────────┬────────────────────┘
                       │
    ┌─────────────┐    │    ┌─────────────┐
    │   기능개선   │◄───┴───►│   변경모듈   │
    └─────────────┘          └─────────────┘
          │                        │
          │                        ▼
          │     SW 기능변경 ────► 변경에 따른 영향범위 ────► 회귀 테스트
          │
          └─────────────────────────────────────────────────────────────
```

- 소프트웨어에 변경이 발생했을 경우, 수정된 부분에 따른 side effect(부작용)나 ripple effect(파급효과) 발생여부 및 영향도를 테스트함

---

## 유형

| 구분 | Reset All Test | Selective Test | Priority Test |
|:-----|:---------------|:---------------|:--------------|
| **수행방법** | 테스트 케이스를 모두 사용하여 테스트 | 영향 범위를 선택적으로 결정하여 테스트 | 핵심 기능을 도출하여 핵심 기능 위주로 테스트 |
| **적용분야** | 금융권 등 고위험 시스템에 적용 | 일반 사용자용 시스템에 적용 | 사용빈도가 낮은 저위험 시스템에 적용 |
| **장점** | 모든 경우를 포함하여 테스트 커버리지가 높음 | 비용 효과성 측면에서 우월 | 테스트 비용 측면에서 가장 낮음 |
| **단점** | 높은 내부 테스트 비용이 소모됨 | 영향범위 선정이 난해하며, 잘못 선정 시 파급효과 발생 | 우선순위 기준 모호성 존재하며, 잘못 선정 시 파급효과 발생 |

- 주어진 예산, 기간, 시스템의 위험도에 따라서 적용할 회귀테스트 유형을 결정함
