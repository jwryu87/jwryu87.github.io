# 뮤테이션 테스트 (Mutation Test)
{: .fs-8 }

9.2 테스트 기법
{: .label .label-yellow }

---

## 핵심 키워드

`뮤턴트(Mutant)` `뮤턴트 연산자` `뮤테이션 스코어` `비버깅(Bebugging Test)`

---

## 정의/개념

결함 추정과 테스트 케이스를 평가하기 위해 프로그램의 구문을 일정한 규칙을 적용하여 변형(Mutant)후 원래 프로그램과 동일한 케이스로 테스트하는 **결함 기반 테스트 기법**

---

## 수행절차

```
                    ┌─────────┐   A 테스트 결과   ┌─────────────┐
                    │   원본   │─────────────────▶│ 테스트결과(A')│──────▶ 배제
                    └─────────┘                   └─────────────┘
   ┌──────────┐          │       같은결과
   │ Testcase │──────────┤
   │   (A)    │          │
   └──────────┘          │       다른결과
  Testcase 목록      ┌─────────┐               ┌─────────────┐
                    │  뮤턴트  │─────────────────▶│ 테스트결과(B)│──────▶ 선정
                    └─────────┘                  └─────────────┘
```

1. **원본 프로그램**에 테스트케이스 실행 → 테스트 결과(A') 도출
2. **뮤턴트(변형)** 프로그램에 동일 테스트케이스 실행 → 테스트 결과(B) 도출
3. **결과 비교**:
   - 같은 결과 → 해당 뮤턴트 **배제** (테스트케이스가 결함을 발견 못함)
   - 다른 결과 → 해당 뮤턴트 **선정** (테스트케이스가 결함 검출 성공)

---

## 뮤테이션 테스트 vs 비버깅 테스트

| 항목 | 뮤테이션 테스트 | 비버깅 테스트 |
|:-----|:---------------|:--------------|
| **목적** | 테스트 케이스의 효율성과 커버리지 평가 | 잔존 오류 측정 및 디버깅 기술의 향상과 검증 |
| **수행 방식** | 돌연변이 생성 | 의도적 오류코드 삽입 |
| **적용 대상** | 테스트 프로세스 자체를 검증하고 강화하는 데 사용 | 디버깅 과정과 도구를 개선하는 데 활용 |
| **결과 활용** | 테스트 케이스의 미비한 부분을 발견하고 이를 강화시키는 데 사용 | 프로그래머가 직접적으로 버그를 찾아내고 해결하는 기술을 연습하고 평가 |

---

## 뮤테이션 스코어

$$
\text{Mutation Score} = \frac{\text{Killed Mutants}}{\text{Total Mutants} - \text{Equivalent Mutants}} \times 100\%
$$

- **Killed Mutants**: 테스트케이스가 탐지한 뮤턴트 수
- **Total Mutants**: 전체 생성된 뮤턴트 수
- **Equivalent Mutants**: 원본과 동일하게 동작하는 뮤턴트 (배제)

---

## 뮤턴트 연산자 예시

| 연산자 유형 | 설명 | 예시 |
|:-----------|:-----|:-----|
| **AOR** | 산술 연산자 교체 | `+` → `-`, `*` → `/` |
| **ROR** | 관계 연산자 교체 | `>` → `>=`, `==` → `!=` |
| **LOR** | 논리 연산자 교체 | `&&` → `\|\|`, `!` → 제거 |
| **ASR** | 할당 연산자 교체 | `+=` → `-=` |
