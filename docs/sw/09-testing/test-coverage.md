# MC/DC (Modified Condition/Decision Coverage)
{: .fs-8 }

9.2 테스트 기법
{: .label .label-yellow }

---

## 핵심 키워드

`조건표` `Condition 커버리지` `Decision 커버리지` `MC/DC 커버리지`

---

## 정의/개념

소프트웨어 화이트박스 테스트에서 개별 조건식이 다른 조건식의 영향을 받지 않고 전체 결정문의 결과에 독립적으로 영향을 미치는 테스트 케이스를 도출하는 테스트 커버리지

---

## 커버리지 범위

### MC/DC 커버리지의 범위

```
        ┌────────────────────────────────────┐
        │         MC/DC Coverage             │
        │  ┌────────────────────────────┐   │
        │  │    Condition    Decision   │   │
        │  │     (조건)       (결정)     │   │
        │  └────────────────────────────┘   │
        └────────────────────────────────────┘
```

| 커버리지 유형 | 설명 |
|:-------------|:-----|
| **Condition (조건) 커버리지** | 개별 조건문이 적어도 한번은 참과 거짓의 결과를 실행하는 경우 |
| **Decision (결정) 커버리지** | 전체 결정문이 적어도 한번은 참과 거짓의 결과를 실행하는 경우 |
| **MC/DC 커버리지** | Condition 커버리지와 Decision 커버리지를 포괄하며 보완하는 관점에서 MC/DC 커버리지 등장 |

---

## 프로세스

### MC/DC 결정 테이블 및 조건식 진리표

```
     IF A OR B
        │
    ┌───┴───┐
    │       │
    C       D
```

| No. | IF A OR B | 개별조건문 ||| 전체결정문 | 테스트 케이스 |
|:---:|:---------:|:--:|:--:|:--:|:----------:|:------------:|
| | | A | B | A OR B | |
| ① | | T | T | T | N |
| ② | | T | F | T | Y |
| ③ | | F | T | T | Y |
| ④ | | F | F | F | Y |

### MC/DC 테스트 케이스 도출 프로세스

| No | 프로세스 | 테스트 케이스 |
|:--:|:---------|:------------:|
| 1. | A조건 기준, 전체 결과에 독립적으로 영향을 미치는 테스트 케이스 도출 | ②, ④ |
| 2. | B조건 기준, 전체 결과에 독립적으로 영향을 미치는 테스트 케이스 도출 | ③, ④ |
| 3. | 1과 2를 모두 만족시키는 테스트 케이스 조합으로 최종 테스트 케이스 도출 | ②, ③, ④ |

- 조건문의 모든 조합에서 현실적이고 유의미한 테스트 케이스의 조합을 도출하는 과정
