# 상태전이 테스팅
{: .fs-8 }

9.2 테스트 기법
{: .label .label-yellow }

---

## 핵심 키워드

`상태` `전이` `상태-이벤트 테이블` `상태전이트리`

---

## 정의/개념

시스템의 상태 및 그 변화에 따른 동작을 테스트하기 위한 **명세기반 테스트 설계 기법**

---

## 상태전이 테스팅 절차

| 단계 | 7요 차 | 특 징 |
|:-----|:-------|:------|
| **1단계** | 상태-이벤트 테이블 작성 | - 테스트를 수행하는 상태명, 이벤트, 정지 이동, 테스트 케이스 도출 시나리오 전이개념 |
| **2단계** | 상태 정의 트리 작성 | - 상태 정의 다이어그램 동작 상태 전이 매핑 |
| **3단계** | 테스트 케이스 작성 | |
| **4단계** | 테스트 시나리오 구성 | |

---

## 상태전이 테이블 설명

### 상태전이 다이어그램

```
        ┌─────────────────────────────────┐
        │ ① 상태전이 다이어그램 설명       │
        │   • 상태전이 다이어그램, 테스트  │
        │     시나리오 도출               │
        └─────────────────────────────────┘

                    ─ 상 태 ─
                  ┌─────────────┐
                  │   상태 1    │
          전 이   │  (State1)   │───▶ 상태 2
                  └─────────────┘    (State2)

  - 상태 전이 다이어그램 동작 상태 전이 명명의 매핑
  - 각 상태 이벤트 발생 시 명령 동작 역디렉스
```

### 테스트 시나리오

- 반응, 무반응 판단하는 테스트 시나리오 도출

| 구분 | 트리 시켜 |
|:-----|:---------|
| 트랜지션 | V1 → V3 → V1 → V4K |
| 트랜지션 | V1 → V3 → VIF → V3 → V2 |

---

## 상태전이 테스팅 장점

| 장점 | 설명 |
|:-----|:-----|
| **명확한 테스트 범위** | 상태와 전이를 기반으로 테스트 범위 명확화 |
| **누락 방지** | 모든 상태와 전이를 커버하여 테스트 누락 방지 |
| **재현성** | 테스트 케이스의 재현성 보장 |
| **문서화** | 상태 다이어그램 및 테이블로 문서화 용이 |
