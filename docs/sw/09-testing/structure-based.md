## 구조기반 테스트의 기법

### 1. 제어구조 시험 (Control Structure Testing)

| 기법 | 설명 | 사례 |
|:-----|:-----|:-----|
| **제어구조 시험<br>(Control Structure Testing)** | - McCabe에 의해 제안된 대표적 White Box Test 기법<br>- 프로그램의 처리 흐름을 제어하는 방법 및 수행 제어를 위해 사용되는 문장의 구조 | - 순차형(순차 구조, Sequence형)<br>- 선택형(분기구조, If Then Else형)<br>- 반복형(반복구조, Do While형) |
| **루프 시험<br>(Loop Testing)** | - 프로그램 루프 구조에 국한해서 실시하는 기법<br>- 루프 시험의 대상 결함: 초기화 결함, 인덱싱 및 증가의 결함, 루프의 경계선에서 나타나는 경계 오류<br>- 루프의 유형: 단순루프, 중첩루프, 연결루프, 비구조적 루프 | - for, while 등<br>- go to 등 |

---

### 2. 구조기반테스트 기법 (커버리지)

| 기법 | 설명 | 예시 |
|:-----|:-----|:-----|
| **구문<br>(Statement)** | 프로그램 모든 문장을 최소 이상 수행 | ![구문](/assets/images/sw/structure-test-1.png) |
| **결정<br>(Decision/Branch)** | 개별 조건 식에 관계없이 전체결정이 참(1)/거짓(0)인 미분적 수행되는지 측정 | 조건A: 참/거짓<br>조건B: 참/거짓 |
| **조건<br>(Condition)** | 전체 결정에 관계없이 개별 조건 식이 각각 참(1)/거짓(0)인 한번씩 수행되는지 측정 | 조건A: 참/거짓<br>조건B: 참/거짓 |
| **조건/결정<br>(Condition/Decision)** | 개별 조건 식과 전체 결정이 각각 참/거짓 한번씩 수행되는지 측정 | 전체: 참/거짓<br>조건A: 참/거짓<br>조건B: 참/거짓 |
| **변경 조건/결정<br>(MC/DC)** | 각 개별 조건식이 다른 개별 조건 식에 무관하게 전체 결과에 독립적으로 영향을 주도록 함<br>- '가'의 경우 조건B가 조건A에 독립적<br>- '나'의 경우 조건A가 조건B에 독립적<br>- '다'의 경우 조건A가 조건B에 독립적이면서 조건B가 조건A에 독립적 | 관계/전체: 조건A/조건B<br>가: 참/참→참<br>나: 참/거짓→거짓<br>다: 거짓/참→거짓 |
| **다중 조건<br>(Multiple Condition)** | 모든 가능한 조합 100% 만족하는지 측정 | 전체: 조건A/조건B<br>1/1/1<br>0/1/0<br>0/0/1<br>0/0/0 |

---

## 커버리지 계층 구조

```
다중 조건 커버리지 (Multiple Condition)
         ↑
    MC/DC 커버리지
         ↑
  조건/결정 커버리지
    ↗         ↖
조건 커버리지   결정 커버리지
    ↖         ↗
    구문 커버리지
```

{: .highlight }
> MC/DC는 안전 필수 시스템(DO-178C, ISO 26262)에서 요구되는 커버리지 수준

---

## MC/DC 상세 설명

**MC/DC (Modified Condition/Decision Coverage)**는 다음 조건을 만족:

1. 모든 **결정(Decision)**이 가능한 모든 결과를 최소 한 번 수행
2. 모든 **조건(Condition)**이 가능한 모든 결과를 최소 한 번 수행
3. 각 조건이 **독립적으로** 결정의 결과에 영향을 미치는 것을 증명

| 조건 수 | 다중 조건 테스트 케이스 | MC/DC 테스트 케이스 |
|:--------|:----------------------|:-------------------|
| 2개 | 4개 (2²) | 3개 |
| 3개 | 8개 (2³) | 4개 |
| N개 | 2ⁿ개 | N+1개 |
