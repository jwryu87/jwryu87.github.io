## 구성요소 상세

| 구분 | 설명 | 주요 항목 |
|:-----|:-----|:----------|
| **Part 1.<br>개념과 정의<br>(Concepts and Definitions)** | - 전체 시리즈에 대한 가이드 제공<br>- 용어 정의, 소프트웨어 테스팅 개념<br>- 8개 단락 구성 | - 소프트웨어 테스팅 소개 및 개념<br>- 조직과 프로젝트 관점 소프트웨어 테스팅<br>- SDLC 모델에서의 일반적 테스팅 프로세스<br>- 위험 기반 테스팅<br>- Test Subprocess, 테스트 사례, 테스트 자동화, 결함관리<br>- 테스트 관리 프로세스 (전략, 모니터링) |
| **Part 2.<br>테스트 프로세스<br>(Test Process)** | - 테스트 프로세스에 관한 부분<br>- 조직, 테스트 관리, 동적 테스트의 세 가지 수준의 다계층 프로세스 모델 설명 | - 다계층 프로세스 모델 (Multi-Layer Process Model)<br>- 조직의 테스트 프로세스 (Organizational Test Process)<br>- 테스트 관리 프로세스 (Test Management Process)<br>- 동적 테스트 프로세스 (Dynamic Test Process) |
| **Part 3.<br>테스트 문서화<br>(Test Documentation)** | - 테스트 문서 견본과 예시 제공<br>- 테스트 프로세스 단계별 산출 문서 작성 방법과 포함될 내용 등 제공 | - 조직의 테스트 프로세스 문서<br>- 테스트 관리 프로세스 문서<br>- 동적 테스트 프로세스 문서 |
| **Part 4.<br>테스트 설계기법<br>(Test Techniques)** | - 소프트웨어 테스팅 기법에 관한 부분<br>- 테스트 설계 및 구현 단계에서 활용 가능한 명세 기반 테스트 설계, 구조 기반 테스트 설계, 경험 기반 테스트 설계 기법 제공 | - 테스트 설계 기법<br>- 명세 기반/구조 기반/경험 기반 테스트 설계 기법<br>- 테스트 커버리지 측정<br>- 명세/구조/경험 기반 테스트 설계 기법의 커버리지 측정 |
| **Part 5.<br>키워드 주도 테스팅<br>(Keyword-Driven Testing)** | - 키워드 주도 테스팅에 대한 소개와 접근 방법 제공<br>- 키워드 주도 테스팅 위한 프레임워크, 도구에 대한 요구사항 다룸 | - 키워드 주도 테스팅 활용<br>- 키워드 주도 테스팅 프레임워크<br>- 키워드 주도 테스팅 이점과 이슈<br>- 기본 키워드와 사례 |
