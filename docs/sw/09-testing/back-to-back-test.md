# 백투백 테스트 (Back-to-Back Test)
{: .fs-8 }

9.3 테스트 종류
{: .label .label-green }

---

## 핵심 키워드

`높은 신뢰성` `설계명세 기반` `ISO26262` `비교테스트`

---

## 정의/개념

두개 이상의 다양한 컴포넌트나 시스템을 동일한 입력 값으로 실행하여 동일한 결과가 나오는지 확인하여, 불일치 할 경우 그 불일치를 서로 비교 분석하는 테스트 기법

---

## 개념도

```
                    ┌─────────────┐
                    │  Test data  │
                    └──────┬──────┘
                           │
              ┌────────────┴────────────┐
              │                         │
              ▼                         ▼
     ┌────────────────┐       ┌────────────────┐
     │    Program     │       │    Program     │
     │   version A    │       │   version B    │
     └────────┬───────┘       └────────┬───────┘
              │                         │
              └────────────┬────────────┘
                           │
                           ▼
                  ┌────────────────┐
                  │    Results     │
                  │   comparator   │
                  └────────┬───────┘
                           │
                           ▼
                  ┌────────────────┐
                  │  Difference    │
                  │    report      │
                  └────────────────┘
```

---

## 프로세스

| No | 프로세스 | 설명 |
|:--:|:---------|:-----|
| 1 | **테스트케이스 생성** | 수행 될 테스트 케이스를 작성 |
| 2 | **테스트 수행** | 테스트 대상에 대한 병렬수행<br>- 작성된 테스트케이스를 사용하여 각각의 테스트하고자 하는 테스트 대상 (Program Version n) 을 병렬적으로 수행 |
| 3 | **테스트 결과 확인** | 테스트 결과값 동일여부 비교 |
| 4 | **테스트 결과분석** | 불일치 시 결과 원인 분석 |

---

## 적용 분야

{: .highlight }
> 자동차 분야에서는 백투백 테스팅 기법을 이용하여 제어모델과 소스코드 간, 품질점검 수행

- ISO 26262 기반의 자동차 기능안전 분야에서 주로 활용
- 높은 신뢰성이 요구되는 임베디드 시스템 테스트
- 설계 명세(모델)와 실제 구현 코드 간의 일치성 검증
