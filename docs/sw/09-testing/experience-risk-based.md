# 경험 기반 테스트 (Experience Based Test)
{: .fs-8 }

9.2 테스트 기법
{: .label .label-yellow }

> 암기: **경탐오체**

---

## 핵심 키워드

`탐색적 테스팅` `오류추정` `체크리스트` `분류 트리`

---

## 정의/개념

유사 어플리케이션이나 기술에서의 경험, 직관, 테스터의 기술 능력으로부터 테스트 케이스를 추출하는 기법

---

## 주요 기법

| 유형 | 설명 | 고려사항 |
|:-----|:-----|:---------|
| **탐색적 테스팅** | 테스트 목표를 포함하는 테스트 차터(Charter)를 기반으로 정해진 시간내에 테스트 설계, 수행, 기록과 학습하는 테스팅 기법 | - 명세가 거의 없고 시간이 부족한 경우<br>- Formal 기법을 보충할 경우 |
| **오류 추정** | Ad-hoc Testing이라고도 불리며, 어떤 특정한 형태의 결함이 발생할 것이라고 예측하고, 이 결함을 발견하기 위한 테스트 케이스를 설계하는 기법<br>- 테스터는 직관과 결함에 기반하여 결함을 예측<br>- 테스터가 시스템에 대한 완전히 이해한다는 전제로 적용되는 기법 | - 테스트의 마지막 단계에 사용 |
| **체크 리스트** | 테스트하고 평가해야 할 내용과 결함을 분류하여 나열해 놓은 목록<br>- 체계적으로 도출되기 보다는 테스트 결함과 노하우를 정리하고 목록화하여 다음 테스팅에서 해당 내용을 누락없이 재활용하는 것을 목적으로 작성 | - 공식적인 테스팅을 보완하는 용도로 활용 |

---

## 기술요소

### 탐색적 테스팅 vs 테스트케이스 기반 테스팅

| 구분 | 탐색적 테스팅 | 테스트케이스 기반 테스팅 |
|:-----|:-------------|:------------------------|
| **설명** | 문서화에 소요되는 시간을 최소화 하여 테스트를 "실행" 하는 것에 집중 | 문서화 기반의 공식적인 수행으로 테스트의 체계적인 "설계" 향상에 집중, 테스트케이스의 재사용 및 공유 |
| **구성도** | Time Boxing → Test Ideas → Note (Tester) | Test Case → Test Product → Test Scenario → Test 결과서 (Tester) |

### 구성요소

| 구성요소 | 탐색적 테스팅 | 테스트케이스 기반 테스팅 |
|:---------|:-------------|:------------------------|
| **테스트 차터** | 각 세션의 임무를 정 | 테스트 계획서 |
| | 인력, 일정, 환경 등 계획 |
| **타임박싱** | 각 세션당 시간제약 (Time Boxing) | 테스트 케이스 |
| | 테스트 내용의 명세화 문서 |
| **테스트 노트** | 머리속으로 작성한 테스트케이스 | 테스트 시나리오 |
| | 테스트 케이스의 흐름 집합 |
| **요약보고** | 세션 종료 후 팀원에게 간략보고 | 테스트 결과서 |
| | 단위/통합테스트 결과서 |
