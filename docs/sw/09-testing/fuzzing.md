# 퍼징 테스트 (Fuzzing Test)
{: .fs-8 }

9.4 기타 테스트
{: .label .label-blue }

---

## 핵심 키워드

`블랙박스 퍼징` `화이트박스 퍼징` `그레이박스 퍼징` `변이기반` `생성기반` `덤` `스마트` `바이너리`

---

## 정의/개념

시스템 안정화 및 보안취약점 제거를 위해, **비정상 값(유효하지 않은 값, 무작 위로 산출된 값)을 입력하여 기대하지 않는 결과를 도출**하여 에러와 잠재되어 있는 취약점을 발견하기 위한 테스트 기법

---

## 수행단계

| 단계 | 설명 |
|:-----|:-----|
| **1) 대상분석** | 분석 대상 시스템의 식별, 특징 분석, 프로그램 실행에 필요한 정보분석 |
| **2) 입력 값 분석** | 오류 유발 가능한 입력 값 선정, 분석, 변이규칙 생성 |
| **3) 테스트케이스 생성** | 입력 값에 대한 테스트 케이스 생성 |
| **4) 프로그램 실행** | 테스트 케이스를 입력하여 프로그램 실행 |
| **5) 예외모니터링** | 문제 발생 시, 로그 수집, 각종 예외 로그 기록 |
| **6) 예외분석** | 문제 발생(예외정보) 항목 점검, 원인 분석, 코드 수정 |

---

## 퍼징 기법

### 처리방법/변조방식

| 기법 | 설명 |
|:-----|:-----|
| **변이기반(Mutation) 퍼징** | 기존 입력값을 변형하여 테스트 케이스 생성 |
| **생성기반(Generation) 퍼징** | 프로토콜/포맷 명세 기반으로 테스트 케이스 생성 |

### 테스팅 기법

| 기법 | 설명 |
|:-----|:-----|
| **블랙박스 퍼징** | 대상 시스템 내부 구조 모름, 입출력만 테스트 |
| **화이트박스 퍼징** | 소스코드 분석 기반 테스트 |
| **그레이박스 퍼징** | 블랙박스 + 화이트박스 혼합 |

---

## 퍼징테스트의 보안취약점 탐지 기법

### Valid Case 기반

```
┌─────────────┐     ┌─────────┐     ┌───────────────┐     ┌─────────┐
│ Valid Case  │     │         │     │ Valid Test    │     │ Device  │
│   기반      │────▶│ Fuzzer  │────▶│    Case       │────▶│  Under  │
│             │     │         │     │   정상 동작    │     │  Test   │
│    탐지     │     │[Test S/W]│     │               │     │[Test 대상]│
└─────────────┘     └─────────┘     └───────────────┘     └─────────┘
```

- Valid Case 기반 테스트, 정상 연결 확인
- 프로토콜 RPC, 명세기반 정상 Case 활용

### Invalid Case 기반 탐지 기법

```
┌─────────────┐     ┌─────────┐     ┌───────────────┐     ┌─────────┐
│Invalid Case │     │         │     │Invalid Test   │     │ Device  │
│   기반      │────▶│ Fuzzer  │────▶│    Case       │────▶│  Under  │
│             │     │         │     │  미처리, 미동작 │     │  Test   │
│  탐지 기법   │     │[Test S/W]│     │  (Skip, Fail) │     │[Test 대상]│
└─────────────┘     └─────────┘     └───────────────┘     └─────────┘
```

- 비정상 입력을 주입하여 테스트, 응답 확인 (목록 있음)
- Case 미 처리시 Skip, 미 응답 Fail 처리
- → Fail 및 Crash Case 재 테스트 수행

## 퍼징 도구

| 도구 | 설명 |
|:-----|:-----|
| **AFL (American Fuzzy Lop)** | 가장 유명한 오픈소스 퍼저, 커버리지 기반 |
| **libFuzzer** | LLVM 프로젝트의 인메모리 퍼저 |
| **Peach Fuzzer** | 상용 퍼징 프레임워크 |
| **Boofuzz** | 네트워크 프로토콜 퍼징 |
| **OSS-Fuzz** | Google의 오픈소스 퍼징 서비스 |
