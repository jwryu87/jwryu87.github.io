# SW 성능 테스트
{: .fs-8 }

9.3 테스트 종류
{: .label .label-green }

---

## 핵심 키워드

`처리지표` `시간지표` `사용자 지표` `Loopback` `Spike` `Stress` `확장성 테스트` `가용성 테스트`

---

## 정의/개념

사용자가 시스템을 사용하기 필요한 성능에 대해 시스템 처리 능력을 검증하는 테스트

---

## 평가 지표

| 구분 | 평가지표 | 설명 |
|:-----|:---------|:-----|
| **처리지표** | TPS (Transaction/Second) | 초당 처리가능한 트랜잭션 |
| | Throughput | 시간 당 처리량 |
| | Resource Usage | 시스템이 작업을 처리함에 있어서의 CPU 및 메모리 등 HW 리소스의 사용량 |
| **시간지표** | Response Time | 요청을 전달한 시간부터 요청에 대한 응답을 받을 때까지 소요된 시간 |
| | Think Time | 사용자가 생각하는 시간 |
| | Request Interval Time | 사용자가 현번 클릭 후 다음 클릭까지 걸리는 시간 |
| **사용자 지표** | Concurrent User | 해당 시스템을 동시에 사용중인 사용자 수 |
| | Active User | 해당 시스템 내 활성화된 사용자 수 |

- 평가지표를 기준으로 목적 및 방법에 따라 다양한 성능 테스트를 수행
- TPS = Active User / MRT (Mean Response Time)

---

## 유형

| 구분 | 유형 | 설명 |
|:-----|:-----|:-----|
| **목적** | 단위 성능테스트 | 대상 시스템을 업무 단위로 각각 테스트 수행 |
| | 복합 성능테스트 | 실 사용자 패턴으로 동시 사용자 및 가중치를 통해 상황을 재현하여 테스트 수행 |
| | 임계 성능테스트 | 시스템이 최대 발휘할 수 있는 성능 측정 |
| | Loopback Test | 시스템을 구성하는 논리적인 아키텍처 상의 컴포넌트들의 overload를 검증하기 위한 성능 시험 |
| **방법** | Spike Test | 모든 동시 단말 사용자가 동시에 거래를 발생시키는 테스트, 해당상황의 시스템을 점검함 |
| | Tier Test | 구체적인 성능 병목 구간을 찾기 위해 개발 소스상의 직접적 변경없이 시험 |
| | 확장성 테스트 | 시스템 증설에 대한 성능 향상 비율 측정 |
| | 가용성 테스트 | Transaction을 특정 기간 동안 발생시키면서 시스템 상황 점검 및 확인 |

- 테스트의 유형 외에 성능테스트를 수행하기 위한 절차가 존재

---

## 절차

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│ 테스트  │    │  성능   │    │  성능   │    │  성능   │    │  성능   │    │  성능   │
│ 요구사항│───►│ 테스트  │───►│ 테스트  │───►│ 테스트  │───►│ 테스트  │───►│ 테스트  │
│  정의   │    │  계획   │    │  설계   │    │  구현   │    │  수행   │    │  종료   │
└─────────┘    └─────────┘    └─────────┘    └─────────┘    └─────────┘    └─────────┘
       │              │              │              │              │
       ▼              ▼              ▼              ▼              ▼
  <계획 단계>    <설계 구현 단계>                        <수행 종료 단계>
    TA 팀          개발 팀                            개발/인프라 팀
```

- 수행 절차는 크게 계획, 설계구현, 수행종료 3가지로 나뉘며 각 단계 별로 세부 절차가 존재
