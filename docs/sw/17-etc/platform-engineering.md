## 핵심 키워드

`공유` `API` `공유인프라 플랫폼` `Cross Functional Team`

---

## 정의/개념

개발자들이 앱 개발에만 집중할 수 있도록, 필요한 **인프라, 도구, 프로세스를 하나의 통합된 플랫폼**으로 제공 기법

---

## 배경

- DevOps는 개발과 인프라를 어떻게 통합해야 하는지에 대한 큰 방향을 제공
- DevOps가 성공적으로 정착하면서 세분화
  - → **SRE, DevSecOps, DevBizOps, 플랫폼 엔지니어링** 모두 DevOps라는 범용 프레임워크에서 파생

---

## 구성 : Cross Functional Team

```
                    ┌─────────────┐
                    │   개발팀    │
                    └──────┬──────┘
                           │
                    ┌──────▼──────┐
                    │   서비스    │
                    └──────┬──────┘
                           │
    ┌──────────────────────┼──────────────────────┐
    │                      │                      │
    ▼                      ▼                      ▼
┌─────────┐         ┌─────────────┐        ┌─────────────┐
│재사용 가능│         │  개발자 도구  │        │ 셀프서비스  │      플랫폼
│ 구성요소  │         │             │        │ 개발자 포털  │   엔지니어링 팀
└─────────┘         └─────────────┘        └─────────────┘
                           │
                    ┌──────▼──────┐
                    │   자동화     │
                    └──────┬──────┘
                           │
                    ┌──────▼──────┐
                    │  복잡한 인프라 │
                    └─────────────┘
```

---

## 핵심 구성요소

| 구성요소 | 설명 |
|:---------|:-----|
| **재사용 가능한 구성요소** | 표준화된 컴포넌트, 모듈, 템플릿 제공 |
| **개발자 도구** | CI/CD 파이프라인, 모니터링, 로깅 도구 통합 |
| **셀프서비스 개발자 포털** | 개발자가 직접 인프라 및 서비스를 프로비저닝 |
| **자동화** | 인프라 프로비저닝, 배포, 운영 자동화 |

---

## DevOps vs 플랫폼 엔지니어링

| 구분 | DevOps | 플랫폼 엔지니어링 |
|:-----|:-------|:-----------------|
| **목적** | 개발-운영 협업 문화 | 개발자 생산성 극대화 |
| **초점** | 프로세스 및 문화 | 플랫폼 및 도구 |
| **대상** | 개발팀 + 운영팀 | 개발팀 (셀프서비스) |
| **산출물** | CI/CD 파이프라인 | Internal Developer Platform (IDP) |

---

## 플랫폼 엔지니어링의 가치

| 가치 | 설명 |
|:-----|:-----|
| **개발자 경험 향상** | 복잡한 인프라 추상화로 개발 집중 |
| **표준화** | 일관된 개발 환경 및 프로세스 제공 |
| **생산성 향상** | 셀프서비스로 대기 시간 감소 |
| **보안 강화** | 기본 보안 정책이 내장된 플랫폼 |
| **비용 절감** | 인프라 중복 제거, 효율적 자원 사용 |
