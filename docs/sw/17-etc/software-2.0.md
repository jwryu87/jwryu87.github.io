## 정의/개념

신경망 학습, 스스로 최적화, 논리 구성, 프로그램 실행 소프트웨어

인공신경망이 데이터를 이용하여 스스로 알고리즘을 생성하는 **데이터 기반의 귀납적 소프트웨어 개발 패러다임**

---

## 소프트웨어 2.0 개발 구조와 기술요인

### 개발 구조

```
Requirements → Analysis → Model Design → Deployment/Coding → Testing → Maintenance
                           ↑                    ↑
                       데이터 분석           파라미터 모델
                           ↓                  최적화
                    Data Preprocessing  →  Model Experiments
```

- 인공신경망이 데이터를 이용하여 스스로 알고리즘을 생성

### 기술요인

| 구분 | 기술요인 | 설명 |
|:-----|:---------|:-----|
| **데이터** | 데이터 중심 AI | - AlphaGo 이후, 딥마인드가 개발한 AlphaGo Zero, AlphaZero, Muzero 등은 알고리즘 스스로 진화하며 발전 |
| | 초거대 언어모델 | - OpenAI의 GPT-3 이후 트랜스포머 기반 초거대 언어모델 경쟁 가속화<br>- 초대형 언어모델의 학습데이터에 존재하는 매우 많은 양의 중복된 부분을 줄임으로써, 효율적이고 더 다양한 표현이 가능한 AI 모델을 생성 |
| **도구의 변화** | MLOps | - 맞춤형 AI 모델 개발과 운영서비스 제공<br>- 다양한 학습모델 생성, 배포, 자동화, 관리 등 지원<br>- AI 데이터 처리 및 관리를 통합 지원 등 |

## 소프트웨어 1.0과 소프트웨어 2.0 비교

| 구분 | 소프트웨어 1.0 | 소프트웨어 2.0 |
|:-----|:---------------|:---------------|
| **시기** | 1940년대 ~ 현재 | 2010년대 초반 ~ 현재 |
| **과정** | 인간이 논리와 조건을 주면 이를 기반으로 인간이 코딩하고 결과 도출 | 데이터와 딥러닝을 통하여 컴퓨터 스스로 논리를 개발 |
| **실행 방식** | 연역적 (Top-Down)<br>(인코딩 후 결과 도출) | 귀납적 (Bottom-Up)<br>(데이터와 ML학습을 통한 알고리즘 도출) |
| **중심** | 모델 플랫폼<br>(Evolving Model, Fixed Data) | 데이터 플랫폼<br>(Evolving Data, Fixed Model) |
| **도구** | DevOps | MLOps |

---

## 핵심 포인트

- **모델 중심의 AI**가 아니라 **데이터 중심의 AI**가 주도하는 소프트웨어 개발 방식으로의 전환
- 소프트웨어 2.0은 프로그램 코드의 양을 줄임으로써 복잡성을 낮출 수 있을 뿐만 아니라, AI 모델 개발 과정을 자동화함으로써 소프트웨어에게 다시 구현해야 하는 부담을 덜어주고 다양한 분야로 인공지능 확산 가속화
