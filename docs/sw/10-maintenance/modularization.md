## 핵심 키워드

`응집도` `결합도` `fan-in` `fan-out`

---

## 정의/개념

소프트웨어 재사용과 동시에 여러 다른 모듈의 개발에 사용될 수 있도록 **기능 단위로 분해하고 추상화 되어 재사용 및 공유 가능**한 수준으로 만들어진 단위

---

## 응집도 (Cohesion)

### 정의

소프트웨어의 한 모듈이 해당 기능을 수행하기 위해 **얼마만큼의 연관된 책임과 아이디어가 뭉쳐 있는지**를 나타내는 정도

### 개념도

```
       "높은 응집도 관계"
         A 모듈                    C 모듈
    모듈 단위로 같은 목적의 기능이 응집
         A -> C 호출
              │
    A -> B    │           B -> C
     호출     │            호출
              ↓
           B 모듈
        ┌────┬────┐
        │ F1 │ F2 │
        ├────┴────┤
        │ F3 │ F4 │
        └────┴────┘
         모듈 내 기능
```

### 응집도 단계 (낮음 → 높음)

| 응집도 | 단계 | 설명 |
|:-------|:-----|:-----|
| **낮음** | 우연적 | - 모듈 간 의미 있는 연관이 없는 기능으로 구성 |
| | 논리적 | - 유사한 성격의 처리 요소들이 한 모듈에서 처리 |
| | 시간적 | - 특정 시간에 처리되어야 할 활동을 모은 모듈 |
| | 절차적 | - 모듈 내 구성 요소들이 순차적으로 수행 |
| | 통신적 | - 동일한 입/출력을 사용하여 다른 기능을 수행 |
| | 순차적 | - 모듈 내 한 활동의 출력값이 다른 활동의 입력값 |
| **높음(좋음)** | 기능적 | - 모듈 내의 모든 기능이 단일 목적으로만 수행 |

> **설계 원칙**: 응집도는 높은 단계의 방향으로 설계하며 결합도는 낮추는 방향으로 설계하는 것이 좋은 소프트웨어의 조건

---

## 결합도 (Coupling)

### 정의

모듈 내부가 아닌 **외부의 모듈과의 연관도** 또는 모듈 간의 **상호 의존성**을 나타내는 정도

### 개념도

```
     낮은 결합도                   높은 결합도
      (A 모듈)                    (B 모듈)         C 모듈
                                                    (c)
       A 모듈                       B 모듈    호출
          │                          │ (b) ──→    E 모듈
          │                          │      호출    (e)
          └─── 호출 ─────────────────┘
                                           호출 ───→ F 모듈
                                                      (f)
     B 모듈의 기능을 수정하려면 C, E, F 모듈의 확인 필요
```

### 결합도 단계 (낮음 → 높음)

| 결합도 | 단계 | 설명 |
|:-------|:-----|:-----|
| **낮음(좋음)** | 자료 | - 모듈 간 I/F로 전달되는 파라미터로 상호작용 |
| | 스탬프 | - 모듈 간 I/F로 배열, 객체, 구조 등이 전달 |
| | 제어 | - 제어 신호를 통해 다른 모듈의 이동 |
| | 외부 | - 외부에서 입력된 데이터 포맷, 통신, 디바이스 I/F 공유 |
| | 공통 | - 전역변수를 참조, 갱신하는 방식으로 상호 작용 |
| **높음** | 내용 | - 모듈 내부 변수, 기능을 다른 모듈에서 사용 |

---

## Fan-In / Fan-Out

응집도와 결합도는 모듈 간의 관계를 각각의 단계별로 정의할 수 있으나 **모듈 간 호출 관계를 깊이(Depth)와 넓이(Width)** 파악하기 위해서는 fan-in, fan-out의 정량적 분석이 필요 함.

| 구분 | 설명 |
|:-----|:-----|
| **Fan-In** | 해당 모듈을 호출하는 상위 모듈의 수 (높을수록 재사용성 높음) |
| **Fan-Out** | 해당 모듈이 호출하는 하위 모듈의 수 (높을수록 복잡도 높음) |
