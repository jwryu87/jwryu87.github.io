# 타임스탬프 순서 기법 (Timestamp Ordering)
{: .fs-8 }

5.3 타임스탬프 기반
{: .label .label-purple }

---

## 핵심 키워드

`논리적 계수` `시스템 시계` `Read_TS(x)` `Write_TS(x)`

---

## 정의/개념

DBMS가 트랜잭션에 유일 식별자인 타임스탬프를 부여하여, 순서대로 처리하는 동시성 제어 기법

---

## 메커니즘 (Timestamp Ordering 메커니즘 및 구현 기술)

```
Read 수행한 트랜잭션 A의        Write-TS(X)
    타임스탬프값      타임스탬프
         │              비교
         ▼              ▼
┌───────────────────────────────────┐
│    TS(A)  │  트랜잭션 A │ Read(X) 시 │
│           │            │   Write-TS(X)│
│ 타임스탬프 부여│         │  쓰기 타임스탬프값│
└───────────────────────────────────┘
         │              ▼
         │        ┌──────────────┐
         │        │  데이터(X)    │
         ▼        │ 읽기 타임스탬프값│
┌───────────────┐ └──────────────┘
│    TS(B)     │       │
│  트랜잭션 B  │◀──────┘
│              │   Write(X) 시
└───────────────┘   타임스탬프 비교
                   Write-TS(X) 및 Read-TS(X)

Write 수행한 트랜잭션 B의
    타임스탬프값
```

- 마지막으로 성공한 읽기 및 쓰기 타임스탬프 값과, 새로운 트랜잭션 타임스탬프 값을 비교하여 동시성 제어

---

## 구현기술 (Timestamp Ordering의 구현기술)

| 구분 | 구현기술 | 설명 |
|:-----|:---------|:-----|
| **타임스탬프 생성<br>방법** | 논리적 계수<br>(Logical Counter) | - 신규 입력 트랜잭션에 계수(Counter)를 1씩 증가 |
| | 시스템 시계<br>(System Clock) | - 신규 입력 트랜잭션에 시스템 시계 값 부여 |
| **타임 스탬프 값** | Read_TS(x) | - Read(x)를 성공적으로 수행한 트랜잭션 타임스탬프 중 가장 큰 값 |
| | Write_TS(x) | - Write(x)를 성공적으로 수행한 트랜잭션 타임스탬프 중 가장 큰 값 |

- DBMS는 트랜잭션이 시스템이 입력된 순서대로 타임스탬프 값을 할당
