# 낙관적 검증(Validation) 기법
{: .fs-8 }

5.5 낙관적 동시성 제어
{: .label .label-purple }

> 암기: **판확기**

---

## 핵심 키워드

`판독` `기록` `확인` `직렬 가능성` `유효성 검사` `Start(Ti)` `Validation(Ti)` `Finish(Ti)` `판독/기록 집합`

---

## 정의/개념

트랜잭션을 실행하는 동안 모든 갱신은 지역 사본에만 반영되고 트랜잭션 종료 시 확인 단계를 통해 직렬 가능성에 위반되지 않으면 실행하고 위반되면 복귀하는 기법

---

## 처리단계

```
          │                    │                    │
          ▼                    ▼                    ▼
    ┌───────────┐        ┌───────────┐        ┌───────────┐
    │Read Phase │        │Validation │        │Write Phase│
    │           │        │  Phase    │        │           │
    └─────┬─────┘        └─────┬─────┘        └─────┬─────┘
          │                    │                    │
          ▼                    ▼                    ▼
    Buffer(메모리)에서 수행   검증(Validation)    디스크에 반영(기록)
```

| 단계 | 설명 |
|:-----|:-----|
| **판독 단계<br>(Read Phase : R)** | - 트랜잭션의 모든 갱신은 사본에 대해서만 수행하고 실제 데이터베이스에 대해서는 미수행 |
| **확인 단계<br>(Validation Phase : V)** | - 판독 단계에서 사본에 반영된 트랜잭션의 실행 결과를 데이터베이스 반영 전 직렬 가능성 위반여부 확인 |
| **기록 단계<br>(Write Phase : W)** | - 확인 단계를 통과하면 트랜잭션의 실행 결과를 데이터베이스에 반영<br>- 확인 단계를 실패하면 실행 결과를 취소하고 트랜잭션은 복귀 |
