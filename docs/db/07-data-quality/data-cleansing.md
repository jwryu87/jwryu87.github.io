# 데이터 클렌징 (Cleansing)
{: .fs-8 }

7.3 데이터 정제
{: .label .label-purple }

> 암기: **정원우차제**

---

## 핵심 키워드

`대상 정의` `Dirty Data 원인` `품질 우선순위` `Bad Data 차단` `Bad Data 제거`

---

## 정의/개념

레코드 세트, 테이블 또는 데이터베이스에서 손상되거나 부정확한 레코드를 감지하고 전처리 과정을 통해 이상치 제거 및 식별, 교체, 삭제, 수정하는 기법

---

## 프로세스

```
   ①              ②              ③              ④              ⑤
┌───────┐     ┌───────┐     ┌───────┐     ┌───────┐     ┌───────┐
│데이터 │     │ RDBMS │     │Missing│     │Mission│     │       │
│베이스 │ ──▶ │ NoSQL │ ──▶ │ data  │ ──▶ │Critical│ ──▶ │  New  │ ──▶ Bad Data
│대상   │     │       │     │Outlier│     │Normal │     │ Rule  │     제거
│ 정의  │     │Dirty  │     │       │     │       │     │       │
│       │     │Data   │     │데이터 │     │Bad Data│     │       │
│       │     │원인파악│     │품질문제│     │유입차단│     │       │
│       │     │       │     │우선순위│     │       │     │       │
└───────┘     └───────┘     └───────┘     └───────┘     └───────┘
```

| 순서 | 프로세스 | 특징 |
|:-----|:---------|:-----|
| ① | 데이터베이스 대상 정의 | 데이터 클렌징 대상 데이터베이스 범위 정립 |
| ② | Dirty Data 원인 파악 | 데이터 결측치 및 이상치 식별 |
| ③ | 데이터 품질 문제 우선순위 | 클렌징 우선 순위 결정 |
| ④ | Bad Data의 데이터베이스 유입 차단 | 잘못된 데이터 입력되는 원인을 찾아 새로운 규칙 생성 |
| ⑤ | 데이터베이스에서 Bad Data 제거 | 데이터베이스 내부의 결측치 및 이상치 제거 |

- Data Ladder, InfoSphere, Winpure, Cloudingo 등의 도구를 이용하여 데이터 클렌징 수행

---

## 데이터 클렌징과 데이터 프로파일링의 비교

| 비교 항목 | 데이터 클렌징 | 데이터 프로파일링 |
|:----------|:-------------|:-----------------|
| **목적** | 데이터 전처리를 통한 이상치 결측치 제거로 데이터 품질 확보 수행 | 통계적 기법 활용한 데이터 구조, 관계 분석 통한 데이터 품질 확보 수행 |
| **분석기술** | 결측치/이상치 제거(Filtering)<br>결측치 변환(Avg, Median, Random)<br>이상치 제거(1.5 × IQR 기준 Box-plot) | 컬럼값 분석(누락값, 허용범위)<br>컬럼패턴 분석(문자열, 날짜유형)<br>컬럼유형 분석(유일값) |
| **도구** | Data Ladder, InfoSphere<br>SAS Data Management, OpenRefine | DataProfiler<br>Microsoft DOC |

- 데이터 클렌징, 프로파일링 기술 적용 통한 고품질 데이터 확보 및 데이터 활용 효율성 향상 기대
