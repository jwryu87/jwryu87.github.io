# 데이터 거버넌스 (Data Governance)
{: .fs-8 }

7.6 데이터 관리 체계
{: .label .label-yellow }

---

## 핵심 키워드

`관리원칙` `조직` `표준` `구조` `흐름` `품질` `데이터베이스` `보안` `관리프로세스`

---

## 정의/개념

전사 차원의 모든 데이터에 대한 정책, 지침, 표준화, 전략을 수립하고 데이터를 관리하는 **조직과 프로세스를 구축**함으로써 고품질의 데이터를 활용하여 기업의 가치 창출을 지원하는 체계

---

## 구성요소

```
┌─────────────────────────────────────────────┐
│            데이터 관리 원칙                   │
├─────────────────────────────────────────────┤
│            데이터 관리 조직                   │
├───────┬───────┬───────┬───────┬───────┬─────┤
│데이터 │데이터 │데이터 │데이터 │데이터 │데이터│
│ 표준  │ 구조  │ 효용  │ 품질  │베이스 │ 보안 │
├───────┴───────┴───────┴───────┴───────┴─────┤
│            데이터 관리 프로세스               │
├─────────────────────────────────────────────┤
│                 인프라                       │
└─────────────────────────────────────────────┘
```

| 구성요소 | 특징 |
|:---------|:-----|
| **데이터 품질관리(DQM)** | 데이터 프로파일링 및 데이터 정제와 같은 작업을 포함하여 데이터 사용방법에 따라 실행 |
| **메타 데이터 관리** | 데이터 검색 중 데이터를 찾고, 분석 도구가 빅데이터를 정확하게 해석하고 사용할 수 있도록 실행 |
| **데이터 주기 관리** | 데이터 생성 및 초기 저장에서부터 데이터가 폐기될 때까지 시스템의 데이터 흐름을 관리하는 정책 수립 필요 |
| **데이터 보안 및 프라이버시** | 데이터 요구 사항 및 정책은 사용자 역할을 기반으로 필요한 데이터 보호 수준을 정의 |

---

## 데이터 거버넌스 성숙도

| 단계 | 1단계 | 2단계 | 3단계 | 4단계 | 5단계 |
|:-----|:------|:------|:------|:------|:------|
| **구분** | 도입 | 프로세스화 | 통합경영 | 위험대응 | 가치창출 |
| **수준** | 데이터품질관리 | 프로세스화 | 경영진 주도 | 정량화 | 성과 공유 |
| **품질부문** | 전사 거버넌스 전략 없음 | 거버넌스 측면 품질관리 진행 | 정형화된 품질 관리 | 데이터를 통한 성과 생물 | |
| **통제부문** | 통제되지 않음 | 의사결정 체계 구축 | 거버넌스에 전사 전략 비전 반영 | 분석결과 신사업 데이터역량 | |
| **조직부문** | 독립적 부서 미 존재 | 담당자 배치 | 독립된 거버넌스 조직 및 위험 관리 수행 | 조직에 경영진 성과관리 | |
