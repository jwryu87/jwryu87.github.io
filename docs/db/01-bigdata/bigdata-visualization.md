## 정의/개념

정보의 목적에 부합하는 효과적인 전달을 위해 수집된 정보를 재조직하고, 시각화하여, 정보전달효과를 극대화하는 프로세스

---

## 프로세스

### ① 정보 구조화

| 프로세스 | 설명 |
|:---------|:-----|
| **주어진 정보를 탐색, 분류, 배열을 통해 구조적으로 그룹핑 하는 활동** | |
| - 데이터 수집 및 탐색 | - 데이터 자료 또는 논거를 수집<br>- 빅데이터 고유의 특성이 훼손되지 않으면서 수집 및 탐색 필요 |
| - 데이터 분류 | - 유사한 것들끼리 그룹핑하는 과정<br>- JSON, XML, CSV, TSV, 구분텍스트 |
| - 데이터 배열 | - 값의 의미에 따라 배치하는 과정<br>- 래치(LATCH) 방법(위치, 알파벳, 시간, 카테고리, 위계) |
| - 관계 탐색(재배열) | - 분류된 데이터를 정보수용자가 인식하기 쉽게 패턴 만드는 과정<br>- 이상값 처리, 차원, 측정값 유형별 관계파악 |

### ② 정보 시각화

| 프로세스 | 설명 |
|:---------|:-----|
| **방대한 양의 정보를 한번에 사용자가 보고, 탐험하고, 이해하도록 시각적 표현방법과 인터렉션 기술을 이용해 추상적 정보를 직관적으로 표현하는 활동** | |
| - 시간, 분포, 관계 시각화 | - 시간과 데이터의 분포 및 관계를 표현하는 기법<br>- 막대, 누적그래프, 파이차트, 스캐터, 버블, 히스토그램 등 |
| - 비교, 변수, 공간 | - 구분, 순서, 비율, 색채 사용과 인지 등의 기본 원리를 사용하여 시각화 |

### ③ 정보 시각 표현

| 프로세스 | 설명 |
|:---------|:-----|
| **정보시각화한 결과물을 그래픽적으로 디자인을 완성시키는 단계** | |
| - 디자인 기본원리 사용 | - 타이포그래피 (서체, 무게, 크기, 스타일, 색채, 간격)<br>- 색상(구분, 순서표현, 비율, 색채사용인지),<br>- 그리드(화면 읽는 방식, 정보의 역피라미드) |
| - 인터렉션 디자인 | - 다양한 유형의 시각 자료를 적용하고, 테스트하여 확인<br>- 강조 및 디테일 표시, 사용자 컨텐츠 선택방식, 사용자 지정시각 매핑변화 |
