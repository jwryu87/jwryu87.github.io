# 데이터베이스 인덱스(Index)의 종류
{: .fs-8 }

9.9 인덱스 유형
{: .label .label-yellow }

---

## 핵심 키워드

**인덱스:** `트리기반` `해시 기반` `함수기반` `정적` `동적` `논리` `물리`

---

## 정의/개념

어떤 파일의 레코드들에 대한 효율적 접근을 위해 <레코드 키 값, 레코드 주소(포인터)> 쌍을 체계적으로 수집하여 관리하는 데이터베이스 오브젝트

---

## 인덱스 종류

| 분류기준 | 인덱스 구조 | 설명 |
|:---------|:-----------|:-----|
| **형태** | 트리 기반 인덱스 | 인덱스(RDBMS는 대부분 B-tree) |
| | 해시 기반 인덱스 | Hash 테이블을 이용하여 데이터 검색<br>=, <, <=, > 연산자만 사용 가능 |
| | 비트맵 인덱스 | 컴퓨터에서 사용하는 최소 단위인 비트를 이용하여 컬럼 값을 저장하고 이를 이용하여 ROWID를 자동으로 생성하는 인덱스 |
| | 함수기반 인덱스 | 사용자 정의 함수 결과를 인덱스로 사용<br>데이터 타입이 상이한 컬럼 간 사용 |
| **목적** | 조인 인덱스 | DW에서 조인 쿼리의 처리가 가능 |
| | 도메인 인덱스 | 사용자 정의의 인덱스 타입을 사용<br>텍스트, 카테고리 인덱스 등 |
| **구조** | 정적 인덱스 | 데이터 파일에 레코드가 삽입되거나 삭제됨에 따라 인덱스의 내용은 변하지만 인덱스 구조 자체는 변경되지 않게 하는 인덱싱 기법 |
| | 동적 인덱스 | 인덱스와 데이터 파일을 블록으로 구성하고, 각 블록에는 나중에 레코드가 삽입될 것을 감안하여 빈 공간을 미리 준비해두는 인덱싱 기법 |
| **논리** | 논리적 인덱스 | 인덱스가 생성될 때 컬럼의 속성과 유형에 따라 분리 |
| | 물리적 인덱스 | 인덱스가 저장되는 데이터 구조에 따라 나누어지는 방법 |
