## 데이터베이스 성능 개선 유형

### 설계 관점

| 구분 | 핵심 | 설명 |
|:-----|:-----|:-----|
| **데이터 구조의<br>논리적 설계** | 논리적 관점의 성능 개선 | 업무적 특성을 반영한 데이터 모델을 바탕으로 반정규화 등을 통한 논리적 구조의 변경 |
| **데이터 구조의<br>물리적 설계** | 실제 저장 구조 관점의<br>성능 개선 | 데이터 파일에 대한 저장 방식의 변경을 통해 성능 개선을 꾀하는 방법 |
| **DBMS** | DBMS 동작을 위한 파라미터 변경 | SGA/PGA 영역, Redo 파일 구성 등 DBMS 운영 측면의 성능 개선 방법 |
| **H/W** | DBMS 운영을 위한 자원 할당 | OS 레벨에서의 CPU, 메모리, N/W, 스토리지 등의 H/W 성능 개선 |

### 구현 관점

| 구분 | 핵심 | 설명 |
|:-----|:-----|:-----|
| **SQL** | 실행 경로 최적화 | I/O 효율 향상, call 최소화 등을 기반으로 옵티마이저가 최고의 성능을 낼 수 있도록 개선하는 방식 |
| **어플리케이션** | DBMS 접근을 위한 요청<br>action 관리 | 요청 트랜잭션 관리, 파싱 부하, Locking 최소화 등 DBMS를 접근하는 응용 프로그램 레벨에서의 성능 개선 |

## 데이터베이스 성능 개선 유형별 기법

| 구분 | 핵심 | 설명 |
|:-----|:-----|:-----|
| **논리 모델링** | 테이블 분할 | 데이터의 량이 증가가 예상되거나 처참 수가 많은 경우 자주 접근 할 컬럼 위주로 분할 설계 |
| | 식별자 지정 | 본질/인조 식별자 정의 등 |
| | 반정규화 실시 | 테이블, 컬럼, 관계 등의 반정규화 실시로 연산 부하 최소화 |
| **물리 모델링** | 적절한 데이터 타입 선정 | 내부 형변환에 의한 성능 저하 예방 |
| | 파티셔닝 | 접근 대상 테이블의 행과 열을 분할하여 필요한 부분만 읽을 수 있도록 개선 |
| | 효율적 인덱스 설정 | 자주 사용되는 컬럼에 대한 조회 성능 개선 |
| | 클러스터 및 IoT 구성 | 데이터의 군집, 정렬된 상태를 활용 |
| | 뷰, 시퀀스 설계 | 뷰, 시퀀스 등의 이용을 통해 SQL 파싱, call 최소화 |
| **DBMS** | I/O 최소화 | 실제 필요한 데이터만 Read |
| | Query off-loading 기법 등 | 지역성을 최대한 활용하여 연산 성능 개선 |
| | Buffer pool 튜닝 | Check point 발생 시의 오버헤드 최소화 |
| | Check point 주기 | Check point 발생 시의 오버헤드 최소화 |
| | Undo Segment 설정 | Undo 영역 크기 조정 |
| **SQL** | 힌트 | 최적의 실행 경로 유도 |
| | 인덱스 활용 | 구성된 인덱스를 이용할 수 있도록 SQL 구조 변경 |
| | 조인 방식 | 최적의 실행 경로 유도 |
