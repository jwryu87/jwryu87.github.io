# 이상현상 (Anomaly)
{: .fs-8 }

3.1 정규화
{: .label .label-purple }

> 암기: **삽삭갱**

---

## 핵심 키워드

`삽입이상` `갱신이상` `삭제이상`

---

## 정의/개념

데이터베이스를 설계할 때 발생하는 논리적인 오류로, 데이터를 삽입, 삭제, 수정하는 과정에서 예상치 못한 문제가 발생하는 현상

---

## 이상현상 3가지와 발생이유

| 구분 | 설명 | 주요원인 |
|:-----|:-----|:---------|
| **삽입이상** | 릴레이션에 레코드를 추가할 때 불필요한 칼럼들의 값 없이는 릴레이션에 새로운 레코드를 삽입하지 못하는 현상 | 완전함수종속 관계가 아닌 경우(부분/이행 함수 종속관계가 있는 경우) |
| **갱신이상** | 하나의 릴레이션에 불필요한 중복 레코드가 포함되어, 모든 레코드를 수정하지 않으면 데이터의 일관성이 저해되는 현상 | 중복 데이터 존재 |
| **삭제이상** | 키 값의 삭제시 의도하지 않았던 전혀 다른 데이터가 삭제되는 현상 | 완전함수종속 관계가 아닌 경우(부분/이행 함수 종속관계가 있는 경우) |

---

## 해결 방안

- 위의 3가지 이상현상을 기준으로 주어진 테이블의 이상현상 발생이유를 설명
- **정규화**를 통해 이상현상 제거
