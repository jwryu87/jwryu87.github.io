{: .fs-8 }

3.2 무결성
{: .label .label-yellow }

> 암기: **상과집튜즉지**

---

## 핵심 키워드

`상태 제약` `과도 제약` `집합 제약` `튜플 제약` `즉시 제약` `지연 제약`

---

## 정의/개념

릴레이션을 조작함에 있어 삽입, 삭제, 갱신 등 트랜잭션의 수행 전과 수행 후에 대한 의미적 관계를 통해 유지하는 특성

---

## 릴레이션 무결성 구현 방법

```
                        ┌─────────────┐
                        │ 데이터베이스 │
                        │    명세     │
                        └──────┬──────┘
                               │
         ┌─────────────────────┼─────────────────────┐
         │                     │                     │
         ▼                     ▼                     ▼
   ┌───────────┐         ┌───────────┐         ┌───────────┐
   │  TRIGGER  │         │Foreign Key│         │  SQL-DDL  │
   └───────────┘         └───────────┘         └───────────┘
   트리거 조건            부모,자식              DDL내에
   만족시 취해야          엔티티간               제약조건을
   하는 조치 명세         튜플관계 명세          명시

                    ┌─────────────────┐
                    │   릴레이션      │
                    │   무결성        │
                    │  구현 방법      │
                    └─────────────────┘
         ┌─────────────────────┼─────────────────────┐
         │                     │                     │
         ▼                     ▼                     ▼
   ┌───────────┐         ┌───────────┐         ┌───────────┐
   │사용자 정의│         │Application│         │  SQL-DCL  │
   │   명세    │         └───────────┘         └───────────┘
   └───────────┘         트랜잭션의             Biz 로직을
                         수행을 제어            직접 Software
                                               를 통해 적용
```

---

## 제약조건 분류

### 1. 변환기준 (동적/정적)

| 구분 | 제약조건 | 설명 |
|:-----|:---------|:-----|
| **변환기준** | 상태 제약 | 특정시점에 만족되어야 하는 정적제약의 성질을 가짐 |
| | 과도 제약 | 데이터베이스 변경 전, 후 모두 만족되어야 하는 동적제약의 성질을 가짐 |

### 2. 범위기준

| 구분 | 제약조건 | 설명 |
|:-----|:---------|:-----|
| **범위기준** | 집합 제약 | 튜플 집합 전체에 관련되어 적용되는 규칙 |
| | 튜플 제약 | 처리되고 있는 튜플에만 적용되는 규칙 |

### 3. 시점기준

| 구분 | 제약조건 | 설명 |
|:-----|:---------|:-----|
| **시점기준** | 즉시 제약 | 연산이 수행되는 즉시 적용되는 규칙 |
| | 지연 제약 | 트랜잭션이 완전히 수행된 후 적용되는 규칙 |

---

## 고려사항

- 동적/정적기준, 적용범위, 적용시점에 따른 릴레이션 무결성 제약
