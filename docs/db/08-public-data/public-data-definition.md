# 공공데이터의 정의 및 포맷별 유형
{: .fs-8 }

8.1 공공데이터 개념
{: .label .label-purple }

---

## 핵심 키워드

`PDF` `XLS` `CSV` `RDF` `LOD` `기계 판독 가능 형태`

---

## 정의/개념

데이터베이스, 전자화된 파일 등 공공기관이 법령 등에서 정하는 목적을 위하여 생성 또는 취득하여 관리하고 있는 데이터

---

## 공공데이터의 포맷별 유형

| 구분 | 기계 판독 가능 형태 | 특징 | 예시 |
|:-----|:-------------------|:-----|:-----|
| **1 단계** | 미충족(공공데이터 포털 등록불가) | 특정 소프트웨어로 읽기 가능<br>자유로운 수정, 변환 불가 | PDF |
| **2 단계** | 최소충족포맷 | 특정 소프트웨어에서 읽기 가능<br>자유로운 수정, 변환 가능 | XLS, JPG, WMV, MP3 |
| **3 단계** | | 모든 소프트웨어에서 읽기 가능<br>자유로운 수정, 변환 가능 | CSV, JSON, XML |
| **4 단계** | 오픈포맷 | URI 기반 데이터 특성, 속성 관계 기술 | RDF |
| **5 단계** | | 웹 상 다른 데이터와 연결, 공유 가능 | LOD |

{: .highlight }
> 기계 판독 가능 형태란 소프트웨어로 데이터의 개별내용 또는 내부구조를 확인하거나 수정, 변환, 추출 등 가공할 수 있는 상태로 공공데이터 포맷별 유형 판단의 기준으로 사용

---

## 포맷별 특징

| 포맷 | 특징 | 활용 |
|:-----|:-----|:-----|
| **PDF** | 문서 형태 보존, 편집 불편 | 보고서, 문서 공유 |
| **XLS/CSV** | 구조화된 데이터, 편집 용이 | 통계, 분석 데이터 |
| **JSON/XML** | 구조화, 기계 판독 가능 | API, 시스템 연계 |
| **RDF** | 의미 기술, 관계 표현 | 시맨틱 웹, 지식그래프 |
| **LOD** | 연결된 데이터, 공유 가능 | 데이터 연계, 통합 |

---

## 공공데이터 개방 단계

| 단계 | 설명 |
|:-----|:-----|
| **1단계** | 데이터 단순 공개 (PDF 등) |
| **2단계** | 구조화된 데이터 제공 (XLS 등) |
| **3단계** | 비독점 개방 포맷 (CSV, XML 등) |
| **4단계** | URI 기반 데이터 식별 (RDF) |
| **5단계** | 다른 데이터와 연계 (LOD) |
