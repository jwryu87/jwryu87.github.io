# NewSQL
{: .fs-8 }

2.1 DB 유형
{: .label .label-purple }

---

## 핵심 키워드

`SQL기반 상호작용` `ACID 지원` `비 잠금 동시성 제어` `노드 단위 고성능` `병렬/비 공유 아키텍처`

---

## 정의/개념

RDBMS의 ACID 특성을 유지하면서 NoSQL의 성능과 확장성을 제공하는 데이터베이스 관리시스템

---

## 관계/기능

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   RDBMS     │     │   NoSQL     │     │   NewSQL    │
│   (SQL)     │ ──▶ │             │ ──▶ │             │
└─────────────┘     └─────────────┘     └─────────────┘
  - ACID 특성         - 수평적 확장성       - ACID 특성
  - SQL 지원          - 고가용성(HA)        - 수평적 확장성
                                          - 고가용성(HA)
                                          - SQL 지원
```

| 구분 | 기능 | 설명 |
|:-----|:-----|:-----|
| **트랜잭션** | - SQL 기반 상호작용 | - App의 DBMS 연계(입력/조회/갱신/삭제) 시<br>SQL 을 사용해 통신 |
| | - ACID 지원 | - RDBMS에서 가장 중요한 트랜잭션<br>커밋(Commit)을 위한 필수 속성인<br>ACID(원자성, 일관성, 고립성, 지속성)을<br>지원해야 함 |
| | - 비잠금 동시성제어 | - 데이터 무결성 처리를 위해 지원하는 트랜잭션<br>동시 제어 잠금 처리와 관련해 기존 방식과 다른<br>Non-locking 구조 지원 |
| **아키텍처** | - 노드단위 고성능 | - 각 단일 DBMS 서버 노드 단위로 확장해<br>고성능을 보장 |
| | - 병렬/비공유 | - 병렬 아키텍처 기반 수행으로 데이터를<br>고성능으로 처리<br>- 분산 처리시 데이터가 각 서버에 중복되지 않고<br>독립적으로 존재 |

---

## 기술요소

| 구분 | 기술요소 | 설명 |
|:-----|:---------|:-----|
| **RDBMS<br>측면** | - 인덱싱 | - 데이터베이스 검색 속도 향상, 테이블 연관, 독립적인 저장 공간 |
| | - MVCC | - 트랜잭션의 다중 버전 동시성 제어로 트랜잭션 직렬화 |
| | - 샤딩 | - 동일 테이블 스키마의 데이터를 다수 데이터베이스에 분산 저장 |
| | - 스키마리스 | - 테이블과 컬럼 스키마 없이 Key-Value 기반 단순 검색, 추가 용이 |
| **NoSQL<br>측면** | - 인메모리 | - 고성능, 서지연 서비스, 버퍼 관리 불필요 |
| | - DB 스케일링 | - scale-out 방식의 유연한 데이터 구조 |

---

## 비교

| 구분 | NewSQL | RDBMS | NoSQL |
|:-----|:-------|:------|:------|
| **ACID 특성** | - ACID 특성 제공 | - ACID 특성 제공 | - ACID 특성 미제공 |
| **BASE 특성** | - BASE 특성 제공 | - BASE 특성 미제공 | - BASE 특성 제공 |
| **스키마** | - Schema-less | - Schema-full | - Schema-less |
| **확장성** | - Scale-out | - Scale-up | - Scale-out |
| **솔루션** | - Volt DB, Spanner | - Oracle, MSSQL | - MongoDB, Redis |
