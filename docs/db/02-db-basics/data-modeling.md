## 핵심 키워드

`요구사항 분석` `개념모델링` `논리모델링` `물리모델링` `ERD` `엔티티` `속성`

---

## 정의/개념

현실 세계의 업무적인 프로세스를 물리적으로 데이터베이스화 하기 위한 과정으로 **개체와 관계를 중심으로 체계적으로 표현하고 문서화하는 기법**이며, 정보 시스템의 중심을 데이터의 관점에서 접근하는 데이터 중심의 분석 방법

---

## 모델링 단계

| 단계 | 절차 | 내용 | 고려사항 |
|:-----|:-----|:-----|:---------|
| **개념<br>모델링** | 주제 영역 정의 | 데이터의 최상위 집합을 구분하는 단계<br>전사적 관점에서 데이터 분류 수행 | 중복 최소화, 확장성 보장, 관련성 및 권의성 확보 |
| | 후보 엔티티 선정 | 엔티티가 될 가능성이 있는 후보 수집 | 임의 제외 금지, 현업참여, 상세화 배제 |
| | 핵심 엔티티 정의 | 후보 중 핵심 엔티티 선별<br>주식별자, 서브타입 등 정의 | 관리여부, 집합여부, 통질성 여부 등 확인 |
| | 관계 정의 | 관련이 있는 엔티티간의 연관성을 정의<br>(1:1, 1:M, M:M 등) | 항상 두 엔티티간 존재, 양방향, 항상 두 개의 관점 존재 |
| | 속성 정의 | 분해 불가능한 최소의 데이터 단위로<br>상세화 속성 후보 도출 후 검증, 확정 | 최소 분할은 필수, 필요시 분할 후 통합 |
| **논리<br>모델링** | 엔티티 상세화 | 식별자 확정, 정규화, M:M관계 해소 | 속성은 하나의 값, 속성간 종속 제거, 참조무결성 규칙 |
| | 이력 관리 정의 | 정보로서의 활용성 추적성 확보, 시점, 선분 이력 | 물리적 성능 고려, 키 및 인덱스 고려 |
| | 물리 환경 조사 | 시스템 구축 환경 조사 및 분석 확장,<br>HW 자원, OS, DBMS | 사용자 관리기법, 백업정책, 보안 정책 고려 |
| **물리<br>모델링** | 논리모델 변환 | 테이블, 속성, 관계, 제약조건 등 전환,<br>슈퍼/서브타입 모델 전환 | PK 최적화, 통합여부 고려, 의사결정을 통한 조정 및 합의 |
| | 반정규화 | 성능, 개발편의성, 운영용이성 확보 | 반정규화 최소화 원칙, SQL 및 인덱스 조정을 고려 |

---

## 모델링 단계 흐름도

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  요구사항   │ → │  개념 모델링  │ → │  논리 모델링  │ → │  물리 모델링  │
│   분석      │    │  (ERD 작성)  │    │  (정규화)    │    │  (테이블 생성)│
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
```

---

## 각 단계별 산출물

| 단계 | 주요 산출물 |
|:-----|:-----------|
| **개념 모델링** | ERD (Entity Relationship Diagram) |
| **논리 모델링** | 정규화된 릴레이션, 스키마 |
| **물리 모델링** | 테이블, 인덱스, 파티션 설계서 |
