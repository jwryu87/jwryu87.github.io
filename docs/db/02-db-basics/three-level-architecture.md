---

## 정의/개념

| 구분 | 정의 |
|:-----|:-----|
| **논리적 독립성** | DBMS가 데이터베이스의 논리적 구조를 변경시키더라도 기존 응용 프로그램에 영향을 주지 않는 성질 |
| **물리적 독립성** | DBMS가 응용 프로그램이나 데이터베이스의 논리적 구조에 영향을 주지 않고 데이터의 물리적 구조를 변경시킬 수 있는 성질 |

## 3단계 데이터베이스 구조 및 스키마

```
          ┌─────────────────────────────────────────┐
 외부 단계 │ External Schema#1 │ External Schema#2 │ External Schema#3 │
          └─────────┬─────────┴─────────┬─────────┴─────────┬─────────┘
                    │                   │                   │    논리적
                    │         ┌─────────┴─────────┐         │    데이터 독립성
                    └─────────┤ Conceptual Schema ├─────────┘
 개념적 단계                   └─────────┬─────────┘
                                        │                        물리적
                                        │                        데이터 독립성
 내부적 단계                  ┌─────────┴─────────┐
                              │   Internal Schema │
                              └───────────────────┘
```

## 스키마별 특성

| 스키마 | 내용 | 관리 정보 |
|:-------|:-----|:----------|
| **외부 스키마**<br>(External Schema) | DB 사용자나 응용 프로그래머가 접근하는 데이터베이스를 정의 | 해당 응용 프로그램이나 사용자에 관련된 개체와 관계 정보 |
| **개념 스키마**<br>(Conceptual Schema) | 범기관적 입장에서 데이터베이스를 정의한 것<br>모든 응용 시스템들이나 사용자들 필요로 하는 데이터를 통합한 조직 전체의 데이터베이스를 기술한 것<br>하나의 데이터베이스 시스템은 하나의 개념스키마만 존재 | 모든 데이터 객체 정보(개체, 관계 및 제약조건)와 객체를 효율적 관리를 위한 필수 정보 (접근권한, 보안정책, 무결성 규칙에 대한 명세) |
| **내부 스키마**<br>(Internal Schema) | 저장 장치 입장에서 데이터베이스 전체가 저장되는 방법을 명세한 것<br>개념 스키마에 대한 저장 구조 정의 | 실제 저장된 내부 레코드 형식, 인덱스 유무, 저장 데이터 항목 표현 방법, 내부 레코드의 물리적 순서 등 |
