## 정의/개념

대용량 데이터베이스 다차원 모델링 위해 사실, 차원 테이블 이용 모델링 기법

---

## 아키텍처: 스타 스키마

```
              ┌─────────────┐
              │ 차원테이블5  │
              └──────┬──────┘
  ┌─────────────┐    │    ┌─────────────┐
  │ 차원테이블1  │────┼────│ 차원테이블4  │
  └──────┬──────┘    │    └──────┬──────┘
         │    ┌──────┴──────┐    │
         └────┤  사실테이블  ├────┘
              │   (노란색)   │
         ┌────┴──────┬──────┴────┐
         │           │           │
  ┌──────┴──────┐ ┌──┴──┐ ┌──────┴──────┐
  │ 차원테이블2  │ │     │ │ 차원테이블3  │
  └─────────────┘ └─────┘ └─────────────┘
```

- 중앙에 **사실 테이블(Fact Table)**, 주변에 **차원 테이블(Dimension Table)** 배치
- 단순한 조인 구조로 빠른 쿼리 성능

---

## 아키텍처: 스노우플레이크

```
┌───────┐                     
│ 시설1 │    ┌──── 대해테이블 ──────┐
│       │    │  이  │시설10│ 이  │ 
│ 대표ID│    │  발상│(DV) │ 등록│
│ 시업1IP│   │      │      │수학11│
│ 시설15│    │      │      │      │
│ 이전16│    └───────┼──────┘     
└───┬───┘      [3차 정규화]        
    │                              
┌───┴───┐                          
│ 사실2 │    ─── DL-Dimension ─────
│       │                          
│ 대    │                          
│       │                          
└───────┘                          
```

- 차원 테이블을 **정규화**하여 세분화
- 데이터 중복 감소, 저장 공간 절약
- 조인이 많아져 쿼리 성능 저하 가능

---

## 구성요소

| 구분 | 구성요소 | 핵심 |
|:-----|:---------|:-----|
| **모델링<br>속성** | 사실 | 도메인 수치 표현 |
| | 차원 | View 제공 |
| | 속성 | 차원 테이블 속성 |
| | 속성 계층 | 차원 내 존재 특성 |
| **테이블** | 사실 테이블 | 차원테이블 수치화된 Data |
| | 차원 테이블 | 사실테이블 다양한 관점 |

---

## 스타 스키마 vs 스노우플레이크 비교

| 구분 | 스타 스키마 | 스노우플레이크 |
|:-----|:-----------|:---------------|
| **정규화** | 비정규화 | 정규화 (3NF) |
| **조인 수** | 적음 | 많음 |
| **쿼리 성능** | 빠름 | 상대적으로 느림 |
| **저장 공간** | 중복으로 인해 큼 | 효율적 |
| **유지보수** | 상대적으로 어려움 | 용이 |
| **사용 환경** | OLAP, 분석용 | DW, 분석용 |
