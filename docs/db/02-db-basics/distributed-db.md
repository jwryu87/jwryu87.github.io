## 핵심 키워드

`위치` `이주` `복제` `병행` `접근` `성능` `규모` `고장 투명성`

---

## 정의/개념

물리적으로는 여러 사이트로 분산되어 있으나 네트워크로 상호 접속해 논리적으로 하나인 데이터베이스(또는 시스템)

---

## 유형

| 투명성 | 설명 |
|:-------|:-----|
| **위치(Location) 투명성** | 사용자가 자원의 물리적인 위치를 모르더라도 자원에 접근 가능 |
| **이주(Migration) 투명성** | 사용자나 응용프로그램의 동작에 영향을 받지 않고 시스템 내에 있는 자원의 이동 가능 |
| **복제(Replication) 투명성** | 자원의 복제를 사용자에게 통지할 필요 없이 자유롭게 수행 |
| **병행(Concurrency) 투명성** | 자원의 위치를 모르더라도 다중 사용자들이 자원을 병행하여 처리하고 공유 가능 |
| **접근(Access) 투명성** | 각 프로세서의 로그인 등과 같은 동작을 사용하여 지역이나 원격자원에 접근 |
| **성능(Performance) 투명성** | 여러 부하에 대해 성능을 증가시키기 위해 시스템을 재구성 가능 |
| **규모(Scaling) 투명성** | 시스템이나 응용프로그램들이 시스템 구조나 응용 알고리즘에 대한 변경 없이 규모에 맞추어 확장 가능 |
| **고장(Failure) 투명성** | 사용자나 응용프로그램이 구성요소의 고장에도 불구하고 작업 완료 가능 |

## 분산 데이터베이스 구조

```
┌─────────────────────────────────────┐
│      Graphic User Interface         │
└─────────────────┬───────────────────┘
                  │
                  ▼
┌─────────────────────────────────────┐     ┌──────────────────────┐
│      Global Query Processor          │     │ Schema Transformation │
└─────────────────┬───────────────────┘     │     Repository        │
                  │                          └──────────────────────┘
                  ▼
         ┌───────────────┐
         │ Global Schema │
         └───────┬───────┘
                 │
                 ▼
┌─────────────────────────────────────┐
│      Schema Transformation Tool      │
└─────────────────┬───────────────────┘
                  │
      ┌───────────┴────────────┐
      │                        │
      ▼                        ▼
┌────────────┐           ┌────────────┐
│Local Schema 1│ • • • •  │Local Schema N│
└──────┬─────┘           └──────┬─────┘
       │                        │
       ▼                        ▼
┌────────────┐           ┌────────────┐
│ Local Data │ • • • • • │ Local Data │
│  Source 1  │           │  Source N  │
└────────────┘           └────────────┘
```
