## 정의/개념

- **회선교환 방식**: 송수신 단말장치 사이에서 데이터 전송할 때마다 통신경로를 설정하여 데이터를 교환하는 방식
- **패킷교환 방식**: 일정한 크기의 데이터 블록인 패킷을 교환기가 수신측 주소에 따라 적당한 통신경로를 선택하여 전송하는 교환방식

---

## 회선교환 vs 패킷교환 비교

| 항목 | 회선교환방식 | 패킷교환방식 |
|:-----|:------------|:------------|
| **회선 효율성** | 전송전 회선 독점 | 회선 다중화로 효율증대<br>경제성 향상 |
| **신뢰성** | 독점사용<br>주변트래픽에 의한 회선 끊김없음 | 우회 기능 보유를 통한 신뢰성 확보 |
| **Busy시** | Busy신호 | 전달 불가시 메시지 저장 |
| **메시지 저장** | 메시지 저장 안함 | 전달시까지 메시지 저장 |
| **데이터 길이** | 비교적 긴 데이터 전송 가능 | 데이터 단위의 길이가 제한됨 |
| **오버헤드 비트** | 없음 | 패킷마다 오버헤드 비트 존재 |
| **유형** | 공간분할 방식, 시분할 방식 | 데이터그램, 가상회선 방식 |
| **사례** | 전화망 | 인터넷망 |
| **장점** | 대용량+고속 데이터 처리 우수<br>고정적인 대역폭을 사용<br>연속적인 데이터 처리에 우수 | 회선의 이용률이 높음<br>에러 및 장애에 강함<br>다양한 통신망에서 사용가능 |
| **단점** | 대역폭 낭비<br>통신비용 고가 | 경로 탐색과정에서 지연이 발생<br>전송량 증가에 따른 지연율 상승<br>패킷헤더 추가 |
