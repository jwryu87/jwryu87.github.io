# SDx (Software Defined Everything)
{: .fs-8 }

6.1 네트워크 관리
{: .label .label-purple }

---

## 핵심 키워드

`SDC` `SDS` `SDDC` `SDSec` `SDR` `SDN` `SDH` `SDVN`

---

## 정의/개념

소프트웨어로 정의되는 모든 기술의 총칭으로, 인프라가 조정됨으로써 상위 단계의 SDX 구현을 통해 SDX생태계가 완성됨

---

## 인프라 측면에서의 SDX 생태계 기술 유형

| 기술유형 | 설명 |
|:---------|:-----|
| **SDC**<br>(Compute) | (정의) 시스템 소프트웨어 Stack의 다양한 레이어에서 서버-컴퓨터 추상화 구현기술<br>- **가상머신**: 클라우드 구현 위한 하이퍼바이저 및 어플리케이션 개발 까지를 포괄하는 SW 스택<br>- **컨테이너 연결**: OS지상에서 독립된 어플리케이션 실행공간을 제공하는 OS가상화, OS가상화진의 컨테이너 간 충돌 스프트웨어<br>- **클라우드 솔루션**: 서버가상화와 SW와 Node-level컨트롤러가 결합한 클라우드 풀 솔루션 |
| **SDS**<br>(Storage) | (정의) 기존 HW인프라에서 제어 및 관리 SW를 분리, 스토리지 리소스 가상화 기술<br>- **Automation**: 설정된 정책에 따라 자동으로 저장장치의 볼륨 생성 및 서비스<br>- **Standard Interface**: 저장장치/서비스 관리, 유지보수를 위한 RESTful API제공<br>- **Virtualized Data Path**: 사용자 요구 및 특성에 부합되는 객체, 파일, 블록 인터페이스 제공, 물리적 공간 할당 및 데이터 배치 등 사용자 고려 접근 경로 제공<br>- **Scalability**: 가용성을 확보하면서 스토리지 인프라 확장<br>- **Data Protection**: 데이터 손실상황에 대비한 보관기능, 특정시점 기준 복원 기능 |
| **SDDC**<br>(Data Center) | (정의) 데이터센터 아키텍처를 SDC, SDS, SDN 기술을 바탕으로 구현한 데이터 센터<br>- **HCI(Hyper converged Infrastructure)**: SDDC 구축 통합솔루션으로 자원을 Open HW, SW를 단일인프라로 구성하는데 사용 |

---

## 보안, 네트워크 및 응용 서비스 측면에서의 SDX 생태계 기술 유형

| 구분 | 기술유형 | 설명 |
|:-----|:---------|:-----|
| **보안** | SDSec<br>(Security) | OpenFlow기술을 기반으로 네트워킹 기술을 컴퓨팅기술을 모델링하여 프로그램(SW)에 의하여 네트워크 제어 및 관리를 용이하게 하는 네트워크 기술 |
| | SDR<br>(Radio) | 통신시스템을 구성하는 기지국과 단말기에서 통신 하드웨어 RF를 지원하던 방식을 소프트웨어 형태로 바꾸는 핵심 기술 |
| **네트워크** | SDN<br>(Network) | OpenFlow기술을 기반으로 네트워킹 기술을 컴퓨팅기술로 모델링하여 프로그램(SW)에 의하여 네트워크 제어 및 관리를 용이하게 하는 네트워크 기술<br>오픈플로우: 네트워크 장비의 패킷포워딩 기능과 컨트롤러 기능을 표준 인터페이스로 분리하여 네트워크 개방성을 제공하는 기술 |
| **응용 서비스** | SDH<br>(Healthcare) | 기존 데이터센터 인프라를 포함하여 프라이버시에 민감한 사용자 데이터가 스토리지에 저장되고 해당 데이터가 유출되지 않도록 안정성 있는 네트워크 및 보안을 제공하는 서비스 |
| | SDVN<br>(Video Network) | SDN의 개념을 방송에 접목, 변하는 비디오 네트워크의 요구 사항을 유연성 있게 처리하고 여러 방송 장비 및 네트워크의 상호운용성을 지원하는 서비스 |

{: .note }
> 기존 인프라 호환성, 보안, 표준화, 책임, 전문 인력 등에 대한 기술 고도화 및 대책 필요
