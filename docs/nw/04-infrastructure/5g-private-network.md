---

## 핵심 키워드

`Type1` `Type2` `Type3` `이음5G`

---

## 정의/개념

특정 기업 등에 속한 사람/사물 접속 연결 허용하고, 필요한 특정 서비스 제공해주는 **맞춤형 사설망**

---

## 구축 유형

| 유형(구축주체) | 도입 방식 | 도입 예 |
|:-------------|:---------|:-------|
| **Type 1**<br>(수요기업) | 수요기업이 자가망 설치자로 신고<br>(수요 기업이 설치 및 서비스 제공) | A사가 사업장에 5G 특화망을 설치, 운영하고, 자사만 사용 |
| **Type 2**<br>(수요기업) | 수요기업이 기간통신사업자로 등록<br>(수요기업 설치 자사·협력사에 서비스 제공) | B사가 자사 건물에 5G 특화망을 설치, 운영하고, 자사 사용 뿐만 아니라 건물 입주자에게 5G 특화망 서비스를 제공 |
| **Type 3**<br>(제 3자) | 제 3자가 기간통신사자로 등록<br>(수요기업 설치 협력사에 서비스 제공) | 대형경기장에 VRAR 기업이 5G 특화망을 설치·운영하고, 관람인원에게 5G 특화망을 이용한 VRAR 서비스를 제공하는 경우 |

{: .note }
> 과기정통부는 5G 특화망 전용 주파수로 28Ghz 대역 600Mhz폭과 4.7Ghz 대역 100Mhz 폭을 최종 결정함.

---

## 주요기능

### 제어부

| 구분 | 주요기능 | 역할 |
|:-----|:---------|:-----|
| **AMF** | Access & Mobility Management Function | 단말 접속시 인증 및 핸드오버, 네트워크 슬라이싱 기능 등 |
| **SMF** | Session Management Function | 단말과 데이터 망간의 세션생성 및 UPF의 라우팅 설정, 과금 데이터 수집, 단말의 IP할당, QoS 제어 등 |
| **PCF** | Policy Control Function | 단말 서비스별 세션관리 등 제어부 기능에 대한 정책 |
| **UDM** | Unified Data Management | 5G 인증 및 키 합의 프로토콜(AKA) 생성, 사용자 식별처리, 액세스 권한부여, 구독 관리 등 |
| **AF** | Application Function | 애플리케이션에 대한 트래픽 라우팅 등 |
| **NEF** | Network Exposure Function | 제 3자등 개발자에 안전하고 진화적인 자체 네트워크 서비스 생성환경 제공 |
| **AUSF** | Authentication Server Function | 네트워크 접속 단말가입자 인증 서버로서 동작 |

### 데이터부

| 구분 | 주요기능 | 역할 |
|:-----|:---------|:-----|
| **UPF** | User Plane Function | 단말 이동에 대한 앵커 포인트 및 단말과 데이터 네트워크 사이의 게이트웨이 역할 수행, 패킷 데이터 처리/전달 등 |

{: .highlight }
> 5G 특화망은 현재 현 정 및 네이버 클라우드에서 특화망 신청 및 승인이 완료되었으며, 활성화를 위해 지속적으로 신청을 받아 확대해 나갈 예정임
