# 링크스테이트 라우팅 (Link State Routing)
{: .fs-8 }

7.1 OSI 7 Layer
{: .label .label-purple }

---

## 핵심 키워드

`SPF 알고리즘` `Link State Packet` `Symbolic Length` `OSPF` `EIGRP`

---

## 정의/개념

한 라우터가 목적지까지 모든 경로를 알고 토폴로지 DB 기반으로 **SPF 알고리즘 이용하여 라우팅 경로를 결정하는 기법**

---

## 메커니즘

| 동작 원리 | 설명 |
|:---------|:-----|
| **① 개별 라우터는 자신과 인접 라우터를 찾음** | Link-State Routing Protocols |
| **② 링크 비용(시간, 대역폭, 트래픽) 계산** | - |
| **③ Link State Packet 구성** | - |
| **④ 위 패킷을 망의 모든 라우터에 전송 (Broadcasting)** | - |
| **⑤ 모든 라우터에서 받은 Link state DB를 구성** | Link-State Packets → Topological Database → SPF Algorithm → Routing Table |
| **⑥ 이 DB를 사용하여, 모든 라우터에 이르는 최단 경로(Shortest paths)를 계산** | Shortest Path First Tree |

---

## 거리벡터 라우팅과 링크스테이트 라우팅 차이점 비교

| 차이점 항목 | 거리벡터 라우팅 | 링크스테이트 라우팅 |
|:----------|:-------------|:-----------------|
| **패킷 전송** | 거리와 방향을 기준으로 전송 | 전체 토폴로지를 고려하여 전송 |
| **주요 메트릭** | Hop count | Symbolic length |
| **경로 설정** | 라우터 간의 거리를 더하여 계산 | 다른 라우터까지의 최단경로 계산 |
| **알고리즘** | 벨만포드 알고리즘 | 다익스트라 알고리즘 |
| **라우팅테이블 갱신 대상** | 인접한 라우터 | 동일 네트워크 내의 모든 라우터 |
| **갱신 시점** | 일정주기 | 링크상태 변화시(이벤트 기반) |
| **교환 정보** | 이웃한 라우터와 라우팅 테이블 교환 | 링크상태 정보만 교환 |
| **라우팅 테이블 정보** | 이웃 라우팅 정보 | 네트워크 전체 |
| **전송 정보** | 자신이 알고 있는 모든 정보(포워딩 테이블 전체) 전송 | 변경된 정보만 전송 |
| **장점** | 1)모든 라우터 정보를 거질 필요가 없어 라우터 테이블 크기 감소<br>2)메모리 절약<br>3)구성 간단, 알고리즘 단순 | 1)컨버전스 타임이 빠름<br>2)라우팅 테이블 오버헤드 감소<br>3)이벤트기반으로 정보교환 트래픽 감소 |
| **단점** | 1)컨버전스 타임이 느림<br>2)주기적인 정보교환 발생, 대역폭 낭비<br>3)네트워크 변화 감지하는 시간 소요 | 1)큰 메모리 필요<br>2)SPF계산으로 HW부하 발생 |
| **프로토콜** | RIP, IGRP | OSPF, EIGRP |
| **라우팅 분류** | IGP | IGP |
