### 🔑 핵심 키워드 3개
- **IGP**: 우리 집 안에서 방 찾기 (내부 라우팅)
- **EGP**: 다른 집으로 가는 길 찾기 (외부 라우팅)
- **AS**: 하나의 조직이 관리하는 네트워크 영역

### 📖 등장배경

> 인터넷은 엄청 크잖아요! 한 회사 안에서 길 찾는 것과, 다른 회사까지 가는 길을 찾는 건 다른 방법이 필요해요. 그래서 **내부용(IGP)**과 **외부용(EGP)** 라우팅 프로토콜이 생겼어요!

### 📝 개념 정의

> - **IGP**: 같은 조직(AS) **안에서** 라우터끼리 길 찾는 방법
> - **EGP**: 서로 **다른 조직(AS) 간**에 길 찾는 방법

### 🏗️ 주요 프로토콜

| 구분 | IGP | EGP |
|:-----|:----|:----|
| 프로토콜 | RIP, OSPF, EIGRP | BGP |
| 특징 | 빠르고 간단 | 안정적, 정책 기반 |
| 비유 | 집 안에서 방 찾기 | 다른 도시로 가는 길 |

## 🎯 기술사 수준 설명
{: #-기술사-수준-설명}

### 📌 핵심 암기 (Quick Reference)

{: .highlight }
> **IGP/EGP**: AS 기준 내부/외부 라우팅
> - (IGP 프로토콜) `RIP-IGRP-EIGRP-OSPF`
> - (EGP 프로토콜) `BGP`
> - (IGP 방식) `DV-LS-Hybrid`
> - (비교) `범알기교속점`

---

### 📝 개념 정의

동적 라우팅 프로토콜: 라우터가 네트워크 토폴로지 변화를 **자동으로 인식**하고, 다른 라우터와 라우팅 정보를 교환하여 **최적 경로를 찾아내는** 프로토콜

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        Routing Protocol                                 │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│         ┌──────────────┐              ┌──────────────┐                 │
│         │Static Routing│              │Dynamic Routing│                 │
│         └──────────────┘              └──────┬───────┘                 │
│                                              │                          │
│                              ┌───────────────┴───────────────┐         │
│                              │                               │         │
│                         ┌────┴────┐                    ┌────┴────┐    │
│                         │   IGP   │                    │   EGP   │    │
│                         └────┬────┘                    └────┬────┘    │
│                              │                              │          │
│              ┌───────────────┼───────────────┐              │          │
│              │               │               │              │          │
│         ┌────┴────┐    ┌────┴────┐    ┌────┴────┐         │          │
│         │Distance │    │  Link   │    │ Hybrid  │         │          │
│         │ Vector  │    │  State  │    │         │         │          │
│         └────┬────┘    └────┬────┘    └────┬────┘         │          │
│              │              │              │               │          │
│          RIP, IGRP        OSPF          EIGRP            BGP         │
└─────────────────────────────────────────────────────────────────────────┘
```

| 구분 | 정의 |
|:-----|:-----|
| **IGP** | 하나의 자율 시스템(AS, Autonomous System) **내부**에서 라우터들이 라우팅 정보를 교환하기 위해 사용하는 프로토콜 |
| **EGP** | 서로 다른 자율 시스템(AS) **간** 라우팅 정보를 교환하기 위해 사용하는 프로토콜 |

---

### 🔍 IGP와 EGP 상세 비교 `범알기교속점`

| 구분 | IGP | EGP |
|:-----|:----|:----|
| 적용 **범**위 | 단일 AS 내부 | 서로 다른 AS 간 |
| 동작 **알**고리즘 | Distance Vector, Link State, Hybrid | Path Vector (BGP) |
| 경로 선택 **기**준 | 홉 수, 대역폭, 지연 등 내부 메트릭 | AS Path, 정책, 사업자 규칙 |
| 정보 **교**환 방식 | 주기적 또는 이벤트 기반 라우팅 정보 교환 | AS 경로를 포함한 정책 기반 교환 |
| 수렴 **속**도 | 빠름 | 상대적으로 느림 |
| **점** (중점) | 내부 효율성, 빠른 최적 경로 탐색 | 외부 정책, 안정성, 확장성 |

---

### 🔧 IGP 기술 요소

| 구분 | 기술 요소 | 설명 |
|:-----|:---------|:-----|
| **프로토콜** | RIP | 거리 벡터 라우팅 방식의 대표 프로토콜 |
| | OSPF | 링크 상태(Link State) 라우팅 방식 |
| | EIGRP | Cisco 독자 기술 기반의 하이브리드 라우팅 프로토콜 |
| **라우팅 방식** | Distance Vector | 단순, 최대 홉수 제한 |
| | Link State | 복잡, 빠르고 안정적인 경로 수립 |
| | Hybrid | 빠른 수렴성, 효율적 경로 탐색 제공 |
| **테이블갱신** | Dynamic Update | 네트워크 변경 시 자동으로 라우팅 테이블 갱신 |

---

### 🌐 EGP 기술 요소

| 구분 | 기술 요소 | 설명 |
|:-----|:---------|:-----|
| **프로토콜** | BGP (Border Gateway Protocol) | 목적지까지의 경로를 각 AS의 목록(AS_PATH) 형태로 관리하여 루프 방지 |
| **라우팅 방식** | 정책 기반 라우팅 | 단순 최단 경로가 아닌, 다양한 정책(경로 제어·보안·우선순위)에 따라 경로 결정 |
| **통신 방식** | 유니캐스트 | TCP 포트 179번 사용 |
| **적용 대상** | 글로벌 인터넷 수준 | 대규모 인터넷 환경에서 수십만 경로를 효율적으로 처리, 수렴 속도는 느리지만 안정성이 매우 높음 |

---

### ⚙️ IGP와 EGP의 효율적 설계 방안

| 구분 | 설계 방안 | 세부 내용 |
|:-----|:---------|:---------|
| **IGP** | 확장성 문제 대응 | OSPF → Area 기반 계층적 설계, IS-IS → 레벨(Level 1/2) 분리 운영 |
| | 빠른 수렴성 확보 | Fast Convergence 튜닝, BFD(Bidirectional Forwarding Detection) 적용 |
| | 보안 강화 | OSPF/IS-IS 인증 활성화(MD5, HMAC-SHA 등), 관리망(Out-of-Band Network) 분리 |
| **EGP** | 라우팅 안정성 확보 | Route Flap Damping, Prefix Limit, 정책 기반 필터링(AS Path, Community) |
| | 트래픽 증가 및 멀티홈 환경 대응 | 트래픽 엔지니어링(TE): Local Preference, MED 값 최적화, Anycast/CDN/MPLS 기반 QoS |

---

### 📈 상위 토픽 계층도

```
네트워크 라우팅
├── 정적 라우팅 (Static Routing)
└── 동적 라우팅 (Dynamic Routing)
    ├── IGP ← 현재 토픽
    │   ├── Distance Vector (RIP, IGRP)
    │   ├── Link State (OSPF, IS-IS)
    │   └── Hybrid (EIGRP)
    └── EGP ← 현재 토픽
        └── BGP (Border Gateway Protocol)
```
