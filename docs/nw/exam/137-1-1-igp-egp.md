### 🔑 핵심 키워드 3개
- **IGP**: 우리 학교(AS) 안에서 길 찾기
- **EGP**: 다른 학교(AS)로 가는 길 찾기
- **AS**: 하나의 관리 영역(학교, 회사 등)

### 📖 쉬운 설명

> 학교 안에서 교실을 찾을 때는 **IGP**(복도 안내도)를 보고, 
> 다른 학교로 갈 때는 **EGP**(지역 지도)를 봐요!
>
> - **IGP** = 우리 회사 내부 네트워크에서 최적 경로 찾기
> - **EGP** = 인터넷 전체에서 다른 회사(ISP)로 가는 경로 찾기

### 🏗️ 간단 비교

| 구분 | IGP | EGP |
|:-----|:----|:----|
| 범위 | 내부 (AS 안) | 외부 (AS 간) |
| 목적 | 빠른 경로 | 안정적 경로 |
| 예시 | RIP, OSPF | BGP |

## 🎯 기술사 수준 설명
{: #-기술사-수준-설명}

### 📌 핵심 암기 (Quick Reference)

{: .highlight }
> **IGP/EGP**: AS 기준, 내부/외부 라우팅
> - (IGP 프로토콜) `RIP-IGRP-EIGRP-OSPF`
> - (EGP 프로토콜) `BGP`
> - (IGP 알고리즘) `DV-LS-Hybrid`
> - (EGP 알고리즘) `Path Vector`

---

### 📝 개념 정의

동적 라우팅 프로토콜의 구조적 차이와 네트워크 설계의 기본 원리를 이해하는지 평가하기 위해 출제

```
                    ┌─────────────────────────┐
                    │   Routing Protocol      │
                    └────────────┬────────────┘
                                 │
              ┌──────────────────┴──────────────────┐
              │                                     │
    ┌─────────▼─────────┐              ┌────────────▼──────────┐
    │  Static Routing   │              │   Dynamic Routing     │
    └───────────────────┘              └───────────┬───────────┘
                                                   │
                              ┌────────────────────┴────────────────────┐
                              │                                         │
                    ┌─────────▼─────────┐                    ┌──────────▼─────────┐
                    │       IGP         │                    │        EGP         │
                    │ (AS 내부 라우팅)   │                    │  (AS 간 라우팅)     │
                    └─────────┬─────────┘                    └──────────┬─────────┘
                              │                                         │
            ┌─────────────────┼─────────────────┐                       │
            │                 │                 │                       │
    ┌───────▼───────┐ ┌───────▼───────┐ ┌───────▼───────┐       ┌───────▼───────┐
    │Distance Vector│ │  Link State   │ │    Hybrid     │       │  Path Vector  │
    └───────┬───────┘ └───────┬───────┘ └───────┬───────┘       └───────┬───────┘
            │                 │                 │                       │
      ┌─────┴─────┐     ┌─────┴─────┐     ┌─────┴─────┐           ┌─────┴─────┐
      │RIP │ IGRP │     │   OSPF    │     │   EIGRP   │           │    BGP    │
      └────┴──────┘     └───────────┘     └───────────┘           └───────────┘
```

| 구분 | 정의 |
|:-----|:-----|
| **IGP** | 하나의 자율 시스템(AS, Autonomous System) **내부**에서 라우터들이 라우팅 정보를 교환하기 위해 사용하는 프로토콜 |
| **EGP** | 서로 다른 자율 시스템(AS) **간** 라우팅 정보를 교환하기 위해 사용하는 프로토콜 |

---

### 🔄 IGP와 EGP 동작 비교

| 구분 | IGP | EGP |
|:-----|:----|:----|
| **적용 범위** | 단일 AS 내부 | 서로 다른 AS 간 |
| **동작 알고리즘** | Distance Vector, Link State, Hybrid | Path Vector (BGP) |
| **경로 선택 기준** | 홉 수, 대역폭, 지연 등 내부 메트릭 | AS Path, 정책, 사업자 규칙 |
| **정보 교환 방식** | 주기적 또는 이벤트 기반 라우팅 정보 교환 | AS 경로를 포함한 정책 기반 교환 |
| **수렴 속도** | 빠름 | 상대적으로 느림 |
| **중점** | 내부 효율성, 빠른 최적 경로 탐색 | 외부 정책, 안정성, 확장성 |

---

### 🏗️ IGP와 EGP 활용 측면 비교

| 구분 | IGP | EGP |
|:-----|:----|:----|
| **확장성** | 중·소규모 네트워크에 적합 | 전 세계 인터넷 수준 대규모 네트워크 가능 |
| **관리 주체** | 단일 관리 기관(조직/ISP 내부) | 다수의 독립된 관리 기관(다른 ISP/AS) |
| **목적** | 내부 네트워크의 효율적 라우팅 및 빠른 최적 경로 탐색 | AS 간 경로 제어, 정책 반영, 안정적 연결 |
| **신뢰성** | 내부 환경에 의존 → 비교적 단순 | 외부와 연결 → 고도의 안정성 및 정책 제어 필요 |
| **주요 프로토콜** | RIP, IGRP, EIGRP, OSPF | BGP |

---

### 🔧 IGP(Interior Gateway Protocol) 기술 요소

| 구분 | 기술 요소 | 설명 |
|:-----|:---------|:-----|
| **프로토콜** | RIP | 거리 벡터 라우팅 방식의 대표 프로토콜 |
| | OSPF | 링크 상태(Link State) 라우팅 방식 |
| | EIGRP | Cisco 독자 기술 기반의 하이브리드 라우팅 프로토콜 |
| **라우팅 방식** | Distance Vector | 단순, 최대 홉수 제한 |
| | Link State | 복잡, 빠르고 안정적인 경로 수립 |
| | Hybrid | 빠른 수렴성, 효율적 경로 탐색 제공 |
| **테이블갱신** | Dynamic Update | 네트워크 변경 시 자동으로 라우팅 테이블 갱신 |

> 속도 및 효율 중시하며, **내부 네트워크 최적화**

---

### 🌐 EGP(Exterior Gateway Protocol) 기술 요소

| 구분 | 기술 요소 | 설명 |
|:-----|:---------|:-----|
| **프로토콜** | BGP (Border Gateway Protocol) | 목적지까지의 경로를 각 AS의 목록(AS_PATH) 형태로 관리하여 루프 방지 |
| **라우팅 방식** | 정책 기반 라우팅 | 단순 최단 경로가 아닌, 다양한 정책(경로 제어·보안·우선순위)에 따라 경로 결정 |
| **통신 방식** | 유니캐스트 | TCP 포트 179번 사용 |
| **적용 대상** | 글로벌 인터넷 수준 | 대규모 인터넷 환경에서 수십만 경로를 효율적으로 처리하며, 수렴 속도는 느리지만 안정성이 매우 높음 |

> 고도의 안정성 및 정책 제어 필요한 **외부 네트워크 연결에 적용**

---

### 📈 상위 토픽 계층도

```
네트워크 라우팅
├── 정적 라우팅 (Static)
└── 동적 라우팅 (Dynamic) ← 현재 토픽
    ├── IGP (AS 내부)
    │   ├── Distance Vector (RIP, IGRP)
    │   ├── Link State (OSPF, IS-IS)
    │   └── Hybrid (EIGRP)
    └── EGP (AS 간)
        └── BGP (Path Vector)
```
