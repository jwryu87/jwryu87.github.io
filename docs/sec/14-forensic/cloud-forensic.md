# 클라우드 포렌식 (Cloud Forensic)
{: .fs-8 }

14.6 포렌식
{: .label .label-purple }

---

## 핵심 키워드

`가상화` `정보 제공 의무` `클라우드 시그니처(Signature)` `사법관할권` `국제 공조`

---

## 정의/개념

클라우드에 존재하는 **전자적 증거물 등을 사법기관에 제출**하기 위해 **클라우드 시그니처(Signature) 데이터 수집, 분석 및 보고서**를 작성하는 과학수사 기법

---

## 클라우드 포렌식 프로세스

```
┌──────────────┐    ┌──────────────┐    ┌──────────────┐    ┌──────────────┐

│ 포렌식 조사 │───▶│  서비스 확인  │───▶│    확인     │───▶│ 존재 여부 확인│
│    시작     │    │              │    │              │    │              │
└──────────────┘    └──────────────┘    └──────────────┘    └──────┬───────┘
                                                                    │
                                                           ┌────────┴────────┐
                                                           │                 │
                                                        No │                 │ Yes
                                                           ▼                 ▼
                                               ┌──────────────┐    ┌──────────────┐
                                               │ 국제 공조    │    │  임의제출    │
                                               │ 클라우드     │    │    요청     │
                                               │ 상세분석 불가 │    │              │
                                               │     요청     │    │              │

                                                                            │
┌──────────────┐    ┌──────────────┐    ┌──────────────┐                   │

│  시그니처   │◀───│  압수권한    │◀───│   데이터    │

       │
```

## 조사 활동별 상세

| 구분 | 조사 활동 | 상세 내용 |
|:-----|:----------|:----------|
| **대상 확인** | 대상 클라우드 서비스 확인 | - SaaS(Google Drive, Naver Mybox, Dropbox 등)<br>- IaaS(Amazon AWS, MS Azure, Google Cloud 등) |
| **증거 확보** | 로그인 정보 확인 | - 웹 브라우저 포렌식 수행<br>- 레지스트리 포렌식 서비스로그분석 수행 |
| **협조 요구** | 사법관할권 내 존재 확인<br>국제 공조 요청<br>임의 제출 요청 | - 클라우드 서비스 제공자의 물리적 위치 확인<br>- 국제형사사법 공조법 활용 증거수집, 압수수색<br>- 포렌식 대상의 ID, Password 정보 등(이용자 동의 필요)<br>- 통신비밀보호법, 전기통신사업법 등 관련 법규 준수 |
| **증거 수집** | 클라우드 압수권한 획득<br>클라우드 계정 접근 차단<br>클라우드 데이터 수집 | - 계정 로그인 Lock, IP 접근 제어 설정<br>- 분산 포렌식 인덱싱, 파일 카빙(Carving), 스테가노그래피(Steganography) |
| **증거 분석** | 클라우드 시그니처 분석 | - Client에 존재하는 클라우드 사용 기록 |

## 클라우드 포렌식 특수성

{: .warning }
> 클라우드 환경은 기존 디지털 포렌식과 달리 데이터의 물리적 위치, 사법관할권, CSP 협조 등 추가 고려사항이 존재

| 구분 | 설명 |
|:-----|:-----|
| **데이터 위치 불확정성** | 클라우드 데이터가 여러 국가/지역에 분산 저장 |
| **사법관할권 문제** | 데이터 소재지와 범죄 발생지가 다를 수 있음 |
| **가상화 환경** | 물리적 증거 확보 어려움, 휘발성 데이터 중요 |
| **CSP 협조 필요** | 클라우드 서비스 제공자의 기술적, 법적 협조 필수 |
| **다중 테넌시** | 다른 사용자 데이터와의 분리 필요 |
