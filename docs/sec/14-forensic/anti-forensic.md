14.5 포렌식

---

## 핵심 키워드

`디가우징` `와이핑` `스테가노그래피` `슬랙공간 은닉` `데이터암호화` `데이터변조`

## 정의/개념

자신에게 불리하게 작용할 가능성이 있는 **디지털 증거를 훼손하거나 숨겨서 포렌식을 방해**하는 일련의 행위

---

## 안티 포렌식 기술

### 데이터 삭제 기술

| 기술 | 설명 | 도구 및 기법 |
|:-----|:-----|:-------------|
| **디가우징 (Degaussing)** | 디스크를 강력한 자기장에 노출시켜 데이터를 영구적으로 삭제하는 기법 | Degausser |
| **와이핑 (Wiping)** | 디스크 영역에 난수 혹은 0으로 덮어쓰는 기법 | Low Level Format |
| **증거데이터 자동삭제** | OS에 의해 생성되는 사용자의 개인 정보 또는 증거물이 될 만한 모든 데이터를 삭제 | 웹페이지, 레지스트리, 쿠키 등 삭제 |

### 데이터 은닉 기술

| 기술 | 설명 | 도구 및 기법 |
|:-----|:-----|:-------------|
| **Steganography** | 데이터의 존재 사실 자체를 숨기는 정보 은닉 기술 | OpenStego, Steghide |
| **디스크내 데이터 은닉** | 파티션으로 인한 슬랙공간 등 파일시스템 구조에서 낭비영역에 데이터 은닉기술 | Slacker, FragFS, RuneFS, KY FS |
| **데이터 암호화** | 파일이나 디렉토리를 암호화하여 아무나 접근할 수 없도록 하는 기술 | EFS, BitLocker, TrueCrypt |
| **데이터 변조** | 원본 파일의 내용과 시스템에서 인식되는 파일 형식을 다르게 함 | 파일 헤더 조작, 날짜 수정 |

## 안티 포렌식 대응기술

| 분류 | 대응 기술 | 상세 설명 |
|:-----|:----------|:----------|
| **스테가노그래피 탐지** | Staganalysis | 눈으로 이미지 변화감지, 통계적 기술 이용해 이미지 픽셀간의 점진적 색상변화 비교하는 방법으로 식별 |
| | 인덱스 기반 탐색 | 모든 파일 훑으면서 키워드를 탐색 |
| **데이터 검색 및 탐지** | BitWise | 디스크내 섹터나 슬랙 공간에서 찾을 수 있는 비할당 영역에 존재하는 간단한 텍스트나 특정 표현 찾기 |
| | Hash 검증 | 알려진 파일을 찾을 때는 MD5, SHA-1과 같은 파일의 해시값을 비교해서 식별 |
| **데이터 복구** | 슬랙 공간 및 미할당 영역 복구 | 유실된 파일헤더를 적절히 재구성하여 본래의 파일로 복구 |
| | 메모리 및 스왑 영역 검색 | - 메모리내에 캐쉬데이터, 라우팅 정보, 프로세스 정보, 응용프로그램이 사용했던 데이트 들이 존재하므로 이를 검색<br>- Swap파일은 가상메모리 시스템에서 사용되는 메모리 공간으로 하드디스크의 일부공간을 메모리처럼 사용하므로 이를 검색 |

## 안티 포렌식 기법 분류

```
┌─────────────────────────────────────────────────────────────────┐
│                    안티 포렌식 기법                              │
├─────────────────────┬─────────────────────┬─────────────────────┤
│    데이터 파괴       │    데이터 은닉       │    흔적 제거         │
├─────────────────────┼─────────────────────┼─────────────────────┤
│ - 디가우징          │ - 스테가노그래피     │ - 로그 삭제          │
│ - 와이핑            │ - 암호화             │ - 히스토리 삭제      │
│ - 물리적 파괴       │ - 슬랙 공간 은닉     │ - 타임스탬프 조작    │
│                     │ - 데이터 변조        │ - 레지스트리 정리    │
└─────────────────────┴─────────────────────┴─────────────────────┘
```

## 포렌식 관점 대응

{: .warning }
> 현재 디지털 포렌식 기술은 파일의 Header나 Footer와 같인 Signature가 없는 경우에는 복원율이 매우 낮은 편임

| 대응 방안 | 설명 |
|:----------|:-----|
| **휘발성 데이터 우선 수집** | 메모리, 네트워크 연결 등 휘발성 증거 먼저 확보 |
| **슬랙 영역 분석** | 삭제된 데이터 흔적 탐지 |
| **타임라인 분석** | 파일 시간 정보 교차 검증 |
| **해시값 DB 활용** | 알려진 파일/악성코드 식별 |
| **스테가노그래피 탐지 도구** | 이미지/파일 내 은닉 데이터 탐지 |
