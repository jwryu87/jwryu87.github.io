# 파일 슬랙 (File Slack)
{: .fs-8 }

14.3 포렌식
{: .label .label-purple }

---

## 핵심 키워드

`램 슬랙` `드라이브 슬랙` `파일시스템 슬랙` `볼륨 슬랙` `클러스터`

---

## 정의/개념

저장매체 **물리적 구조와 논리적 구조 차이**로 발생하는 **램 슬랙과 드라이브 슬랙의 공간**

---

## 개념도

```
┌──────────────────────────────────────────────────────────────────┐
│                                                                  │
│                                                                  │
│                      Cluster (2048 Bytes)                        │
│                                                                  │
│                                                   File Slack     │
│    ┌────────────────────────────────────────────────────────┐   │
│    │           │                                            │   │
│    │   RAM     │              Drive Slack                   │   │
│    │   Slack   │                                            │   │
│    │           │                                            │   │
│    └────────────────────────────────────────────────────────┘   │
│       sector 1     sector 2     sector 3     sector 4           │
│    ├───────────┼───────────┼───────────┼───────────┤           │
│                                                                  │
│        used area              wasted area                        │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

- 슬랙 공간의 종류에는 램 슬랙, 파일 시스템 슬랙, 볼륨 슬랙, 파일 슬랙, 드라이브 슬랙 5가지가 존재함

---

## 슬랙 유형별 설명

| 구분 | 핵심 기술 | 설명 |
|:-----|:----------|:-----|
| **램 슬랙 (RAM Slack)** | 섹터에 남는 비할당 영역 | - 섹터의 크기가 512 byte이고 100 byte의 데이터가 할당되어 있으면 나머지 412 byte는 0으로 채우게 되며 램 슬랙 공간이라고 부름<br>- 램에 저장된 데이터가 저장매체에 기록될 때 섹터 단위로 기록이 되는 특징이 있어서 램 슬랙이라고 함 |
| **파일 슬랙 (File Slack)** | 램슬랙과 드라이브 슬랙 합친 부분 | - 저장매체의 물리적인 공간을 사용하기 위해서는 파일시스템을 생성해야 함<br>- 이때 파일시스템의 크기는 클러스터의 크기에 따라 결정되는데 이때 파일시스템 마지막 부분에 사용할 수 없는 영역이 발생하기도 함<br>- 최근에는 파일시스템 슬랙 부분도 악성코드를 은닉하는데 많이 사용됨 |
| **드라이브 슬랙 (Drive Slack)** | 데이터를 할당하고 남은 비할당 영역 | - 드라이브 슬랙은 낭비되는 빈 영역이지만 다른 용도로 사용할 수 없기 때문에 악의 적인 목적으로 다른 데이터(악성 코드 등)를 기록해 두는 경우도 있음 |
| **파일시스템 슬랙 (File System Slack)** | 저장매체의 물리적인 공간을 사용하기 위해서는 파일시스템을 생성 | - 이때 파일시스템의 크기는 클러스터의 크기에 따라 결정되는데 이때 파일시스템 마지막 부분에 사용할 수 없는 영역이 발생함<br>- 최근에는 파일시스템 슬랙 부분도 악성코드를 은닉하는데 많이 사용되고 있음<br>- 파일시스템의 할당 크기와 파티션 크기의 차이로 발생되는 낭비 공간 |
| **볼륨 슬랙 (Volume Slack)** | 전체 볼륨 크기와 할당된 파티션 크기의 차이로 발생하는 낭비 공간 | - 파티션의 크기는 임의로 변경이 가능하기 때문에 다른 슬랙과 다르게 볼륨 슬랙의 공간은 임의로 변경할 수 있음 |

---

## 보안 관점

{: .warning }
> 슬랙 영역은 악성코드 은닉에 자주 사용되므로 디지털 포렌식 수행 시 반드시 검사해야 함

| 활용 | 설명 |
|:-----|:-----|
| **악성코드 은닉** | 드라이브 슬랙, 파일시스템 슬랙에 악성코드 저장 |
| **데이터 복구** | 삭제된 파일의 흔적이 슬랙 영역에 남아있을 수 있음 |
| **포렌식 증거** | 슬랙 영역 분석으로 과거 데이터 흔적 발견 가능 |
