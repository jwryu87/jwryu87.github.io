# 디스크 이미징 (Disk Imaging)
{: .fs-8 }

14.4 포렌식
{: .label .label-purple }

---

## 핵심 키워드

`Disk to Disk` `Disk to File` `비트 단위 복제` `무결성` `쓰기방지장치`

---

## 정의/개념

**디지털 포렌식 조사**를 위해 조사 **저장매체의 사본을 생성**하는 기술

---

## 이미징 방식 비교

### Disk to Disk 방식

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│    ┌─────────────┐                        ┌──────────────┐     │
│    │             │                        │   사본A      │     │
│    │   디스크    │      ┌─────────┐      │   Hard       │     │
│    │   이미징    │──────│  원본   │──────│   Disk       │     │
│    │   장비      │      │  Hard   │      └──────────────┘     │
│    │             │      │  Disk   │                           │
│    │             │      └─────────┘      ┌──────────────┐     │
│    │             │           │           │   사본B      │     │
│    └─────────────┘           └───────────│   Hard       │     │
│                                          │   Disk       │     │
│                                          └──────────────┘     │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

| 구분 | 설명 |
|:-----|:-----|
| **디스크 원본 복제** | 컴퓨터 하드 디스크의 내용물을 다른 디스크나 이미지 파일로 직접적으로 복제하여 사본을 만드는 방식 |
| **하드웨어 장비 사용** | 단독으로 복제 디스크를 생성할 수 있는 포렌식 장비인 디스크 이미징 장비를 하용하여 복제 진행 |
| **컴퓨터 미연결** | 직접적인 컴퓨터에 디스크 연결하는 작업없이 다른 하드디스크에 사본 생성 |

- 하드웨어를 통한 직접적인 복제 방법 외 File 복제 방식 존재

### Disk to File 방식

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│   ┌─────────┐     ┌──────────┐     ┌──────────┐    ┌────────┐ │
│   │         │     │          │     │          │    │ 이미징 │ │
│   │  원본   │────▶│ 쓰기방지  │────▶│  디스크   │───▶│ 파일   │ │
│   │ 디스크  │     │   장치   │     │  이미징   │    │ 생성   │ │
│   │         │     │          │     │   SW     │    │        │ │
│   └─────────┘     └──────────┘     └──────────┘    └────────┘ │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

| 구분 | 설명 |
|:-----|:-----|
| **개별 이미지 생성** | 디스크에 존재하는 파일을 개별 파일별로 이미지로 생성 진행 |
| **소프트웨어 사용** | 디스크 이미징 소프트웨어를 활용하여 사전 분석 작업 진행 후 복제 |
| **직접 분석진행** | 디스크 이미징 완료 후 이미지 파일에서 직접적으로 분석 진행 |

- 하드웨어를 통한 직접적인 복제 방법 외 File 복제 방식 존재

---

## 이미징 방식 비교표

| 구분 | Disk to Disk | Disk to File |
|:-----|:-------------|:-------------|
| **복제 형태** | 물리 디스크 → 물리 디스크 | 물리 디스크 → 이미지 파일 |
| **주요 장비** | 전용 이미징 장비 (Duplicator) | 이미징 소프트웨어 + 쓰기방지장치 |
| **장점** | 빠른 복제 속도, 독립 운용 | 파일 형태로 관리 용이, 다수 복제 용이 |
| **단점** | 물리 디스크 필요 | 소프트웨어 의존, 분석 환경 필요 |
| **대표 도구** | Tableau, Logicube | FTK Imager, EnCase, dd |

---

## 이미징 수행 원칙

| 원칙 | 설명 |
|:-----|:-----|
| **무결성 보장** | 해시값(MD5, SHA256) 생성으로 원본과 사본 동일성 검증 |
| **쓰기방지** | Write Blocker 사용으로 원본 변경 방지 |
| **문서화** | 이미징 과정, 시간, 담당자 등 상세 기록 |
| **검증** | 이미징 완료 후 해시값 비교 검증 |
