### 🔑 핵심 키워드 3개
- **SIEM**: 보안 이벤트를 모아서 경고해주는 시스템
- **SOAR**: 보안 사고를 자동으로 처리해주는 시스템
- **플레이북**: 자동 대응을 위한 미리 만든 시나리오

### 📖 등장배경

> 회사에 보안 담당자가 하루 종일 수천 개의 보안 경고를 받아요. 그런데 대부분은 거짓 경고인데, 진짜 위험한 건 놓치면 안 되죠. 그래서 **SIEM**이 나왔어요 - 모든 보안 로그를 모아서 분석하고 경고해주는 시스템이에요. 그런데 경고를 받아도 사람이 일일이 확인하고 처리하느라 시간이 오래 걸려요. 그래서 **SOAR**가 나왔어요 - 위험한 경고가 오면 자동으로 차단하고 대응해주는 시스템이에요!

### 📝 개념 정의

> **SIEM**은 **"보안 이벤트를 모아서 분석하고 경고해주는 모니터링 시스템"**이에요.
> 
> **SOAR**는 **"보안 사고를 자동으로 처리하고 대응해주는 자동화 시스템"**이에요.
> 
> 비유하자면, SIEM은 **"화재 경보기"**처럼 위험을 알려주고, SOAR는 **"스프링클러"**처럼 자동으로 불을 끄는 거예요!

### 🏗️ 기술요소 (2그룹)

**그룹 1: 핵심 차이점**

| 구분 | SIEM | SOAR |
|:-----|:-----|:-----|
| 목적 | 보안 이벤트 모니터링 및 경고 | 보안 사고 자동 대응 |
| 데이터 | 로그 수집 및 분석 | 위협 인텔리전스 활용 |
| 자동화 | 제한적 (경고만) | 높음 (자동 대응) |
| 대응 | 수동 분석 후 수동 조치 | 자동 분석 후 자동 조치 |

**그룹 2: 연계 효과**

| 연계 | 설명 |
|:-----|:-----|
| SIEM → SOAR | SIEM이 경고하면 SOAR가 자동 대응 |
| 통합 운영 | SIEM으로 탐지, SOAR로 대응하는 완전 자동화 |
| 시너지 | 탐지 정확도 향상 + 대응 시간 단축 |

### ⭐ 차별점 키워드

> **SIEM + SOAR = 완전 자동화**: SIEM이 위험을 찾으면 SOAR가 자동으로 막아주는 완벽한 조합!

### 📋 6하원칙 요약

| 구분 | 내용 |
|:-----|:-----|
| **누가** | 보안 관리자, SOC 운영자 |
| **무엇을** | 보안 이벤트 모니터링(SIEM), 자동 대응(SOAR) |
| **언제** | 24/7 실시간 모니터링 및 대응 |
| **어디서** | SOC(Security Operations Center) |
| **왜** | 대규모 보안 이벤트 처리, 대응 시간 단축 |
| **어떻게** | 로그 수집·분석(SIEM) → 자동 대응(SOAR) |

## 🎯 기술사 수준 설명
{: #-기술사-수준-설명}

{: .highlight }
> **SIEM vs SOAR**: 로그수집분석, 위협인텔리전스, 자동대응 → 보안 관제 플랫폼
> - (SIEM) `목로그이사보자`
> - (SOAR) `목자오사보자`
> - (비교항목) `목로그이 사보자 / 오통집 분자유`

---

### 📝 개념 정의

대규모 SOC 환경에서 보안 운영 자동화를 위한 시스템 연계 사례가 증가하면서, SIEM과 SOAR의 차이점과 통합 활용 방안이 중요해졌다.

| 구분 | SIEM | SOAR |
|:-----|:-----|:-----|
| 개념 | 이기종 엔드포인트의 이벤트를 수집, 파싱, 분석 후 시각화하여 보안 관리자에게 단일창으로 Alert 하거나 대시보드를 지원하는 모니터링 솔루션 | 위협 인텔리전스 등을 동적 플레이북 기반으로 분석하고, 보안 정책 자동 관리 및 인시던트 대응 자동화 등을 수행하는 통합 보안 솔루션 |
| 등장배경 | 대규모 보안 이벤트 수집 및 분석 필요성 | SIEM 이벤트 기반 수동 대응의 자동화 필요성 |
| 구성요소 | SOA(보안 오케스트레이션), SIRP(보안 인시던트 대응 플랫폼), TIP(위협 인텔리전스 플랫폼)의 통합 |

---

### 🏗️ 상세 비교 `목로그이사보자오통집분자유`

| 구분 | SIEM | SOAR |
|:-----|:-----|:-----|
| **목**적 | 이벤트 대시보드 가시화, 위협 탐지 및 조사 | 보안 정책 자동 관리, 인시던트 자동 대응 |
| **로**그 관리 | 다양한 소스에서 보안 로그 데이터를 중앙 집중화 | 로그 관리 없음 |
| **그**래프/시각화 | 대시보드 제공, 실시간 모니터링 | 대시보드 없음 (워크플로우 중심) |
| **이**벤트 상관관계 | 각각의 이벤트간 패턴 분석, 시그니처 기반 분석 | 상관관계 분석 없음 (플레이북 기반) |
| **사**건 모니터링 | 보안 모니터링 후 경고 생성 | 경고 생성 없음 (자동 대응 중심) |
| **보**고 | 보안 사건 상세 보고서 생성 | 보고서 생성 없음 |
| **자**동화 | 제한적 자동화 수행 | 사전 정의된 작업과 워크플로우 실행 |
| **오**케스트레이션 | 없음 | 다양한 보안도구 간 기술 통합, 사건 조사와 협업을 위한 워룸 제공, 반복 작업 자동화 |
| **통**합 | 다양한 데이터 소스 통합 | 다양한 보안 도구와 기술 통합 |
| **집**중 영역 | 보안 이벤트 데이터 분석 | 사건 대응 프로세스 자동화 및 조정 |
| **분**석 방식 | 파싱된 인덱스에 로그 대입, 시그니처 기반 분석 | 행위 기반 분석, 플레이북 기반 분석 |
| **자**동화 수준 | 낮음 (수동 개입 요구) | 높음 (자동화 워크플로우 지원) |
| **유**지관리 | 파싱 및 시그니처 업데이트 필요 | 플레이북 최적화 필요 |

---

### 📊 운영 요구사항 및 데이터 비교

| 구분 | SIEM | SOAR |
|:-----|:-----|:-----|
| **운영 요구사항** | 로그 파싱 및 Alert 레벨 결정 | 대용량 스토리지 연계 |
| **수집 데이터** | syslog(네트워크, 시스템) | log, 이벤트, 위협 인텔리전스 |
| **데이터 유형** | 정형 데이터 | 정형 + 비정형 데이터 |
| **대응 절차** | 수동 분석 후 수동 조치 | 자동 분석 후 자동 조치 |

---

### 🔄 매커니즘 비교

**SIEM 매커니즘:**
```
엔드포인트 + 보안 솔루션 + 네트워크 노드
    ↓ (syslog, 로그 파싱)
SIEM
    ↓ (Alert, 대시보드)
보안 관리자
```

**SOAR 매커니즘:**
```
위협 인텔리전스
    ↓ (수집)
SOAR
    ├─ 위협 분석/결정
    │   ├─ 1. 시그니처
    │   └─ 2. 동적 플레이북
    └─ 오케스트레이션 → 보안 솔루션
        ↓ (자동 업데이트)
    SOAR
```

        ↓

           │

### 🔗 SIEM, SOAR 통합 구성

```
시스템 Data 및 이벤트
    ↓
SIEM 데이터 저장
    ├─→ 이벤트 알림 ──┐
    │                │
    └─→ SOAR 위협 대응 프로세스 자동화
                    │
                    └─→ 사건 대응 ──┐
                                   │
                            보안 담당자
```

**통합 효과:**
- SIEM으로 수집한 데이터 기반 SOAR에서 위협에 대응하는 프로세스로 기업 보안 강화 수행
- SIEM 이벤트를 기반으로 실시하던 수동 대응을, SOAR를 통해 자동화

---

### 🔗 XDR과의 연계 효과

**XDR, SIEM, SOAR 연계:**
```
SIEM ──(실시간 로그)──→ XDR ──(위협 데이터)──→ SOAR
  ↑                        │                      │
  └──(심층 포렌식)─────────┘                      │
                                                  │
                                            (오케스트레이션)
```

**XDR 내부 기능:**
- 다중 상관분석
- 위협 시뮬레이션
- 인시던트 스코어링
- (AI/ML 기반)

**연계 효과:**
- XDR을 추가 연계하면 엔드포인트·네트워크·클라우드까지 포함한 다층 위협 탐지가 가능하며, AI 기반 분석을 통해 탐지 정확도가 향상됨
- 특히 SOAR는 XDR과 연계할 때 더욱 정교한 자동 대응이 가능하며, 보안 운영 효율성을 극대화할 수 있음

---

### ⭐ 차별점 키워드

> **SIEM + SOAR + XDR = 완전 자동화 보안 운영**
> - SIEM: 탐지 (Detection)
> - XDR: 심층 분석 (Deep Analysis)
> - SOAR: 자동 대응 (Automated Response)
> - 세 가지를 통합하면 탐지부터 대응까지 완전 자동화된 보안 운영 체계 구축 가능

---

### 📈 상위 토픽 계층도

```
보안 운영 (Security Operations)
├── 보안 모니터링
│   ├── SIEM ← 현재 토픽
│   ├── XDR
│   └── 로그 관리
├── 보안 자동화
│   ├── SOAR ← 현재 토픽
│   ├── 플레이북
│   └── 오케스트레이션
├── 위협 인텔리전스
│   ├── TIP
│   └── 위협 정보 공유
└── 인시던트 대응
    ├── SIRP
    └── 포렌식
```

### ✅ 학습 체크리스트

- [ ] SIEM과 SOAR의 개념 차이를 설명할 수 있다
- [ ] SIEM의 목적(이벤트 모니터링)과 SOAR의 목적(자동 대응)을 구분할 수 있다
- [ ] SIEM의 로그 관리 기능과 SOAR의 로그 관리 부재를 비교할 수 있다
- [ ] 이벤트 상관관계 분석 방식(시그니처 vs 플레이북) 차이를 설명할 수 있다
- [ ] 자동화 수준(낮음 vs 높음) 차이를 설명할 수 있다
- [ ] 오케스트레이션 기능의 유무를 구분할 수 있다
- [ ] SIEM과 SOAR 통합 구성 및 연계 효과를 설명할 수 있다
- [ ] XDR과의 연계를 통한 다층 위협 탐지 방안을 설명할 수 있다
- [ ] 비교 항목 12개 이상을 암기할 수 있다 (`목로그이사보자오통집분자유`)
