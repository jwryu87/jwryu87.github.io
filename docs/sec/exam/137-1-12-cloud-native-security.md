### 🔑 핵심 키워드 3개
- **4C**: Cloud, Cluster, Container, Code 4개 계층
- **DevSecOps**: 개발+운영+보안을 하나로
- **제로 트러스트**: 아무도 믿지 않고 항상 검증

### 📖 등장배경

> 요즘 회사들이 클라우드에서 컨테이너로 서비스를 만들어요. 그런데 기존 보안은 "회사 안은 안전, 밖은 위험"이라는 생각이었는데, 클라우드에서는 경계가 없어졌어요! 그래서 **처음부터 끝까지, 모든 계층에서 보안을 챙기자**는 새로운 방식이 필요해졌어요.

### 📝 개념 정의

> 클라우드 네이티브 보안은 **"클라우드 환경 전체에 걸쳐 보안을 자동화하고 통합하는 것"**이에요.
> 
> 비유하자면, 집 전체에 **CCTV, 잠금장치, 경보기를 모든 방에 설치하는 것**과 같아요!

### 🏗️ 기술요소 (2그룹)

**그룹 1: 4C 계층**

| 계층 | 쉬운 설명 | 보안 대상 |
|:-----|:---------|:---------|
| Cloud | 클라우드 인프라 | AWS, Azure 등 |
| Cluster | 쿠버네티스 클러스터 | Pod, Node 관리 |
| Container | 컨테이너 이미지 | Docker 이미지 |
| Code | 애플리케이션 코드 | 소스코드, 라이브러리 |

**그룹 2: 보안 도구**

| 도구 | 역할 | 쉬운 설명 |
|:-----|:-----|:---------|
| CSPM | 클라우드 설정 점검 | 설정 오류 찾기 |
| CWPP | 워크로드 보호 | 컨테이너 지키기 |
| SAST/DAST | 코드 취약점 분석 | 코드 검사 |

### ⭐ 차별점 키워드

> **CNAPP**: 클라우드 네이티브 보안 도구들을 하나로 합친 통합 플랫폼!

### 📋 6하원칙 요약

| 구분 | 내용 |
|:-----|:-----|
| **누가** | DevSecOps 팀, 클라우드 보안 담당자 |
| **무엇을** | 인프라, 클러스터, 컨테이너, 코드 |
| **언제** | 개발 → 배포 → 운영 전 주기 |
| **어디서** | 클라우드 네이티브 환경 |
| **왜** | 동적 환경에서 경계 기반 보안 무력화 |
| **어떻게** | 4C 계층별 보안, DevSecOps, 제로 트러스트 |

## 🎯 기술사 수준 설명
{: #-기술사-수준-설명}

{: .highlight }
> **클라우드 네이티브 보안**: 4C, DevSecOps, 제로트러스트
> - (계층) `4C` (Cloud-Cluster-Container-Code)
> - (특징) `동Dev제` (동적구조, DevSecOps, 제로트러스트)
> - (원칙) `3R` (Rotate, Repair, Repave)

---

### 📝 개념 정의

클라우드 환경의 동적 구조와 컨테이너·마이크로서비스·API 중심 아키텍처로 기존 경계 중심 보안이 무력해짐에 따라,
인프라·어플리케이션·컨테이너·코드 등 **클라우드 네이티브 환경 전반에 걸쳐 보안을 자동화하고 통합하는 프레임워크**

| 구분 | 설명 |
|:-----|:-----|
| 정의 | 개발~운영 전체 수명주기에 보안을 내재화하는 접근 방식 |
| 목표 | 애플리케이션과 인프라 전반에 대한 지속적, 자동화된 보안 |
| 핵심 | 4C 계층 구조를 통해 기존 경계기반 보안의 한계 극복 |

---

### 🔄 4C 계층 개념도

```
┌─────────────────────────────────────────────────────────┐
│  Cloud (클라우드) - 인프라 보안 / 인프라스트럭처 보안      │
│  ┌─────────────────────────────────────────────────┐   │
│  │  Cluster (클러스터) - 클러스터 컴포넌트 / 앱 보안   │   │
│  │  ┌─────────────────────────────────────────┐   │   │
│  │  │  Container (컨테이너) - 가상화 컨테이너 보안  │   │   │
│  │  │  ┌─────────────────────────────────┐   │   │   │
│  │  │  │  Code (코드) - Application 코드 보안  │   │   │   │
│  │  │  └─────────────────────────────────┘   │   │   │
│  │  └─────────────────────────────────────────┘   │   │
│  └─────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────┘
```

### 🏗️ 구성요소

#### 4C 계층별 보안 `Cloud-Cluster-Container-Code`

| 계층 | 보안 대상 | 보안 위협 | 대응 방안 |
|:-----|:---------|:---------|:---------|
| **C**loud | API서버, 노드, etcd | 권한 남용, 구성 오류 | IAM, 최소권한, 암호화, CIEM |
| **C**luster | 쿠버네티스 워크로드 | API 노출, 파드 탈출 | RBAC, Pod 보안정책, 감사 로깅 |
| **C**ontainer | 컨테이너 이미지 | 루트 권한, 런타임 위협 | 취약점 스캔, CWPP, 불변 인프라 |
| **C**ode | 애플리케이션 코드 | 취약 라이브러리, 미암호화 | SAST/DAST, SBOM, 시큐어 코딩 |

#### 클라우드 네이티브 보안 특징 `동Dev제`

| 특징 | 설명 | 상세 |
|:-----|:-----|:-----|
| **동**적 구조 기반 | 컨테이너, 마이크로서비스, IaC | FaaS/BaaS 기반 아키텍처 활용 |
| **Dev**SecOps | CI/CD 파이프라인에 보안 통합 | SAST/DAST 이용, 보안 내재화 조직문화 |
| **제**로 트러스트 | 항상 검증, 최소 권한 | SDP, 마이크로 세그멘테이션, 인증 강화 |

### 🔧 보안 솔루션 (CNAPP)

```
┌─────────────────────────────────────────────────────┐
│            CNAPP (Cloud-Native App Protection)       │
├─────────────┬─────────────┬─────────────────────────┤
│  Artifact   │  Security   │   Runtime Protection    │
│   스캐닝    │    보안     │      런타임 보호         │
├─────────────┼─────────────┼─────────────────────────┤
│    CSNS     │ IaC 스캐닝  │         CWPP            │
│    CIEM     │    CSPM     │   Cloud Configuration   │
└─────────────┴─────────────┴─────────────────────────┘
```

| 솔루션 | 풀네임 | 역할 |
|:------|:------|:-----|
| CSPM | Cloud Security Posture Management | 클라우드 설정 오류 탐지 |
| CWPP | Cloud Workload Protection Platform | 워크로드 보호 |
| CIEM | Cloud Infrastructure Entitlement Mgmt | 클라우드 권한 관리 |

---

### 📋 대응/운영 전략 5가지

| 전략 | 설명 |
|:-----|:-----|
| Shift-Left Security | 개발 초기 단계부터 보안 검증 |
| Runtime Monitoring | 실행 중 컨테이너, 서버리스 모니터링 |
| Policy-as-Code | 보안 정책을 코드화하여 CI/CD 적용 |
| Zero Trust | "항상 검증, 최소 권한" 접근 원칙 |
| Continuous Compliance | 규제·보안 기준 실시간 준수 검증 |

---

### ⭐ 차별점 키워드

> **3R 원칙 (Rotate, Repair, Repave)**
> - **Rotate**: 자격증명, 키 주기적 교체
> - **Repair**: 취약점 발견 시 즉시 패치
> - **Repave**: 문제 발생 시 새로 구축 (불변 인프라)
