### 🔑 핵심 키워드 3개
- **디지털 흔적**: 컴퓨터 사용 기록이 남는 것
- **증거 데이터**: 범죄 수사에 쓸 수 있는 데이터
- **타임스탬프**: 언제 했는지 시간 기록

### 📖 등장배경

> 경찰이 범인의 컴퓨터를 압수했는데, 범인이 파일을 다 지웠대요. 그런데 수사관이 "삭제해도 흔적은 남아요~" 하면서 웹 방문 기록, 최근 열었던 문서 목록, 로그인 기록 등을 찾아냈어요. 이렇게 **지웠다고 생각한 것들도 어딘가에 남아있는 흔적**을 '아트팩트'라고 해요.

### 📝 개념 정의

> 아트팩트는 **"디지털 기기에서 사용자가 뭘 했는지 보여주는 흔적"**이에요.
> 
> 비유하자면, 눈 내린 날 발자국처럼 **지나간 길이 다 보이는 것**과 같아요!

### 🏗️ 기술요소 (2그룹)

**그룹 1: 아트팩트 종류**

| 종류 | 쉬운 설명 | 예시 |
|:-----|:---------|:-----|
| 시스템 아트팩트 | 컴퓨터가 자동으로 기록하는 것 | 윈도우 레지스트리, 이벤트 로그 |
| 응용 프로그램 아트팩트 | 프로그램이 남기는 기록 | 브라우저 방문기록, 카톡 대화 |
| 파일 아트팩트 | 파일 관련 기록 | 바로가기 파일, 최근 문서 목록 |
| 메모리 아트팩트 | RAM에 남아있는 정보 | 실행 중인 프로그램, 암호화 키 |

**그룹 2: 분석 도구**

| 도구 | 용도 | 쉬운 설명 |
|:-----|:-----|:---------|
| EnCase | 디스크 분석 | 하드디스크 속 증거 찾기 |
| Volatility | 메모리 분석 | RAM 속 정보 분석 |
| Wireshark | 네트워크 분석 | 인터넷 통신 기록 보기 |
| Cellebrite | 모바일 분석 | 스마트폰 데이터 추출 |

### ⭐ 차별점 키워드

> **증거 보관 연속성**: 증거가 수집부터 법정까지 조작되지 않았음을 증명하는 것!

### 📋 6하원칙 요약

| 구분 | 내용 |
|:-----|:-----|
| **누가** | 디지털 포렌식 수사관, 보안 담당자 |
| **무엇을** | 디지털 기기의 사용 흔적(아트팩트) |

| **어디서** | 컴퓨터, 스마트폰, 서버 등 |
| **왜** | 범죄 증거 확보, 사건 재구성 |
| **어떻게** | 전문 도구로 수집 → 무결성 검증 → 분석 |

## 🎯 기술사 수준 설명
{: #-기술사-수준-설명}

{: .highlight }
> **아트팩트**: 디지털흔적, 증거연속성, 타임스탬프
> - (절차) `식우수무문`
> - (유형) `시응파메`
> - (기술) `수보분`
> - (도구) `E-V-W-C`

---

### 📝 개념 정의

디지털 기기에서 **사용자 행위·시스템 동작으로 생성된 흔적 데이터**(로그, 캐시, 레지스트리)로,  
삭제 후에도 잔존하며 **법적 증거로 활용**

---

### 🏗️ 구성요소

#### 아트팩트 유형 `시응파메`

| 유형 | 설명 | 예시 |
|:-----|:-----|:-----|
| **시**스템 | 운영체제가 자동 생성·관리하는 흔적 | 레지스트리, 이벤트 로그, syslog |
| **응**용프로그램 | 소프트웨어에서 발생하는 흔적 | 방문기록, 캐시, 쿠키, 대화 로그 |
| **파**일 | 파일 및 디렉터리 관련 흔적 | 바로가기(LNK), Thumbs.db, 최근 문서 |
| **메**모리 | 휘발성 데이터 흔적 | 메모리 덤프, 암호화 키, 세션 로그 |

#### 대표 분석 도구 `E-V-W-C`

| 도구 | 분석 대상 | 특징 |
|:-----|:---------|:-----|
| **E**nCase/FTK | 디스크/파일 | 삭제 파일 복구, 로그·레지스트리 추출 |
| **V**olatility | 메모리 | 실행 프로세스, 네트워크 흔적 확인 |
| **W**ireshark | 네트워크 | C&C 통신 추적, 세션 분석 |
| **C**ellebrite | 모바일 | 통화·메시지·앱 데이터 추출 |

---

### 🔄 수집 절차 `식우수무문`

```
아트팩트 식별 → 수집 우선순위 → 아트팩트 수집 → 무결성 검증 → 문서화 및 보관
(Identification)  (Prioritization)  (Acquisition)   (Verification)  (Documentation)
      │                  │                │               │               │
      ▼                  ▼                ▼               ▼               ▼
  유형 선정          휘발성 우선        전문 도구        해시 검증      증거 연속성
 (로그,레지스트리)      확보            활용           (MD5,SHA)        확보
```

---

### 🔧 분석 기술 `수보분`

| 구분 | 주요 기술 | 상세 내용 |
|:-----|:---------|:---------|
| **수**집 | 비트 단위 이미징 | 저장매체를 원본 그대로 복제 |
| | 라이브 수집 | 메모리, 네트워크 세션 등 휘발성 확보 |
| **보**존 | 해싱(MD5, SHA) | 해시값으로 증거 변경 여부 검증 |
| | 쓰기 방지 장치 | 원본 저장매체 수정 방지 |
| **분**석 | 타임라인 분석 | MAC Time 기반 행위 재구성 |
| | 파일 시스템 분석 | LNK, Prefetch, Thumbs.db 등 추출 |

---

### ⭐ 차별점 키워드

> **증거 보관 연속성 (Chain of Custody)**
> - 증거 수집부터 법정 제출까지 무결성·연속성 보장
> - 해시 검증, 문서화를 통해 '생성 증거'의 신뢰성 확보

---

### 📈 상위 토픽 계층도

```
정보보안
├── 디지털 포렌식
│   ├── 아트팩트 (Artifact) ← 현재 토픽
│   ├── 증거 수집 절차
│   ├── 파일 카빙 / 파일 슬랙
│   └── 안티 포렌식
└── 침해사고 대응
    └── SOAR, 로그 분석
```
