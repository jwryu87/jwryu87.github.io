### 한 줄 정의
> 네트워크 보안 인프라의 핵심 장비로, **경계보안**(Firewall, IDS, IPS)과 **암호화 통신**(VPN)을 담당하는 보안 시스템

### 쉬운 비유 🏰
> **성(城)을 지키는 방어 체계**와 같아요!
> - 🧱 **Firewall**: 성벽 (출입 통제)
> - 👁️ **IDS**: 감시탑 (침입 탐지만, 차단 안 함)
> - 🛡️ **IPS**: 경비병 (침입 탐지 + 차단)
> - 🔐 **VPN**: 비밀 통로 (암호화된 안전한 연결)

### 왜 필요한가? (3줄 요약)
1. **경계보안 중심**: 외부 위협으로부터 내부 시스템 보호
2. **보안 패러다임 변화**: 제로 트러스트, 클라우드 기반 보안으로 진화
3. **원격 근무 확산**: VPN, SSL, ZTNA 사용 원격접속 보안 구조의 중요성 급증

## 1. 네트워크 보안 인프라의 중요성 설명

{: .highlight }
> **네트워크 보안 인프라**: 경계보안, 제로트러스트, 원격접속 보안
> - (중요성) `보안패러다임변화`, `거버넌스`, `원격근무`
> - (핵심장비) `Firewall-IDS-IPS-VPN`

### 나. 네트워크 보안 인프라의 중요성

| 중요성 | 설명 |
|:-------|:-----|
| **보안 패러다임 변화** | 경계보안 중심에서 제로 트러스트, 클라우드 기반 보안으로 진화 |
| **보안 거버넌스, 관리체계 수립의 필요성 증가** | ISO 27001 도입 시 각 인프라 이해 필요 |
| **클라우드 환경 원격 근무 확산** | VPN, SSL, ZTNA 사용 원격접속 보안 구조의 중요성 급증 |

**핵심 요약**: 외부 연결에 대하여 VPN, Firewall, IDS, IPS로 보안 시스템을 구축함으로써 내부 시스템 보호

## 2. 침입차단시스템(Firewall), 침입탐지시스템(IDS) 설명

{: .highlight }
> **Firewall & IDS**: 첫 번째 방어선, 탐지 중점
> - (Firewall 기능) `접사트트감프N`
> - (Firewall 유형) `패상프`
> - (IDS 기능) `경세실R`
> - (IDS 유형) `HND`

### 가. "첫 번째 방어선", 침입차단시스템(Firewall) 설명

| 구분 | 설명 |
|:-----|:-----|
| **정의** | 외부로부터 불법 침입과 내부의 불법 정보 유출을 방지하고, 내·외부 네트워크의 상호 간 영향을 차단하기 위한 **보안시스템** |
| **개념** | 내부 네트워크와 외부 네트워크 간의 **트래픽 흐름을 제어**하여 허용된 통신만 통과시키고 비인가된 접근을 차단하는 보안 게이트웨이 장비 |

#### Firewall 기능 `접사트트감프N`

| 기능 | 설명 |
|:-----|:-----|
| **접**근제어 | 특정 송신원의 주소 혹은 발신원 주소 등을 가진 패킷 통과를 제한하는 기능. 송·수신자의 IP 주소, TCP, UDP의 포트번호를 바탕으로 패킷 필터링. 외부에서 내부 네트워크에 접속하는 것을 패킷 필터링을 이용하여 통제. 방화벽이 지나는 모든 패킷을 정의된 보안 정책에 의해 막음 |
| **사**용자 인증 | 트래픽에 대한 사용자의 신분을 증명하는 기능 |
| **트**래픽 암호화 | 네트워크 통신 트래픽 정보의 암호화 및 복호화 기술 |
| **트**래픽 로그 | 트래픽에 대한 접속 정보 기록, 네트워크 사용에 따른 유용한 통계 정보 |
| **감**사추적 기능 | 보안 정책에 따라 사용자 및 중요 데이터의 감사 및 추적과 다양한 유출 경로를 통한 유출 방지 |
| **프**록시 기능 | Application 계층에 대한 필터링 기능(OSI 7 Layer). 프록시 서버: 프록시 서버는 네트워크 IP 주소를 대체하며, 실제 IP 주소를 인터넷 상에서 효과적 은닉. 프록시 서버의 종류: 웹 프록시, 회선/애플리케이션 수준 게이트웨이 등이 있음 |
| **N**AT(주소변환) | Network Address Translation. 발신자 호스트의 IP 주소나 목적지 호스트의 IP 주소를 전송 단계에서 변경하여 전달하는 기능. 내부(사설 주소)와 외부(공인 주소)의 주소 변환(Mapping) 기능 |

#### Firewall 유형 `패상프`

| 유형 | 설명 |
|:-----|:-----|
| **패**킷 필터형 | L3 네트워크 계층 기반 기본형 방화벽 |
| **상**태기반 검사형 | 세션 계층 상태까지 추적하는 진화형 방화벽 |
| **프**록시형 | L7 응용 계층 기반, 콘텐츠까지 확장된 지능 방화벽 |

**참고**: 최근에는 통합 보안 기능, 제로트러스트 원칙을 지원하는 **NGFW(Next-Generation Firewall)** 널리 사용

### 나. "탐지"에 중점, 침입탐지시스템(IDS) 설명

| 구분 | 설명 |
|:-----|:-----|
| **정의** | 비인가된 사용자가 자원의 **무결성(integrity)**, **기밀성(confidentiality)**, **가용성(availability)**을 저해하는 일련의 행동들과 보안 정책을 위반하는 행위, 즉 **침입(intrusion)을 실시간으로 탐지**하는 시스템 |
| **개념** | 네트워크나 시스템에서 발생하는 이벤트를 모니터링하고, **잠재적인 비정상적 행위나 공격 패턴을 탐지**하여 관리자에게 알림을 전송하는 보안 인프라 시스템 |

#### IDS 기능 `경세실R`

| 기능 | 설명 |
|:-----|:-----|
| **경**보 | 경보, 이메일 발송, SNMP Trap 발송 등으로 통보 |
| **세**션 차단 | 의심스러운 행위 감지 시에 해당 세션 차단 |
| **실**시간 탐지 | 시스템이나 네트워크를 모니터링하여 실시간 침입 탐지 기능 |
| **R**eporting | 통계적 분석 및 reporting 기능 |

#### IDS 기능 상세

| 기능 | 설명 |
|:-----|:-----|
| **로그 분석 및 경보** | 탐지 결과를 관리자에게 실시간 전송, 로그 저장 |
| **패턴 기반 탐지** | 알려진 공격 시그니처와 비교하여 탐지 |
| **이상 행위 탐지** | 정상 트래픽 / 패턴과 비교하여 비정상 행위를 탐지 |
| **하이브리드 방식** | 패턴 기반 및 이상 행위 탐지 병행 |

#### IDS 유형 `HND`

| 유형 | 설명 |
|:-----|:-----|
| **H**IDS (Host-based IDS) | 개별 서버나 호스트의 로그, 프로세스를 감시 |
| **N**IDS (Network-based IDS) | 네트워크 구간의 트래픽을 모니터링 |
| **D**IDS (Distributed IDS) | 여러 IDS를 통합 관리 및 분석 |

#### IDS 탐지 모델 기반

| 유형 | 탐지 방식 | 설명 |
|:-----|:---------|:-----|
| **데이터 소스 기반** | 호스트 기반 (HIDS) | 서버에 직접 설치됨에 따라 네트워크 환경과 무관, 감사자료 다양, 정확한 침입탐지 가능 |
| | 네트워크 기반 (NIDS) | 네트워크 세그먼트당 하나의 탐지기만 설치, 설치 용이, 독립적으로 네트워크에서 실행되어 운영서버의 성능저하 없음 |
| **침입탐지 모델 기반** | 오용침입탐지 (Misuse) | 특정 공격에 관한 기존의 축적된 지식을 바탕으로 패턴 설정, 패턴과 축약 가공된 데이터를 비교, 일치 시 불법 침입 간주, 새로운 공격탐지를 위해 지속적으로 새로운 공격패턴 갱신 필요 |
| | 이상침입탐지 (Anomaly) | 사용자의 행동패턴을 분석, 정상 사용패턴과 비교해 이상 패턴 발 견을 침입으로 간주, 인공지능 알고리즘 사용 |

**참고**: IDS는 **False Positive(오탐)**, **False Negative(미탐)** 관리 중요. 탐지만 할 뿐 유해 트래픽에 대해 자동으로 차단하는 시스템은 존재하지 않음

## 3. 침입방지시스템(IPS), 가상사설망(VPN) 설명

{: .highlight }
> **IPS & VPN**: 능동형 보안 시스템, 원격접속 보안 네트워크
> - (IPS 기능) `정패이실`
> - (IPS 유형) `HN`
> - (VPN 기능) `인암무터키`
> - (VPN 유형) `ISM`

### 가. "능동형 보안 시스템", 침입방지시스템(IPS) 설명

| 구분 | 설명 |
|:-----|:-----|
| **정의** | 침입탐지시스템의 오판(False Positive)과 미탐(Miss Detection)의 문제 해결을 위해 등장한 정보시스템 네트워크에서 **침입탐지와 침입을 사전 예방하는 능동형 정보보호시스템** |
| **개념** | **IDS+Firewall 기능을 결합**한 형태로, 네트워크 트래픽을 **실시간 분석하여 공격을 탐지**하고, 탐지된 공격을 **자동으로 차단**하는 보안 시스템 |

#### IPS 기능 `정패이실`

| 기능 | 설명 |
|:-----|:-----|
| **정**책 기반 제어 | 네트워크 보안 정책에 따른 접근 제어 |
| **패**턴 기반 탐지 및 차단 | 알려진 공격 시그니처와 비교하여 탐지 및 **실시간 차단** |
| **이**상 행위 탐지 및 차단 | 정상 패턴과 비교하여 비정상 행위를 탐지 및 **실시간 차단** |
| **실**시간 대응 및 로그 보고 | DDoS, 스캐닝 실시간 차단 및 **SIEM 연계 가능** 보고서 추출 |

#### IPS 기능 상세

| 기능 | 설명 |
|:-----|:-----|
| **실시간 자동 대응** | 침입 및 바이러스 감염 등 상황에 맞는 실시간 자동 대응 기능 제공 |
| **실시간 분석** | 광대역 통신망 패킷을 실시간 분석 기능 제공 |
| **미확인 공격 탐지** | 알려지지 않는 공경에 대한 탐지 기능 제공 |
| **세션 기반 탐지** | 패킷 기반 탐지 외에 세션 기반 탐지 기능 제공 |

#### IPS 유형 `HN`

| 유형 | HIPS (호스트 기반 IPS) | NIPS (네트워크 기반 IPS) |
|:-----|:----------------------|:------------------------|
| **설명** | 특정 서버 기반으로 애플리케이션 소프트웨어 형태로 설치, 시큐어 OS와 유사기능 수행 | 방화벽처럼 네트워크 인라인모드로 설치되어 공격을 차단해 주는 기능 수행, 정책에 근거한 패턴 탐지 |
| **특징** | 서버기반의 커널 조작 등 불법적인 서버 침입 방지 | DDoS, 버퍼오버플로우 등 네트워크 기반 공격에 빠른 대응과 사전 차단 및 방비 |
| **목적** | 시스템 불법 침입 및 바이러스, 해킹 등 호스트 기반 보호 | 네트워크 기반의 공격 등 과도한 트래픽 공격방지 |
| **장점** | 커널 상에서 불법 응용 실행 방지, 업데이트 및 패치 관리 용이 | 하드웨어 기반의 빠른 대응속도, 대부분 자동 수행 기능 |
| **단점** | Agent 설치 비용 증가, 정상적인 응용 프로그램 방해 가능성 발생 | 네트워크상의 단일 실패점 존재로 여분의 장비 필요, 보안업데이트에 의존함 |

**참고**: HIPS는 에이전트 설치 비용 증가 이슈, NIPS는 SPOF 문제 발생 가능하므로, 목적에 맞는 IPS 장비 구축 요망

### 나. "원격접속 보안 네트워크", 가상사설망(VPN) 설명

| 구분 | 설명 |
|:-----|:-----|
| **정의** | 터널링(Tunneling) 기법을 사용하여 공중망에 접속해 있는 두 네트워크 사이의 연결을 마치 **전용회선을 이용해 연결한 것과 같은 효과를 내는 가상 네트워크 기법** |
| **개념** | 공중망을 이용하여, 암호화 / 인증 / 무결성 기반으로 **기업 내부 전용망과 동일한 수준의 보안성을 제공하는 가상의 사설 통신 네트워크** |

#### VPN 주요 기능 `인암무터키`

| 기능 | 설명 |
|:-----|:-----|
| **인**증 (Authentication) | 사용자/단말의 신원 확인(ID, 인증서, OTP 등) |
| **암**호화 (Encryption) | 통신 데이터의 기밀성 보장(AES, 3DES, DES 등) |
| **무**결성 (Integrity) | 데이터 위/변조 방지(HASH 사용) |
| **터**널링 (Tunneling) | 인터넷 상에서의 가상의 정보 흐름 통로, IPSec |
| **키** 관리 | 키의 안전한 분배를 위한 관리 매커니즘, **IKE 프로토콜** |

#### VPN 유형 `ISM`

| 유형 | 설명 | 특징 |
|:-----|:-----|:-----|
| **I**PSec VPN | IPSec 프로토콜(3계층)을 이용하여 전용 VPN장비로 구현 | Site-to-Site, Site-to-Remote방식 구현, 다양한 환경에 적용, 고객이 애플리케이션과 독립적으로 운영, **네트워크 계층, RFC 2401 표준** |
| **S**SL VPN | 보안통신 프로토콜(SSL)을 통해 VPN 구현 (HTTPS) | 전송계층 ~ 응용계층(4~7 계층), 다수의 원격사용자, 웹기반 어플리케이션, 운영 환경에 적합, **원격접속 VPN**, 웹브라우저 기반 |
| **M**PLS VPN | 패킷 스위칭 기술인 MPLS 환경을 통해 VPN 구현 | 데이터 링크 계층(2계층), 트래픽 제어 기술 제공(QoS), 높은 보안성, **레이블 사용 패킷 전달**, 인증 암호화 미제공, IPSec과 혼합 |

**참고**: 본사-지사간 인트라넷을 접속하기 위한 VPN은 주로 MPLS 방식과 IPSec 방식을 사용하며, Client 단위로 언제 인트라넷을 접속하기 위해 응용 프로그램을 통한 SSL VPN 방식 사용

**참고**: IPsec VPN의 핵심 프로토콜로 **IKE, AH, ESP, SA** 존재하며 IPS, 방화벽 등과 정합성 있게 구성해야 함

## 4. Firewall, IDS, IPS, VPN 차이 비교

{: .highlight }
> **4가지 보안 장비 비교**: 목적, 작동방식, 대응방식, 위치
> - (비교항목) `목작대위주보`

### 가. Firewall, IDS, IPS, VPN 차이 비교표 `목작대위주보`

| 항목 | Firewall (방화벽) | IDS (침입탐지시스템) | IPS (침입방지시스템) | VPN (가상사설망) |
|:-----|:------------------|:-------------------|:-------------------|:----------------|
| **목**적 | 트래픽 통제 및 접근 제어 | 침입/비정상행위 탐지 및 경보 | 침입 탐지 및 **실시간 차단** | 통신 암호화 및 안전한 연결 |
| **작**동 방식 | 규칙 기반 트래픽 허용/차단 | 트래픽 모니터링 및 패턴 분석 | 트래픽 모니터링 및 탐지+**차단** | 터널링·암호화 통한 연결 |
| **대**응 방식 | 허용/차단 | 탐지·알림 (차단하지 않음) | 탐지·알림 + **실시간 차단 (방어)** | 암호화·인증으로 안전한 접속 |
| **위**치 | 네트워크 경계 | 방화벽 뒤·네트워크 경계 등 | 방화벽 뒤·네트워크 경계 등 | 사용자와 내부 네트워크 사이 |
| **주**요 기능 | IP·포트·프로토콜 기반 제어 | 공격/침입 탐지, 로그 분석 | 공격/침입 **즉시 차단** | 외부에서 내부 접속시 정보 암호화 |
| **보**안 수준 | 기본 트래픽 통제 | 탐지 중심 (수동적) | **능동 대응 (차단)** | 데이터 기밀성/무결성 확보 |

## ⭐ 차별점 키워드

### 🏆 NGFW (Next-Generation Firewall)
> **차세대 방화벽**
> - 기존 패킷 필터링 + 딥 패킷 인스펙션(DPI) + 애플리케이션 인식
> - IPS, 안티바이러스, URL 필터링 등 **통합 보안 기능** 제공
> - 제로트러스트 원칙 지원

### 🏆 ZTNA (Zero Trust Network Access)
> **제로트러스트 기반 네트워크 접속**
> - VPN의 한계를 극복한 차세대 원격접속 보안
> - "Never Trust, Always Verify" 원칙
> - 사용자/디바이스 단위로 접근 제어
