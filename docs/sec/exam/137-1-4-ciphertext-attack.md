### 🔑 핵심 키워드 3개
- **암호 해독**: 비밀번호 없이 암호를 풀려는 시도
- **평문/암호문**: 원본 메시지 / 암호화된 메시지
- **키(Key)**: 암호를 풀 수 있는 열쇠

### 📖 등장배경

> 친구에게 비밀 편지를 보냈는데, 다른 애가 그 편지를 가로챘어요. 편지는 암호로 써있어서 읽을 수 없지만, 그 애가 여러 방법으로 암호를 풀려고 해요. 이렇게 **암호화된 메시지를 가지고 원래 내용이나 비밀 키를 알아내려는 시도**를 '암호문 공격'이라고 해요.

### 📝 개념 정의

> 암호문 공격은 **"암호 키 없이 암호를 풀려는 해킹 시도"**예요.
> 
> 비유하자면, 자물쇠 열쇠 없이 **여러 방법으로 금고를 열려는 것**과 같아요!

### 🏗️ 기술요소 (2그룹)

**그룹 1: 공격 유형 4가지**

| 공격 | 쉬운 설명 | 난이도 |
|:-----|:---------|:------|
| 암호문 단독 공격 | 암호문만 가지고 푸는 것 | 가장 어려움 |
| 알려진 평문 공격 | 일부 원문을 알고 푸는 것 | 어려움 |
| 선택 평문 공격 | 원하는 문장을 암호화해서 분석 | 보통 |
| 선택 암호문 공격 | 원하는 암호를 복호화해서 분석 | 쉬움 |

**그룹 2: 대응 방안**

| 단계 | 방법 | 쉬운 설명 |
|:-----|:-----|:---------|
| 예방 | 강력한 암호화 | AES 같은 튼튼한 암호 사용 |
| 탐지 | 모니터링 | 이상한 시도 감시 |
| 대응 | 차단/격리 | 공격 발견 시 막기 |
| 개선 | 보안 강화 | 더 안전하게 업그레이드 |

### ⭐ 차별점 키워드

> **커크호프 원칙**: 암호 시스템의 보안은 알고리즘이 아닌 **키(Key)의 비밀성**에 의존해야 한다!

### 📋 6하원칙 요약

| 구분 | 내용 |
|:-----|:-----|
| **누가** | 해커, 공격자, 암호 분석가 |
| **무엇을** | 암호문, 암호 키 |
| **언제** | 도청, 데이터 탈취 시 |
| **어디서** | 네트워크, 저장장치, 통신 채널 |
| **왜** | 기밀 정보 탈취, 인증 우회 |
| **어떻게** | 통계 분석, 패턴 분석, 브루트포스 |

## 🎯 기술사 수준 설명
{: #-기술사-수준-설명}

{: .highlight }
> **암호문 공격(COA)**: 암호문만으로 평문·키 해독 시도, 고전암호 취약, 현대암호 기준점
> - (특징) `제패고현` (공격) `빈패통기` (대응) `강암패적`

---

### 📝 개념 정의

**암호화된 데이터(암호문)만 가지고** 원래의 평문이나 암호화 키를 유추하려고 시도하는 공격 기법

---

### 🏗️ 특징 `제패고현`

| 특징 | 설명 |
|:-----|:-----|
| **제**한된 정보 | 공격자는 암호문만 가지고 있어 **가장 어려운 공격** 형태 |
| **패**턴 분석 기반 | 암호문에서 반복되는 패턴, 통계적 특성을 활용 |
| **고**전 암호 취약 | 단순 치환(Substitution)이나 전치(Transposition) 암호는 COA에 취약 |
| **현**대 암호학 기준점 | 현대 암호 알고리즘은 **COA에 안전하지 않으면 실사용 불가** |

---

### ⚔️ 공격 방법 `빈패통기`

| 공격 방법 | 설명 |
|:---------|:-----|
| **빈**도 분석 (Frequency Analysis) | 문자나 블록의 등장 횟수를 기반으로 평문 추정 |
| **패**턴 매칭 (Pattern Matching) | 반복되는 암호문 블록 → 평문 구조 추측 |
| **통**계적 분석 (Statistical Analysis) | 암호문 길이, 분포 등을 이용해 암호 기법 특성 유추 |
| **기**계학습 기반 공격 | ML/DL 활용하여 암호문 패턴에서 평문 특성 추출 시도 |

---

### 🛡️ 대응 방안 `강암패적`

| 대응 방안 | 설명 | 상세 |
|:---------|:-----|:-----|
| **강**력한 암호 알고리즘 | AES, RSA, ECC 등 검증된 알고리즘 | 단순 치환/전치 암호 사용 금지 |
| **암**호학적 안전성 보장 | Semantic Security | 동일 평문 → 암호화마다 다른 암호문 생성 |
| **패**딩 및 난수 사용 | 무작위 패딩 추가 | 암호화 전 패턴 노출 방지 |
| **적**절한 운영 모드 선택 | Block Cipher Modes | CBC, CTR, GCM 같은 안전한 모드 사용 |

---

### ⭐ 차별점 키워드

> **커크호프 원칙 (Kerckhoff's Principle)**
> - 암호 시스템의 보안은 **알고리즘의 비밀이 아닌 키(Key)의 비밀성**에 의존
> - 현대 암호는 COA에 안전해야 실용적으로 사용 가능

---

### 📈 상위 토픽 계층도

```
정보보안
├── 암호화 기술
│   ├── 대칭키 암호 (AES, DES)
│   ├── 비대칭키 암호 (RSA, ECC)
│   └── 해시 함수 (SHA, MD5)
├── 암호 분석/공격
│   ├── 암호문 공격 (COA) ← 현재 토픽
│   ├── 알려진 평문 공격 (KPA)
│   ├── 선택 평문 공격 (CPA)
│   └── 선택 암호문 공격 (CCA)
└── 키 관리
    └── PKI, 키 생명주기 관리
```
