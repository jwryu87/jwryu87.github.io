### 🔑 핵심 키워드 3개
- **공표 전**: 아직 알려지지 않은 취약점
- **패치 전**: 수정 프로그램 배포 전
- **0일**: 방어할 시간이 0일(Zero Day)

### 📖 등장배경

> 해커가 보안 취약점을 먼저 발견하면, 회사가 고치기 전까지 마음대로 공격할 수 있어요. 이때 방어할 시간이 **"0일"**이라서 제로데이라고 불러요. 최근 생성형 AI 등장으로 취약점 찾기가 더 쉬워져서 위험이 커지고 있어요!

### 📝 개념 정의

> 제로데이 취약점은 **"보안 패치가 나오기 전에 공격당하는 취약점"**이에요.
> 
> 비유하자면, **집 열쇠를 잃어버렸는데 자물쇠를 바꾸기 전**에 도둑이 들어오는 것과 같아요!

### 🏗️ 기술요소 (2그룹)

**그룹 1: 대응 프로세스**

| 단계 | 활동 | 쉬운 설명 |
|:-----|:-----|:---------|
| 1단계 | 취약점 발견 | 문제 찾기 |
| 2단계 | 보고 및 분석 | 원인 파악 |
| 3단계 | 솔루션 개발 | 해결책 만들기 |
| 4단계 | 패치 배포 | 수정 프로그램 전달 |
| 5단계 | 시스템 업데이트 | 사용자가 설치 |

**그룹 2: 예방 방법**

| 방법 | 설명 | 쉬운 비유 |
|:-----|:-----|:---------|
| 최신 업데이트 | SW/OS 최신 유지 | 백신 접종 |
| 필수 앱만 사용 | 불필요 SW 삭제 | 집 출입구 줄이기 |
| 방화벽 설정 | 네트워크 보호 | 담장 세우기 |
| 보안 교육 | 직원 역량 강화 | 경비원 훈련 |

### ⭐ 차별점 키워드

> **버그바운티**: 취약점을 먼저 찾아주면 보상금을 주는 제도!

| **어떻게** | 랜섬웨어, 스피어피싱 등 |

## 🎯 기술사 수준 설명
{: #-기술사-수준-설명}

### 📌 핵심 암기 (Quick Reference)

{: .highlight }
> **제로데이 취약점**: 공표 전, 패치 전, 0일 방어
> - (대응) `발분개배업`
> - (예방-기술) `최방` (최신패치, 방화벽)
> - (예방-관리) `필교` (필수앱, 교육)

---

### 📝 개념 정의

생성형AI 등장으로 제로데이 공격이 쉬워지고 비용도 낮아지는 추세에 따라,
소프트웨어의 **공표되지 않은 또는 패치되지 않은 보안 취약점 이용, 방어 시간 0일인 공격**

| 구분 | 설명 |
|:-----|:-----|
| 제로데이(Zero Day) | 취약점 발견~패치 배포 사이 방어 가능 일수 = 0일 |
| 공격 특징 | 알려지지 않아 탐지·방어 불가, 피해 심각 |
| 대표 공격 | 랜섬웨어, 스피어피싱 |

### 🔄 대응 프로세스 `발분개배업`

```
┌────────────┐    ┌────────────┐    ┌────────────┐
│ 1. 취약점  │───▶│ 2. 보고 및 │───▶│ 3. 보안    │
│    발견    │    │    분석    │    │ 솔루션개발 │
└────────────┘    └────────────┘    └─────┬──────┘
                                          │
        ┌─────────────────────────────────┘
        ▼
┌────────────┐    ┌────────────┐
│ 4. 보안    │───▶│ 5. 사용자  │
│ 패치 배포  │    │시스템 업뎃 │
└────────────┘    └────────────┘
```

| 단계 | 활동 | 설명 |
|:-----|:----|:-----|
| **발**견 | 새로운 취약점 발견 | 버그바운티, 보안연구원 |
| **분**석 | 취약점 보고 및 분석 | 영향도, 위험도 평가 |
| **개**발 | 새로운 보안 솔루션 개발 | 패치, 핫픽스 제작 |
| **배**포 | 보안 패치 배포 | 공식 업데이트 릴리즈 |
| **업**데이트 | 사용자 시스템 업데이트 | 최종 적용 |

| 구분 | 공격 | CVE | 상세 설명 |
