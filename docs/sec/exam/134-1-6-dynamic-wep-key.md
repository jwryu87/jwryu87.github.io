### 🔑 핵심 키워드 3개
- **동적 WEP 키**: 주기적으로 바뀌는 무선랜 암호화 키
- **RC4 알고리즘**: 스트림 암호화 방식
- **EAP-TLS**: 서버와 클라이언트가 서로 인증하는 방식

### 📖 등장배경

> 옛날 무선랜은 같은 암호 키를 계속 써서 해킹당하기 쉬웠어요. 해커가 40만~100만 번만 시도하면 키를 찾아낼 수 있었죠. 그래서 **"키를 자주 바꾸면 해커가 못 찾겠지?"**라는 아이디어로 나온 게 동적 WEP 키예요. 세션마다, 기기마다 다른 키를 쓰고 주기적으로 바꿔서 보안을 강화한 거예요!

### 📝 개념 정의

> 동적 WEP 키는 **"주기적으로 자동으로 바뀌는 무선랜 암호화 키"**예요.
> 
> 비유하자면, **자동으로 자물쇠를 바꾸는 금고**처럼 키가 자동으로 바뀌어서 더 안전한 거예요!

### 🏗️ 기술요소 (2그룹)

**그룹 1: 정적 WEP vs 동적 WEP**

| 구분 | 정적 WEP | 동적 WEP 키 |
|:-----|:---------|:-----------|
| 키 변경 | 고정된 키 사용 | 세션마다, 기기마다 다른 키 |
| 보안성 | 취약 (크랙 공격에 취약) | 강화 (주기적 키 변경) |
| 인증 | 공유키 인증 | EAP-TLS 상호인증 |

**그룹 2: 암호화 방식**

| 방식 | 설명 | 비트 수 |
|:-----|:-----|:--------|
| 40Bit WEP | 64비트 암호화 (40Bit WEP + 24Bit IV) | 64비트 |
| 104Bit WEP | 128비트 암호화 (104Bit WEP + 24Bit IV) | 128비트 |
| RC4 알고리즘 | 대칭키 스트림 암호화 | - |

### ⭐ 차별점 키워드

> **주기적 키 갱신**: 키가 유출되어도 잠깐 동안만 유효해서 보안이 강화됨!

### 📋 6하원칙 요약

| 구분 | 내용 |
|:-----|:-----|
| **누가** | 무선 단말, 액세스 포인트(AP), 인증 서버 |
| **무엇을** | 주기적으로 변경되는 WEP 암호화 키 |
| **언제** | 세션마다, 주기적으로(예: 30분마다) |
| **어디서** | IEEE 802.11b 무선통신 환경 |
| **왜** | 정적 WEP 키의 크랙 공격 취약점 보완 |
| **어떻게** | EAP-TLS 인증 → 동적 키 생성 → RC4 암호화 |

### 📝 개념 정의

무선통신의 보안취약점 이해, 무선통신 보안기술 및 발전의 이해를 위해 출제되었다.

정적 WEP 키 방식의 평문 공격 및 크랙 취약점을 보완하기 위해,
각 사용자 세션마다 주기적으로 다른 WEP 키를 제공하는 무선통신 보안기술이다.

**표준:**
- IEEE 802.11b 무선통신 표준

**암호화 알고리즘:**
- 대칭키 스트림 암호화 RC4 알고리즘

**주요 취약점 보완:**
- WEP 크랙 공격 취약점
- 패킷 스니핑 취약점
- 64Bit/128Bit WEP 키는 40만~100만 번의 키 공격으로 크랙 가능

---

### 🏗️ 기술요소 `인암`

| 구분 | 기술요소 | 상세 설명 |
|:-----|:---------|:----------|
| **인**증 측면 | EAP-TLS 사용 | 서버 인증서와 개인 인증서를 상호 교환하는 무선랜 연결 방식 |
| **인**증 측면 | WEP KEY 동적 적용 | 세션별, 기기별로 다른 WEP 암호화 적용, 주기적 WEP 변경 |
| **인**증 측면 | 데이터링크 계층 보안 | MAC 주소 등록 접근제어를 통한 보안성 및 기밀성 강화 |
| **암**호 측면 | RC4 알고리즘 적용 | 대칭키 스트림 암호화 RC(Ron's Code) 4 알고리즘 적용 |
| **암**호 측면 | 40Bit WEP | 64비트 암호화 방식 (40Bit WEP + 24Bit IV: 초기벡터) |
| **암**호 측면 | 104Bit WEP | 128비트 암호화 방식, RC4 암호화 알고리즘의 Key 값에 적용 |

### 🔄 정적 WEP에서 동적 WEP 키로의 발전

```
정적 WEP
├─ 여러 사용자가 동일한 WEP 키 사용
├─ 고정된 WEP 키 인증
└─ 크랙 공격에 취약한 보안기술
    ↓ (WEP 키 크랙공격 취약점 개선)
동적 WEP KEY
├─ RC4 알고리즘을 통한 암호화
├─ 세션별, 기기별 WEP 사용
└─ 주기적 WEP 키 생성
```

**주요 개선사항:**
- IEEE 802.11B에 적용된 동적 WEP 키는 대칭키 스트림 암호화 RC4 암호화 알고리즘을 사용
- 세션마다, 기기마다 다른 WEP 키 적용
- 주기적 키 갱신으로 보안 강화

---

### 🔐 암호화 과정 `공무키초`

```
초기벡터(IV) + 공유키 WEP키
    ↓
암호키 형성
    ↓
WEP PRNG (RC4)
    ↓
키 스트림 생성
    ↓
평문 + ICV (무결성체크)
    ↓
XOR 연산
    ↓
암호문 + IV
```

**암호화 단계:**

1. **공**유키와 난수 발생기를 이용하여 RC4 알고리즘을 통한 키스트림을 생성 (데이터 암호화 지원)
2. **무**결성을 보장하기 위해 CRC-32 알고리즘으로 구성되는 ICV를 사용
3. **키**스트림과 평문과 ICV 값이 합해진 데이터를 XOR 연산 수행하여 암호문 생성
4. **초**기벡터(IV) 값을 생성된 암호문에 추가
5. 완성된 암호문에 프레임 헤더와 FCS를 붙여 전송

---

### 🔓 복호화 과정

```
암호문에서 IV 추출
    ↓
IV + 공유키 WEP키
    ↓
WEP PRNG (RC4)
    ↓
키 스트림 생성
    ↓
키 스트림 XOR 암호문
    ↓
평문 + ICV
    ↓
평문 → CRC-32 → ICV'
    ↓
무결성체크 (ICV = ICV'?)
    ↓
평문 (성공) / failed IVC (실패)
```

**복호화 단계:**

1. 송신자가 보낸 패킷에서 IV 값 추출
2. 추출한 IV 값과 WEP 키를 이용하여 난수 발생기로부터 난수를 생성하여 키 스트림 형성
3. 키 스트림과 암호문을 XOR 연산하여 복호화 수행
4. 복호화된 평문을 CRC-32 연산하여 계산한 ICV'와 원본 ICV를 비교하여 무결성 확인
5. 무결성 검증이 완료되면 평문 수신

### 🔄 인증 절차 `인인암성`

```
클라이언트                    AP
    │                        │
    │── 인증 요청 ──────────→│
    │                        │
    │←─ 인증용 문자열 전송 ──│ (인증용 IV 생성)
    │   (Challenge)          │
    │                        │
    │── 암호화된 문자열 전송 ─→│
    │   (RC4키로 인증용 IV   │
    │    암호화)             │
    │                        │
    │←─ 성공/실패 여부 통보 ──│ (인증용 IV 암호문
    │                        │  복호화 확인)
```

**인증 단계:**

1. **인**증 요청: 클라이언트가 AP에 인증 요청 전송
2. **인**증용 문자열 전송(Challenge): AP가 인증용 IV 생성 후 클라이언트에 전송
3. **암**호화된 문자열 전송: 클라이언트가 RC4 키로 인증용 IV를 암호화하여 AP에 전송
4. **성**공/실패 여부 통보: AP가 인증용 IV 암호문을 복호화하여 확인 후 결과 통보

### 🏗️ 동적 WEP 키 구성요소

| 구성요소 | 설명 |
|:---------|:-----|
| 키 관리 프로토콜 | IEEE 802.1aa 인증 프로토콜을 이용한 키 관리 |
| 키 갱신 주기 | 네트워크 관리자가 설정한 주기(예: 30분마다)에 따라 키 자동 갱신 |
| 키 분배 | 인증 서버가 클라이언트 기기와 액세스 포인트(AP) 간에 새로운 WEP 키를 안전하게 분배 |

---

### 📊 동적 WEP 키 특징

| 특징 | 설명 |
|:-----|:-----|
| 보안 강화 | 키를 주기적으로 변경함으로써 키가 유출되더라도 잠깐 동안만 유효해 보안 강화 |
| 중앙 관리 | 네트워크 관리자가 중앙에서 키를 관리하고 배포할 수 있어 관리 효율성 증가 |
| 사용자 인증 | IEEE 802.1aa 프로토콜을 통해 사용자 인증을 강화하여 네트워크 접근을 제어 |

### 📈 무선통신 보안기술 발전 단계

| 단계 | 보안 기술 | 설명 |
|:-----|:----------|:-----|
| 1단계 | 비 보안 방식 | 모든 무선 단말과 통신을 허락하는 방식 |
| 2단계 | WEP 보안 방식 | WEP 공유하는 공유키 인증, WEP 암호화 |
| 3단계 | IEEE-802.1x 보안 | 인증서버를 이용한 EAP-MD5 인증수행 |
| 4단계 | **동적 WEP 보안** | **EAP-TLS 이용 상호인증, 동적 WEP 키 인증** ← 현재 토픽 |
| 5단계 | WPA 보안 | TKIP 사용, WPA-PSK 암호방식(10000 패킷당 키 변경) |
| 6단계 | RSN 보안 | CCMP 알고리즘, 기본알고리즘 정의 |
| 7단계 | 이동보안 | IAPP(Inter AP Protocol)적용, 안전한 이동성보장 |
| 8단계 | 무선네트워크 보안 | 무선네트워크 전체 보호, 무선랜 통신보안목표 |

**IEEE 802.11i 이후:**
- WPA, WPA2, WPA3 등으로 발전하며 무선통신의 보안기술이 적용
- WIFI-6, WIFI-7(EHT)에는 SAE, 192Bit 암호화 기술 등이 적용된 WPA3가 탑재되어 보안성이 향상된 무선통신 보안기술이 적용
