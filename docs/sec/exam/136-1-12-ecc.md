### 🔑 핵심 키워드 3개
- **타원곡선**: 특별한 수학 곡선 위의 점들
- **공개키 암호**: 공개키/비밀키 한 쌍 사용
- **짧은 키**: RSA보다 작은 키로 같은 보안

### 📖 등장배경

> RSA 암호는 키가 너무 커서 스마트폰이나 IoT처럼 작은 기기에서 쓰기 어려웠어요. 그래서 **더 작은 키로 같은 보안 수준**을 가진 암호가 필요했고, 1985년에 타원곡선 암호가 제안됐어요. 지금은 비트코인, TLS 등에서 널리 사용돼요!

### 📝 개념 정의

> ECC는 **"타원곡선이라는 수학 그래프 위의 점들을 이용한 암호화"**예요.
> 
> 비유하자면, 미로에서 **앞으로 가기는 쉽지만 뒤로 추적하기는 불가능**한 것처럼, 공개키에서 비밀키를 알아내기 매우 어려워요!

### 🏗️ 기술요소 (2그룹)

**그룹 1: 타원곡선 연산**

| 연산 | 설명 | 쉬운 비유 |
|:-----|:-----|:---------|
| 점 덧셈 | P + Q = R | 두 점 연결해서 새 점 찾기 |
| 스칼라 곱 | kP = P + P + ... | 같은 점 여러 번 더하기 |
| 무한원점 | ∞ (영점 역할) | 0처럼 더해도 변화 없음 |

**그룹 2: 키 구성요소**

| 요소 | 의미 | 쉬운 설명 |
|:-----|:-----|:---------|
| G | 기본점 | 출발점 (모두 아는 점) |
| d | 개인키 | 비밀 숫자 |
| P | 공개키 | P = d × G (결과 점) |

### ⭐ 차별점 키워드

## 🎯 기술사 수준 설명
{: #-기술사-수준-설명}

### 📌 핵심 암기 (Quick Reference)

{: .highlight }
> **ECC**: 유한체, 타원곡선, 이산대수
> - (방정식) `y² = x³ + ax + b`
> - (연산) `덧스무` (덧셈, 스칼라곱, 무한원점)
> - (키생성) `P = d × G`

---

### 📝 개념 정의

블록체인, IoT, 무선환경 등 경량 암호화 수요 증가에 따라,
유한체(Finite Field) 위 타원곡선의 **점 연산 기반, 짧은 키로 RSA 동등 보안 제공하는 공개키 암호**

| 구분 | 설명 |
|:-----|:-----|
| 수학적 기반 | 타원곡선 이산대수 문제 (ECDLP) |
| 특징 | RSA 대비 짧은 키, 빠른 연산, 낮은 전력 소모 |
| 활용 | 암호화폐(비트코인), TLS 1.3, 모바일 보안, IoT |

---

### 📐 타원곡선 수학 구조

**타원곡선 방정식**

```
y² = x³ + ax + b   (유한체 Fp 위에서 정의)

     │    ●-R        P + Q = R
     │   /           (P와 Q를 지나는 직선이
   Q─┼──●            곡선과 만나는 점의
     │ /             X축 대칭점이 R)
   P─●──────
     │      R = P+Q
```

**도메인 파라미터**

| 파라미터 | 설명 |
|:--------|:-----|
| p | 유한체의 소수, 연산 모듈로 범위 결정 |
| a, b | 타원곡선 방정식의 계수 |
| G | 생성점(Generator Point), 기본 공개 시작점 |
| n | G의 순서 (G를 n번 더하면 무한원점 도달) |
| h | Cofactor, n × h = 곡선 위 전체 점의 수 |

---

### 🔢 암호 연산 요소 `덧스무`

| 연산 | 표기 | 설명 |
|:-----|:----|:-----|
| **덧**셈 | P + Q | 타원곡선 위 두 점을 더하는 연산 |
| **스**칼라 곱 | kP | 점 P를 k번 더하는 연산, ECC 핵심 연산 |
| **무**한원점 | ∞ | 덧셈의 항등원 역할, 특수한 점 |

**연산 상세**

| 연산종류 | P ≠ Q (Addition) | P = Q (Doubling) |
|:--------|:-----------------|:-----------------|
| 기울기 λ | (y₂-y₁)/(x₂-x₁) | (3x₁² + a)/(2y₁) |
| 결과 x₃ | λ² - x₁ - x₂ | λ² - 2x₁ |
| 결과 y₃ | (x₁-x₃)λ - y₁ | (x₁-x₂)λ - y₁ |

---

### 🔐 키 생성 구조 `P = d × G`

```
┌────────────────────────────────────────────┐
│         타원곡선 키 생성                    │
│                                            │
│   G (기본점) ──────┐                       │
│   [공개, 고정]      │                       │
│                    ▼                       │
│   d (개인키) ─── × ──→ P = d × G (공개키)  │
│   [비밀, 난수]           [공개]             │
│                                            │
│   ※ P와 G로 d를 역산하는 것은 불가능!       │
│      (이산대수 문제)                        │
└────────────────────────────────────────────┘
```

| 구분 | 요소 | 설명 |
|:-----|:----|:-----|
| 공개 | G (기본점) | 타원곡선 위 특정 고정된 점 |
| 비밀 | d (개인키) | 임의의 비밀 정수, 난수로 생성 |
| 공개 | P (공개키) | P = d × G, Point Doubling으로 생성 |

**보안 원리**
- d와 G로 P 계산: **쉬움** (순방향)
- P와 G로 d 역산: **불가능** (역방향, ECDLP)

---

### ⚔️ ECC vs RSA 비교

| 구분 | ECC | RSA |
|:-----|:----|:----|
| 기반 구조 | WPKI (무선) | PKI |
| 수학적 기반 | 타원곡선 이산대수 | 큰 수의 소인수분해 |
| 키 크기 (동등 보안) | 256bit | 3072bit |
| 속도 | 빠름 | 느림 |
| 적용 분야 | 모바일, IoT, 블록체인 | 기존 인프라 환경 |
| 알고리즘 | ECDH, ECDSA | RSA, DSA |

---

### 📊 주요 알고리즘

| 알고리즘 | 용도 | 설명 |
|:--------|:-----|:-----|
| **ECDH** | 키 교환 | ECC 기반 Diffie-Hellman |
| **ECDSA** | 디지털 서명 | ECC 기반 전자서명 (비트코인 사용) |
| **ECIES** | 암호화 | ECC 기반 통합 암호화 체계 |

---

### 🔮 발전 방향: 아이소제니(Isogeny) 기반 암호

| 구분 | 내용 |
|:-----|:-----|
| 개념 | 서로 다른 두 타원곡선 간의 구조적 관계를 정의하는 함수 |
| 특징 | ECC보다 더 복잡한 수학 구조 |
| 목적 | 양자 컴퓨터 내성 암호 (Post-Quantum) |

### 📈 상위 토픽 계층도

```
암호화 알고리즘
├── 대칭키 암호
│   ├── AES, DES
│   └── ARIA, SEED
└── 비대칭키 암호 (공개키)

```
