### 🔑 핵심 키워드 3개
- **의도(Intent)**: "이렇게 해줘"라고 말하면 자동으로 설정
- **자연어 처리(NLP)**: 사람 말을 컴퓨터가 이해
- **폐쇄 루프(Closed-Loop)**: 계속 확인하고 자동 조정

### 📖 등장배경

> 5G와 IoT 시대가 되면서 네트워크에 연결되는 기기가 **폭발적으로 증가**했어요. 수천, 수만 개의 장비를 사람이 일일이 설정하기엔 너무 복잡하고 실수도 많이 생겨요. 그래서 **"내 의도만 말하면 AI가 알아서 네트워크를 설정해주는"** 기술이 필요해졌어요!

### 📝 개념 정의

> IBN은 **"네트워크야, 이렇게 동작해줘"라고 의도만 말하면, AI가 알아서 설정하고 계속 확인해주는 똑똑한 네트워크 기술**이에요.
>
> 비유하자면, **"에어컨아, 시원하게 해줘"라고 말하면 알아서 온도를 조절하는 것**과 같아요! 일일이 "25도로 맞춰줘"라고 말할 필요 없이, 내 의도(시원하게)만 전달하면 돼요.

### 🏗️ 기술요소 (2그룹)

**그룹 1: 의도 파악 기술**

| 기술 | 쉬운 설명 | 역할 |
|:-----|:---------|:-----|
| NLP | 사람 말을 컴퓨터가 이해 | 자연어로 된 정책/요구사항 분석 |
| LLM | ChatGPT 같은 대규모 언어 모델 | 의도를 더 정확하게 파악 |
| Embedding | 언어를 숫자로 변환 | 언어의 유사도 분석 가능 |

**그룹 2: 네트워크 자동화 기술**

| 기술 | 쉬운 설명 | 역할 |
|:-----|:---------|:-----|
| SDN | 소프트웨어로 네트워크 제어 | 중앙에서 네트워크 설정 변경 |
| Closed-Loop | 계속 확인하고 자동 조정 | 의도대로 동작하는지 검증 |
| Automation | 자동 설정 | 수동 작업 없이 설정 적용 |

### ⭐ 차별점 키워드

> **Closed-Loop Intent Control**: 의도를 설정하고 → 모니터링하고 → 검증하고 → 피드백하는 **폐쇄 루프 자동화**!

### 📋 6하원칙 요약

| 구분 | 내용 |
|:-----|:-----|
| **누가** | 네트워크 관리자, NMS(Network Management System) |
| **무엇을** | 네트워크 정책, 설정, 구성 |
| **언제** | 5G/IoT 시대, 대규모 네트워크 관리 필요 시 |
| **어디서** | 기업 네트워크, 데이터센터, 클라우드 환경 |
| **왜** | 복잡한 수동 관리의 어려움, 오동작 위험성 감소 |
| **어떻게** | 자연어 의도 → AI 분석 → 자동 설정 → 모니터링 → 피드백 |

## 🎯 기술사 수준 설명
{: #-기술사-수준-설명}

{: .highlight }
> **IBN(Intent-Based Networking)**: 의도 기반, 자동화, 폐쇄 루프
> - (등장배경) `5N` (5G 서비스 상용화, NW 관리 복잡성)
> - (매커니즘) `명번적모검` (의도 명령, 번역, 적용, 모니터링, 검정)
> - (기술요소) `NLES` + `SOP로캐` + `모VFC`
> - (참조모델) `RFC-ZSM-NW`

---

### 📝 개념 정의

**자연어 기반**으로 표현된 네트워크 관리 정책이나 서비스 요구사항을 **NLP 기술**을 활용해 분석하고 사용자 **의도(Intent)**를 파악해 최적의 네트워크 설정 파라미터를 도출, 적용하는 네트워크 관리 기술

| 구분 | 설명 |
|:-----|:-----|
| **정의** | 네트워크 관리자의 의도(Intent)를 AI 기술로 파악하여 유선망과 무선망 설정을 **자동으로 수행**하는 네트워크 기술 (30자 이내) |
| **목적** | 복잡한 네트워크의 자동화된 관리 및 오류 최소화 |
| **표준** | IETF NMRG(Network Management Research Group)에서 정립한 **RFC 9315** 기반 기술 |

---

### 🔄 등장배경 `5N`

| 배경 | 설명 |
|:-----|:-----|
| **5**G 서비스 상용화 | 다양한 형태의 IoT 기기 등장으로 인한 디바이스 수 기하급수적 증가, 서비스 대역폭 증가와 네트워크 디바이스 수 증가에 따른 데이터 양 크게 증가 중 |
| **N**W 관리 복잡성 | 복잡한 NW의 수동적인 관리의 복잡성, 어려움, 오동작의 위험성 증가, 네트워크 복잡성, 운영 어려움으로 인해 관리자/운영자 수동 관리 방식에서 자동화 관리의 필요성이 크게 증가 |

### 🔄 IBN 매커니즘 (Closed-Loop Intent Control)

```
┌─────────────────────────────────────────────────────────────────┐
│               Closed-Loop Intent Control                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│       ┌───────┐     ┌───────┐                                   │
│       │ User  │     │  NMS  │                                   │
│       └───┬───┘     └───┬───┘                                   │
│           │             │                                        │
│           └──────┬──────┘                                        │
│                  │ Intent (의도)                                 │
│                  ▼                                               │
│   High-level ┌─────────────────┐                                │
│   Policy     │  Translation/   │◀──────── Feedback              │
│       │      │  Optimization   │                                │
│       │      └────────┬────────┘                                │
│       ▼               │                                          │
│  ┌──────────────┐     │     ┌──────────────┐                    │
│  │ Activation/  │     │     │ Monitoring/  │                    │
│  │Configuration │◀────┴────▶│  Assurance   │                    │
│  └──────┬───────┘           └──────┬───────┘                    │
│         │ Low-level                │ Monitoring                  │
│         │ Policy                   │ Data                        │
│         ▼                          ▼                             │
│  ┌─────────────────────────────────────────┐                    │
│  │           Infrastructure                 │                    │
│  └─────────────────────────────────────────┘                    │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

#### 매커니즘 동작절차 `명번적모검`

| 구분 | 기술 요소 | 설명 |
|:-----|:---------|:-----|
| 의도 **명**령 (Intent command) | NMS, User | 네트워크 관리 시스템(NMS), 네트워크 관리자(Admin, Client), 원하는 서비스의 목적 위한 **선언적인 명령** |
| 의도 **번**역 (Intent translation) | Translation, Optimization | **High-level Policy**, 음성/텍스트 형태의 의도 번역 |
| | LLM | 거대 언어 모델(Large Language Model), 컴퓨터 스크립트 형태 **자동 번역** 목적 |
| 의도 **적**용 (Intent deployment) | Activation, Configuration | **Low-level Policy**, 인프라 요청된 네트워크 서비스 수행 목적 |
| | Virtual Machine, Container | 가상화 시스템 활성화, 네트워크 변경 의도 적용 준비 및 수행 |
| 의도 **모**니터링 (Intent monitoring) | Monitoring | 네트워크 모니터링 데이터 수집 |
| | Assurance | 데이터 분석 및 정합성 검증 |
| 의도 **검**정 및 재설정 (Intent validation & reconfiguration) | Validation | 네트워크 서비스 의도대로 동작 여부 검정 |
| | Feedback | 정책 보완 및 새로운 정책 생성하여 피드백 |

> 모니터링과 피드백 기반의 재설정 과정 필요한 **폐쇄 루프 의도 제어(Closed-Loop Intent Control)** 기술 형태

---

### 🏗️ 기술요소

#### Intent Engine `NLES`

| 기술요소 | 설명 |
|:--------|:-----|
| **N**LP | 자연어 기반 정책/요구사항 분석 기술 |
| **L**LM | 거대 언어 모델 기반으로 자연어 기반 의도 파악 정확도 향상 |
| **E**mbedding | 자연어 벡터 데이터 변환, 자연어 유사도 분석 가능 |
| 강화 학**습** | 네트워크 부하 분산 기술 |

#### 네트워크 설정 기술 `SOP로캐`

| 기술요소 | 설명 |
|:--------|:-----|
| **S**DN | Software Defined Network, H/W, S/W 분리를 통한 네트워크 관리 효율 증가 기술 |
| **O**penFlow | SDN의 주소 테이블 관리, 패킷 관리 |
| **P**4 | Programming Protocol-independent Packet Processor, 스위치/라우터의 포워딩 테이블 수정, 신규 패킷 헤더 처리 |
| **로**드 밸런싱 | 네트워크 부하 분산 기술 |
| **캐**싱 | 사용 빈도 높은 데이터 저장/사용, 저지연 네트워크 구현 |

#### NW State Data Store / Validation of Intent `모VFC`

| 기술요소 | 설명 |
|:--------|:-----|
| **모**니터링 | 네트워크 상태 모니터링, 이상(Anomaly) 감지 수행 |
| **V**alidation | 사용자 의도대로 네트워크 설정 여부 검증/확인 |
| **F**eedback | 인공지능 모델 파라미터 재설정에 활용 |
| **C**losed Loop | 폐쇄 제어 루프 통한 사용자 의도 보장 |

### 📊 네트워크 자동화를 위한 참조모델 `RFC-ZSM-NW`

| 단체명 | 참조모델 | 설명 |
|:-------|:--------|:-----|
| IETF | **RFC** 9315 | NMRG(Network Management Research Group)에서 정립 |
| ETSI | **ZSM** | ETSI ZSM 표준화 그룹의 차세대 관리 참조 모델 |
| 3GPP | **NW**DAF | 3GPP에서 네트워크 자동화 위한 참조 모델 구성 |

> 5G, 6G와 인공지능 기술 발전 따라 자동화된 네트워크 기술 활용도 증가 예상

---

### ⭐ 차별점 키워드

> **Closed-Loop Intent Control**: 의도 명령 → 번역 → 적용 → 모니터링 → 검정 → 피드백의 **폐쇄 루프 자동화**로, 네트워크가 사용자 의도대로 동작하는지 **지속적으로 검증하고 자동 조정**하는 핵심 메커니즘!
