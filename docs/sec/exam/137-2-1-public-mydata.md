## 1. 공공 마이데이터 서비스의 개념 및 구성요소 `정보본제`

{: .highlight }
> **공공 마이데이터**: 정보주체 중심, 동의 기반, 안전한 제공·활용 → 공공 데이터 활용 서비스
> - (개념) `전자정부법 제43조의 2`
> - (구성요소) `정보본제`
> - (등장배경) `데분디개`

### 가. 개념 정의

**전자정부법 제43조의 2**에 근거한 공공 마이데이터 서비스는, 정보주체의 **제공요구권**에 따라 보유기관이 **개인 행정정보**를 정보주체 본인 또는 지정한 제3자에게 제공하는 서비스

| 구분 | 설명 |
|:-----|:-----|
| **법적 근거** | 전자정부법 제43조의 2 |
| **핵심 권리** | 정보주체의 제공요구권 |
| **제공 대상** | 정보주체 본인 또는 지정한 제3자 |
| **제공 정보** | 개인 행정정보 (주민등록, 건강보험, 고용보험, 세금, 복지, 자격 등) |

### 나. 등장배경

| 배경 | 설명 |
|:-----|:-----|
| **데이터 주권 강화 요구** | 최근 데이터 주권 강화 요구 증가 |
| **분산 행정정보의 비효율** | 여러 기관에 분산된 행정정보의 비효율적 관리 |
| **디지털 전환 가속화** | 디지털 전환 가속화에 따른 맞춤형 서비스 필요 |
| **개인정보 보호 필요성** | 개인정보 보호 필요성에서 비롯하여 공공 마이데이터 시장 활성화를 위한 활용 방안 모색 필요 |

### 다. 공공 마이데이터 서비스 흐름도

```
┌─────────────────┐
│  국민 (정보주체)  │
└────────┬─────────┘
         │
         │ ① 행정정보 제공 요구
         │
         ▼
┌─────────────────────────┐
│ 국민 행정정보 보유 기관    │
│ (본인정보 보유기관)        │
└────────┬─────────────────┘
         │
         ├─── ② 행정정보 제공 ────┐
         │                        │
         │                        ▼
         │              ┌─────────────────────────┐
         │              │ 행정정보 활용 서비스     │
         │              │ 제공 기관 (제공기관)     │
         │              └─────────────────────────┘
         │
         └─── ② 행정정보 제공 ────▶ 국민 (정보주체)
```

### 라. 구성요소 `정보본제`

| 구성요소 | 설명 | 세부 내용 |
|:---------|:-----|:----------|
| **정**보주체 (개인, 국민) | 데이터의 주권을 가진 당사자 | 본인의 동의에 따라 행정·공공 데이터를 제공·이용할 수 있도록 통제 |
| **보**유기관 (공공데이터 제공자) | 국민의 행정정보를 보관·관리하는 기관 | 법령 근거에 따라 본인 동의 시 데이터를 제공 |
| **제**공기관 (데이터 활용·서비스 기관) | 정보주체의 동의에 따라 데이터를 전달받아 서비스 제공 | 행정 서비스(맞춤형 복지·민원 자동완성)뿐 아니라 민간 서비스(금융, 취업, 헬스케어 등)로 확대 가능 |

**핵심 구조**: 정보주체(개인)가 본인정보 보유기관(공공기관)에 동의하면, 그 데이터가 제공기관(서비스 기관)으로 안전하게 전송·활용되는 구조

## 2. 행정 서비스 측면 공공 마이데이터 활용 방안 `맞행통취생`

{: .highlight }
> **행정 서비스 활용**: 맞춤형, 간소화, 통합
> - (생애주기) `전청청장노`
> - (활용방안) `맞행통취생`

### 가. 생애주기 맞춤형 공공 마이데이터 활용 방안

| 생애주기 | 활용 방안 |
|:---------|:----------|
| **전세대** | 의료기관 간 의료정보 전송, 통신요금 합리적 관리 |
| **청소년기 (10대)** | 맞춤형 학습 및 대입 멘토링, 병원 치료 및 봉사활동 증명 |
| **청년기 (20,30대)** | 맞춤형 일자리 추천, 청년 맞춤형 주거정보 안내 |
| **장년기 (40,50대)** | 만성질환 관리, 에너지 사용량 관리 |
| **노년기 (60대 이상)** | 독거노인 응급 알림, 장기요양 내역 자녀에게 알림 |

### 나. 행정 서비스 편의 및 통합을 위한 활용 방안 `맞행통취`

| 구분 | 주요 활용 내용 | 기대 효과 |
|:-----|:--------------|:----------|
| **맞**춤형 복지 제공 | 소득·가족·건강 데이터를 기반으로 복지 자격 자동 판별, 필요한 서비스 선제 제공 | 사각지대 해소, 국민 체감도 향상 |
| **행**정 절차 간소화 | 정부24 등에서 민원 서류 자동 작성·제출, 증빙자료 자동 연계 | 국민 불편 감소, 행정 효율성 제고 |
| **통**합 조회 서비스 | 세금·연금·건강·자격증 정보를 원스톱으로 제공 | 국민 편의성 증대, 데이터 접근성 강화 |
| **취**약계층 지원 강화 | 고령자·장애인·저소득층에 맞춤형 지원 정책 설계 및 연계 | 사회안전망 강화, 포용적 행정 실현 |
| **생**애주기 맞춤 행정 | 출산·이사·은퇴 등 이벤트 발생 시 필요한 공공서비스 패키지 제공 | 개인별 맞춤형 서비스, 정책 만족도 향상 |

## 4. 효과적인 공공 마이데이터 활용방안을 위한 고려사항 `법기이정`

{: .highlight }
> **고려사항**: 법제도, 기술보안, 이용자 중심, 정책운영
> - (고려사항) `법기이정`

### 가. 법·제도적 측면

| 고려사항 | 설명 |
|:---------|:-----|
| **법적 정합성 확보** | 기존 법률(개인정보보호법, 전자정부법, 신용정보법)과의 충돌 최소화, 향후 마이데이터 기본법과의 정합성 확보로 안정적 법적 근거 마련 |
| **제도적 기반 강화** | 국민의 '데이터 이식권'을 명확히 정의하여 자유로운 데이터 이동·제공 가능, 간편하고 투명한 동의 관리 제도적 메커니즘 구축 |

### 나. 기술·보안적 측면

| 고려사항 | 설명 |
|:---------|:-----|
| **데이터 표준화·상호운용성** | 기관별 상이한 데이터 구조·형식을 표준화, API 기반 상호운용성 확보로 공공-민간 데이터 연계 원활화 |
| **보안·신뢰성 강화** | 데이터 전송·저장 시 고도화된 보안 기술(암호화, 접근통제, 변조방지, 로그 감사) 적용, ISMS-P 등 보안 인증제도를 통한 신뢰성 확보 |

### 다. 이용자 중심 측면

| 고려사항 | 설명 |
|:---------|:-----|
| **이용자 친화적 UX 제공** | 모든 국민이 쉽게 이해하고 사용할 수 있는 직관적 UI/UX 설계, 복잡한 동의·조회·전송 과정을 단순화하여 이용자 편의성 극대화 |

### 라. 정책·운영적 측면

| 고려사항 | 설명 |
