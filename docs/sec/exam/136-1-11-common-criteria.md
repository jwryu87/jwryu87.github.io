### 🔑 핵심 키워드 3개
- **보안 인증**: IT 제품이 안전한지 검사받는 것
- **EAL 등급**: 보안 수준을 1~7로 나눈 것
- **국제 표준**: 전 세계가 같은 기준으로 평가

### 📖 등장배경

> 나라마다 보안 제품을 평가하는 기준이 달랐어요. A나라에서 합격한 제품이 B나라에서는 다시 검사받아야 했죠. 그래서 전 세계가 **"같은 기준으로 평가하고, 서로 인정하자!"**고 만든 게 CC(공통평가기준)예요. 한 번 인증받으면 여러 나라에서 인정받을 수 있어요!

### 📝 개념 정의

> CC는 **"IT 보안 제품의 안전성을 평가하는 국제 표준"**이에요.
> 
> 비유하자면, 자동차 **안전 등급**처럼 보안 제품에도 **1~7등급**을 매기는 거예요!

### 🏗️ 기술요소 (2그룹)

**그룹 1: 핵심 용어**

| 용어 | 의미 | 쉬운 설명 |
|:-----|:-----|:---------|
| TOE | 평가 대상 | 검사받을 제품 |
| PP | 보호 프로파일 | 공통 평가 기준 |
| ST | 보안 목표 명세서 | 제품별 평가 기준 |
| EAL | 평가 보증 등급 | 보안 수준 1~7 |

**그룹 2: EAL 등급**

| 등급 | 수준 | 쉬운 설명 |
|:-----|:-----|:---------|
| EAL1 | 기능 시험 | 기본 테스트 |
| EAL4 | 체계적 설계 | 일반 상용 제품 수준 |
| EAL7 | 정형 검증 | 최고 보안 수준 |

### ⭐ 차별점 키워드

> **CCRA**: CC 인증을 국가간 상호 인정하는 협정!

### 📋 6하원칙 요약

| 구분 | 내용 |
|:-----|:-----|
| **누가** | 과기정통부(정책), ITSCC(인증), KISA 등(평가) |
| **무엇을** | IT 보안 제품의 보안성 |
| **언제** | 보안 제품 출시 전 인증 |
| **어디서** | 인증된 평가기관 |
| **왜** | 국제 상호인정, 신뢰성 확보 |
| **어떻게** | ISO/IEC 15408 기준으로 EAL 등급 평가 |

## 🎯 기술사 수준 설명
{: #-기술사-수준-설명}

{: .highlight }
> **CC(Common Criteria)**: ISO/IEC 15408, CCRA, EAL
> - (구성) `소기보` (Part1 소개, Part2 기능, Part3 보증)
> - (용어) `TOE-PP-ST-EAL`
> - (국내체계) `정인평` (정책-인증-평가)

---

### 📝 개념 정의

정보보호 제품의 보안성을 평가하는 국제표준의 중요성이 대두됨에 따라,
IT제품 및 시스템의 **보안성평가 수준을 표준화, 평가결과의 국가간 상호인정(CCRA)을 위해 단일화한 국제표준이자 인증 제도**

| 구분 | 설명 |
|:-----|:-----|
| 관련 표준 | ISO/IEC 15408 (Common Criteria) |
| 관련 법령 | 지능정보화 기본법 제58조 (정보보호시스템에 관한 기준 고시 등) |
| 특징 | EAL 등급제(1~7), CCRA 회원국간 상호 평가 인정 |

---

### 🏗️ CC 구성요소 (3 Part) `소기보`

| Part | 명칭 | 핵심요소 | 특징 |
|:-----|:-----|:--------|:-----|
| Part1 | **소**개 및 일반 모델 | TOE, PP, ST 정의 | 평가대상, 보호프로파일, 보안목표 명세 |
| Part2 | 보안 **기**능 요구사항 (SFR) | 기능 구성요소 카탈로그 | IT 제품이 제공해야 할 보안 기능 표준화 |
| Part3 | 보안 **보**증 요구사항 (SAR) | 보증 구성요소, EAL | 보안 기능의 신뢰성 평가 요구사항 |

---

### 📊 핵심 용어 `TOE-PP-ST-EAL`

| 용어 | 풀네임 | 설명 |
|:-----|:------|:-----|
| **TOE** | Target of Evaluation | 인증·평가의 직접적인 대상인 보안 제품 |
| **PP** | Protection Profile | 독립적이고 여러 제품에 적용가능한 공통 평가 기준 |
| **ST** | Security Target | 특정 제품의 보안 기능을 표현한 평가 기준 자료 |
| **EAL** | Evaluation Assurance Level | TOE가 얼마나 정형화되었는지 평가하는 등급 수준 |
| **SFR** | Security Functional Requirement | 제품 구현 시 필요한 보안 기능 요구사항 |
| **SAR** | Security Assurance Requirement | 등급에 부합함을 증명하는 보증 요구사항 |

---

### 📈 평가보증등급 (EAL 1~7) `기구체체준준정`

| 등급 | 등급설명 | 주요 산출물 |
|:-----|:--------|:-----------|
| EAL1 | **기**능적인 시험 | 기능명세서, 설명서 |
| EAL2 | **구**조적인 시험 | 기본 설계서, 기능 시험서 |
| EAL3 | **체**계적인 시험 및 검사 | 생명주기 문서, 개발보안문서, 오용 분석서 |
| EAL4 | **체**계적인 설계·시험·검토 | 상세 설계서, 보안 정책서 |
| EAL5 | **준**정형화된 설계·시험 | 보안기능 전체 코드, 준정형적 분석서 |
| EAL6 | **준**정형화된 설계 검증·시험 | 전체 소스 코드, 정형 검증 보고서 |
| EAL7 | **정**형화된 설계 검증·시험 | 정형적으로 검증된 설계 명세 |

---

### 🏛️ 국내 CC 체계 `정인평`

```
┌─────────────┐
│   정책기관   │ ← 과학기술정보통신부
└──────┬──────┘
       │ 평가·인증 정책
       ▼
┌─────────────┐     ┌─────────────┐
│   인증기관   │◀───│   평가기관   │
│   (ITSCC)   │     │ KISA, TTA 등│
└─────────────┘     └─────────────┘
       │                   │
       ▼                   ▼
  인증서 발급          평가 수행
```

| 구분 | 기관 | 역할 |
|:-----|:-----|:-----|
| **정**책기관 | 과학기술정보통신부 | 평가·인증 정책 수립 |
| **인**증기관 | 국가보안기술연구소(NSR) IT보안인증사무국(ITSCC) | 인증서 발급, 평가기관 관리 |
| **평**가기관 | KISA, KoSyAs, KSEL, TTA, KTC 등 | 실제 평가 수행 |

---

### 🔄 CC 인증 흐름

```
PP(기준) → ST(평가기준자료) 작성 → TOE(제품) 구현 → EAL 등급 획득
(공통기준)   (제품별 기준)         (평가대상)      (인증등급)
```

---

### ⭐ 차별점 키워드

> **CCRA (Common Criteria Recognition Arrangement)**
> - CC 인증결과의 국가간 상호인정 협정
> - 한 번 인증받으면 회원국 모두에서 인정
> - 2014년부터 IT보안인증사무국(국가보안기술연구소)에서 운영
