## 1. OWASP LLM Top 10이 제시된 배경과 주요 특징

{: .highlight }
> **OWASP LLM Top 10 배경**: 기술 확산, 공격 표면 변화, 정보 유출 가능성
> - (배경) `기공정` (기술 확산, 공격 표면 변화, 정보 유출 가능성)
> - (특징) `LLM특화`, `Top10`, `커뮤니티`

### 가. OWASP LLM Top 10이 제시된 배경 `기공정`

| 구분 | 배경 | 설명 |
|:-----|:-----|:-----|
| **기**술의 확산 | LLM 서비스 대중화 | ChatGPT 등 LLM 기반 서비스의 대중화 및 활용 서비스 증가 |
| **공**격 표면의 변화 | 기존 보안 한계 | 전통적 웹 보안 모델은 자연어 프롬프트, 응답 기반 행동 설계 취약사항에 대응하는데 한계가 존재 |
| | 신종 위협 등장 | 입력 프롬프트를 조작하거나 출력 결과를 악용하는 Prompt Injection, 응답 조작 등의 신종 위협 등장 |
| **정**보 유출 가능성 | 데이터 보호 필요 | 훈련 데이터 민감정보나 입력된 개인 정보에 대한 보호 필요 |

> ChatGPT, Bard 등 LLM 기반 서비스 확산에 따라 기존 OWASP Top 10만으로는 대응할 수 없는 **AI 고유의 보안 위협**이 부각됨. 특히 프롬프트 인젝션, 민감정보 노출, 학습 데이터 조작 등 LLM 구조 특화된 취약점에 대응하기 위해 **2023년 OWASP에서 별도 LLM Top 10**을 제정

### 나. OWASP LLM Top 10 주요 특징

| 구분 | 특징 | 설명 |
|:-----|:-----|:-----|
| **구조적 측면** | LLM 특화 보안 | LLM 시스템 고유 입력/출력/학습 구조 반영 제시 |
| | Top 10 포맷 유지 | 기존 OWASP TOP 10 위협 가이드라인 구조 유지 |
| | 커뮤니티 기반 | 학계, 보안 전문가 참여 위해 공개 기반의 가이드라인 |
| | 구조적 구성 | Top 10 위협 유형 체계화, 위협을 명확히 10가지 유형으로 분류하고 각 항목에 대해 개념, 위협, 대응방안 제공 |
| | 광범위 적용성 | 챗봇, API, 플러그인 연동형 서비스, 검색/요약/분류 등 생성형 AI 기반 시스템 전반에 적용 가능 |
| **사용성 측면** | 프롬프트 보안 | 프롬프트 입력/출력 제어에 대한 명확한 대응방안 제시 |
| | 시나리오 중심 | 각 시나리오에 대한 공격 예시 및 영향 제시 |
| | 기존 보안 통합기능 | 기존 웹/모바일/클라우드 보안 체계 통합 연계성 고려 |
| | 목적 지향성 | LLM 보안 위협 특화 지침, 기존 OWASP Top 10과 달리 LLM 기반 시스템에서 발생하는 특수한 위협을 식별하고 대응하기 위해 별도 정의됨 |
| | 실무 중심성 | 개발·운영자가 적용 가능한 기준 제시, 단순 이론적 위협이 아닌 실제 응용 시스템에서 활용 가능한 보안 설계 및 운영 체크리스트 제공 |
| | 최신성 | 2023년 이후 신기술·공격 트렌드 반영, 프롬프트 인젝션, 출력 조작, 학습 데이터 오염 등 최근 공격 유형 포함 |
| | 주기 대응 | 설계·개발·운영 전 단계 고려, 위협 분석뿐 아니라 설계, 테스트, 배포, 운영 단계별 대응 방안을 포함함 |

> OWASP TOP 10 for LLM에는 기존 OWASP 10대 위협과 견주어 인공지능 애플리케이션을 개발하거나 사용할 때 염두에 두어야 할 주요 보안 이슈에 대한 가이드가 제시됨

## 2. OWASP LLM에서 제시된 주요 보안 위협

{: .highlight }
> **주요 보안 위협**: 입출력 위협, 모델 위협, 시스템 위협
> - (위협 10가지) `인민공 대출과 프벡정무`
> - (LLM01~05) `인민공대출` (인젝션, 민감정보, 공급망, 데이터중독, 출력처리)
> - (LLM06~10) `과프벡정무` (과도한대행, 프롬프트노출, 벡터취약점, 잘못된정보, 무제한소비)

### 가. OWASP LLM Top 10 보안 위협 `인민공 대출과 프벡정무`

| NO | 보안 취약점 | 설명 |
|:---|:-----------|:-----|
| **LLM01** | 프롬프트 **인**젝션 (Prompt Injections) | 사용자 입력(프롬프트)을 악의적으로 조작하여 LLM의 행동이나 출력 결과를 의도와 다르게 변경하는 취약점 |
| **LLM02** | **민**감한 정보 공개 (Sensitive Information Disclosure) | LLM이 민감한 개인 정보, 기밀 데이터 또는 독점 알고리즘 정보를 의도치 않게 노출하는 취약점 |
| **LLM03** | **공**급망 취약점 (Supply Chain Vulnerabilities) | LLM 개발 및 운영에 사용되는 서드파티 구성 요소, 데이터셋, 및 사전 학습 모델에서 발생하는 공급망 취약점 |
| **LLM04** | **데**이터 및 모델 중독 (Data and Model Poisoning) | 학습 데이터나 모델 파라미터를 악의적으로 변조하여 취약점을 주입하는 공격 |
| **LLM05** | 부적절한 **출**력 처리 (Improper Output Handling) | LLM의 출력이 충분히 검증, 정제, Sandboxing 되지 않을 경우 발생하는 문제 |
| **LLM06** | **과**도한 대행 (Excessive Agency) | LLM이 지나치게 자율적인 행동을 수행하도록 허용. 인간의 직접적인 통제 없이 예기치 않은 결과나 악의적 행동 발생 |
| **LLM07** | 시스템 **프**롬프트 노출 (System Prompt Leakage) | LLM이 내부 지시사항이나 운영 설정 정보를 의도치 않게 외부에 공개하는 취약점 |
| **LLM08** | **벡**터와 임베딩 취약점 (Vector and Embedding Weaknesses) | RAG 시스템에서 사용되는 벡터 표현 및 임베딩 기법의 결함으로 인한 문제. 부정확한 검색 결과, 조작된 문맥, 또는 민감 데이터 노출 발생 |
| **LLM09** | 잘못된 **정**보 (Misinformation) | LLM이 사실과 다른, 또는 왜곡된 정보를 생성하여 잘못된 결정을 유도하는 취약점. 환각(hallucination) 및 학습 데이터의 편향 등이 주요 원인으로 작용하며, 법적, 평판, 안전 문제를 야기 |
| **LLM10** | **무**제한 소비 (Unbounded Consumption) | LLM이 과도하고 통제되지 않은 요청을 처리함으로써 시스템 자원(메모리, CPU, 비용 등)이 고갈되는 취약점 |

> LLM 기반 앱은 데이터 및 사용자 입력 처리 방식의 독특한 특성으로 인해 기존 IT 시스템과 다른 보안 위협에 노출됨

### 나. 보안 위협 분류

| 분류 | 위협 항목 | 설명 |
|:-----|:---------|:-----|
| **입출력 위협** | LLM01, LLM02, LLM05 | 프롬프트 및 응답 처리 과정에서 발생하는 위협 |
| **모델 위협** | LLM04, LLM08, LLM09 | 학습 데이터, 모델 구조 관련 위협 |
| **시스템 위협** | LLM03, LLM06, LLM07, LLM10 | 인프라, 권한, 공급망 관련 위협 |

## 3. LLM 애플리케이션 설계, 운영 시 OWASP LLM 활용한 보안 대응방안

{: .highlight }
> **보안 대응방안**: 설계 단계, 운영 단계
> - (설계) `입출외학출핵플실사민토`
> - (운영) `악응구재위명내벡사요`

### 가. 설계 측면의 OWASP LLM 활용한 보안 대응방안

| 보안 취약점 | 설계할 때 대응방안 | 설명 |
|:-----------|:------------------|:-----|
| **프롬프트 인젝션** | 입력 필터링 및 정규화 구조 설계 | 위험 키워드 제거, 시스템 명령과 사용자 명령 분리 설계 |
| **민감한 정보 공개** | 출력 마스킹 및 데이터 분류 체계 설계 | 민감 정보 탐지 후 자동 치환 또는 제거 구조 반영 |
| **공급망 취약점** | 외부 구성요소 무결성 검증 및 SBOM 관리 | 모델, 플러그인, 데이터 출처에 대한 검증 프로세스 포함 |
| **데이터 및 모델 중독** | 학습 데이터 수집·전처리 기준 수립 | 신뢰 가능한 출처에서 수집하고 변조 여부 검증 설계 포함 |
| **부적절한 출력 처리** | 출력 검증 정책 설계 및 허용 응답 포맷 정의 | 불필요하거나 위험한 응답 유형 차단 규칙 사전 정의 |
| **과도한 대행** | 모델 실행 권한 최소화 및 명령어 화이트리스트 설정 | LLM이 외부 시스템을 제어할 수 없도록 제한 설계 |
| **시스템 프롬프트 노출** | 시스템 프롬프트 비공개 설계 및 프롬프트 분리 구조 | 내부 지시문이 유출되지 않도록 프롬프트 영역을 분리 구성 |
| **벡터와 임베딩 취약점** | 민감한 벡터 저장 제한 및 안전한 인덱스 설계 | 벡터 DB에 저장되는 정보에 대한 접근 제어 포함 |
| **잘못된 정보** | 모델 응답 출처 명시 및 검증 기능 설계 | 모델의 주관적/허위 응답에 대해 출처 정보 함께 출력 |
| **무제한 소비** | 토큰 수/요청 크기 제한 구조 설계 | 프롬프트 크기, 요청 속도, 응답 길이에 대한 제한 설정 |

### 나. 운영 측면의 OWASP LLM 활용한 보안 대응방안

| 보안 취약점 | 운영할 때 대응방안 | 설명 |
|:-----------|:------------------|:-----|
| **프롬프트 인젝션** | 악성 입력 탐지 로직 운영 및 경고 시스템 연계 | 입력 로그를 기반으로 프롬프트 인젝션 패턴 탐지 및 차단 |
| **민감한 정보 공개** | 응답 감시 및 PII 검출 시스템 연동 | 출력값에서 개인정보 또는 기밀 데이터 자동 검출 후 마스킹 |
| **공급망 취약점** | 구성요소 취약점 스캐닝 및 자동 업데이트 | 플러그인/라이브러리 버전 및 보안 이슈 실시간 감시 |
| **데이터 및 모델 중독** | 재학습 시 이상 패턴 탐지 및 품질 점검 | 학습 후 응답 품질 저하 시 데이터 소스 확인 프로세스 가동 |
| **부적절한 출력 처리** | 위험 응답 사후 필터링 및 편집 정책 적용 | 응답 내 금칙어 탐지 시 자동 블록 또는 수정 처리 |
| **과도한 대행** | 명령 실행 로깅 및 관리자 승인 시스템 운영 | 모델의 명령 실행 기록을 분석하고 비인가 실행 차단 |
| **시스템 프롬프트 노출** | 내부 지시문 유출 감시 및 응답 로그 분석 | 응답에 시스템 프롬프트 노출 여부 실시간 탐지 및 경고 |
| **벡터와 임베딩 취약점** | 벡터 DB 접근 모니터링 및 쿼리 로그 분석 | 임베딩 기반 검색 요청의 민감도 분석 및 필터 적용 |
| **잘못된 정보** | 사용자 피드백 수집 및 응답 정정 프로세스 운영 | 잘못된 정보 제공 시 수정 요청 기능과 검토 절차 마련 |
| **무제한 소비** | 요청 수·토큰 수 실시간 제한 및 경고 시스템 | 반복 요청, 과도한 토큰 소비에 대해 속도 제한 또는 차단 처리 |

> OWASP LLM 보안 가이드라인에 대한 적용을 위해 기업에서 실무적 측면의 대비가 필요함. AI 기술의 급격한 발전에 따라 다각화된 위협에 사전, 사후 대응할 수 있는 대응 전략 필요

---

## 4. LLM의 보안 위협 대응 전략

{: .highlight }
> **대응 전략**: 안전한 서비스, 실시간 탐지, 접근제어
> - (전략) `Dev-EDR-MFA-ZT`

### 가. 대응 전략 `Dev-EDR-MFA-ZT`

| 대응전략 | 세부 전략 | 설명 |
|:---------|:---------|:-----|
| **안전한 AI 서비스 구축 및 사용** | **Dev**SecOps | 보안을 개발 프로세스에 통합하고, 보안 테스트와 정기적인 점검으로 취약점을 조기에 제거 |
| | 사전 대응책 마련 | AI 모델, 데이터, 서드 파티 소프트웨어 등 다양한 요소에서 발생 가능한 보안 위협을 분류하고 사전 대응책 마련 |
| **실시간 위협 탐지/분석/대응** | **E**ndpoint **D**etection and **R**esponse | 행위기반으로 악성코드의 이상 행위 탐지, 주기적으로 위협 정보 업데이트 |
| **안전한 자격증명관리 및 접근제어 강화** | 다중인증(**MFA**) | 자격증명의 외부 노출을 철저히 차단하고, 다중인증(MFA)을 적용해 보안을 강화 |
| | 제로 트러스트(**ZT**) | 탈취된 자격증명으로도 검증되지 않은 사용자의 접근을 차단하는 제로 트러스트 환경을 구축 |

> 변화하는 환경에 맞게 정보보안 서비스를 고도화해서 다양한 위협들에 대응할 수 있는 체계 필요

## ⭐ 차별점 키워드

### 🏆 Prompt Injection (프롬프트 인젝션)
> **LLM01: 가장 대표적인 LLM 보안 위협**
> - 사용자 입력(프롬프트)을 악의적으로 조작하여 LLM의 행동이나 출력 결과를 변경
> - 기존 SQL Injection과 유사한 개념이나 자연어 기반으로 진화
> - 대응: 입력 필터링, 시스템 명령과 사용자 명령 분리, 프롬프트 샌드박싱

### 🏆 RAG 보안 (Vector and Embedding)
> **LLM08: RAG 시스템의 벡터 DB 보안**
> - Retrieval-Augmented Generation 시스템에서 벡터 표현 및 임베딩 취약점
> - 부정확한 검색 결과, 조작된 문맥, 민감 데이터 노출 발생
> - 대응: 벡터 저장 제한, 안전한 인덱스 설계, 접근 제어
