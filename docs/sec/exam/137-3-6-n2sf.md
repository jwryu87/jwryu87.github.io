### 📌 한 줄 정의
> 업무 중요도에 따라 **기밀(C)/민감(S)/공개(O)** 등급으로 분류하고, 등급별 차등화된 보안 대책을 적용하는 **국가 망 보안 프레임워크**

### 🎭 쉬운 비유
> 공항 보안 시스템을 생각해보세요. 일반 구역은 자유롭게 이동하고(공개), 탑승 구역은 검색을 거치며(민감), 조종실은 특별 인가자만 접근합니다(기밀). N2SF도 이처럼 정보의 중요도에 따라 보안 수준을 차등 적용합니다.

### ❓ 왜 필요한가? (3줄 요약)
1. **기존 망분리 정책의 한계**: 원격근무, 클라우드, 생성형 AI 확산으로 획일적 망분리 정책에 한계 발생
2. **업무 효율성 저하**: 인터넷 단절로 인한 업무 효율 저하와 공공데이터 활용 제약
3. **보안성과 활용성 균형**: 민감 정보는 엄격히 보호하면서 일반 자료는 활용성을 강화하는 유연한 보안체계 필요

### 1. 국가 망 보안체계(N2SF)의 개념

{: .highlight }
> **N2SF 개념**: C/S/O 등급분류, 자율적 보안, 망분리 완화
> - (정의) `CSO차보`
> - (추진배경) `기정대`
> - (기대효과) `모보AI`

#### 가. N2SF의 정의 및 개념

| 구분 | 내용 | 비고 |
|:-----|:-----|:-----|
| **정의** | 정부 및 공공기관 전산망 업무 중요도에 따라<br>기밀(C), 민감(S), 공개(O) 등급 분류 후<br>차등화된 보안 대책을 적용하는 보안 프레임워크 | 2025년 1월 발표 |
| **목적** | 보안성과 데이터 활용성 동시 충족 | 망분리 정책 개선 |
| **적용대상** | 국가·공공기관 정보시스템 | 자율적·점진적 추진 |

#### 나. N2SF의 추진배경

| 구분 | 추진배경 | 내용 |
|:-----|:---------|:-----|
| **기**존 정책 운영 | 망분리 정책 시행 | 2006년부터 국가·공공기관 대상 망분리 정책 도입,<br>WannaCry 랜섬웨어 등 글로벌 위협 대응 |
| **정**책의 한계 | 업무환경 변화 | 원격근무, 클라우드, 생성형 AI 확산으로<br>망분리 환경에 한계 발생 |
| **대**응 필요성 | 개선 요구 증가 | 획일적 망분리 한계 극복,<br>업무 중요도 따라 보안 수준 차등 적용 필요 |

#### 다. C/S/O 등급분류 체계

| 등급 | 명칭 | 설명 | 보안 수준 |
|:-----|:-----|:-----|:---------|
| **C** | 기밀<br>(Classified) | 유출 시 국가 안보·국익에<br>심각한 영향을 미치는 정보 | 강력한 통제 중심 |
| **S** | 민감<br>(Sensitive) | 유출 시 개인·조직에<br>피해가 발생할 수 있는 정보 | 적정 수준 보안 |
| **O** | 공개<br>(Open) | 공개 가능하거나<br>이미 공개된 정보 | 활용 중심 |

### 2. 국가 망 보안체계(N2SF) 적용 절차

{: .highlight }
> **5단계 적용 절차**: 준비, 등급분류, 위험식별, 보안대책수립, 적절성평가
> - (5단계) `준등위보적`
> - (핵심) `자율적·사전 예방 중심`

#### 나. 5단계 적용 절차 상세

| 단계 | 절차 | 주요 활동 | 설명 |
|:-----|:-----|:---------|:-----|
| **1** | **준**비<br>(Prepare) | - N2SF 적용계획 수립<br>- 기관의 업무 기능 분석<br>- 업무정보, 정보시스템,<br>  정보서비스 식별 | 보안등급 분류 및 위협 식별을 위한<br>기초 정보와 범위 정의 단계 |
| **2** | **등**급분류<br>(Categorize) | - 업무정보 C/S/O 등급 분류<br>- 정보시스템 C/S/O 등급 분류 | 업무 및 시스템의 중요도에 따라<br>기밀(C), 민감(S), 공개(O) 등급 지정 |
| **3** | **위**험식별<br>(Identify) | - 정보서비스 구성요소 모델링<br>- '위치-주체-객체' 기반<br>  C/S/O 평가<br>- 보안원칙(정보 이동·저장) 적용 | 이질적인 보안등급 간 혼용을 탐지하여<br>보안 위협 요소를 식별 |
| **4** | **보**안대책수립<br>(Select) | - 보안통제 항목 선택<br>- 필요시 통제 조정<br>- 보안통제 구현계획 수립 | 식별된 위협에 대응하기 위한<br>적절한 보안대책 선정 및 실행 계획 수립 |
| **5** | **적**절성평가조정<br>(Assess) | - 각 단계별 결과 평가<br>- 미비점 보완 및 조정<br>- 적절성 결과 승인 및 보고 | 전 과정을 검토하여 보안체계의<br>완성도와 실효성을 확인하고 승인 |

#### 다. 적용 절차의 특징

| 특징 | 설명 |
|:-----|:-----|
| **자율적 추진** | 각 기관이 자율적·점진적으로 N2SF 적용 |
| **사전 예방 중심** | 보안 위협을 사전에 식별하고 대응 |
| **순환 프로세스** | 미비점 발견 시 보완하여 지속적 개선 |
| **국가정보원 검토** | 각 단계 산출물은 국가정보보안 기본지침에 따라 보안성 검토 시 제출 |

### 3. 국가 망 보안체계(N2SF) 적용 시 고려사항

{: .highlight }
> **고려사항**: 기술적 측면, 관리적 측면
> - (기술적) `모보` (모델링 측면-보안 측면)
> - (관리적) `프거` (프로세스 측면-거버넌스 측면)

#### 가. 기술적 측면 고려사항

| 구분 | 고려사항 | 내용 |
|:-----|:---------|:-----|
| **모**델링<br>측면 | C/S/O 등급별<br>보안통제 적용 | - 업무정보·시스템을 기밀(C), 민감(S), 공개(O)로 분류 후<br>  등급별로 차등 통제 적용<br>- 기밀은 강력 통제 중심, 공개는 활용 중심 |
| | Legacy 시스템<br>호환성 | - 기존 레거시 시스템(온프레미스, 구형 DB 등)과<br>  N2SF 연동 시 보안취약점 최소화<br>- 망연계 장비·프로토콜 보안 적용,<br>  점진적 이관 계획 수립 |
| **보**안<br>측면 | 다중요소 인증<br>·최소권한 | - 관리자/사용자 계정에 MFA 적용<br>- 최소권한 원칙 준수, 불필요 계정 자동 비활성화<br>- 접근권한 모니터링 |
| | 암호화·격리<br>·흐름제어 | - 저장·이동 중 데이터 암호화<br>- 전송 구간 보안, 프로세스 격리<br>- 네트워크 경계 통제 및 화이트리스트 기반 통신 허용 |

#### 나. 관리적 측면 고려사항

| 구분 | 고려사항 | 내용 |
|:-----|:---------|:-----|
| **프**로세스<br>측면 | 표준화된 절차<br>및 산출물 관리 | - 준비(업무/시스템 식별), C/S/O 등급분류,<br>  위험식별, 보안대책 수립, 적절성 평가의<br>  5단계 절차 운영 및 산출물 관리 |
| | 역할·책임 정의 | - 각 단계별 책임자(CISO 등) 지정<br>- 자체 심의위원회 구성해 보안대책 승인<br>  및 적절성 평가 수행 |
| **거**버넌스<br>측면 | 규정·예산·인력<br>고려 | - 보안통제 선택·구현 시 기술요건뿐 아니라<br>  법령, 지침, 예산, 인력 등 비기술적 요소도<br>  반영해 추진 |
| | 보안문화 정착 | - N2SF 각 단계의 활동을 기관 보안업무<br>  절차에 반영<br>- 교육·훈련을 통해 구성원 보안 인식 강화 |

#### 다. 성공적 정착을 위한 제언

| 구분 | 제언 |
|:-----|:-----|
| **단계적 전환** | 각 기관의 현실과 환경을 반영한 자율적·점진적 전환 |
| **보안·활용 균형** | 기밀 정보는 엄격히 보호, 공개 정보는 활용성 강화 |
| **지속적 개선** | 업무절차에 N2SF 활동을 자연스럽게 정착시켜 효과 극대화 |
| **산업 활성화** | AI·보안기술 등 디지털 산업 육성 및 활성화 기여 |
