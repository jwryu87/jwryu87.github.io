## 정의/개념

하이퍼바이저 환경에서의 **가상머신 내부정보 분석 기반 침입 탐지** 및 **클라우드 기반 보안 서비스** 기술

---

## 하이퍼바이저 기반 가상화 침입 대응 기술

### 1. VMI (Virtual Machine Introspection) - 가상머신 내부정보 분석 기반 침입 탐지

| 항목 | 내용 |
|:-----|:-----|
| **침입대응기술** | 가상머신 내부정보 분석 기반 침입 탐지 (VMI) |
| **동작원리** | Virtual H/W → VM → Command → Virtual Machine Monitor → Response |
| **설명** | 하이퍼바이저를 통하여 각 가상머신의 가상 CPU 레지스터, 가상 메모리의 내용, 파일 I/O 활동 등의 내부 정보에 대한 접근 및 분석을 통하여 가상머신의 악성행위를 탐지 |

### 아키텍처

```
┌─────────────────────────────────────────────────────────────────┐
│                    Virtual Machine Monitor                       │
│  ┌──────────┐    ┌──────────┐    ┌────────────────────────────┐ │
│  │    VM    │    │    VM    │    │   Virtual Machine Monitor  │ │
│  │          │    │          │    │   (Command / Response)     │ │
│  └──────────┘    └──────────┘    └────────────────────────────┘ │
│  ├──────────────────────────────────────────────────────────────┤ │
│  │                     Hypervisor                               │ │
│  ├──────────────────────────────────────────────────────────────┤ │
│  │                    Physical H/W                              │ │
└─────────────────────────────────────────────────────────────────┘
```

---

### 2. Agentless - 가상보안 어플라이언스

| 항목 | 내용 |
|:-----|:-----|
| **개념** | 각 가상머신 내에서 에이전트 방식으로 동작하지 않고 별도의 특별한 권한을 가진 보안 전용의 가상머신 상에서 동작하는 보안 어플라이언스 |
| **특징** | Special Authority를 가진 Security VM이 Hypervisor 위에서 동작 |

### 아키텍처

```
┌─────────────────────────────────────────────────────────────────┐
│  ┌──────────┐    ┌──────────┐    ┌──────────────────────────┐  │
│  │    VM    │    │    VM    │    │      Security VM        │  │
│  │          │    │          │    │   (Special Authority)   │  │
│  └──────────┘    └──────────┘    └──────────────────────────┘  │
│  ├──────────────────────────────────────────────────────────────┤ │
│  │                     Hypervisor                               │ │
│  ├──────────────────────────────────────────────────────────────┤ │
│  │                    Physical H/W                              │ │
└─────────────────────────────────────────────────────────────────┘
```

---

### 3. SECaaS - 클라우드 기반보안 서비스

| 항목 | 내용 |
|:-----|:-----|
| **개념** | 보안 기능을 클라우드 서비스 형태로 제공하는 것으로 클라우드 인프라 및 서비스를 위한 보안 서비스뿐만 아니라, 클라우드가 아닌 기존의 IT 환경을 위한 보안 기능 또한 클라우드 서비스화 하는 것을 포함 |
| **구성** | SECaaS가 DDoS, Spam, Virus를 Drop하고 Trusted packet만 Internal System으로 전달 |

### 아키텍처

```
┌─────────────────────────────────────────────────────────────────┐
│                         인터넷                                   │
│     ┌──────────┐                                                │
│     │  DDoS    │                                                │
│     │  Spam    │                                                │
│     │  Virus   │                                                │
│     │  Normal  │                                                │
│     │  packet  │                                                │
│     └────┬─────┘                                                │
│          ↓                                                       │
│     ┌──────────┐                                                │
│     │  SECaaS  │→ Drop (DDoS, Spam, Virus)                     │
│     └────┬─────┘                                                │
│          ↓                                                       │
│     Trusted packet                                               │
│          ↓                                                       │
│     ┌──────────────────┐                                        │
│     │ Internal System  │                                        │
│     └──────────────────┘                                        │
└─────────────────────────────────────────────────────────────────┘
```

## 기술 비교

| 구분 | VMI | Agentless | SECaaS |
|:-----|:----|:----------|:-------|
| **위치** | 하이퍼바이저 레벨 | 전용 보안 VM | 클라우드 |
| **방식** | 내부정보 분석 | 에이전트리스 | 서비스형 |
| **장점** | 깊은 수준의 분석 | 관리 용이성 | 확장성, 비용 효율 |
| **적용** | 가상머신 보안 | 가상화 환경 전체 | IT 인프라 전체 |

---

## 최신 트렌드

- 최근 클라우드에서의 보안 기술은 **SECaaS**, 즉 클라우드 서비스로서의 보안 서비스 형태로 제공될 것이 요구되고 있으며
- N/W의 복잡성을 줄이고 유연하고 동적인 제어를 가능하게 하는 **SDN(Software Defined Networking)** 기술이 클라우드 데이터 센터에 적용되고 있는 추세임
