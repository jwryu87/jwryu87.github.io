# 랜섬 디도스 (Ransom DDoS)
{: .fs-8 }

13.3 DDoS
{: .label .label-red }

---

## 핵심 키워드

`대역폭 공격` `자원 소진 공격` `Web/DB 부하 공격` `협박 메일` `금전 요구`

---

## 정의/개념

**금전 미지불 시 IT 인프라 마비 협박 DDoS 공격 기법**

- 금융사 등 공격 대상에 협박성 메일 발송
- 자신들의 협박을 믿게 하거나, 공격할 수 있는 능력이 있다는 것을 알리기 위한 과정
- 소규모, 단기간 수행 후 더 큰 공격을 하겠다고 예고 후 금전 요구
- 최근 디도스 공격 대행 서비스의 등장으로 과거보다 공격이 더 수월해짐

---

## 공격절차

### 랜섬 디도스의 공격 절차 및 기술요소

```
                     공격 대상
                  ┌─────────────┐
                  │Target Target│
1. 대상 선정 후   │Target Target│   - 금융사 등 공격 대상에 협박성 메일 발송
   협박 메일 전송 │             │   - 메일 내용: 공격자 정보, 공격 기한, 금전 요구
                  └─────────────┘
       Attacker
          │
          │        ┌─────────┐
2. 사전 공격       │ Target  │     - 자신들의 협박을 믿게 하거나, 공
   (특징: 소규모,  │         │       격할 수 있는 능력이 있다는 것을
    단기간)        └─────────┘       알리기 위한 과정
       Attacker ───▶   │           - 소규모, 단기간 수행 후 더 큰 공
          │        DDoS 공격          격을 하겠다고 예고 후 금전 요구
          │
          │
3. 본 공격        ┌─────────┐     - 실제 공격을 발생시키는 단계
                  │ Target  │     - 봇넷으로 악용 가능한 IoT 기기
       Attacker ──│         │       보급 증대로 테라급 규모의 디도
          │       └─────────┘       스 공격 가능
       DDoS 공격
    (UDP 증폭 반사 공격)
```

## 공격유형

### 랜섬 디도스의 공격 유형

| 구분 | 대역폭 공격 | 자원 소진 공격 | Web/DB 부하 공격 |
|:-----|:-----------|:--------------|:----------------|
| **사용 Protocol** | UDP, ICMP, TCP, GRE 등 | TCP | HTTP, HTTPS |
| **공격 유형** | ICMP/UDP Flooding 공격, Fragmentation 공격, UDP 기반 반사공격 (DNS, NTP, CLDAP, SSDP 등) | SYN Flooding, Flag Flooding, TCP Open Flooding 등 | 비정상적인 HTTP Request, HTTP Get Flooding등 |
| **공격 효과** | 동일 네트워크에서 사용중인 모든 시스템 위/변조 여부 | 대상서버, 보안장비, 네트워크 장비 등 과부하 발생 | 대상 Web/DB 서버 과부하 발생 |
| **IP 위/변조 여부** | 위/변조 가능 | 위/변조 가능 | 위/변조 가능 |
| **비고** | 일시적으로 대량의 트래픽을 발생 | 대역폭 공격에 비해 작은 트래픽으로도 서비스 과부하 발생 | 정상 세션을 맺은 후 과도한 HTTP 요청으로 Web/DB서버의 과부하를 유도함 |

- 기업 중 금융권은 대량의 개인(신용)정보를 보관하고 있고 서비스 가용성이 중요한 업무로 랜섬 디도스 공격의 주요 타겟이 되어 각별한 대응방안 마련이 필요함

---

## 대응방안

### 랜섬 디도스의 대응방안

| 구분 | 대응방안 | 설명 |
|:-----|:---------|:-----|
| **백업 체계 및 대응계획 수립** | 백업 체계 및 대응계획 수립 | - 데이터 안전한 백업 체계 수립<br>- 랜섬웨어 발생 시 대응 절차 및 R&R 수립 |
| **랜섬웨어 속면** | SandBox/EDR | - 행위기반 SandBox와 EDR을 이용하여 알려지지 않은 공격 및 변종 악성 코드 검사 |
| | Anti-Virus | - Anti-Virus 최신 버전 설치 |
| | 침해사고 신고 | - 랜섬웨어 침해대응센터에 즉시 신고 |
| **디도스 측면** | F/W, IPS/IDS | - 운영 장비 공격 대응을 위한 IP 및 N/W Packet 모니터링 |
| | SIEM | - 통합 관제 시스템 구축을 통한 NW 모니터링 및 사전 위협 알림 |
| | IP, Port 차단 | - DDoS 트래픽을 유발하는 IP 및 공격 서버의 IP, 서비스 Port 차단 |
| | 임계치 설정 | - 트래픽 유형별 임계 설정하여 과도한 TCP 패킷 유급 차단 |

- 기업은 더 고도화된 공격에 대비하여 사전 보안 대책을 수립하고 지속적으로 개선하여 기업 데이터 유지

## 랜섬 DDoS vs 랜섬웨어 비교

| 구분 | 랜섬 DDoS | 랜섬웨어 |
|:-----|:---------|:---------|
| **목적** | 서비스 마비 협박 | 데이터 암호화 협박 |
| **공격 방식** | DDoS 공격 | 파일 암호화 |
| **피해 범위** | 서비스 가용성 | 데이터 무결성 |
| **복구 방법** | 공격 차단 후 정상화 | 복호화 키 필요 |
| **금전 요구 시점** | 공격 전/중 | 암호화 후 |
