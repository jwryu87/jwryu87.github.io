# 미라이 봇넷 (Mirai Botnet)
{: .fs-8 }

13.5 DDoS
{: .label .label-red }

---

## 핵심 키워드

`IoT 기기 좀비화` `봇넷 사용` `기본 패스워드` `Telnet` `포트 23/2323`

---

## 정의/개념

**IoT 기기 좀비화**, 대량의 트래픽 유발, **봇넷 사용 DDoS 공격 기법**

- 기존 좀비 PC를 이용한 DDoS 공격 방식에서 다양한 사물(IoT)기기를 활용한 공격 방식으로 진화

---

## 대응방안

### Mirai 봇넷 대응방안

| 시점 | 대응 방안 |
|:-----|:---------|
| **Mirai 봇넷 감염시 대응방안** | - 네트워크 연결을 끊는다<br>- 네트워크와 인터넷이 끊기면 재부팅한다. Mirai 악성코드는 독적 메모리에서 동작하기에 재부팅하여 메모리를 초기화하여 해결<br>- 재부팅이 끝나면 장치에 접근할 때 사용하는 암호를 강력한 암호로 변경<br>- 강력한 암호로 변경했으면 다시 네트워크에 연결하여 서비스를 실행. 만약 암호를 변경하기 전에 네트워크에 연결하면 다시 감염될 수 있음<br>- 반드시 모든 기본 비밀번호를 강력한 비밀번호로 변경. 기본 사용자 계정과 비밀번호는 인터넷에서 쉽게 찾을 수 있음 |
| **Mirai 봇넷 예방방안** | - 사물인터넷 기기를 주기적으로 보안 업데이트<br>- UPnP(Universal Plug and Play)가 필요 없는 기기는 이 기능을 비활성화<br>- 보안 기능을 제공하거나 대중성이 있는 회사에서 사물인터넷 기기를 구매<br>- 가정과 기업에 설치된 기기와 애플리케이션의 기능 숙지. 또한 공개 와이파이를 사용하지 않고 홈 네트워크에서 동작하는 와이파이를 사용<br>- 원격 터미널을 사용하는 사물인터넷 기기에 접근할 때 자주 사용하는 포트를 모니터링<br>- 포트 48101로 통신하는 의심스러운 트래픽을 발견하면 즉시 대응 방안을 통해 악성코드를 제거. Mirai는 48101 포트를 사용하여 공격자에게 추가 정보를 전송하고 이 정보를 기반으로 추가적인 악성코드 배포를 시도함 |

## Mirai 봇넷 동작 원리

| 단계 | 설명 |
|:-----|:-----|
| **1. 스캔** | 기본 자격증명(ID/PW)을 사용하는 IoT 기기 탐색 |
| **2. 감염** | Telnet(23, 2323) 포트로 접근하여 악성코드 설치 |
| **3. C&C 연결** | 명령제어(C&C) 서버에 연결 |
| **4. 공격 대기** | 공격 명령 수신 대기 |
| **5. DDoS 실행** | 대규모 DDoS 공격 수행 |

---

## 주요 특징

| 특징 | 설명 |
|:-----|:-----|
| **IoT 기기 타겟** | CCTV, 라우터, DVR 등 IoT 기기 감염 |
| **기본 패스워드 악용** | 공장 초기 설정 패스워드 사용 기기 공격 |
| **메모리 상주** | 재부팅 시 악성코드 제거 가능 |
| **대규모 공격** | 테라비트급 DDoS 공격 가능 |
