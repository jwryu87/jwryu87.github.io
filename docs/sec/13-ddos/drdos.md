# DRDoS (Distributed Reflection DoS)
{: .fs-8 }

13.1 DDoS
{: .label .label-red }

---

## 핵심 키워드

`Reflector 이용` `추적 불가` `Agent 불필요` `IP Spoofing`

---

## 정의/개념

**Agent 설치 없이**, 네트워크 통신 프로토콜 구조 취약성 이용, **정상적 시스템 Agent 활용** 서비스 거부 공격 기법

---

## DRDoS 의 공격기법

### 가. DRDoS 의 공격 절차

```
┌─────────────────────────────────────────────────────────────────────┐
│  ① 서버는 변조된 공격대상                                            │
│     IP로 syn-ack 패킷 전송                                          │
│                                    syn                              │
│  ┌────────────────────────┐    ─────▶    ┌─────────────────┐       │
│  │   반사서버(정상 TCP서버  │              │                 │       │
│  │      또는 라우터)       │    ◀─────    │     공격자      │       │
│  └────────────────────────┘    syn-ack   │   10.0.0.10     │       │
│              │                           └─────────────────┘       │
│              │ syn                                                  │
│              │                                                      │
│              ↓                                                      │
│  ┌────────────────────────┐                                        │
│  │        공격대상         │                                        │
│  │    192.168.100.200     │                                        │
│  └────────────────────────┘                                        │
│                                                                      │
│  ② 공격대상 IP로 출발지 IP를                                         │
│     변조(192.168.100.200)하여 이                                     │
│     syn패킷 전송                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 나. DRDoS 의 공격기법 상세설명

| 구성요소 | 세부 요소 | 설명 |
|:---------|:---------|:-----|
| **공격절차** | 출발지 IP 변조 | - 공격자는 출발지 IP 를 Victim IP 로 Spoofing 하여 SYN 패킷을 공격 경유지 서버로 전송 |
| | Victim 서버로 응답 | - SYN 패킷을 받은 경유지 서버는 Spoofing 된 IP (Victim 서버)로 SYN/ACK 를 전송 |
| | 서비스 불가 | - Victim 서버는 수 많은 SYN/ACK 를 받게 되어 다운 |
| **주요특징** | Reflector 이용 | - 정상적인 선량한 TCP 서버로부터 반사<br>- Syn/Ack 패킷은 정상적인 상태 |
| | 공격자 추적불가 | - 공격경로 지속 변경 공격 |
| | Agent(봇) 불필요 | - DDoS 의 Agent 감염없이 정상서버 이용 |
| | 경로차단 어려움 | - 특정 서버가 아닌 다양한 경로로 수신 |

- DRDoS는 방어와 대응이 DDoS 보다도 어렵고 까다로움

## 대응방안

### DRDoS 의 대응방안

| 구분 | 설명 |
|:-----|:-----|
| **Victim 서버의 IP, Port 필터링** | - 공격 대상이 되고 있는 Victim 서버 IP 와 port(서비스)를 포기하고, 다른 서비스 서버 이용 |
| **반사서버의 무차별 이용방지** | - 반사서버에서 연결을 완료하지 않은 Syn 출처 IP 를 조사하여 블랙리스트 운용하여 공격 머신을 차단 |
| **ISP 의 필터링** | - 반사공격을 위한 트래픽 생성을 IP 주소의 위조에 의해 좌우<br>- ISP 의 제어 가능한 네트워크 내부에서 위조된 패킷을 인터넷 에 유입 불가하도록 체제 수립<br>- 배출 필터링(egress filtering) 실시 |
| **공격 플랫폼 제한** | - 미가공 소켓(RAW Socket) 등 공격에 이용될수 있는 API 사용 불가하도록 처리 |

- 근본적인 방어가 아닌 임시적으로 공격 대상 서비스를 버리고 다른 서비스를 살리는 형태로 대응에 한계가 있음
- 서버/서비스 운용시 DRDoS 등에 악용되지 않도록 사전대비 및 철저한 보안의식 필요

## DDoS vs DRDoS 비교

| 구분 | DDoS | DRDoS |
|:-----|:-----|:------|
| **봇넷** | 필요 (좀비PC) | 불필요 |
| **반사서버** | 미사용 | 사용 |
| **IP Spoofing** | 선택적 | 필수 |
| **추적** | 가능 | 매우 어려움 |
| **대응** | 상대적 용이 | 매우 어려움 |
