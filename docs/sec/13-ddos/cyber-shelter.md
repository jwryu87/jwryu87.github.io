# DDoS 사이버 대피소
{: .fs-8 }

13.6 DDoS
{: .label .label-green }

---

## 핵심 키워드

`중소기업기본법 제2조` `시행령 제3조` `무료지원 서비스` `DNS 정보 변경` `트래픽 우회`

---

## 정의/개념

**DDoS 트래픽을 대피소로 우회 유도 중소기업 무료지원 서비스**

- 사이버대피소는 피해 웹사이트로 향하는 DDoS 트래픽을 대피소로 우회하여 분석, 차단함으로써 정상적으로 운영될 수 있도록 하는 중소기업 무료지원 서비스

---

## 서비스 개요

| 항목 | 내용 |
|:-----|:-----|
| **서비스대상** | "중소기업기본법 제2조" 및 "중소기업기본법 시행령 제3조"에 해당하는 중소기업 |
| **서비스** | 방어 서비스 적용 후 1개월 |
| **이용기간** | 재공격이 예상되는 경우 지속적으로 1개월 연장 |

- 전체 500Gbps 규모의 연동망으로 대규모 공격 방어 가능, 24시간 365일 대응 통한 서비스 안전성 확보

## 대응방안

### DDoS 사이버 대피소 서비스의 DDoS 대응기법

| 구분 | 확인사항 | 설명 |
|:-----|:---------|:-----|
| **관리적 측면** | HTTP 계열 방어정책 | - X-Forwarded-For 설정으로 식별된 IP에 대해 60초 기준 1개 IP에서 같은 URL을 100회 요청시 60분간 차단되며, 임계치 임의 수정 가능 |
| | WhiteList 관리 | - 관리자 IP 혹은 지속적으로 요청이 많은 정상적인 URL 및 기반의 예외 설정 |
| | 이상행위 IP 관리 | - 웹서버 구간 내 이상행위 시도 좀비PC IP에 대한 사이버대피소 방어 서비스 인입 구간 차단 설정 |
| **기술적 측면** | DNS 정보 변경 | - DNS CNAME처리 또는 DNS A Record 변경 통해 해외에서 유입되는 트래픽의 차단 |
| | CACHE 적용 | - 정적 컨텐츠인 멀티미디어 파일 확장자의 경우 캐시 서비스 통해 속도 저하 문제 대응 |
| | 부하분산 | - Least Connection 방식의 Load Balancing 기술 적용 |

- 국제적 협력 통한 대역폭 소진, 자원 소진, 웹/DB 부하 공격 별 대응 기법 및 노하우 공유

## 서비스 절차

| 단계 | 내용 |
|:-----|:-----|
| **1. 사전 등록** | 웹사이트 정보 및 서비스 환경 사전 등록 |
| **2. 공격 감지** | DDoS 공격 발생 감지 |
| **3. DNS 변경** | DNS 정보 변경으로 트래픽 우회 |
| **4. 공격 방어** | 대피소에서 악성 트래픽 필터링 |
| **5. 정상화** | 공격 종료 후 정상 서비스 복구 |

---

## 서비스 특징

| 특징 | 설명 |
|:-----|:-----|
| **무료 서비스** | 중소기업 대상 무료 지원 |
| **24시간 운영** | 365일 24시간 모니터링 및 대응 |
| **대규모 방어** | 500Gbps 규모 공격 방어 가능 |
| **신속한 적용** | 긴급 상황 시 빠른 적용 가능 |
