# LAND Attack
{: .fs-8 }

13.4 DDoS
{: .label .label-red }

---

## 핵심 키워드

`Source IP Spoofing` `Loop` `Buffer Over Flow` `Null Session`

---

## 정의/개념

공격자가 **타겟의 IP 주소를 변경한 TCP SYN 패킷**을 보내 타겟 시스템이 응답을 무한반복 하면서 **Null Session을 생성**하여 서비스를 거부상태로 만드는 공격 기법

---

## 공격기법

### LAND Attack 공격기법 설명

```
┌─────────────────────────────────────────────────────────────────────────┐
│                              ① IP 주소 획득                             │
│                                                                         │
│     Src IP       Dst IP       Data                Target                │
│  ┌─────────┐                               ┌─────────────────┐         │
│  │192.168.0.30│ 192.168.0.30   Data    ───▶│  (192.168.0.30) │         │
│  └─────────┘                               └─────────────────┘         │
│                                                                         │
│                                                                         │
│     Attacker                                        반복                │
│   (192.168.0.10)                                   버퍼 오버플로우 발생  │
│        │                                                                │
│        │   ③ Target IP로                                               │
│        │   Src IP 스푸핑     192.168.0.30   192.168.0.30   Data        │
│        │   (192.168.0.30)                                               │
│        │                                                                │
│        │   ④ Src/Dst IP가 같은                                          │
│        │   패킷 전송                                                    │
└─────────────────────────────────────────────────────────────────────────┘
```

| 구분 | 공격단계 | 설명 |
|:-----|:---------|:-----|
| **① IP 주소 획득** | IP 주소 획득 | - ping 명령어 등을 통해 타겟 IP 주소 획득 |
| **② Src IP 스푸핑** | Src IP 스푸핑 | - 획득한 IP 주소로 공격자의 IP 변조 수행 |
| **③ Src/Dst IP가 같은 패킷 전송** | 패킷 전송 | - 타겟 IP를 Dst IP로 하여 Src/Dst IP가 동일한 패킷 전송<br>- 이 때 타겟은 Response 를 자기자신한테 수행 |
| **④ 반복, 널 세션 생성, 버퍼 오버플로우 발생** | 반복, 널 세션 생성 | - Src IP 가 타겟 IP 이므로 자기 자신한테 계속해서 반복 응답<br>- 혹은 새로운 연결 생성 위한 널 세션 생성 |
| **⑤ 서비스 마비** | 버퍼 오버플로우 발생 | - 루핑, 세션이 계속 저장, 할당된 버퍼 메모리를 초과하는 현상 발생 |
| **⑥ 서비스 마비** | 서비스 마비 | - 버퍼 오버플로우로 인한 시스템 마비, 서비스 제공 불가 |

- 다양한 Device를 이용해 명령을 전송하고, 실시간으로 분석하여 빠른 대응이 가능

## 대응방안

### 공격기법에 대한 보안 대책

#### 가. 사전 보안 대책

| 구분 | 보안 대책 | 설명 |
|:-----|:---------|:-----|
| **관리적** | 보안 정책 수립 | - DDoS 대비 기업 내부 보안 정책 및 거버넌스 구축 |
| | 모의 훈련 수행 | - 분기/반기 별 DDoS 대비 모의 훈련 수행 및 결과 개선함 |
| | 대응계획 수립 | - 사전 공격에 대응할 수 있는 R&R 및 최적 대응을위한 계획 수립<br>- DDoS에 발생 시 대응 가능한 TF(Task Force) 사전 구성 |
| | 백업체계 수립 | - 장애 발생 후 데이터 복원을 위한 데이터, OS 백업 진행 |
| **기술적** | F/W, IPS/IDS | - 운영장비의 공격 대응을 위한 IP 및 N/W Packet 모니터링 |
| | SIEM | - 통합된 관제 시스템 구축을 통한 NW 모니터링 및 사전 위협 알림 |
| | 스크러빙센터 | - 금융보안원이 제공하는 SECaaS 형태의 DDoS 공격 대응이 가능한 스크러빙센터 서비스 사전 구축 |
| | CDN 서비스 | - DDoS 공격 시 NW 전송 용량을 쉽게 증설할 수 있도록 CDN 서비스 구축 |

- DDoS 공격에 대비하여 보안 대책 수립하고 공격이 들어오 이후 사후 보안 대책을 수립하여야 함

#### 나. 공격기법의 사후 보안 대책

| 구분 | 보안 대책 | 설명 |
|:-----|:---------|:-----|
| **관리적** | 재발 방지 수립 | - 공격 기법 분석 후 기법에 맞는 재발 방지 대책 수립 후 적용<br>- 수립된 재발 방지 기반으로 기존 보안 정책 Update |
| | 사고 대응 처리 | - 장애 서비스 복구 및 서비스 이용 고객에 대한 대응 |
| | 유관기관 공조 | - KISA 등 유관기관 협조를 받아 빠른 원인 분석 및 사후 대응 진행 |
| **기술적** | IP, Port 차단 | - DDoS 트래픽을 유발하는 IP 및 공격 서버의 IP, 서비스 Port 차단 |
| | 패킷 필터링 | - 라우터 등에서 소스 주소와 목적지 주소가 동일한 경우 패킷 차단 |
| | 임계치 설정 | - 트래픽 유형별 임계 설정하여 과도한 TCP 패킷 유입 차단 |

## LAND Attack 특징

| 특징 | 설명 |
|:-----|:-----|
| **IP Spoofing 필수** | Source IP를 Target IP로 위조 |
| **루핑 유발** | 자기 자신에게 응답 반복 |
| **버퍼 오버플로우** | 메모리 고갈로 시스템 마비 |
| **탐지 용이** | Src IP = Dst IP 패턴으로 탐지 가능 |

---

## 유사 공격 비교

| 공격 유형 | LAND Attack | SYN Flood | Smurf Attack |
|:---------|:-----------|:----------|:-------------|
| **IP Spoofing** | Src = Dst | Src 위조 | Src 위조 |
| **프로토콜** | TCP | TCP | ICMP |
| **특징** | 루핑 유발 | 반개방 연결 | 브로드캐스트 악용 |
