# 사이버 디셉션 (Cyber Deception)
{: .fs-8 }

12.3 보안솔루션
{: .label .label-purple }

---

## 핵심 키워드

`Decoy(미끼)` `Trap(함정)` `Deception System` `허니팟` `샌드박스`

---

## 정의/개념

**NW, Endpoint 등 위협이 주변 확산 전 Decoy, Trap 이용 실시간 유인, 탐지, 예방 보안 기술**

- 제로데이, 원데이 공격과 같은 최고 수준의 공격자를 실시간으로 탐지, 분석, 범위 수행
- Decoy(미끼): 쿠키, 서버접근정보, 로그인정보, 자동생성기술
- Trap(함정): EndPoint, Network, OS
- Deception System: NW탐지, 포렌식, UI

---

## 구성도/구성요소

### 구성도

```
┌────────────────────────────────────────────────────────────────────────┐
│                                                          EndPoint      │
│                           Firewall                          │         │
│   Malicious  ──────────────▶  ──────────▶  Decoy   ──▶   T  NW        │
│     User          IDS/IPS      접근유도              R     │         │
│                                                      A    OS         │
│                   행위기록                           P     │         │
│                                               Deception System       │
│    Benign     ──────────────▶  ──────────▶             (탐지/대응)    │
│     User          NW            정상접근                             │
│                 Detection                     Real System            │
└────────────────────────────────────────────────────────────────────────┘
```

### 구성요소

| 분류 | 구성요소 | 설명 |
|:-----|:---------|:-----|
| **Decoy(미끼)** | 쿠키 | 스크립트의 기록서(HTTP), 로컬에 저장되는 기록 정보 파일 |
| | 서버접근정보 | IP 주소, MAC, 접속시간 정보, 접속기간 정보 |
| | 로그인정보 | 계정정보, 접근 경로, 권한 정보, 사용 명령어 |
| | 자동생성기술 | 방화벽 및 유사 내부정보를 활용한 접근 포인트 생성 |
| **Trap(함정)** | EndPoint | 에이전트 정보, 방화벽, 경로별 정보 유도, USB, 포트 |
| | Network | 가상 유도 NW, 트래픽 생성, Access Control, 이메일 정보 |
| | OS | Root 정보, 사용자 인증, 접근제어 허가, 메모리 유도 |
| **Deception System** | NW 탐지 | 이상접근, 미끼기반 모니터링, 랜섬웨어 확인, 현상파악 |
| | 포렌식 | 정당성 확보를 위한 근거 추출, 미상현상에 대한 원인 분석 |
| | User Interface | 실시간 트래픽 모니터링, 포토 등 다경로 시각화 수행/제공 |

## 비교

### 사이버디셉션과 허니팟 비교

| 구분 | 사이버디셉션 | 허니팟 |
|:-----|:-----------|:-------|
| **대응범위** | 내/외부 접근 | 외부접근 |
| **라이선스** | 라이선스 최소화 | 가상화 등 많은 라이선스 필요 |
| **오탐율** | 오탐율 최소화(제로화) | 오탐지 높음 |
| **탐지방식** | 포괄적 탐지(자동화 구현/확장) | 로그분석 기반 |
| **확장** | 포괄적 배치 | 제한적 배치 |

- 정해진 특정 중심의 IT 자산만 HoneyNet으로 구축할 수 있었으나 사이버디셉션은 IoT 및 IIoT, 의료기, PLC, 리눅스 등 다양한 시스템을 에뮬레이션 할 수 있어서 IoT 보안에서의 활용도가 매우 높음

---

## 사이버 디셉션 기술 유형

| 유형 | 설명 |
|:-----|:-----|
| **미끼(Decoy)** | 가짜 데이터, 문서, 크리덴셜 배치 |
| **함정(Trap)** | 공격자 행위 유도 및 모니터링 |
| **허니팟(Honeypot)** | 가짜 시스템으로 공격자 유인 |
| **허니토큰(Honeytoken)** | 추적 가능한 가짜 데이터 |

---

## 적용 효과

| 효과 | 설명 |
|:-----|:-----|
| **조기 탐지** | 공격 초기 단계에서 탐지 가능 |
| **공격자 분석** | 공격 기법, 도구, 의도 파악 |
| **오탐 최소화** | Decoy 접근 = 무조건 공격 |
| **시간 확보** | 실제 자산 보호를 위한 대응 시간 확보 |
