# 제로 트러스트 (Zero Trust)
{: .fs-8 }

12.1 보안솔루션
{: .label .label-purple }

---

## 핵심 키워드

`지능형 방화벽` `DevSecOps` `SOAR` `Secure SSO` `Micro-Segmentation`

---

## 정의/개념

모든 네트워크 자산, 위험 요소 전제, **접근 통제**, **권한제어 무신뢰 기반** 과립형 경계

---

## 개념도/적용 절차

### 가. 제로 트러스트 모델 적용 방안

| 단계 | 항목 | 내용 |
|:-----|:-----|:-----|
| 1 | **중요데이터 식별** | 자산 중요도, 데이터 민감도 |
| 2 | **민감데이터 흐름 설계** | 데이터 Flow 최적화 |
| 3 | **Micro-Perimeters 설계** | 보안 경계 설정, 정책/Tool 구성 |
| 4 | **보안분석, 모니터링** | 지속 모니터링, 체계 구축 |
| 5 | **보안자동화 오케스트레이션** | 자동화 기반 운영 효율화 |

- 시스템 설계 시 내부에서 외부로 진행, 식별된 주요 자산 및 데이터의 **Micro-Segmentation** 기반 보안 경계 설정
- 네트워크, 엔드포인트, 클라우드, 파일 및 문서 등에서 멀웨어나 취약점을 탐지해 제거하고, 네트워크와 어플리케이션 접속을 시도할 때 사전에 허가 받은 자 또는 하가된 장치인지 인증하며 모든 과정을 모니터링하고 이상행위를 탐지하는 프로세스로 진행함
- 선제방어, 위협탐지, 대응전략을 제로트러스트 보안 프로세스에 통합해 중단 없는 보안 모델 구축

## 주요기술/비교

### 나. 제로 트러스트 모델 주요 기술요소

| 구분 | 기술요소 | 특징 |
|:-----|:---------|:-----|
| **보안 강도 강화 측면** | 지능형 방화벽 | - 의미/구문 분석 수준 데이터 분석 수행<br>- SQL Injection, CSRF, 악성 파일 실행 등 차단 |
| | DevSecOps<br>SOAR SIEM, EDR | - SaaS, PaaS, IaaS 환경 배포, 서비스 보안 강화<br>- SOAR, SIEM, EDR 기반 연계 분석 |
| **사용성, 운용성 제공 측면** | Secure SSO | - 데이터 접근 통제 및 편의성 동시 고려<br>- MFA(Multi Factor Authentication) 활용 |
| | 컬럼 단위 암호화 | - 데이터 암호화 범위 전체 → 컬럼단위 암호화<br>- 최소 범위, 최소 인원 대상, 최단 시간 노출 |

- 기존 경계 보안과 달리 세분화/복잡화된 경계 관리를 위해 **Orchestration**, 분석, 인증의 연계 및 자동화 구축 필요

## 제로 트러스트 모델과 기존 Inner Trust 모델 비교

| 비교 항목 | 제로 트러스트 모델 | Inner Trust 모델 |
|:---------|:-----------------|:-----------------|
| **시스템 운영방식** | 클라우드 기반 서비스 호스팅 | 온프레미스(On-Premise) |
| **보안 정책** | 과립형 경계 시행<br>(Granular Perimeter enforcement) | 경계 보안<br>(Perimeter Security Model) |
| **시스템 설계** | 내부(자산 식별) → 외부 설계 | 외부 → 내부 설계 |
| **검증/모니터링 대상** | 엔드포인트<br>자산/데이터 단위, 모든 트래픽 | 외부 접속, 트래픽 |

- 기업 도입 목적에 따른 아키텍처, 인증절차 선정 중요. 대표적 참조 모델로 구글의 **BeyondCorp** 존재
- 최근 클라우드, IoT 로 제로 트러스트를 확장시켜 엣지 컴퓨팅을 통한 제로 트러스트 결합하고 엣지에서 인공지능 분석하여 위협을 사전에 제거하여 보안에 대한 리소스를 효율화하여 비즈니스 속도를 높일 수 있음

---

## 제로 트러스트 핵심 원칙

| 원칙 | 설명 |
|:-----|:-----|
| **Never Trust, Always Verify** | 항상 검증, 신뢰하지 않음 |
| **최소 권한** | 필요한 최소한의 접근 권한만 부여 |
| **Micro-Segmentation** | 네트워크를 작은 세그먼트로 분리 |
| **지속적 검증** | 접근 시마다 검증 수행 |
