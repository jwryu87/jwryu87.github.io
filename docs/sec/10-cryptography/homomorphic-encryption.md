# 동형암호 (Homomorphic Encryption)
{: .fs-8 }

10.10 암호화
{: .label .label-purple }

---

## 핵심 키워드

`부트스트래핑` `스퀘싱` `RAD78` `BGN05` `Gen09` `암호문 연산`

---

## 정의/개념

**데이터 손실/유출 없이, 암호화된 상태**, 부트스트래핑, 스퀘싱 **4세대 암호화**

- 암호화된 데이터에 대해 복호화 없이 연산을 수행하고, 연산 결과도 암호화된 상태로 유지
- 평문 연산 결과와 암호문 연산 결과가 동일

---

## 메커니즘

### 동형암호의 메커니즘과 원리

```
         발전                   동형암호
                ─────────────────────────
                    ↑
                  공개키
                    │
                  대칭키
                    │
                 Password


동형 암호의 메커니즘과 원리:

┌─────────────────────────────────────────────────────────┐
│                                                         │
│   평문:      8    +    10    =    18                    │
│           mod4       mod4       mod4                    │
│                                                         │
│                  x           x+A                        │
│            mod n      mod n    "행렬식"    mod n        │
│                                                         │
│   암호문:    0    +    2     =    2                     │
│             X          Y          3ᵦ        4           │
│                                                         │
│   - 평문 연산 결과와 암호문 연산 결과가 동일             │
└─────────────────────────────────────────────────────────┘
```

---

## 구성요소

### 동형암호 유형 및 알고리즘

| 구분 | 구성요소 | 특징 |
|:-----|:---------|:-----|
| **유형** | 부분적 동형암호 | 한 유형 연산만 지원 |
| | 준 동형암호 | **덧셈과 곱셈 지원** |
| | 완전 동형암호 | **덧셈, 곱셈, XOR 지원** |
| **설계원리** | 부트스트래핑 | 암호화된 비밀키 이용, 노이즈 감소 암호문 생성 |
| | 스퀘싱 | 노이즈 증가 감소, 평문 변형되지 않도록 구성 |
| **알고리즘** | RAD78 | 최초 동형암호 |
| | BGN05 | 덧셈과 한번의 곱셈 연산 보존 가능 |
| | Gen09 | 격자 기반, 난수화된 에러 사용 |

---

## 동형암호 발전 세대

| 세대 | 특징 | 대표 알고리즘 |
|:-----|:-----|:-------------|
| **1세대** | 부분 동형암호 | RSA, ElGamal |
| **2세대** | 준 동형암호 | BGN |
| **3세대** | 완전 동형암호 (노이즈 이슈) | Gentry09 |
| **4세대** | 완전 동형암호 (노이즈 해결) | BFV, CKKS, TFHE |

## 장단점

| 구분 | 내용 |
|:-----|:-----|
| **장점** | - 데이터 프라이버시 보장<br>- 복호화 없이 연산 가능<br>- 클라우드 환경에 적합 |
| **단점** | - 연산 속도 느림<br>- 암호문 크기 증가<br>- 구현 복잡성 |
