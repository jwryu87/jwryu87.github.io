# VPN (Virtual Private Network)
{: .fs-8 }

16.3 네트워크 보안
{: .label .label-yellow }

---

## 핵심 키워드

`터널링` `PKI` `IPSec` `DES` `RSA` `SSL VPN`

---

## 정의/개념

인터넷 같은 **공중망에서 터널링 기법을 사용**하여 **보안성 있는 연결로 안전하게 통신**을 할 수 있는 보안 솔루션

---

## 개념도

```
┌───────────────────────────────────────────────────────────────────────┐
│                                                                       │
│    ┌──────┐                                              ┌──────────┐│
│    │      │    Layer 3: IPSec, Layer 4~7: SSL           │          ││
│    │ 본사 │    ┌──────────────────────────────┐         │   지사   ││
│    │      │◀───│                              │────────▶│          ││
│    │ VPN  │    │   공중망(Public Network)     │         │   VPN    ││
│    │      │    │                              │         │  Client  ││
│    │ 기업 │    │    Layer 2: L2F, L2TP, PPTP, MPLS      │          ││
│    │      │    └──────────────────────────────┘         │          ││
│    └──────┘                                              └──────────┘│
│                                                                       │
│                Remote User, Mobile User 등의 원격 사용자              │
│                                                                       │
└───────────────────────────────────────────────────────────────────────┘
```

- 통신을 위는 양측에는 VPN 장비를 갖추어야 함
- 계층별로 IPSec VPN, SSL VPN 터널링 구성으로 전체 시스템, 네트워크 보안성을 확보

## VPN 구현기술

### 인증

| 상세 | 설명 |
|:-----|:-----|
| **터널링** | - PKI (Public Key Infrastructure) 기반의 비 대칭 키를 사용하여 세션을 성립하기 때문에 End-to-End 인증 효과<br>- 추가적으로 MD-5, SHA-1 등의 해쉬 함수를 통한 메시지 다이제스트 구현으로 대상 노드 및 데이터 무결성 인증 |

### 암호화

| 상세 | 설명 |
|:-----|:-----|
| **터널링** | - 인터넷 상에서 가상의 정보통로 통로<br>- 패킷을 사전에 암호화하는 방법을 규정한 IPSec이 업계표준 |
| **기밀성** | - 기밀성 보장을 위한 메커니즘<br>- 전송중인 정보의 공개 방지(DES,SEED 등 사용)<br>- End-to-End 간의 세션 키를 기반으로 암호화/복호화 수행 |
| **알고리즘** | - DES, 3-DES, AES, SEED 등의 블록 암호화 알고리즘 사용 |

### 키 관리

| 상세 | 설명 |
|:-----|:-----|
| **키 분배** | - 사전에 공유한 암호화 키의 안전한 분배를 위한 키 안전관리 메커니즘<br>- IKE(Internet Key Exchange: IPSec 를 암호화하는데 사용 |
| **키 교환** | - 비대칭 키의 교환을 위해 RSA 프로토콜이 사용되며, End-to-End 간에 SA (Security Association)이 성립된 이후에는 해당 비 대칭 키로, 데이터의 암호화에 사용 되는 세션 키(대칭 키, 비밀 키)를 생성 및 교환 |

---

## VPN 유형

| 유형 | 상세 | 설명 |
|:-----|:-----|:-----|
| **IPSec VPN** | IP 프로토콜의 일부인 IPSec 프로토콜을 이용하여 VPN 구현 | - 네트워크계층(3계층)에서 동작<br>- Site-to-Site(전용랜), Site-to-Remote(원격접속) 방식으로 구성 |
| **SSL VPN** | 보안통신 프로토콜(SSL)을 통해 VPN 구현 | - 전송계층(4계층)~응용계층(7계층)에서 동작<br>- Site-to-Remote (원격접속) 방식으로 구성 |
| **MPLS VPN** | 패킷 스위칭 기술인 MPLS 환경을 통해 VPN 구현 기술 | - 데이터링크 계층(2계층)에서 동작<br>- 시간에 민감한 어플리케이션 운영 환경(음성, 동영상) |

- 이외 유형으로 VoVPN, Mobile VPN의 유형도 존재, 일반적으로 IPSec, SSL VPN 사용

---

## IPSec VPN vs SSL VPN 비교

| 구분 | IPSec VPN | SSL VPN |
|:-----|:----------|:--------|
| **동작 계층** | 3계층 (네트워크) | 4~7계층 (전송~응용) |
| **클라이언트** | 전용 SW 설치 필요 | 웹 브라우저 사용 |
| **접속 방식** | Site-to-Site, Remote | Remote 중심 |
| **보안 범위** | 전체 네트워크 | 특정 애플리케이션 |
| **구축 복잡도** | 높음 | 낮음 |
| **유연성** | 낮음 | 높음 |
