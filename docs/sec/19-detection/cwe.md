# CWE (Common Weakness Enumeration)
{: .fs-8 }

19.5 탐지/대응
{: .label .label-purple }

---

## 핵심 키워드

`MITRE` `잠재적 보안 위협` `보안 약점 분류` `CWSS`

---

## 정의/개념

미국 MITRE 기관에서 일반적 SW에서 공통적으로 발생하는 **잠재적 보안 약점을 사전식으로 분류하고 표준화**한 체계

---

## 특징

| 구분 | 설명 |
|:-----|:-----|
| **개념** | 미국 MITRE 기관에서 일반적 소프트웨어에서 공통적으로 발생하는 잠재적인 보안 약점(weakness)을 사전식으로 분류하고 표준화한 체계 |
| **목적** | 보안 약점 분류하기 체계, 카테고리 정리 제공 |
| **분류기준** | 뷰, 카테고리, 취약점, 복합요소 |
| **선정기준** | 정량평가(CWSS) |
| **체계내용** | 취약점 정의, 설명, 플랫폼, 빈도, 예제코드 |

---

## 분류체계

| 분류 | 설명 |
|:-----|:-----|
| **CWE 699** | 소프트웨어 개발에서 자주 사용되거나 발생하는 취약점으로 구성 |
| **CWE 1000** | 취약점에 대한 연구를 용이하게 하기 위한 분류(연구자, 개발자 활용) |
| **CWE 700** | 7PK(Seven Pernicious Kingdoms), 행안부 시큐어 코딩 가이드 분야 |
| **CWE 900** | CWE/SANS Top 25 |
| **CWE 928** | OWASP TOP 10, 웹 공격에 대한 Top 리스트 |

{: .note }
> CWE의 보안약점 그룹을 구체화한 것이 보안 취약점 코드인 CVE

---

## CWE vs CVE 비교

| 구분 | CWE | CVE |
|:-----|:----|:----|
| **정의** | 보안 약점 (Weakness) | 보안 취약점 (Vulnerability) |
| **범위** | 일반적/추상적 | 구체적/특정 |
| **예시** | CWE-79 (XSS) | CVE-2021-44228 (Log4Shell) |
| **용도** | 개발자 교육, 코드 점검 | 취약점 관리, 패치 |
| **관계** | 원인 | 결과 |
