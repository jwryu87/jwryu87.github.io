# 정보보호 거버넌스 (Information Security Governance)
{: .fs-8 }

19.4 탐지/대응
{: .label .label-purple }

---

## 핵심 키워드

`책임` `전략` `획득` `성과` `준거` `행동`

---

## 정의/개념

조직의 **정보보호 목표 달성을 위해 최고 경영층부터 일반 직원까지 모두 참여**하여 정보보호 활동을 **체계적으로 관리** 정책

---

## 6대 원칙

| 구분 | 설명 |
|:-----|:-----|
| **책임 (Responsibility)** | - 조직원은 주어진 책임과 권한을 이해하고 수용<br>- 역할과 권한은 반드시 책임을 수반해야 함 |
| **전략 (Strategy)** | - 비즈니스 전략과 정보보호 전략의 연계가 필요함<br>- 전략적 연계는 현재와 미래의 요구/전략을 고려해야 함 |
| **획득 (Acquisition)** | - 투명한 의사결정과 절차를 통해 정보보호 자산을 구매<br>- 자원관리와 위험관리에 해당 |
| **성과 (Performance)** | - 요구되는 서비스의 수준과 품질을 유지해야 함<br>- 비즈니스의 요구에 부합하는 성과 제공 |
| **준거 (Conformance)** | - 컴플라이언스 관리에 대한 항목<br>- 정보보호 규정/법규 준수 (위험관리에 해당함) |
| **행동 (Human behavior)** | - 정보보호 정책/시행/결과에 있어 인간적인 이해와 행동 방식을 이해해야 함 |

- 정보보호 거버넌스의 6대 원리 충족을 위해서는 일련의 프로세스가 필요함

---

## 핵심 프로세스 설명

| 구분 | 설명 | 산출물 |
|:-----|:-----|:-------|
| **평가** | - 변경으로 인한 보안 목적 달성 영향도 평가<br>- 프로세스 변경시 사전 평가 수행 | 평가 결과서 |
| **지시** | - 보안 목적 및 전략 달성에 필요한 사항 제시<br>- 자원할당, 우선순위, 정책승인, 위험관리 등 | 실행 결과서 |
| **모니터링** | - 보안관리 활동에 대한 진단과 점검 수행<br>- 적절한 성과지표를 수립하여 관리해야 함 | 모니터링 보고 |
| **의사소통** | - 이해관계자의 필요에 따라 보안내용을<br>- Govering body 와 이해관계자간 공유 | EDM 보고서 |
| **감사** | - 감사 의뢰/결과에 대한 문서화 및 피드백<br>- 독립적인 Governing body 위원회가 수행 | 감사 결과서, 피드백 |

- 정보보호 거버넌스의 구성은 크게 도메인과 프레임워크로 구분할 수 있음

---

## ISO/IEC 27001과 연계

| 구분 | ISO/IEC 27001 | 정보보호 거버넌스 |
|:-----|:--------------|:-----------------|
| **범위** | 정보보안 관리 | 정보보안 방향/감독 |
| **대상** | 관리자/담당자 | 경영진/이사회 |
| **역할** | 실행 | 의사결정/감독 |
| **산출물** | ISMS 인증 | 거버넌스 프레임워크 |
