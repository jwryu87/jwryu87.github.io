## 생애주기

### IoT 장치의 전주기 단계별 보안 고려사항

```
┌─────────────────────────────────────────────────────────────────────┐
│                           F/W갱신   S/W갱신                         │
│                              ↓         ↓                            │
│  ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐ ┌──────┐                      │
│  │ 제조 │→│ 조립 │→│ 보안 │→│운영/ │→│ 폐기/│                      │
│  │      │ │      │ │ 설정 │ │ 실행 │ │ 이관 │                      │
│  └──────┘ └──────┘ └──────┘ └──────┘ └──────┘                      │
│     │         │         │         │         │                      │
│  제조                 장치    고장   이관                           │
│     │         │      대체    수리   시스템                          │
│  제설치                              │                              │
│     ↓         ↓         ↓         ↓                               │
│  IoT 장치 설계/개발 보안 | 배포/재설치/재구성 보안 | IoT 장치 운영/관리/폐기 보안│
└─────────────────────────────────────────────────────────────────────┘
```

- IoT 장치 및 서비스의 전주기 단계에 걸쳐 보안의 잠재적 위험요소와 취약점을 고려해 공통 보안 7대 원칙 수립

---

## 7대원칙

### IoT 공통 보안 7대 원칙 (정안초프취관침)

| 단계 | IoT 공통 보안 원칙 | 설명 |
|:-----|:-------------------|:-----|
| **설계/개발** | ① 정보보호와 프라이버시 강화를 고려한 IoT 제품·서비스 설계 | - "Security by Design" 및 "Privacy by Design" 기본 원칙 준수<br>- 제품의 설계 단계부터 보안 내재화 및 프라이버시 침해 위험 요소를 분석해 지속적 점검하고, 선제적 대응 |
| | ② 안전한 SW 및 HW 개발기술 적용 및 검증 | - 시큐어 코딩, 소프트웨어, 어플리케이션 및 소프트웨어 보안성 검증 및 시큐어 하드웨어 장치 활용 |
| | ③ 안전한 초기 보안설정 방안 제공 | - "Secure by Default" 기본 원칙 준수<br>- 초기 설치 단계와 고장 수리 후 재 설치 단계에서 기본으로 설정되는 파라미터 값이 가장 안전한 설정 값 유지 |
| **배포/설치/구성** | ④ 안전한 설치를 위한 보안 프로토콜 준수 및 안전한 프로토콜 사용 | - 통신 및 플랫폼에서 검증된 보안 프로토콜 사용 (암호/인증/인가 기술) |
| | ⑤ 제품·서비스 취약점 패치 및 업데이트 지속 이행 | - IoT 제품 제조사와 서비스 제공자는 보안 취약점 발견되면 보안 패치를 신속히 배포할 수 있는 사후 조치 방안 마련 |
| **운영/관리/폐기** | ⑥ 안전 운영관리를 위한 정보보호 및 프라이버시 관리체계 마련 | - 사용자 정보 취득·사용·폐기의 전주기 정보의 보호 및 프라이버시 관리 |
| | ⑦ IoT 침해사고 대응체계 및 책임 추적성 확보 방안 마련 | - 보안 사고에 대비한 침입탐지와 사고 시 분석 및 책임추적성 확보 |

---

## 7대원칙 요약

| 원칙 | 키워드 |
|:-----|:-------|
| **정** | 정보보호와 프라이버시 강화 |
| **안** | 안전한 SW/HW 개발기술 |
| **초** | 초기 보안설정 방안 |
| **프** | 프로토콜 준수 |
| **취** | 취약점 패치 및 업데이트 |
| **관** | 관리체계 마련 |
| **침** | 침해사고 대응체계 |
