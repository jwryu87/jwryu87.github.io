# 공격표면관리 (Attack Surface Management, ASM)
{: .fs-8 }

17.1 보안정책
{: .label .label-purple }

---

## 핵심 키워드

`ASM` `EASM` `Outside-In 방식` `네트워크 경계` `자산 식별` `취약점 관리`

---

## 정의/개념

민감한 데이터를 포함하여 전송 및 처리하는 **외부 디지털 자산을 지속적으로 검색, 인벤토리, 분류, 우선순위를 지정**하는 보안 모니터링

---

## 공격 표면 관리 방법

### 관리 3가지 주요 개념

| 구분 | 주요 개념 | 설명 |
|:-----|:----------|:-----|
| **관리** | 사물(Things) | 사물, 사람을 다루는 과정 및 리더, 관리자 및 전문가 안내 |
| | 사람(People) | 조직이 인터넷 연결과 관련 위험 쉽게 최소화 |
| **취약성** | 무단 액세스 | 유출된 전자 메일 주소, 사용자 이름, 암호, 로그인 페이지 |
| | 남용되는 모든 정보 | 열린 포트, 데이터베이스, 회사 기밀, 도메인 정보 |
| **온라인 존재** | 디지털 발자국 | 인터넷과 상호작용을 통한 디지털 발자국 남김 |
| | 인공물(Artefacts) | 모든 해커가 남용할 수 있는 인공물 생성 |

### 단계별 공격 표면 관리 방법

| No | 관리 방법 | 설명 |
|:---|:----------|:-----|
| ① | 디지털 발자국 발견 | - 외부 공격 표면 관리 위해 인터넷 공개된 모든 자산 식별<br>- 알지 못하는 자산 존재 위험으로 발견 단계가 중요 |
| ② | 자산 재고 및 분류 | - 자산 유형, 기술적 특성, 비즈니스 중요도, 올바른 레이블<br>- 지속적 업데이트되는 자산 유지 관리 및 보호 필요 |
| ③ | 지속적인 보안 모니터링 | - 디지털 자산의 지속적 업데이트에 따른 추적의 어려움<br>- 지속적인 디지털 자산 24*365 모니터링 체계 구축 |
| ④ | 자산 사정 및 사고 모니터링 | - 해커가 만든 악의적 및 자기성 자산 포함 여부 모니터링<br>- 공격 벡터 대한 가시성 확보 위한 이벤트 모니터링 |
| ⑤ | 위험감지 및 식별 | - 디지털 자산에 발생할 수 있는 위험 식별<br>- 권한 조치 취하기 위해 외부 공격 표면 관리 필요 |

- 효율적인 공격 표면 관리를 위해서는 공격 표면(Attack Surface) 최소화 필요

---

## 기술요소

### 탐지

| 핵심 기술 | 설명 |
|:----------|:-----|
| **Auto Discover** | 기업의 외부, 내부 IT 자산 자동 식별 기능<br>IPv4, IPv6, Cloud, SaaS, Open Port, SSL인증서 등 |
| **Threat Visualizer** | Graph 기반 시각화 기능 제공<br>자산간의 상관관계 및 위험 판단 |

### 관리

| 핵심 기술 | 설명 |
|:----------|:-----|
| **Alert Function** | 자산변화 모니터링<br>실시간 경보(E-Mail, SMS) |
| **Playbook** | 공격 유형에 따른 대응방안 표준화 |
| **Threat Management** | CWE, CVE 기반 보안 위험 관리<br>정책기반 관리, 사용자 기반 관리 등 |
| **3rd Party API** | SIEM, SOAR 솔루션 API 연동 |

- 효율적인 ASM을 위해서는 AS(Attack Surface) 최소화 필요

---

## 대응방안

| 구분 | 설명 |
|:-----|:-----|
| **인프라측면** | - 제로 트러스트(Zero Trust)관점에서 보안 아키텍처 설계 및 구현(Network Micro-Segmentation, SDP, IAP)<br>- 지점 및 디바이스 보안강화 방안수립 |
| **데이터측면** | - 무분별한 데이터 스프롤(Data Sprawl)대응을 위해 데이터 접근대상자의 업무별 권한 부여시 인증(Authentication)과 인가(Authorization) 강화<br>- 데이터 암·복호화 적용<br>- 프라이버시 보존기술(PETs, Privacy Enhancing Technologies)를 통한 데이터 보안강화 |
| **소프트웨어측면** | - DevSecOps를 통한 개발 및 운영, 보안의 자동화된 보안라이프사이클 수립<br>- 시큐어코딩, 소프트웨어 테스팅 고도화<br>- 보안인식 제고 및 기본보안수칙 실행을 위한 행태과제 도입 |
| **사용자측면** | - 모의해킹 및 취약점 점검을 통한 보안이슈 발견 및 해소<br>- 엔드 포인트부터 네트워크 경계까지 체계적인 보안체계 수립을 통한 취약점 관리, 위협정보 활용 등을 바탕으로 공격표면 최소화 노력 필요 |
