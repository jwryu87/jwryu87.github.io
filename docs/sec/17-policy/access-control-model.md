# 접근통제 보안모델
{: .fs-8 }

17.3 보안정책
{: .label .label-purple }

---

## 핵심 키워드

`벨라파둘라 모델` `비바 무결성 모델` `클락-윌슨 무결성 모델` `기밀성` `무결성`

---

## 정의/개념

허가되지 않은 **자원의 사용과 허가되지 않은 방법을 통한 자원 사용을 제어**하는 보안 모델

---

## 벨라파둘라 모델 (BLP, Bell-LaPadula Model)

### 정의

허가된 **비밀정보에 허가되지 않은 방식의 접근을 금지하는 기밀성**을 집행하는 상태머신모델/정보흐름모델(최초로 수학적 검증을 거친 모델)

### 보안규칙

| 규칙 | 설명 |
|:-----|:-----|
| **1) No Read Up (Simple security property)** | 상향읽기금지, 하향읽기허용 |
| **2) No Write Down (*-property)** | 하향기록 금지, 상향기록 허용 |
| **3) Strong *-property** | 동일한 레벨에서만 읽기/쓰기 가능 |

### 접근 권한 매트릭스

| | 상급 보안 계층 |
|:--|:--|
| **상급 보안 계층** | ❌ |
| **할당된 보안 접근 계층** | 읽기: OK, 쓰기: ❌, 읽기/쓰기: ❌ |
| **하급 보안 계층** | 읽기: OK, 쓰기: ❌, 읽기/쓰기: ❌ |

### 특징

| 장점 | 단점 |
|:-----|:-----|
| 기밀성 유지에 매우 강함 | 무결성 유지에 문제점 노출 |

---

## 비바 무결성 모델 (Biba Integrity Model)

### 정의

비인가자에 대한 **데이터 변조방지(무결성)**을 위한 상업용 모델로, 상태머신모델 (수학적 검증을 거친 모델)

### 보안규칙

| 규칙 | 설명 |
|:-----|:-----|
| **1) Simple Integrity** | 하향읽기 불가, 상향읽기 허용 |
| **2) * Integrity property** | 상향쓰기 금지, 하향쓰기 허용 |
| **3) Invocation property** | 주체는 높은 무결성을 갖는 주체에게 서비스 요청을 할 수 없음 |

### 접근 권한 매트릭스

| | 데이터 손상 | 무결성 훼손 |
|:--|:--|:--|
| **상급 보안 계층** | 읽기: OK, 쓰기: ❌ | 서비스 명령 전송: ❌ |
| **할당된 보안 접근 계층** | 읽기: OK, 쓰기: OK | 서비스 명령 전송: OK |
| **하급 보안 계층** | 읽기: ❌, 쓰기: OK | 서비스 명령 전송: OK |

### 특징

BLP의 단점인 무결성을 보장

---

## 클락-윌슨 무결성 모델 (Clark-Wilson Integrity Model)

### 정의

무결성의 **3가지 목표를 제시**함으로써 더 심오하고 실제적인 무결성을 제시하는 상업적 모델

### 규칙

| 규칙 | 설명 |
|:-----|:-----|
| **1) 정확한 트랜잭션** | 데이터를 하나의 일치 상태에서 다른 일치상태로 변경하는 일련의 동작 |
| **2) 무결성의 3가지 목표 제시** | 무결성의 3가지 목표를 다음과 같이 제시 |

### 무결성 3가지 목표

| 목표 | 설명 |
|:-----|:-----|
| **1) 비인가자 수정 방지** | 권한 없는 사용자의 데이터 수정 금지 |
| **2) 내/외부 일관성 유지** | 정확한 트랜잭션을 통한 데이터 일관성 유지 |
| **3) 합법적인 사람에 의한 불법적 수정 방지** | 직무분리를 통한 내부 통제 (참고: 비바 모델에서는 비인가자 수정 방지만 제시함) |

---

## 보안모델 비교

| 구분 | 벨라파둘라(BLP) | 비바(Biba) | 클락-윌슨 |
|:-----|:----------------|:-----------|:----------|
| **목적** | 기밀성 | 무결성 | 무결성 |
| **적용 분야** | 군사/정부 | 상업용 | 상업용 |
| **핵심 원칙** | No Read Up, No Write Down | Simple Integrity, * Integrity | 정확한 트랜잭션, 직무분리 |
| **수학적 검증** | O | O | X |
| **실용성** | 낮음 | 중간 | 높음 |

