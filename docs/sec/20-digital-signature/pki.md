# PKI (Public Key Infrastructure)
{: .fs-8 }

20.1 전자서명
{: .label .label-purple }

---

## 핵심 키워드

`CA` `RA` `CRL` `Directory` `X509` `LDAP` `OCSP`

---

## 정의/개념

인증기관의 **전자 서명된 인증서 분배를 통해** 공중망 기반 전자거래에서 상대방을 인증하고 **데이터 안전한 교환을 위한 공개키 기반 인프라**

---

## 공개키 기반 구조의 목적

| 목적 | 주요 내용 | 요소 기술 |
|:-----|:---------|:----------|
| **인증 (Authentication)** | 사용자에 대한 확인, 검증(공개키 인증) | Certificate, X509 |
| **기밀성 (Confidentiality)** | 송수신 정보에 대한 암호화 | AES, SEED, 3DES |
| **무결성 (Integrity)** | 송수신 정보의 위/변조 방지 | SHA, MD5 |
| **부인봉쇄 (Non-Repudiation)** | 송수신 사실에 대한 부인 방지 | 전자서명 |
| **접근제어 (Access Control)** | 허가된 수신자만 정보에 접근 가능 | DAC, MAC, RBAC |
| **키 관리 (Key Management)** | 공개키 발급, 등록, 관리, 폐기 | LDAP |

## 구성요소

| 구성요소 | 주요 기능 |
|:---------|:---------|
| **인증기관 CA (Certification Authority)** | 인증 정책 수립, 인증서 및 인증서 폐기 목록 관리(생성, 공개, 취소, 재발급), 다른 CA 와 상호 인증, CRL 등록 및 인증 절차 작성 |
| **등록기관 RA (Registration Authority)** | 사용자 신원 확인, 인증서 요구를 승인, CA 에 인증서 발급 요청, PKI 를 이용하는 Application 과 CA 간 인터페이스 제공 ex) 은행 공관사 |
| **CRL (Certificate Revocation List)** | 인증서 폐기 목록, 인증서의 지속적인 유효함을 점검하는 도구<br>폐지 사유: 디지털서명의 개인키 노출, 개인키 분실, 인증서 효력정지 |
| **Directory** | 인증서, 암호키에 대한 저장, 관리, 검색 등의 기능, PKI 관련 정보 공개<br>디렉토리 표준 형식으로 X.500(DAP)과 이를 간략화 시킨 LDAP 이 있음 |
| **CPS (Certification Practice Statement)** | 인증서 실무 준칙 문서, PKI 를 구현하기 위한 절차를 상세히 설명해 놓은 문서로 CA 와 운영을 통제하는 실제된 업무의 규정 |
| **X509** | X.500 디렉토리 서비스에서 서로 간의 인증을 위해 개발된 것, CA 에서 발행하는 인증서를 기반으로 한 공개키 인증서 표준 포맷 |
