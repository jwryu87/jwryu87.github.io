# 식별(Identification)과 인증(Authentication)
{: .fs-8 }

9.1 보안인증
{: .label .label-purple }

---

## 핵심 키워드

`식별: 고유 식별자, ID, 주민번호` `인증: 지식, 소유, 존재, 행위` `MFA`

---

## 정의/개념

| 구분 | 정의 |
|:-----|:-----|
| **식별(Identification)** | 인증 서비스에 개인 스스로 확인시키기 위하여 ID 등 고유 식별자를 이용하는 **정보 주체 활동** |
| **인증(Authentication)** | 식별된 주체의 신원을 검증하기 위해 지식, 소유, 존재, 행위 인증 방식 이용하는 **정당한 사용자 증명 활동** |

---

## 차이점

### 개인 식별과 사용자 인증의 차이점

| 차이점 | 개인 식별 | 사용자 인증 |
|:-------|:---------|:-----------|
| **개념도** | A → B → C (일방향) | A ↔ B ↔ C ↔ D (양방향) |
| **개념도 설명** | 사용자 A가 사용자 B와 협조하여 자신이 을 증명할 수 있으나 제3자 C는 자신이 사용자 일을 증명할 수 있으나 C는 자신이 A인것처럼 가장 불가하고 B는 제3자 D에게 A인것처럼 가장할 수 없는 것 |
| **종류** | 고유식별번자(주민등록번호, ID) | Password, 생체, 토큰, 움직임 |

---

## 인증 방식 4가지 유형

### III. 인증 방식에 따른 4가지 유형 및 유형별 특징

| 유형 | 특징 | 기법 |
|:-----|:-----|:-----|
| **지식** | - 주체가 알고 있는 것(Something you know)<br>- 비밀번호 크기와 랜덤성에 의해 보안성의 안전도 결정<br>- 지식이 노출될 경우 비인가자 인증 가능 취약점 존재 | - Password<br>- I-PIN |
| **소유** | - 주체가 가지고 있는 것(Something you have)<br>- 고정된 정보의 위험성으로 동기/비동기 방식 OTP 사용<br>- 분실의 위험성 존재하여 비인가자 인증 가능 취약점 존재 | - 신분증<br>- OTP<br>- 토큰 |
| **존재** | - 주체를 나타낼 수 있는 것(Something you are)<br>- 생체 정보를 서버에 저장후 인증시 사용<br>- 인증 속도 및 정확도(1차, 2차 오류) 고려 필요 | - 홍채 인식<br>- 지문 인식<br>- 얼굴 인식 |
| **행위** | - 주체가 수행하고 있는 것(Something you do)<br>- 주체 고유의 미세하게 다른 습관 기준 인증<br>- 무자각 인증으로 주체의 별도 인증 행위 불필요 | - 서명, 걸음걸이<br>- 말하는 속도<br>- 음성 패턴 |

- 단일 인증 방식의 취약점 해결 위해 **MFA** 및 다중 인증 불편함 해소 위한 **통합 인증 체계** 사용

---

## 사용자 인증 시 보안 요구사항

| 요구사항 | 방안 |
|:---------|:-----|
| **기밀성** | 암호화 |
| **무결성** | 디지털 서명 |
| **가용성** | Anti-DDoS |
| **방식** | SHA-1 |
| **저장** | Hash |
| **관리** | RBAC |

---

## MFA (Multi-Factor Authentication)

| 유형 | 설명 |
|:-----|:-----|
| **2FA** | 2가지 인증 요소 조합 |
| **MFA** | 3가지 이상 인증 요소 조합 |
| **적응형 인증** | 컨텍스트 기반 인증 강도 조절 |
