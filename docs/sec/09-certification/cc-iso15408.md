# CC (Common Criteria) / ISO15408
{: .fs-8 }

9.2 보안인증
{: .label .label-purple }

---

## 핵심 키워드

`ISO/IEC 15408` `ISO/IEC 18045` `SFR` `SAR` `EAL` `PP` `TOE` `SP` `CCRA` `등급제(EAL1~EAL7)`

---

## 정의/개념

신청기관이 개발한 **정보보호제품에 구현된 보안기능의 안정성과 신뢰성을 보증**하여 안심하게 제품을 사용할 수 있도록 지원하는 제도

(국가마다 서로 다른 정보보호시스템 평가기준을 연동하고 평가결과를 상호인증하기 위해 제정된 **정보보호시스템 평가기준**)

---

## 구성

### ISO/IEC 15408 시리즈 구성

| 표준 | 타이틀 | 주요내용 |
|:-----|:-------|:---------|
| **15408-1** | Part 1: Introduction and general model | - ISO/IEC 15408 시리즈의 모든 부분에 대한 개요, 용어, 약어 설명<br>- 평가대상(TOE) 핵심 개념, 보호 프로파일(PP), 보안대상(ST) 설명 |
| **15408-2** | Part 2: Security functional components | - 정보보호 제품의 보안 기능 요구사항(SFR, Security Function Requirement)을 충족하는 기능 구성요소 카탈로그 포함 |
| **15408-3** | Part 3: Security assurance components | - 정보보호 제품의 보증 요구사항(SAR, Security Assurance Requirement)을 정의<br>- ISO/IEC 15408-5에 포함된 평가보증수준 및 기타 패키지가 구성 되는 개별보증요소와 보호프로파일(PP), PP-구성, PP-모듈 및 보안 대상(ST)의 평가기준을 포함 |
| **15408-4** | Part 4: Framework for the specification of evaluation methods and activities | - 객관적이고 반복 가능하며 재현 가능한 평가 방법과 평가 활동을 명시하기 위한 표준화된 프레임워크<br>- SFR과 SAR 기준으로 평가방법 및 평가활동 도출 가이드라인 |
| **15408-5** | Part 5: Pre-defined packages of security requirements | - 공통 사용을 지원하는 데 유용한 것으로 확인된 보안 보증 및 보안 기능 요구사항의 패키지들 제공 |
| **18045** | Methodology for IT security evaluation | - ISO/IEC 15408 시리즈 평가를 수행하기 위해 평가자가 수행해야 하는 최소 조치를 정의 |

---

## 용어

### CC인증 평가내용 및 구성요소

| 구분 | 용어 | 설명 |
|:-----|:-----|:-----|
| **CC인증 평가내용** | SFR (Security Functional Requirement) | 제품 구현 시에 필요한 요구사항을 평가, 보안기능요구사항 |
| | SAR (Security Assurance Requirement) | 등급에 부합함을 증명, 보안보증요구사항 |
| **CC인증 구성요소** | TOE (Target of Evaluation) | 인증 및 평가의 직접적인 대상인 보안 제품 |
| | PP (Protection Profile) | 독립적이고 여러 제품에 적용가능한 공통 평가 기준 |
| | ST (Security Target) | 특정 제품의 보안 기능을 표현한 평가 기준 자료 |
| | EAL (Evaluation Assurance Level) | TOE가 얼마나 정형화 되었는지에 대해 평가받는 등급 수준 |

---

## EAL 등급

### 평가보증등급 (EAL1 ~ EAL7)

| 등급 | 명칭 | 설명 |
|:-----|:-----|:-----|
| **EAL1** | 기능 시험 | 기능 명세서에 근거하여 독립적인 시험 수행 |
| **EAL2** | 구조 시험 | 개발자 시험 결과 및 취약성 분석 추가 |
| **EAL3** | 체계적 시험 및 검사 | 개발 환경의 보안성, 형상관리 검사 추가 |
| **EAL4** | 체계적 설계, 시험 및 검토 | 상위 수준 설계서의 정형화된 표현 추가 |
| **EAL5** | 준정형화 설계 및 시험 | 상세 설계서의 정형화된 표현 추가 |
| **EAL6** | 준정형화 검증 설계 및 시험 | 구현의 정형화된 표현 및 검증 추가 |
| **EAL7** | 정형화 검증 설계 및 시험 | 완전 정형화된 설계 및 검증 |

- 일반적으로 상용 제품은 **EAL2~EAL4** 수준으로 평가
- 국가/군사용 제품은 **EAL5 이상** 요구

## CCRA (Common Criteria Recognition Arrangement)

| 항목 | 설명 |
|:-----|:-----|
| **정의** | CC 평가결과 상호인정협정 |
| **목적** | 한 국가에서 받은 CC인증이 다른 회원국에서도 인정 |
| **회원국** | 30개국 이상 참여 (한국 포함) |
| **인정 범위** | EAL4까지 상호인정 (EAL5 이상은 국가별 별도 인정) |
