# 클라우드 서비스 보안인증 (CSAP)
{: .fs-8 }

9.3 보안인증
{: .label .label-purple }

---

## 핵심 키워드

`클라우드컴퓨팅법` `정보보호 정책` `14개 통제분야` `IaaS/SaaS/PaaS`

---

## 정의/개념

클라우드 서비스 제공자가 제공하는 서비스에 대해 정보보호 기준 준수 **평가/인증제도**

---

## 법적근거

- **클라우드컴퓨팅법 제23조제2항**

---

## 핵심내용

### 14개 통제분야

| # | 통제분야 | 통제항목 |
|:--|:---------|:---------|
| 1 | **정**보보호 정책 및 조직 | 정보보호 정책, 정보보호 조직 |
| 2 | **인**적보안 | 내부인력 보안, 외부인력 보안 |
| 3 | **자**산관리 | 자산 식별, 위험관리 |
| 4 | 서비스 **공**급망 관리 | 공급망 관리정책, 공급망 변경관리 |
| 5 | **침**해사고관리 | 침해사고 절차, 사후관리 |
| 6 | 서비스 **연**속성 관리 | 장애대응, 서비스 가용성 |
| 7 | **준**거성 | 법 준수, 보안 감사 |
| 8 | **물**리적 보안 | 물리적 보호구역, 장비보호 |
| 9 | **가**상화 보안 | 가상화 인프라, 가상 환경 |
| 10 | **접**근통제 | 접근통제 정책, 접근권한 관리 |
| 11 | **네**트워크 보안 | 네트워크 보안 |
| 12 | **데**이터 보호 및 암호화 | 데이터 보호, 암호화 |
| 13 | **시**스템 개발 및 도입 보안 | 시스템 분석 설계, 시스템 도입 보안 |
| 14 | **국**가기관 등의 보안요구사항 | 추가 요구사항 |

> 암기: **정인자공침연준물가접네데시국**

---

## 평가/인증 절차

| 단계 | 절차 | 수행 내용 |
|:-----|:-----|:---------|
| **준비** | 1) 시스템, 서비스 구축 | 클라우드 서비스, 보안시 구축 |
|  | 2) 사전 컨설팅 | 필요 자료, 인증 형태 검정 |
|  | 3) 평가, 인증 신청 | 자료가/인증 기관에 서류 제출 |
| **평가** | 4) 평가, 인증 계약 체결 | 범위, 기간 등 협의 |
|  | 1-1) 서면/현장 평가 | 내역 검정, 대면심사 인터뷰 |
|  | 1-2) 취약점 진단 | 기술/누락 검정, 인도법 |
|  | 1-3) 모의 침투 테스트 | 침투 시나리오 작성, 수행 |
| **인증** | 2) 보안조치 요청 | 검사 보완, 보완 요청 |
|  | 3) 보완조치 결과 제출 | 30일 이내 검치 |
|  | 4) 심의 및 정점 | 보완건가 현황 |
| **사후** | 1) 인증위원회 개최 | 적합 대상 파악 보안 요구 |
|  | 2) 인증서 발급 및 취득 | 인증서 발급 및 수령 |

- 인증 유효기간: **3년**
- 사후심사: 인증일로부터 **1년마다 2~6개월** 소요

---

## 인증 유형

| 유형 | 대상 | 특징 |
|:-----|:-----|:-----|
| **IaaS** | 서버, 스토리지, 네트워크 인프라 | 가상 자원 제공 |
| **SaaS** | 소프트웨어 애플리케이션 | 표준등급/간편등급 구분 |
| **PaaS** | 개발 플랫폼 | 개발환경 제공 |

---

## SaaS 등급

| 구분 | 표준등급 | 간편등급 |
|:-----|:---------|:---------|
| **대상** | 민감정보 처리 | 일반정보 처리 |
| **인증 범위** | 전체 통제항목 | 일부 통제항목 |
| **심사 수준** | 심층 심사 | 간소화 심사 |
