# ISMS-P (정보보호 및 개인정보보호 관리체계 인증)
{: .fs-8 }

9.5 보안인증
{: .label .label-purple }

---

## 핵심 키워드

`정보통신망법 제47조` `개인정보 보호법 제32조의2` `KISA` `3개 영역` `101개 인증기준`

---

## 정의/개념

개인정보 흐름, 정보보호 영역 인증, **KISA 인증 정보보호 인증제도**

---

## 법적근거

| 법령 | 조항 |
|:-----|:-----|
| **정보통신망법** | 제47조 (ISMS) |
| **개인정보 보호법** | 제32조의2 (PIMS) |

---

## 인증대상

### 의무대상

| 구분 | 대상 | 특징 |
|:-----|:-----|:-----|
| **ISP** | - 서울특별시 및 모든 광역시 ISP 제공자 | 전기통신사업자 |
| **IDC** | - 집적정보통신 시설 사업자 | 데이터센터 |
| **병원, 학교** | - 연간 매출액 또는 세입 **1500억 이상**<br>- 상급병원, 재학생 수 **1만명 이상** | 대규모 기관 |
| **정보통신 서비스 제공자** | - 전년도 매출액 **100억원 이상**<br>- 전년도 말 3개월 간 일평균 이용자 수 **100만명 이상** | 대형 서비스 |

### 자율신청

| 구분 | 대상 |
|:-----|:-----|
| **자발적 구축 운영 기관** | 연계결과 정보 주체 수 |

## ISMS vs ISMS-P 차이점

| 구분 | ISMS | ISMS-P |
|:-----|:-----|:-------|
| **인증기준** | 80개 | 101개 |
| **영역** | 2개 영역 | 3개 영역 |
| **개인정보** | 미포함 | 포함 (21개 항목) |
| **적용** | 정보보호 중심 | 정보보호 + 개인정보보호 |

---

## 인증 절차

| 단계 | 내용 |
|:-----|:-----|
| **1. 신청** | 인증 신청서 제출 |
| **2. 심사 계약** | 심사 범위, 일정 협의 |
| **3. 서면 심사** | 문서 검토 |
| **4. 현장 심사** | 실제 운영 현황 점검 |
| **5. 보완** | 결함사항 보완 |
| **6. 인증위원회** | 인증 심의 |
| **7. 인증서 발급** | 인증서 교부 |
