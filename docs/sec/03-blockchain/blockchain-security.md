## 정의/개념

블록체인 플랫폼을 구축할 때 고려해야 하는 **성능 기준**과 **보안 고려 기준**

---

## 블록체인 플랫폼 성능 기준

| 구분 | 기준 | 고려사항 |
|:-----|:-----|:---------|
| **처리성능** | 블록 사이즈, 트랜잭션 사이즈 | 초당 처리량(TPS) 결정 |
| **네트워크 성능** | 노드 구성 방식, 네트워크 환경 | 노드 간 통신 지연 최소화 |
| **블록 확정 성능** | 합의 방식, 노드 분산 | 블록 생성 및 확정 시간 |
| **참조 성능** | 블록 구조, DB 구성 | 데이터 조회 성능 |

---

## 블록체인 플랫폼 보안 고려 기준

### 기밀성 (Confidentiality)

| 항목 | 고려사항 |
|:-----|:---------|
| **접근관리** | 역할 기반 접근 통제 |
| **데이터 암호화** | 저장 및 전송 데이터 암호화 |

### 무결성 (Integrity)

| 항목 | 고려사항 |
|:-----|:---------|
| **멤버 관리** | 노드 참여/탈퇴 관리 |
| **해시 체인** | 블록 간 해시 연결로 위변조 방지 |

### 부인방지 (Non-Repudiation)

| 항목 | 고려사항 |
|:-----|:---------|
| **합의 방식** | PBFT, PoW, PoS 등 합의 알고리즘 |
| **하드 포크 정책** | 포크 발생 시 대응 정책 |

---

## 블록체인 보안 위협 유형

| 위협 유형 | 설명 | 대응방안 |
|:----------|:-----|:---------|
| **51% 공격** | 과반수 해시 파워 장악 | PoS, DPoS 도입 |
| **Sybil 공격** | 다수의 가짜 노드 생성 | 신원 검증 강화 |
| **이중 지불** | 동일 자산 이중 사용 | 합의 알고리즘 강화 |
| **스마트 컨트랙트 취약점** | 코드 결함 악용 | 보안 감사, 시큐어 코딩 |
