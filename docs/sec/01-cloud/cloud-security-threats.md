# 클라우드 컴퓨팅 보안 위협
{: .fs-8 }

1.1 클라우드 보안
{: .label .label-red }

---

## 핵심 키워드

`하이퍼바이저 레벨 문제` `가상 스위치 보안 사각지대` `Multi-Tenancy` `사법관할권` `ISO 27017` `ISO 27018`

---

## 정의/개념

클라우드 컴퓨팅 환경에서 발생할 수 있는 **기술적, 관리적 보안 위협**과 이에 대한 **대응 기술 및 표준**

---

## I. 클라우드 환경 보안 필요성

- 데이터의 집중화로 인한 보안 위협 증가
- 가상화 환경의 새로운 공격 벡터 출현
- 다중 테넌시(Multi-Tenancy) 환경의 격리 문제
- 책임 공유 모델에 따른 보안 관리 복잡성

---

## II. 클라우드 컴퓨팅 보안 위협

### 가. 기술적 측면 보안 위협

| 구분 | 위협 | 설명 |
|:-----|:-----|:-----|
| **가상머신 측면** | 하이퍼바이저 레벨 문제 | 가상머신 상호작용, VM Escape 공격 |
| **인프라 측면** | 가상 스위치 보안 사각지대(IDS/IPS) | Multi-Tenancy 환경의 네트워크 보안 |

### 나. 관리적 측면 보안 위협

| 구분 | 위협 | 설명 |
|:-----|:-----|:-----|
| **책임소재 측면** | 사법관할권 | 책임 소재, 상이한 법체계 |
| **표준화 측면** | 상이한 표준화 | 인증제도 부재 |

> (간글) 특히, 가상화 핵심 기술 **하이퍼바이저**

---

## III. 하이퍼바이저 기반 가상화 침입 대응 기술

### 가. 가상머신기반

| 기술 | 설명 |
|:-----|:-----|
| **VMI** | Virtual Machine Introspection |
| **Agentless** | 에이전트 없는 보안 모니터링 |

### 나. 클라우드 서비스 기반

| 기술 | 설명 |
|:-----|:-----|
| **SECaaS** | Security as a Service |
| **CWPP** | Cloud Workload Protection Platform |
| **CASB** | Cloud Access Security Broker |
| **CSPM** | Cloud Security Posture Management |
| **SASE** | Secure Access Service Edge |

> (간글) 관리적 측면 표준화/보안 인증 도입 필요

---

## IV. ISO 27017, 27018

### 가. ISO 27017

클라우드 서비스에 대한 **정보보안 통제 지침**
- ISO/IEC 27002 기반 클라우드 보안 확장
- 클라우드 서비스 제공자(CSP)와 고객의 역할 정의

### 나. ISO 27018

클라우드 환경의 **개인정보보호 지침**
- PII(개인식별정보) 보호 요구사항
- GDPR 준수 지원

> (마무리 간글) **KISA 인증, CSAP**

---

## 클라우드 보안 책임 공유 모델

| 서비스 모델 | 고객 책임 | CSP 책임 |
|:------------|:----------|:---------|
| **IaaS** | 데이터, 애플리케이션, OS | 가상화, 네트워크, 스토리지, 물리 |
| **PaaS** | 데이터, 애플리케이션 | OS, 가상화, 네트워크, 스토리지, 물리 |
| **SaaS** | 데이터, 사용자 관리 | 애플리케이션, OS, 가상화, 네트워크, 스토리지, 물리 |
