## 핵심 키워드

`검색` `암호화` `접근통제` `데이터 손실 방지` `로깅` `이상 탐지` `SW(Agent, API)` `HW(Public, Private)`

---

## 정의/개념

기업이 이용/제공하는 SaaS, 접근 통제, 암호화, 로깅, 감사 보안기능을 제공하는 **클라우드 보안 기술**

---

## 아키텍처

### CASB의 기능 구성 아키텍처

```
┌─────────────────────────────────────────────────────────────┐
│                    Cloud Services                            │
│  ┌─────────┐    ┌─────────┐    ┌─────────┐                  │
│  │  PaaS   │    │  SaaS   │    │  IaaS   │                  │
│  └─────────┘    └─────────┘    └─────────┘                  │
└─────────────────────────────────────────────────────────────┘
                          ↑
                    ┌─────┴─────┐
                    │   CASB    │
                    │(Proxy/API)│
                    └─────┬─────┘
                          ↑
┌─────────────────────────────────────────────────────────────┐
│                      기업/사용자                              │
│  ┌─────────┐    ┌─────────────────┐    ┌─────────────┐      │
│  │  기업   │────│   컴플라이언스   │────│ 렌더링션    │      │
│  │ 가시성  │    │    시스템 통합   │    │API 액세스   │      │
│  └─────────┘    └─────────────────┘    └─────────────┘      │
└─────────────────────────────────────────────────────────────┘
```

- 접속 통제뿐 아니라 SaaS 어플리케이션 내에서 일어나는 행위 통제와 컴플라이언스 등 종합적 제공

## 핵심기능/유형

### CASB의 핵심 기능 (검암접데로이)

| 핵심 기능 | 상세 설명 | 핵심 기능 |
|:----------|:----------|:----------|
| **클라우드 서비스의 검색 및 위험 평가** | 각 서비스의 위험 평가와 회사에서 사용중인 모든 클라우드 서비스 검색 | Risk assessment |
| **암호화** | 기업 제어 암호화 키를 사용 클라우드 데이터 암호화 | 기밀성, 가용성 보장 |
| **접근 통제** | 사용자, 장치 또는 위치의 컨텍스트를 기반으로 클라우드 서비스에 대한 접근 제어 | 정책, 모델, 메커니즘 |
| **데이터 유출방지** | 내부 정보 유출방지 또는 사용중인 DLP 솔루션과의 통합을 통해 클라우드로 이동하는데 데이터 DLP 적용 | 데이터 유출 방지 |
| **로깅 및 감사** | 잠재적으로 발생할 수 있는 위험에 대비하거나, 이미 발생된 위험에 대한 증거를 확보하기 위해서 로깅 및 감사 기능 무결성 보장 제공 | 추적 및 무결성 보장 |
| **이상 탐지** | 행위 정보를 수집, 학습시킴으로써 보안사고로의 전이를 모형화한 후, 새롭게 수집되는 행위 들을 관찰하여 이상 행위 탐지 | 행위적 침해 대응 |

- 클라우드 서비스 이용자와 클라우드 서비스 사이에 위치하여 독립적으로 보안 기능을 수행

## CASB의 주요 유형 (APPA: 아파)

| 항목 | 에이전트형 | 프라이빗 | 퍼블릭 | API 형 |
|:-----|:-----------|:---------|:-------|:-------|
| **형상** | SW | HW | HW | SW |
| **설치위치** | 이용자 단말 내 | 이용자 조직 내 | 클라우드 서비스 플랫폼 내 | 클라우드 서비스 플랫폼 내 |
| **이용목적** | 클라우드 서비스 | 클라우드 서비스 | 클라우드 서비스 | 클라우드 서비스 |
| **주체** | 이용자 | 이용 기업 | 제공자 | 앱 개발사 |
| **주요기능** | 접근통제 | 접근통제, DLP | 접근통제 | 접근 통제 |
| **SSL 여부** | 불필요 | 처리 | 불필요 | 불필요 |
| **특징** | 이용자 OS에 따라<br>별도 개발 | 에이전트 설치<br>필요 | 클라우드 서비스<br>규모에 따라<br>과부하 고려 | APP 환경 별로<br>별도 개발 |

- 유형에 따라 위치 및 형상의 차이가 존재 → 시스템/서비스의 구성에 따라 선택하여 이용 필요
