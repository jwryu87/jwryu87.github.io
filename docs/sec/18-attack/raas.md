# RaaS (Ransomware as a Service)
{: .fs-8 }

18.2 공격기법
{: .label .label-red }

---

## 핵심 키워드

`익명 네트워크` `서비스` `다크웹` `비트코인` `수익 배분`

---

## 정의/개념

다크웹과 같은 **익명 네트워크**를 이용하여 비용만 지급하면 랜섬웨어 공격을 할 수 있도록 **서비스 형태로 제공**되는 랜섬웨어

---

## 공격 프로세스

```
┌─────────────────────────────────────────────────────────────────────┐
│                                                                      │
│      ┌─────────┐                              ┌─────────┐           │
│      │         │  1. 랜섬웨어 구매의뢰          │         │           │
│      │         │  ────────────────▶          │         │           │
│      │         │  2. 랜섬웨어 코드 제공         │         │           │
│      │  공격자  │  ◀────────────────          │  제작자  │           │
│      │         │  5. 수익금 배분               │         │           │
│      │         │  ────────────────▶          │         │           │
│      │         │  6. 업데이트 제공             │         │           │
│      └────┬────┘                              └─────────┘           │
│           │                                                          │
│           │ 3. 랜섬웨어 공격                                          │
│           │    금전요구                                               │
│           │                                                          │
│           ↓                                                          │
│      ┌─────────┐                                                     │
│      │         │                                                     │
│      │  피해자  │ 4. 비트코인 지급                                     │
│      │         │                                                     │
│      └─────────┘                                                     │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

---

## 공격 단계별 상세 내용

| 관계 | 단계 | 상세 내용 |
|:-----|:-----|:---------|
| **공격자↔제작자** | 랜섬웨어 구매 및 코드 제공 | ① 공격자는 제작자에게 랜섬웨어 구매 의뢰<br>② 제작자는 공격자에게 랜섬웨어 Tool 제공 |
| **공격자↔피해자** | 공격 및 비트코인 제공 | ③ 공격자는 피해자의 자료 암호화 후 복호화 대가로 금전 요구<br>④ 복호화 키를 받는 조건으로 비트코인 등 대가를 지급 |
| **공격자↔제작자** | 수익 배분 및 업데이트 | ⑤ 수익에 대해 일정비율로 분배<br>⑥ 지속적으로 업데이트 및 애프터서비스 지원 |

---

## RaaS 대응방안

| 대응 방안 | 설명 |
|:---------|:-----|
| **백업 수행** | 관리자 보안의식 강조, 서버 및 Secure OS, Anti-Virus 설치 |
| **사용자 교육** | Mail Filtering, Web 방화벽, 백업 서버 망 분리 |
| **최신 패치** | 연관 기관과 지속적인 정보 공유, 사후 처리, 재발 방지 |
| **화이트리스트 기반 관리** | 사용자 접속/프로세스 실행을 원천적으로 차단 |

---

## RaaS vs 일반 랜섬웨어

| 구분 | 일반 랜섬웨어 | RaaS |
|:-----|:------------|:-----|
| **제작** | 공격자 직접 제작 | 제작자가 제공 |
| **기술력** | 높은 기술 필요 | 기술 불필요 |
| **비용** | 개발 비용 | 구독/수수료 |
| **업데이트** | 자체 관리 | 제작자 제공 |
| **진입 장벽** | 높음 | 낮음 |

---

## RaaS의 비즈니스 모델

| 모델 | 설명 |
|:-----|:-----|
| **구독형** | 월정액 지불 |
| **수익 배분형** | 몸값의 일정 비율 공유 |
| **라이선스형** | 일회성 구매 |
| **무료형** | 무료 제공 후 수익 공유 |

---

## 주요 RaaS 그룹

| 그룹 | 특징 |
|:-----|:-----|
| **REvil(Sodinokibi)** | 이중 갈취, 고액 요구 |
