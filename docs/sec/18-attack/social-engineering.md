# 사회공학 (Social Engineering)
{: .fs-8 }

18.5 공격기법
{: .label .label-red }

---

## 핵심 키워드

`Research` `Developing Trust` `Exploiting Trust` `Utilizing Trust`

---

## 정의/개념

인간 상호 작용 깊은 **신뢰를 바탕으로 사람들을 속여 정상 보안 절차 파괴** 비기술적 침입수단

---

## 공격 절차

| 단계 | 설명 |
|:-----|:-----|
| **Research** | 관계 형성, 정보 수집 |
| **Developing Trust** | 유대 형성, 인물 식별 |
| **Exploiting Trust** | 감정 호소, 결정 요구 |
| **Utilizing Trust** | 책임 회피, 목적 수행 |

{: .note }
> (간략) 지속적 탐색 및 신뢰 형성 통해 시스템 보안을 내부자 도움으로 회피

---

## 공격 유형

### 유대 관계 기반 사회공학

| 기법 | 설명 |
|:-----|:-----|
| **직접접근** | 대면을 통한 정보 획득 |
| **도청** | 대화 내용 엿듣기 |
| **훔쳐보기** | 화면, 키보드 엿보기 (Shoulder Surfing) |
| **설문조사** | 설문을 가장한 정보 수집 |
| **Piggybacking** | 승인된 사람을 따라 출입 |

### 컴퓨터 기반 사회공학

| 기법 | 설명 |
|:-----|:-----|
| **포렌식** | 폐기된 문서/장비에서 정보 수집 (Dumpster Diving) |
| **악성코드 전송** | 악성 링크/첨부파일 전송 |
| **크롤링** | 웹사이트 정보 자동 수집 |
| **피싱** | 위장 사이트/이메일로 정보 탈취 |
| **파밍** | DNS 조작으로 위장 사이트 유도 |

{: .warning }
> (간략) 사람을 대상으로 하기 때문에 인력, 조직, 프로세스, 기술 측면 대응체계 구축 필요

---

## 대응방안

### 인력 및 조직 측면

| 대응 방안 | 설명 |
|:----------|:-----|
| **보안거버넌스** | 보안 정책 및 절차 수립 |
| **내부통제** | 직무 분리, 권한 관리 |
| **상호인증** | 요청자 신원 확인 절차 |
| **보안 교육** | 정기적 인식 교육 실시 |

### 기술 및 프로세스 측면

| 대응 방안 | 설명 |
|:----------|:-----|
| **DRM** | 문서 유출 방지 |
| **접근제어** | 최소 권한 원칙 적용 |
| **비식별화** | 민감 정보 마스킹 |
| **MFA** | 다중 인증 적용 |

---

## 사회공학 vs 기술적 공격

| 구분 | 사회공학 공격 | 기술적 공격 |
|:-----|:-------------|:-----------|
| **대상** | 사람 | 시스템 |
| **방식** | 심리 조작 | 취약점 공격 |
| **방어** | 교육, 절차 | 기술적 보안 |
| **탐지** | 어려움 | 상대적 용이 |
| **비용** | 저렴 | 고가 |
