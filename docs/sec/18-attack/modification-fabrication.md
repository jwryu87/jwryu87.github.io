# 변조(Modification)와 위조(Fabrication)
{: .fs-8 }

18.3 공격기법
{: .label .label-red }

---

## 핵심 키워드

- **변조**: 중간가로채기
- **위조**: 불법정보생성

---

## 정의/개념

- **변조(Modification)**: 정보를 **중간 가로채서 일부 또는 전부를 변경**하여 수신자에게 전송하는 보안 공격 기법
- **위조(Fabrication)**: **불법정보를 생성**하여 수신자에게 전송하는 보안 공격 기법

---

## 개념도

### 변조(Modification)

```
┌─────────┐                           ┌─────────┐
│         │        1.원본             │         │
│  Alice  │─────────────────────────▶│   Bob   │
│         │                           │         │
└─────────┘                           └─────────┘
     │                                     ▲
     │         3.변조정보                   │
     └─────────────────────────────────────┘
                      │
               ┌──────┴──────┐
               │   공격자    │
               │ 2.불법적 수정│
               └─────────────┘
```

### 위조(Fabrication)

```
┌─────────┐                           ┌─────────┐
│         │                           │         │
│  Alice  │                           │   Bob   │
│         │                           │         │
└─────────┘                           └─────────┘
                                           ▲
                2.위조정보                  │
     ┌─────────────────────────────────────┘
     │
┌────┴────┐
│  공격자  │
│1.불법적 생성│
└─────────┘
```

## 변조와 위조 상세 비교

| 항목 | 변조 | 위조 |
|:-----|:-----|:-----|
| **목적** | 존재하는 정보의 불법적 수정 | 원본과 식별 불가능한 불법정보 생성 |
| **공통점** | 무결성 훼손 | 무결성 훼손 |
| **CRUD** | Update | Create |
| **대상** | 시스템 자원 또는 Data 대상 공격 | 송신자 또는 시스템 대상 공격 |
| **사례** | 기존 사업체의 출금계좌를 공격자의 계좌로 불법 변경 | 존재치 않는 사업체를 신규 등록하며 공격자의 계좌를 신규 등록 |

- 변조는 데이터 자체의 공격이며, 위조는 시스템의 신뢰성 훼손 또는 불법 접근 후 임의 정보 생성
- 변조와 위조는 무결성을 훼손하는 보안의 중요 문제로 대응이 요구

## 대응방안

### 변조 공격 대응

| 항목 | 공격 기법 | 대응 방안 |
|:-----|:----------|:----------|
| **파일 수정** | 파일 수정 | - 파일 접근 권한 관리<br>- 다중 인증 및 보안 통제 메커니즘 운용 |
| **프로그램 코드 변경** | 프로그램 코드 변경 | - 프로그램 버전 변경시 승인 절차<br>- Hash 기반 원본 변조 검증 |

### 위조 공격 대응

| 항목 | 공격 기법 | 대응 방안 |
|:-----|:----------|:----------|
| **허위 메시지 생성** | 허위 메시지 생성 | - 소유자 생성 인람과 타임스탬프<br>- 접근 통제 모델 기반 관리 |
| **순서 변경 공격** | 순서 변경 공격 | - Sequencing 관리 및 오류 점검<br>- TLS 및 공개키 암호화 세션 적용 |

## 무결성 보장 기법

| 기법 | 설명 |
|:-----|:-----|
| **해시 함수** | MD5, SHA-256 등을 통한 원본 무결성 검증 |
| **디지털 서명** | 전자서명을 통한 발신자 인증 및 변조 탐지 |
| **MAC** | 메시지 인증 코드를 통한 무결성 확인 |
| **타임스탬프** | 시간 정보 기록으로 시점 증명 |
| **이중화** | 원본과 복사본 비교를 통한 변조 탐지 |
