# 랜섬웨어 (Ransomware)
{: .fs-8 }

18.1 공격기법
{: .label .label-red }

---

## 핵심 키워드

`백업방식` `C&C 서버` `파일 암호화` `가상화폐` `조정방복`

---

## 정의/개념

중요 데이터 암호화, 시스템 사용 불가능, **가상화폐 요구 악성 소프트웨어**

---

## 랜섬웨어 감염 키워드

| 항목 | 설명 |
|:-----|:-----|
| **확장자명 변경** | 파일손상 |
| **CPU, RAM 사용량 증가** | 암호화 연산 수행 |
| **백신 프로그램 오류** | 보안 솔루션 무력화 |
| **윈도우 백업시점 제거** | 복구 방해 |

---

## C&C 서버 (Command & Control)

원격지 명령 및 악성코드 제어 서버 (좀비PC 제어)

---

## 랜섬웨어 감염 절차

| 단계 | 내용 |
|:-----|:-----|
| ① | SNS 접속 |
| ② | 악성코드 다운 |
| ③ | 랜섬웨어 동작 |
| ④ | 문서 파일 검색 |
| ⑤ | 암호화 수행 |
| ⑥ | 결제유도 |

---

## 랜섬웨어 유형

| 유형 | 설명 |
|:-----|:-----|
| **이메일 첨부파일** | 악성 파일 첨부 |
| **ActiveX 취약점** | 브라우저 취약점 악용 |
| **쿼타임 플레이어** | 미디어 플레이어 악용 |
| **아이콘 위장** | 아이콘 변조 |
| **SNS** | 소셜 미디어 링크 |

---

## 백업 방법: 조정방복

- **절차**: 백업 조직 구성 → 백업 정책 수립 → 백업 방식 결정 → 복구 훈련

### 기법 (종범방보)

| 기준 | 종류 | 설명 |
|:-----|:-----|:-----|
| **시스템 종료 기준** | Cold | 시스템 종료 후 백업 |
| | Hot | 운영 중 백업 |
| | Warm | 부분 운영 중 백업 |
| **백업 범위 기준** | Full | 전체 백업 |
| | Differential | 차등 백업 |
| | Incremental | 증분 백업 |
| **백업 방식 기준** | Logical | 논리적 백업 |
| | Physical | 물리적 백업 |
| **백업 보관 기준** | Local | 로컬 저장 |
| | Remote | 원격 저장 |

---

## 랜섬웨어 대응방안

### 가. 관리적 (설임외정)

| 구분 | 대응방안 |
|:-----|:---------|
| **설비** | 백업체계, 모의훈련 |
| **임직원** | 접근통제, 교육 |
| **외부** | 버그바운티, 전파 |
| **정책** | 분리 보관, 클라우드 이용 |

### 나. 기술적

| 구분 | 대응방안 |
|:-----|:---------|
| **NW** | IPS/IDC, 사이버 디셉션 |
| **Device** | EDR, Whitelist |
| **Platform** | SOAR, Secure SDLC |
| **Endpoint** | 업데이트, 백신 |

---

## 랜섬웨어 주요 종류

| 종류 | 특징 |
|:-----|:-----|
| **WannaCry** | SMB 취약점 악용, 자가 전파 |
| **Petya** | MBR 암호화, 부팅 불가 |
| **Locky** | 스팸 메일 통한 유포 |
| **CryptoLocker** | 비트코인 요구 |
| **Ryuk** | 표적 공격, 고액 요구 |
