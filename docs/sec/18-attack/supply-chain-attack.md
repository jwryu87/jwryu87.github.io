# 공급망 공격 (Supply Chain Attack)
{: .fs-8 }

18.4 공격기법
{: .label .label-red }

---

## 핵심 키워드

`펌웨어 공격` `인프라 공격` `위변조` `SW 개발 공급과정` `악성코드 배포`

---

## 정의/개념

특정 기업, 기관 등의 **HW 및 SW 개발 공급과정에 침투**하여 제품의 **악의적 변조, 악성코드 배포 행위, 해킹**하는 공격기법

---

## 공급망 공격과정 설명

| 공격 절차 | 공격 대상 | 설명 |
|:---------|:----------|:-----|
| **① 개발환경침투** | SW 개발 서버 | - 원격 근무자의 권한 통한 SW의 개발 서버 침투<br>- SW 개발서버, 업데이트 서버, 관리 서버, 개발자 PC |
| **② 변조 업데이트 파일유포** | 업데이트 파일 | - 업데이트 모듈 위장, 파일의 변조 파일유포<br>- 인증서 탈취, 관리 모듈 취약점, 코드 패치 등 |
| **③ 개인사용자 및 기업내부 확대** | 고객 PC 및 서버 | - 개인 사용자 PC 및 기업 내부 서버/PC 감염<br>- 공급망 내부 사용 시스템의 업데이트 서버 |
| **④ 타겟 설정 / 범위 확대** | 내부/외부 타 서버 | - 설정 타겟 또는 내부 기밀 서버 또는 관리자 서버 침투<br>- 백도어 설치, 랜섬웨어, 악성 코드 삽입, C&C 연계 |
| **⑤ 추가 악성코드 감염** | 외부 영향 PC/서버 | - 공급망 연계 회사나 고객사 관리 PC 및 서버 침투<br>- 지속적인 타겟 검색 및 정보 유출 범위 확대 |

- 컴퓨터 제조회사 ASUS이 소프트웨어 펌웨어 업데이트 서버가 해킹당하면서 수백만 대 컴퓨터 멀웨어가 설치되는 사례가 있었음

## 공격유형

| 구분 | 보안 위협 유형 | 공격 기법 및 취약점 |
|:-----|:-------------|:-------------------|
| **HW 구성요소 (IC칩, PCB, 펌웨어)** | 하드웨어 트로이 공격(HT) | - 공급망 공정과정에서 마이크로칩 등이 포함된 회로 (Circuit block)를 이용하여 공격 |
| | 부채널 공격(SCA) | - 암호 모듈이 포함된 전자장비에서 암호 알고리즘이 수행되는 시점에 공정보를 획득 |
| | FPGA 공격 | - 원로 수정이 가능한 FPGA의 특성을 이용하여 추가 기능 삽입 후 특정 조건 충족 시 동작 |
| **업데이트 서버** | 위변조 패치 파일 배포 | - 업데이트 서버의 파일 및 명령어 배포 기능을 이용하여 위변조된 패치 파일 배포 |
| **인프라 구성요소 (CI/CD)** | CI/CD 구성 도구 취약점 공격 | - 이슈관리도구(IRA 등)의 원격코드실행 및 관리자 포털 접근 취약점 공격을 통해 탈취한 코드서명인증서로 악성프로그램 인증을 통해 부정접근 검증 회피 후 가장 공격에 활용 |
| **인증서 탈취** | 인증서 탈취 | - 부정접근 검증 회피 후 가장 공격에 활용 |
| **접근제어 우회 (원격/인가)** | 관리자 페이지 노출 | - '개발망', 운영망 등에서 관리기능을 제공하는 페이지의 취약점 및 계정정보 유출 |
| **원격접속 프로토콜 접근** | 계정 탈취 | - Telnet, SSH, FTP, RDP, VPN 등 내부망 접근이 가능한 원격 프로토콜 및 서비스프로그램의 접근계정 악용 |

## 대응방안

### 기술적 대응

| 구분 | 대응 방안 | 설명 |
|:-----|:----------|:-----|
| **인증서 관련 대응** | 빌드 인증서관리 시스템 구축<br>인증시 사용로그기 기록 | - 코드 서명 악용을 수행하는 시스템 및 인증서 관리<br>- 코드서명을 위해 인증서 사용 시 작업일지 기록 |
| **개발시스템 관리** | 시스템 망분리<br>접근통제 | - 개발 네트워크 분리<br>- 작업 수행하는 시스템간 접근차 및 접근 차단<br>- 외부 인터넷 접속 시, 화이트리스트 기반 프로세스만 |

### 관리적 대응

| 구분 | 대응 방안 | 설명 |
|:-----|:----------|:-----|
| **업데이트 서비스 제공** | 무결성 검증<br>업데이트 모니터링 | - 실행파일 비실행 파일 업데이트 정체라면 무결성<br>- 무결성 검증값을 갖을 수 있는 기법 적용 |
| **로그 관리** | 로그 관리 | - 개발시스템, 연동서 관리 시스템 모든 로그<br>- 주요 정보의 관리 기록 |
| **비상연락망 구축** | 비상연락망 구축 | - 신속한 대응체계 마련을 위한 비상연락망구축 |
| **정보보안체계 수립** | 정보보안/보안감사 모니터링<br>공급망 프로세스 지속적 개선 | - 정보보안/보안감사리체계 이행 및 대응 체계<br>- 프로세스 점검 위험, 취약점 모니터링<br>- 지속적인 성찰 및 보안 거버넌스체계 마련 |
