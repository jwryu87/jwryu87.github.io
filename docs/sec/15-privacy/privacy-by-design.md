# 개인정보보호 중심 설계 (Privacy by Design)
{: .fs-8 }

15.3 개인정보보호
{: .label .label-purple }

> 암기: **사초내균/생가존** (사후→사전, 초기설정부터, 내재한 설계, 균형, 생애주기, 가시성, 존중)

---

## 핵심 키워드

`사전예방` `프라이버시 보호를 내재한 설계` `개인정보 생애주기 전체에 대한 보호`

---

## 정의/개념

IoT, 인공지능, 빅데이터 등 신기술로 인한 **개인정보 위협이 증가함에 따라** 제품이나 서비스를 개발할 때 **기획단계에서부터 개인정보 침해요인을 예상하거나 침해가능성에 대비**하여 설계하는 방식

---

## 7대 원칙

| 번호 | 원칙 | 내용 |
|:-----|:-----|:-----|
| 1 | **사후 조치가 아닌 사전 예방** | 프라이버시 침해사고가 발생한 뒤 조치하는 것이 아닌 침해 사건을 예상하고 사전 예방 |
| 2 | **초기설정부터 프라이버시 보호조치** | IT시스템 또는 사업 진행과정에서 개인정보가 보호될 수 있도록 자동으로 기본 설정하여 프라이버시 최대 보장 |
| 3 | **프라이버시 보호를 내재한 설계** | 프라이버시 보호를 설계에 내재화함으로써 프라이버시를 IT시스템 또는 개인정보 처리와 통합 적용 |
| 4 | **프라이버시 보호와 사업기능의 균형** | 서비스 제공을 위한 기능성, 편리성 등과 프라이버시 보호 중 어느 하나도 포기하지 않고 프라이버시의 안전한 보호와 사업의 기능성 두 가지 모두 확보하기 위한 노력<br>- 제로섬이 아닌 포지티브섬 |
| 5 | **개인정보 생애주기 전체에 대한 보호** | 개인정보의 수집·이용·저장·제공·파기 전 단계에 걸쳐 보호될 수 있도록 안전조치를 적용 |
| 6 | **개인정보 처리과정에 대한 가시성 및 투명성 유지** | 개인정보 처리과정에 대해 정보주체가 완전하고 명확하게 이해하도록 하여 신뢰성 제고 |
| 7 | **이용자 프라이버시 존중** | 프로그램, 프로세스 등에서 명시적인 보호체계가 없더라도 사용자의 프라이버시를 보장하기 위한 활동 수행 |

## 적용 절차

```
┌────────────┐
│   파 악    │  수집·이용하려는 전체 데이터 현황을 파악
└─────┬──────┘
      │
      ▼
┌────────────┐
│   분 석    │  개인정보 항목과 유형(식별자, 속성정보 등)을 분석
└─────┬──────┘
      │
      ▼
┌────────────┐
│   결 정    │  개인정보 항목별 수집근거(동의 등)와 활용방식(가명처리 등) 결정
└─────┬──────┘
      │
      ▼
┌────────────┐
│   예 방    │  '개인정보 처리 흐름도 작성' 사전수행, 위험성·침해요인 분석, 대안마련
└────────────┘
```

## PbD vs 기존 접근방식

| 구분 | 기존 접근방식 | Privacy by Design |
|:-----|:-------------|:------------------|
| **시점** | 사후 대응 | 사전 예방 |
| **범위** | 특정 기능/시스템 | 전체 생애주기 |
| **초기 설정** | 수동 설정 필요 | 자동으로 프라이버시 보호 |
| **관점** | 규제 준수 중심 | 이용자 중심 |
| **목표** | 침해 최소화 | 프라이버시 내재화 |

---

## 적용 사례

| 분야 | 적용 사례 |
|:-----|:----------|
| **IoT** | 데이터 수집 최소화, 익명화 기본 설정 |
| **AI** | 학습 데이터 비식별화, 설명 가능한 AI |
| **모바일 앱** | 최소 권한 요청, 선택적 동의 |
| **클라우드** | 암호화 기본 적용, 접근 통제 강화 |
| **빅데이터** | 가명/익명 처리, 목적 외 이용 차단 |
