# NFT 마켓 플레이스 보안 취약점
{: .fs-8 }

2.2 블록체인 보안
{: .label .label-red }

---

## 핵심 키워드

`스마트 컨트랙트` `DDoS` `SSRF` `콜드 월렛` `DID` `EIP 712`

---

## 정의/개념

NFT 마켓플레이스에서 발생하는 다양한 **보안 취약점**과 이에 대한 **대응방안**

---

## 취약점 분류

### 사회공학적 측면

| 취약점 | 설명 |
|:-------|:-----|
| **내부거래 유출** | 내부자에 의한 정보 유출 |

### 피싱/해킹

| 취약점 | 설명 |
|:-------|:-----|
| **스미싱** | SMS를 이용한 피싱 |
| **피싱** | 가짜 사이트를 통한 자격증명 탈취 |

### 마켓

| 취약점 | 설명 |
|:-------|:-----|
| **익스플로잇** | 취약점 악용 공격 |
| **웹 플러그인 취약점** | 브라우저 확장 프로그램 취약점 |

### 블록체인

| 취약점 | 설명 |
|:-------|:-----|
| **키 관리** | 개인키 탈취, 분실 |
| **스마트 컨트랙트** | 스마트 컨트랙트 취약점 |
| **합의 가로채기** | 51% 공격, 합의 알고리즘 공격 |

### 플랫폼

| 취약점 | 설명 |
|:-------|:-----|
| **DDoS** | 분산 서비스 거부 공격 |
| **SSRF** | Server Side Request Forgery |

### 디지털 자산

| 취약점 | 설명 |
|:-------|:-----|
| **랜섬웨어** | 암호화 협박 공격 |

---

## 대응방안

### 사회공학적 측면

| 대응방안 | 설명 |
|:---------|:-----|
| **접근제어** | 역할 기반 접근 통제 |
| **MFA** | 다중 인증 적용 |

### 피싱/해킹

| 대응방안 | 설명 |
|:---------|:-----|
| **NFT 보안정책 가이드** | 보안 인식 교육 및 정책 수립 |

### 마켓

| 대응방안 | 설명 |
|:---------|:-----|
| **에어드랍 패치** | 취약점 패치 적용 |
| **EIP 712** | 타입화된 구조적 데이터 해싱 표준 |

### 블록체인

| 대응방안 | 설명 |
|:---------|:-----|
| **콜드 월렛** | 오프라인 지갑 사용 |
| **시큐어 코딩** | 스마트 컨트랙트 보안 코딩 |
| **DID** | 탈중앙화 신원 인증 |

---

## EIP 712 개요

**EIP 712 (Ethereum Improvement Proposal 712)**
- 타입화된 구조적 데이터의 해싱 및 서명을 위한 표준
- 사용자가 서명하는 데이터를 명확하게 표시
- 피싱 공격 방지에 효과적
