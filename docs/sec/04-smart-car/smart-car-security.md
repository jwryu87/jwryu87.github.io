## 정의/개념

스마트카의 **센서**, **네트워크**, **백엔드 인프라** 측면에서의 보안 취약점과 대응방안

---

## III. 스마트카 취약점

### 가. 센서 측면 보안 위협

| 센서 유형 | 취약점 | 설명 |
|:----------|:-------|:-----|
| **인지 센서** | GPS 스푸핑 공격 | 위치 정보 조작 |
| | LiDAR 펌웨어 변조 | 센서 데이터 조작 |
| **판단 센서** | ECU 원격 해킹 | Electronic Control Unit 침투 |
| | 5G MEC MITM 공격 | 중간자 공격 |
| **제어 센서** | ADAS 시스템 오작동 | 첨단 운전자 지원 시스템 방해 |
| | 리버스 엔지니어링 | 펌웨어 분석 및 취약점 발굴 |

### 나. 네트워크 측면 보안 위협

| 구분 | 취약점 | 설명 |
|:-----|:-------|:-----|
| **외부 통신 측면** | Replay 공격 | 이전 통신 데이터 재전송 |
| | 스마트폰 App Brute Force Attack | 무차별 대입 공격 |
| **내부 통신 측면** | CAN 위변조 | 차량 내부 네트워크 조작 |
| | LIN 패킷 조작 공격 | Local Interconnect Network 공격 |

### 다. 백엔드 인프라 측면 보안 위협

| 구분 | 취약점 | 설명 |
|:-----|:-------|:-----|
| **관제서버** | DDoS 공격 | 서비스 거부 공격 |
| | 데이터 위·변조 | 관제 데이터 조작 |

---

## IV. 스마트카 취약점 대응방안

### 가. 센서 측면 대응방안

| 센서 유형 | 대응방안 | 기술/방법 |
|:----------|:---------|:----------|
| **인지 센서** | PVT 알고리즘 검증 | Position, Velocity, Time 검증 |
| | PUF 공정설계 도입 | Physical Unclonable Function |
| **판단 센서** | AUTO SAR 적용 | 표준 소프트웨어 아키텍처 |
| | 데이터 암호화 적용 | E2E 암호화 |
| **제어 센서** | 차량 XAI 도입 대응 | 설명 가능한 AI |
| | 시큐어 리부트 | 보안 부팅 |

### 나. 네트워크 측면 대응방안

| 구분 | 대응방안 | 기술/방법 |
|:-----|:---------|:----------|
| **외부 통신 측면** | PKI 기반 보안 모듈 | 공개키 기반 인증 |
| | 키 스트레칭 알고리즘 적용 | 비밀번호 강화 |
| **내부 통신 측면** | 모니터링 NSG 분리 | 네트워크 세그먼트 분리 |
| | 패킷 검출 확인 | 이상 패킷 탐지 |

### 다. 백엔드 인프라 측면 대응방안

| 구분 | 대응방안 | 기술/방법 |
|:-----|:---------|:----------|
| **관제서버** | Anti-DDoS | DDoS 방어 시스템 |
| | TLS 적용 | 전송 계층 보안 |

```
└─────────────────────────────────────────────────────┘
```
