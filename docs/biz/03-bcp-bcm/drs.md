## 정의/개념

정보시스템에 대한 **비상 대비체계 유지**와 각 업무 조직 별 비상사태에 대비한 **복구계획 수립**을 통한 업무 연속성을 유지할 수 있는 체제

---

## 구성도/기술요소

### DR 구성도

```
                    ┌──────────┐        전용선         ┌──────────┐
                    │주센터    │ ←──────────────────→ │Secondary │
        전용선      │Main Center│                      │ Center   │
       ←────→      │  HW/SW   │        DWDM          │  HW/SW   │
                    │ 스토리지 │ ←─ 데이터 복제 ───→ │ 스토리지 │
                    │ Network  │     거점정보         │ Network  │
                    └──────────┘     기업정보          └──────────┘
                         │             │                    │
                         │         전용선                   │
                         │             │                    │
                         └─────────────┼────────────────────┘
                                      │
                              ┌───────┴───────┐
                              │   제휴업체    │
                              └───────────────┘
```

### 기술 요소

| 기술 요소 | 설명 |
|:---------|:-----|
| **HA (High Availability)** | 최단 복구를 위한 H/W Clustering 또는 Stand-By 형태 |
| **FT (Fault Tolerant)** | 실시간 복구가 가능한 Dual 시스템 |
| **IP-SAN** | SAN Traffic을 IP Packet에 실어 전송 |
| **DWDM** | Dense Wavelength Division Multiplexing<br>고속의 Data 전송을 위한 통신,고밀도 파장분할 다중화 |

---

## DR 유형

| 구분 | 내용 | 목표시간 |
|:-----|:-----|:---------|
| **Mirror Site** | 양 센터에서 동시에 데이터를 처리 및 운영하며, 재해발생 시 즉시 대체 가동 | RTO = 0 |
| **Hot Site** | 주 전산센터와 거의 동일한 수준의 자원 확보<br>데이터 실시간 이중화 | RTO <= 2H |
| **Warm Site** | 주요업무처리를 위한 일부 장비를 구비하고, 재해발생 시 주요 업무만 복구하여 운영하는 형태 | RTO <= 1W |
| **Cold Site** | 기본시설만 확보하고 재해 발생 시 전산기기 및 네트워크를 구축하여 대체 가동 | 수개월 |

---

## 운영 형태별 DR센터 유형

| 유형 | 내용 |
|:-----|:-----|
| **상호계약형** | 유사기관 간에 상호 백업 방식<br>평소 상호 원조 관계를 갖고 비상시 지원하는 형태 |
| **공동이용형** | 여러 기관이 공동 출자하여 백업센터 건립 운영 |
| **외부위탁형** | 재해 복구 전문 서비스 기관에 위탁 운영 |
| **독자구축형** | 독자적인 백업센터 구축/운영 |
