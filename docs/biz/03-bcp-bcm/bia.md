## 절차

### IV. BIA의 주요절차

| 주요 절차 | 주요 활동 | 설명 |
|:---------|:---------|:-----|
| **주요 업무 프로세스 식별** | 조직의 주요 업무 프로세스 파악 | - 업무 프로세스의 계층적 식별<br>- 주요 업무 프로세스의 식별<br>- 업무 프로세스간의 상호 연관성 분석<br>- 업무간 선후관계 / 참조관계 분석 |
| **재해 유형별 발생 가능성** | 재해 발생원인의 상대적 빈도 판단 | - 재해원인 별 발생일수 판단, 재해발생원인의 상대적 빈도 및 원인판단으로 발생 가능성 판단 |
| **재해 시 업무 프로세스 중단에 따른 손실평가** | 업무 중단에 따른 주요 손실은 고객 서비스와 직결된 프로세스에서 발생 | - 정략적 평가 : 유형의 피해, 고객서비스 중단에 따른 단위 시간당 손실 규모 판단<br>- 정성적 평가 : 무형의 피해, 상/중/하 평가 및 프로세스간 연계성 매트릭스 활용 |
| **업무 중요성 우선순위 및 복구대상 업무범위 설정** | 정량적/정성적 분석결과를 바탕으로 우선순위 부여 | - 업무 연관성 분석 결과를 적용, 업무 상관도 조사 결과를 근거로 우선순위 부여 |
| **주요 업무 프로세스별 복구시간 결정** | 업무복구 우선순위를 통해 재해복구 대상 그룹 선정 | - 복구대상 업무범위에 대해 업무 중단허용시간<br>- 업무수행시의 수집 기록되는 데이터의 중요성에 입각하여 업무손실허용시점 |

---

## 관련 용어

| 용어 | 설명 |
|:-----|:-----|
| **RSO (Recovery Scope Objective)** | 복구대상이 되는 **핵심업무** : 계정계, 정보계, 대외계, 단순백업 |
| **RPO (Recovery Point Objective)** | **핵심업무 복구**를 위한 복구수준 : 특정 백업 시점 데이터 복구, 전일마감 |
| **RTO (Recovery Time Objective)** | **업무재개**를 위한 필요시간 : 0시간, 2시간, 8시간, 24시간 |
| **RCO (Recovery Communication Objective)** | **네트워크 복구** 수준 : 본사, 영업점, 전 영업점 |
| **BCO (Backup Center Objective)** | **백업센터 구축** 목표 : 자체 2nd 센터에 재해시스템 복구 |
