{: .label .label-red }

> 암기: **계비복설운**

---

## 핵심 키워드

`BIA` `RA`

---

## 정의/개념

비즈니스 중단 없이, **지속적 수행**, IT 인프라, 인적 자원, 물적 자원 **재해 예방 및 복구 계획** 수립 체계

## 수립절차

| 절차 | 주요 활동 | 설명 |
|:-----|:---------|:-----|
| **프로젝트 계획수립** | 목표 및 범위 설정 | - 업무연속성계획을 위한 프로젝트 계획 수립<br>- 수행범위, 조직, 일정, 자원 등에 대한 정의 |
| **업무영향 분석 및 위험평가** | 업무영향분석(BIA) | - 통제되지 않고 구체화되지 않은 사건들의 잠재적 영향 식별<br>- 전사 차원의 업무 영향도를 분석<br>- 최대허용가능중단시간(MTPD, Maximum Tolerable Period of Disruption), 최소사업연속성목표(MBCO, Minimum Business Continuity Objective), 복구목표시간(RTO, Recovery Time Objective), 복구목표시점(RPO, Recovery Point Objective) 등 설정 |
| | 위험평가(RA) | - 중요성과 발생가능성에 근거하여 잠재적 업무중단에 따른 우선순위를 부여<br>- 현재 수준과 목표 수준 간 발생할 수 있는 차이 분석 |
| **복구전략 수립** | 우선순위 산정 | - 우선순위활동과 이들을 지원하는 자원들의 안정화, 연속성, 재개 등을 고려한 복구전략을 수립 |
| | 자원요구 산정 | - 인력, 정보, 데이터, 건물, 작업환경, 시설, 장비, ICT 등의 요구 되는 자원을 산정<br>- 자원 요구사항 추가 검토 |
| **BCP 설계 및 개발** | 조직화 및 구현 계획 | - 조직 내 지휘, 통제, 의사소통 구조 구축<br>- 업무복구계획을 위한 프레임워크 개발, R&R 명확화 |
| | 재해복구 대책 구현 | - 업무프로세스 복구에 필요한 공간, 시스템 등의 예비를 마련<br>- DR센터 구축 및 데이터 백업 방법 결정<br>- 비용대비 효과적인 면을 고려하여 위험감소 대책 구현 |
| | 업무복구계획 개발 | - 기업 업무연속성을 위한 조직구조 결정 및 마스터플랜 수립<br>- 긴급 대응계획, 피해평가 계획 등 구체적 계획 수립 |
| | 교육 및 모의훈련 | - 재해 등 위험에 대응하는 방식 전파<br>- 비상시 사용되는 프로세스나 시스템에 대한 모의훈련 |
| **운영 및 유지보수** | 모니터링 및 성과 평가 | - 조직에 적합한 정성적, 정량적 모니터링 척도 수립<br>- 업무연속성 정책 및 목표에 대한 달성 정도 모니터링 |
| | 내부감사와 지속적 개선 | - 업무연속성계획에 대한 보증과 평가에 대한 내부감사<br>- 내외부 환경변화에 따른 업무연속성계획의 지속적 개선 |

> - 업무연속성계획은 업무연속성관리시스템(BCMS, ISO 22301)을 기반으로 하여 수립
> - 최근 IT시스템의 업무연속성을 위해 클라우드 서비스를 활용한 DRaaS가 활성화되어 운영
