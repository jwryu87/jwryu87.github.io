## 정의/개념

**ITSM 체계적 지원** 위해 DevOps, Agile 등 지원 통한 실용적 기반을 제공하고 **가치 기반 체계**를 지원하는 Best Practice

---

## 구성요소

### IV. ITIL 4.0 의 구성요소

| 구분 | 보안 기능 요소 | 설명 |
|:-----|:-------------|:-----|
| **4차원모델** | Organization and People | - ITSM 부서의 조직 구조와 인력의 기술 및 역량에 중점<br>- 목표 달성을 위해 서비스 중심 문화 창달<br>- 단절(silo)을 식별하고 인력별 적정 역량과 기술의 보유 강조 |
| | Information and Technology | - 정보와 지식, 기술, 요소간의 관계의 3 가지 핵심 영역 관리<br>- 자동화, 정보보안 통제 등의 이슈 대응 |
| | Partners and Suppliers | - 조직과 파트너 간의 계약(contract) 및 합의(agreement) 중점<br>- 전략적 초점, 기업 문화, 비용, 외부적 문제 등의 요소 기반 공급자 선정 |
| | **Value Streams and Processes** | - 조직이 수행하는 활동, 활동 구성 방법 중점으로 소비자 요구 충족 위한 활동 중점<br>- 외부요인 6가지 영역(PESTLE) 활용 |
| | Guiding Principles | - ITIL 4의 7가지 가이드 원칙 |
| | Governance | - 지시(Direct) -> 모니터링(Monitor)->평가(Evaluate)를 통해, 거버넌스 개선 적용 |
| **Service Value System** | Service Value Chain (SVC) | - 비즈니스 요구 사항에 대해 가치 기반 피드백 수행<br>- Plan, Engage, Design and Transition, Obtain/Build, Deliver and Support, Improve |
| | Practices | - 일반/서비스/기술 관리의 34 개 Practice 로 구성 |
| | Continual Improvement | - ITSM 계획(initiative) 성공을 위한 지속적인 개선 적용 |

> ITIL 4.0 적용시 Co-Create 실현 위해 가치 집중, 유기적 일과 사고하기 등의 7 대 적용 원칙 고려 필요

## ITIL 3.0 vs ITIL 4.0 비교

### II. ITIL 3.0과 ITIL 4.0 상세 비교

| 비교 항목 | ITIL 3.0 | ITIL 4.0 |
|:---------|:---------|:---------|
| **핵심구조** | Service Life Cycle | Service Value Chain |
| **구성** | 26개 Process | 34개 Practice |
| **원칙** | 9대 원칙 | 7대 원칙 |
| **관점** | 사람(People), 프로세스(Process), 파트너(Partner), 제품(Product) | 4차원 모델 |
| **특징** | ITSM 구축을 위한 프로세스 중심 | 서비스 가치와 가치 기반의 연결 중심 |
| **방법론** | 전통적 방법론 적합 | 애자일 방법론 적합 |
| **연도** | 2007년 출시, 2011년 Minor Update | 2019년 2월 |

> ITIL 4.0 성공적인 적용을 위해 7대 원칙 적용 필요
