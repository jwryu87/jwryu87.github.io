# SLA (Service-Level Agreement)
{: .fs-8 }

4.2 SLA
{: .label .label-yellow }

---

## 핵심 키워드

`SOW` `SLM` `SLO` `SLR` `Penalty` `Incentive`

---

## 정의/개념

**IT 아웃소싱**, 서비스 수준 보장, **정량적 측정 서비스 수준** 계약

---

## 주요지표

### II. SLA의 주요지표

| 구분 | 주요지표 | 설명 |
|:-----|:---------|:-----|
| **하드웨어** | 서비스 가동률 | - 서비스 시간 동안 제공 가용성 목표 비율<br>- 서비스 가동률(%) = (1 - 장애시간/서비스시간) * 100 |
| | 동일 장애 발생율 | - 기존 발생된 하드웨어 장애가 재발생된 비율<br>- 동일 장애 발생율(%) = (동일장애발생건수/총장애발생건수)*100 |
| **소프트웨어** | 오류 건수 | - 소프트웨어 버그, 오작동 등 오류 발생 건수 |
| | SR 적기 처리율 | - SR(Service Request)의 요청한 완료일 이내 서비스 제공 비율<br>- SR 적기 처리율(%) = (적기 처리된 SR건수/전체 SR건수) * 100 |
| **네트워크** | 네트워크 가동률 | - 서비스 시간 동안 제공 네트워크 가용성 목표 비율<br>- 네트워크 가동률(%) = (1 - 장애시간/네트워크 가동시간) * 100 |
| | 네트워크 장애건수 | - 스위치, 라우터 등의 문제로 발생된 장애건수 |
| **고객만족도** | 고객 만족도 점수 | - 정보시스템 사용자(고객) 만족도 점수(100점 만점) |

> SLA 관리 성과를 높이기 위한 **Penalty/Incentive, SIP, Annual Reset** 관리체계 존재

---

## 관리체계

### III. SLA 관리체계

| 구분 | 핵심 기술 | 설명 |
|:-----|:---------|:-----|
| **Penalty/Incentive** | Penalty | SLA에 명시된 서비스 수준 미달성시 부과되는 벌금 |
| | Incentive | SLA에 명시된 서비스 수준 초과달성시 지급되는 보상 |
| **SIP** | SLA 수준 개선 로드맵 | - SIP(Service Improvement Plan)<br>- 서비스 수준 검토 및 관리 프로세스 개선 제안 |
| **Annual Reset** | 목표 수준 조정, 매년 1회 | - IT 환경 변화를 고려한 사용자와 공급자 간의 계약 수준 변경<br>- 사용자, 공급자 간의 협의 하에 기존 서비스 수준 지속 가능 |

> SLA 관리체계와 ITSM, ITIL의 IT 관리 방법 접목으로 계약 투명성 및 수준 고도화 달성
