# OKR (Objectives and Key Results)
{: .fs-8 }

2.9 OKR
{: .label .label-blue }

> 암기: **집정추도**

---

## 핵심 키워드

`Objectives` `Key Results`

---

## 정의/개념

조직 차원에서 목표를 설정하고 그 결과를 추적할 수 있도록 도와주는 **성과 중심 목표 설정**, 실행 프레임워크

---

## 구성요소

### 원칙

| 구분 | 세부 | 설명 |
|:-----|:-----|:-----|
| **원칙** | 집중 | 주요 과제에 집중, 모든 구성원이 정확하게 의사 소통 |
| | 정렬 | 자신의 목표와 기업의 목표를 연결, 구성원의 목표를 투명 하게 공개, 공유 |
| | 추적 | 상황 추적을 통한 현재 상태 및 문제 발생 시 해결 방안 빠르게 모색 |
| | 도전 | 자신의 한계를 시험하고 실패할 자유를 부여, 도전-실패-수정-도전 반복 |

### 항목

| 구분 | 세부 | 설명 |
|:-----|:-----|:-----|
| **항목** | Objectives | 조직의 Mission 이 달성되기 위 해 자신이 설정하는 목표 |
| | Key Results | 정의된 Objective 가 달성되었는지 판단하기 위한 핵심 지표 |

### 프로세스

| 구분 | 세부 | 설명 |
|:-----|:-----|:-----|
| **프로세스** | Define | OKR에 대해 직원들과의 공감 확보 및 진행 기간, 목표와 KR 설정 |
| | Measure | Score point, 3 status level 등을 통한 직관적으로 진행 상태 측정 |
| | Wrap-up | 1:1 미팅을 통해 측정 결과의 효율적 개선 방안 도출 및 반영 |

---

## 주요미팅/원칙

| 항목 | OKR | 설명 |
|:-----|:----|:-----|
| **주요 미팅** | Company Meeting | 당기 KR's(Key Results) 결과를 공유하고, 다음분기 KR's 발표 |
| | STAFF Meeting | 당기 진행상황을 모니터링 하고 평가<br>다음 분기 KR's의 임시버전(Draft) 개발 |
| | One-on-One Meeting | Bottom-up으로 OKR 수립 및 진행상황 점검 및 장애요소를 식별 |
| | 최대 5개 목표와 4개 성과지표 | 최대 5개의 목표(Objectives)와 각 목표 별 최대 4개의 성과 지표 (Key results)를 넘지 않도록 함. |
| **원칙** | Bottom-up 60% | 60%의 목표는 Bottom-up을 통해 설정함 |
| | 상호합의 | Top-down의 일방적인 목표설정이 아닌 상호 합의된 목표 |
| | 60%~70% 달성목표 | 60%~70% 달성을 목표로 함, 40% 미만 BAD |
| | 평가 연동 금지 | 평가와 연동하지 않도록 함 |
