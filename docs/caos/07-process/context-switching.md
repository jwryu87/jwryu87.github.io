# 문맥교환 (Context Switching)
{: .fs-8 }

7.1 프로세스
{: .label .label-purple }

---

## 핵심 키워드

`PCB` `프로세스 상태 저장/복원` `오버헤드`

---

## 정의/개념

**문맥교환(Context Switching)**은 CPU가 현재 실행 중인 프로세스의 상태를 저장하고, 다음 실행할 프로세스의 상태를 복원하는 **프로세스 전환 과정**입니다.

---

## 문맥교환 과정

1. 현재 프로세스의 상태를 PCB에 저장
2. 다음 실행할 프로세스 선택 (스케줄러)
3. 선택된 프로세스의 PCB에서 상태 복원
4. 새 프로세스 실행

---

## PCB (Process Control Block)

| 항목 | 설명 |
|:-----|:-----|
| **프로세스 ID** | 프로세스 식별자 |
| **프로세스 상태** | 준비, 실행, 대기, 종료 등 |
| **프로그램 카운터** | 다음 실행할 명령어 주소 |
| **레지스터** | CPU 레지스터 값들 |
| **메모리 정보** | 페이지 테이블, 세그먼트 테이블 |
| **입출력 상태** | 열린 파일, 장치 정보 |

---

## 문맥교환 오버헤드

- CPU 레지스터 저장/복원
- 메모리 맵 갱신
- 캐시 무효화
- TLB 플러시
