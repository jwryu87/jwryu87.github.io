## 정의/개념

**모니터(Monitor)**는 세마포어 타이밍 오류 해결, **고수준 프로그래밍**, 운영체제 수준 **동시성 제어 기법**입니다.

---

## 세마포어와 모니터 관계

```
                    ┌─────────────────────────────────────┐
                    │   - 갑골절차에 대한 이론적 기반 제공  │
                    └─────────────────────────────────────┘
                                     │
       ┌─────────┐    동시성    ┌─────────┐
       │ 세마포어 │ ──────────▶ │  모니터  │
       └─────────┘     제어     └─────────┘
            │                        │
  - Low Level 기법 저급      - High Level 기법 저공
  - OS/개발 수준 저급        - 프로그래밍 언어 수준 제공
            │                        │
  - 오류수정어려  - 개발편의          │
            │                        │
       ┌────┴────┐                   │
       │ E-이밍 문제 │      언어 수준의     │
       │   발생    │      타이밍 해결      │
       └─────────┘                   │
  - 부정확하거 사용시               - 한 순간에 하나의
    탐지 곤란한 타이밍 오류발생        스레드만 실행
```

## 모니터 메커니즘

```
                    ┌──────────────┐
                    │   공유자원    │
                    └──────┬───────┘
                           │
                           ▼
    ┌──────────┐    ┌─────────────────────┐    ┌──────────────┐
    │ 배타 동기를│──▶│ 공유자원에 접근하는   │──▶│ 조건 동기를   │
    │ 위한 Queue│    │ 함수를 최대 한 놈만  │    │ 위한 Queue   │
    └──────────┘    │   들어갈 수 있다.    │    └──────────────┘
                    └──────────┬──────────┘
                               │
                               ▼
                             exit
```

## 모니터 구성요소

| 구분 | 상세구분 | 설명 |
|:-----|:---------|:-----|
| **구성요소** | 임계영역 | 스레드가 접근하는 공유자원과 공유함수가 존재하는 공간 |
| | 배타동기 큐 | 하나의 스레드만 공유자원에 접근할 수 있게 하는 작용을 하는 공간 |
| | 조건동기 큐 | 진입 스레드가 블록 되면서 새 스레드가 진입 가능하게 하는 공간 |
| **함수 (JAVA)** | synchronized | 배타 동기를 선언하는 함수 |
| | wait | 진입한 스레드를 조건 동기 큐에 블록 시키는 함수 |
| | notify, notifyAll | 블록 된 함수를 깨워 임계구역에 접근하도록 허용하는 함수 |

---

## 세마포어 vs 모니터 비교

| 구분 | 세마포어 | 모니터 |
|:-----|:---------|:-------|
| **추상화 수준** | Low Level | High Level |
| **제공 주체** | OS, 개발자 | 프로그래밍 언어 |
| **타이밍 오류** | 발생 가능 | 해결됨 |
| **동시 실행** | P, V 연산으로 제어 | 한 순간에 하나만 실행 |
