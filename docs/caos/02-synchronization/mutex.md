# 뮤텍스 (Mutex)
{: .fs-8 }

2.2 도구
{: .label .label-green }

---

## 핵심 키워드

`그림에 임계구역(Critical Session) 표시` `Wait(Mutex)` `Signal(Mutex)`

---

## 정의/개념

**뮤텍스(Mutex)**는 **Locking, Unlocking** 운영체제 수준 **동시성 제어 기법**입니다.

---

## 뮤텍스 메커니즘

```
┌────────────────────────────────┐
│            뮤텍스              │
│                                │
│  do {                          │
│      ┌──────────────────┐      │
│      │   Wait( mutex )  │      │
│      └────────┬─────────┘      │
│               │                │
│      ┌────────┴─────────┐      │
│      │  critical section│      │
│      └────────┬─────────┘      │
│               │                │
│      ┌────────┴─────────┐      │
│      │  signal( mutex ) │      │
│      └──────────────────┘      │
│                                │
│  } while(TRUE)                 │
│                                │
│  - Locking, Unlocking 사용     │
│  - 임계영역에 단 하나의 스레드만│
│    수행 가능하고 진입할 때      │
│    locking 나올때 unlocking    │
│    수행함                       │
└────────────────────────────────┘
```

---

## 뮤텍스 특징 (세마포어와 비교)

| 구분 | 세마포어 | 뮤텍스 |
|:-----|:---------|:-------|
| **동시획득** | 세마포어 여러 개 획득 가능 | 두 스레드가 동시 획득 불가 |
| **연산기법** | P(S), V(S) 연산 | Locking, Unlocking 수행 |
| **특징** | Binary/Counting 세마포어 사용 | 한 시점에 하나의 프로세스만 실행 |
| **구현코드** | `Semaphore S;`<br>`P(S); // 검사역할, S--`<br>`임계구역`<br>`V(S); // 증가역할, S+` | `do {`<br>`  wait(mutex)`<br>`  임계구역`<br>`  signal(mutex)`<br>`} while(true)` |
| **주체** | - 세마포어는 소유 불가능<br>- 세마포어 소유하지 않은 프로세스가 세마포어 해제 가능 | - 뮤텍스는 소유가 가능함<br>- 뮤텍스를 소유한 스레드만 unlocking 가능함 |

---

## 핵심 차이점

- 세마포어와 뮤텍스의 특징을 이해하고 사용하는 것이 중요하며 이 외 **모니터**와 **스핀락**으로 다중 프로세스의 동기화 문제해결이 가능함

| 특징 | 세마포어 | 뮤텍스 |
|:-----|:---------|:-------|
| **소유권** | 없음 | 있음 (소유한 스레드만 해제 가능) |
| **값 범위** | 0 이상의 정수 | 0 또는 1 (Binary) |
| **용도** | 여러 자원 관리 | 단일 자원 보호 |
