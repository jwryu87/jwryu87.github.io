`미러링` `스트라이핑` `해밍` `패리티` `분산 패리티` `0+1` `1+0`

---

## 정의/개념

**RAID**는 디스크의 가용성 및 성능 향상을 위해 **미러링** 및 **스트라이핑** 기술 이용하는 **디스크 고가용성 기술**입니다.

---

## RAID 레벨 비교

| 구분 | 특징 | 최소 드라이브 수 |
|:-----|:-----|:---------------|
| **RAID 0** | - 블록 레벨 스트라이핑(Striping) 방식<br>- I/O 성능은 드라이브 수에 비례<br>- 오류 검출 및 복구 기능 없음 | 2개 |
| **RAID 1** | - 디스크 미러링(Mirroring) 방식<br>- 가용성과 성능 고려<br>- 읽기 성능 : N배 향상 (N: 드라이브 수) | 2개 |
| **RAID 2** | - 비트 레벨 스트라이핑<br>- Hamming Code 활용한 ECC 기능 제공<br>- Overhead 발생 | 3개 |
| **RAID 3** | - 바이트 레벨 스트라이핑<br>- 패리티 디스크 사용<br>- 읽기/쓰기 성능 : N-1 배 향상 (N:드라이브 수) | 3개 |
| **RAID 4** | - 블록 레벨 스트라이핑<br>- 패리티 디스크 사용<br>- 읽기/쓰기 성능 : N-1 배 향상 (N:드라이브 수) | 3개 |
| **RAID 5** | - 블록 레벨 스트라이핑<br>- 분산 패리티 사용<br>- 읽기 성능 : N배 향상 (N:드라이브 수)<br>- 쓰기 성능 : 최대 N-1 배 향상 | 3개 |
| **RAID 6** | - 블록 레벨 스트라이핑<br>- 듀얼 분산 패리티 사용<br>- 읽기 성능 : N배 향상 (N:드라이브 수)<br>- 쓰기 성능 : 최대 N-2 배 향상 | 4개 |
| **RAID 10** | - 미러링 + 블록 레벨 스트라이핑<br>- 패리티 사용 안 함<br>- RAID 01 보다 장애 발생시 복구시간이 빠르며 안정적임 | 4개 |
| **RAID 01** | - 블록 레벨 스트라이핑 + 미러링<br>- 패리티 사용 안 함<br>- 가용성, 성능/안정성 측면에서 RAID 10 보다 낮음 | 4개 |

---

## RAID 선택 가이드

| 용도 | 권장 RAID |
|:-----|:---------|
| **성능 중시** | RAID 0 |
| **가용성 중시** | RAID 1, RAID 10 |
| **용량 효율성** | RAID 5, RAID 6 |
| **균형** | RAID 10 |
