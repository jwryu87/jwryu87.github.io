# 인터럽트 (Interrupt)
{: .fs-8 }

10.1 인터럽트
{: .label .label-purple }

---

## 핵심 키워드

`하드웨어 인터럽트` `소프트웨어 인터럽트` `인터럽트 벡터` `ISR` `폴링`

---

## 정의/개념

**인터럽트(Interrupt)**는 CPU가 프로그램 실행 중 예외 상황이 발생하여 처리가 필요한 경우, 현재 작업을 중단하고 발생된 상황을 우선 처리하는 **이벤트 처리 방식**입니다.

---

## 인터럽트 유형

| 유형 | 설명 | 예시 |
|:-----|:-----|:-----|
| **하드웨어 인터럽트** | 외부 장치에서 발생 | 키보드, 마우스, 타이머 |
| **소프트웨어 인터럽트** | 프로그램에서 발생 | 시스템 콜, 예외 |

---

## 인터럽트 처리 과정

1. 인터럽트 발생
2. 현재 상태 저장
3. 인터럽트 벡터 확인
4. ISR(Interrupt Service Routine) 실행
5. 상태 복원 및 복귀

---

## 폴링 vs 인터럽트

| 구분 | 폴링 | 인터럽트 |
|:-----|:-----|:---------|
| **방식** | CPU가 주기적으로 확인 | 장치가 CPU에 알림 |
| **효율성** | CPU 시간 낭비 | 효율적 |
| **응답성** | 느림 | 빠름 |
