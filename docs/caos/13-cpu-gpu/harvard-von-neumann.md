## 핵심 키워드

`프로그램 저장 방식` `메모리 구조` `병목 현상`

---

## 정의/개념

컴퓨터 메모리 구조에 대한 두 가지 아키텍처 방식

---

## 비교

| 구분 | 폰노이만 아키텍처 | 하버드 아키텍처 |
|:-----|:-----------------|:---------------|
| **메모리 구조** | 명령어와 데이터 공유 | 명령어와 데이터 분리 |
| **버스 구조** | 단일 버스 | 별도 버스 |
| **장점** | 단순, 유연 | 병렬 접근, 빠른 속도 |
| **단점** | 병목 현상 | 복잡한 설계 |
| **용도** | 범용 컴퓨터 | DSP, 마이크로컨트롤러 |

---

## 폰노이만 병목 현상

- 명령어와 데이터가 같은 메모리 사용
- 동시 접근 불가
- CPU 성능 제한

---

## 수정된 하버드 아키텍처

- 현대 CPU에서 사용
- 캐시 레벨에서 명령어/데이터 분리
- 메인 메모리는 통합
