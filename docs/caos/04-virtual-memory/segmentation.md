{: .fs-8 }

4.2 기법
{: .label .label-green }

---

## 핵심 키워드

`가변 크기 분할` `내부 단편 최소화` `limit` `base`

---

## 정의/개념

**세그멘테이션 기법(Segmentation)**은 메모리를 **가변 크기 분할**, 논리적 주소 세그먼트 할당, **내부 단편 최소화**, **메모리 관리 기법**입니다.

---

## 세그멘테이션 메커니즘

```
                              limit  base
                               │      │
                               ▼      ▼
                         ┌──────────────┐
                         │              │
                         │   segment    │
       ┌────┬────┐       │    table     │
  CPU ─│ s  │ d  │──────▶│              │
       └────┴────┘       └──────┬───────┘
                                │
                            yes │
                         ┌──────┴──────┐
                         │     d       │
                         │   ─────     │
                         │    <        │
                         └──────┬──────┘
                                │
                            no  │
                                ▼
                    trap: addressing error
                                │
                                │ yes
                                ▼
                        physical memory

  S : segment number, d : offset
```

---

## 세그멘테이션 프로세스

| # | 프로세스 |
|:--|:---------|
| 1 | 세그먼트 주소 이용, **세그먼트 번호** 확보 |
| 2 | 세그먼트 테이블에서 **base**, **limit** 확보 |
| 3 | 프레임번호와 페이지 오프셋으로 **메모리 주소** 확인 |

---

## 페이징 vs 세그멘테이션 비교

| 구분 | 페이징 | 세그멘테이션 |
|:-----|:-------|:------------|
| **분할 크기** | 고정 크기 | 가변 크기 |
| **내부 단편화** | 발생 가능 | 없음 |
| **외부 단편화** | 없음 | 발생 가능 |
| **논리적 분할** | 물리적 분할 | 논리적 분할 (코드, 데이터, 스택) |
| **테이블** | 페이지 테이블 | 세그먼트 테이블 |
