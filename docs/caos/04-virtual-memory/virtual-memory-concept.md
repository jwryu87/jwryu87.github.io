# 가상메모리 개념
{: .fs-8 }

4.1 개념
{: .label .label-purple }

---

## 핵심 키워드

`멀티프로세스` `주소공간` `보조기억장치`

---

## 정의/개념

**가상메모리(Virtual Memory)**는 메모리 크기보다 큰 프로세스를 사용하기 위해, **프로세스일부만 메모리에 적재**, **나머지 보조기억장치 적재** 기법입니다.

---

## 필요성

| 필요성 | 설명 |
|:-------|:-----|
| **멀티프로세스 고유주소** | 다수의 프로세스가 동시에 실행 될 수 있는 주소공간을 제공 |
| **프로세스 자원 보호** | 각 프로세스 고유의 메모리 자원을 보호 |
| **주소공간 공유** | 필요할 때 프로세스 사이의 주소공간을 공유 가능 |
| **주소관리 유연성** | 주소공간을 프로그래머에게 투명하게 관리 |

---

## 핵심 원리

{: .highlight }
> **물리적인 메모리 크기 제약과 유연성을 확보하기 위해서** 가상메모리 메커니즘을 사용

---

## 가상메모리 특징

| 특징 | 설명 |
|:-----|:-----|
| **메모리 확장** | 물리 메모리보다 큰 프로세스 실행 가능 |
| **메모리 보호** | 프로세스 간 메모리 영역 분리 |
| **주소 변환** | 가상 주소 → 물리 주소 변환 |
| **요구 페이징** | 필요한 페이지만 메모리에 로드 |
