---

## 핵심 키워드

`고정 크기 프레임 분할` `외부 단편 최소화` `page table`

---

## 정의/개념

**페이징 기법(Paging)**은 메모리를 **고정 크기 프레임 분할**, 페이지 번호와 프레임 번호 할당, **외부 단편 최소화**, **메모리 관리 기법**입니다.

---

## 페이징 기법 메커니즘

```
     페이징 기법에서의 주소 바인딩 과정

                    Paging
     ┌───────────────────────────────────────┐
     │   Logical           Physical         │
     │   Address           Address          │
     │                                      │
     │              ┌─────────┐ 10000..00   │
     │              │         │             │
     │  CPU ──▶ P │ d │──▶ f │ d │──▶ Physical │
     │              │         │    Memory   │
     │              └─────────┘ 11111..11   │
     │                                      │
     │           P ┌─────────┐              │
     │             │         │              │
     │             │    f    │              │
     │             │         │              │
     │             └─────────┘              │
     │             Page table               │
     └───────────────────────────────────────┘

     <P, d> -> <f, d>

     P : 페이지 번호
     d : 변위
     f : 프레임 번호(또는 프레임 시작 주소)
```

---

## 페이징 기법 프로세스

| # | 프로세스 |
|:--|:---------|
| 1 | 논리주소 이용, **페이지 번호** 확보 |
| 2 | 페이지 테이블에서 **프레임 번호** 확보 |
| 3 | 프레임번호와 페이지 오프셋으로 **메모리 주소** 확인 |

---

## 페이지 테이블

| 구성요소 | 설명 |
|:---------|:-----|
| **페이지 번호 (P)** | 가상 주소의 페이지 식별자 |
| **프레임 번호 (f)** | 물리 메모리의 프레임 위치 |
| **변위 (d)** | 페이지/프레임 내 오프셋 |
| **유효 비트** | 페이지가 메모리에 있는지 표시 |
| **참조 비트** | 최근 참조 여부 |
| **수정 비트** | 페이지 수정 여부 |

---

## 페이징 특징

| 특징 | 설명 |
|:-----|:-----|
| **외부 단편화 해결** | 고정 크기 분할로 외부 단편화 없음 |
| **내부 단편화 발생** | 마지막 페이지에서 발생 가능 |
| **페이지 테이블 필요** | 주소 변환을 위한 테이블 관리 |
| **TLB 활용** | 주소 변환 속도 향상 |
