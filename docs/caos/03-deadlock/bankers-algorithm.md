# 은행가 알고리즘 (Banker's Algorithm)
{: .fs-8 }

3.1 개념/해결
{: .label .label-yellow }

---

## 핵심 키워드

`AVAILABLE` `MAX` `NEED` `ALLOCATE`

---

## 정의/개념

**은행가 알고리즘(Banker's Algorithm)**은 안전상태 허용, 불안전 상태 회피 **교착상태 회피 알고리즘**입니다.

---

## 은행가 알고리즘 프로세스

| # | 발생조건 |
|:--|:---------|
| 1 | 준비 |
| 2 | 자원할당 요청 |
| 3 | 안정여부 |
| 4 | 자원할당 |
| 5 | 할당거부 |

---

## 은행가 알고리즘 자료구조

| 자료구조 | 구성 | 표기 | 설명 |
|:---------|:-----|:-----|:-----|
| **AVAILABLE (가용 자원)** | 길이가 m인벡터 | AVAILABLE[j] = k | - 사용 가능한 자원의 수<br>- rj 유형 자원 중 k 개의 자원이 가용 상태 |
| **MAX (최대 요구)** | n X m 행렬 | MAX[i, j] = k | - 각 프로세스의 자원에 대한 최대 요구량<br>- 프로세스 Pi는 자원 유형 Rj에 대해 총 k개 요청 가능 |
| **NEED (추가 요구)** | n X m 행렬 | NEED[i, j] = k | - 현재 할당되어 있는 자원의 수<br>- 프로세스 Pi는 작업 종료를 위해 자원 유형 Rj의 자원을 총 k개 필요 |
| **ALLOCATE (할당 자원)** | n X m 행렬 | ALLOCATE[i, j] = k | - 각 프로세스가 향후 더 요구할 수 있는 자원의 수<br>- 프로세스 Pi는 현재 자원 유형 Rj의 자원을 총 k개 할당 받은 상태 |

---

## 안전 상태 판단

```
NEED[i] = MAX[i] - ALLOCATE[i]
```

- 프로세스가 요청한 자원이 AVAILABLE보다 작거나 같으면 할당
- 그렇지 않으면 안전하지 않으므로 대기

---

## 은행가 알고리즘 한계점 및 대응방안

| # | 한계점 | 대응방안 |
|:--|:-------|:---------|
| 1 | 일정한 수 자원, 사용자 유지 | 스왑공간 할당 : 점유/대기 부정 전략 |
| 2 | 최대 자원 요구량 사전 파악 | 프로세스 자원 : 회피 전략 |
| 3 | 모든 요구 시간 내 수행 및 반환 | 주메모리 : 선점 허용 전략 |
| 4 | | 내부자원 : 예방 방법 |
