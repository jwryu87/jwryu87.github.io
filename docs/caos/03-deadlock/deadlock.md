## 정의/개념

**교착상태(Deadlock)**는 멀티 프로세스 환경 하 다수의 프로세스가 특정 **자원할당**을 무한정 기다리는 **무한 대기 상태**입니다.

---

## 교착상태 메커니즘

```
                R₁
            ┌───●───┐
            │       │
            ▼       │
         ┌─────┐    │
         │ P₁  │◀───┘
         └──┬──┘
            │
            │
         ┌──┴──┐
         │ P₂  │
         └──┬──┘
            │
            ▼
            R₂
```

P₁은 R₁을 점유하고 R₂를 요청, P₂는 R₂를 점유하고 R₁을 요청하여 순환 대기 발생

---

## 교착상태 발생조건

| 구분 | 발생조건 | 설명 |
|:-----|:---------|:-----|
| **자원 특성** | 상호배제 | 한 번에 하나의 프로세스만 자원 사용 |
| | 비선점 | 다른 프로세스가 사용 중인 자원 강제 회수 불가 |
| **프로세스 특성** | 점유와 대기 | 자원을 점유한 상태에서 다른 자원 대기 |
| | 환형 대기 | 프로세스들이 순환 형태로 자원 대기 |

**4가지 조건이 모두 만족할 때 교착상태 발생**

---

## 교착상태 해결방안

| 구분 | 구성요소 | 핵심 |
|:-----|:---------|:-----|
| **예방** | 4가지 조건 부정 | 발생조건 중 하나 이상 제거 |
| **회피** | 은행가 알고리즘 | 안전 상태 체크 |
| | Wait-die, Wound-wait | Timestamp 기준 교착상태 회피 |
| **발견** | 자원할당 그래프 | 프로세스 자원 간 그래프 표현 |
| | Wait for graph | 교착상태 검출 |
| **복구** | 프로세스 강제 종료 | 프로세스 Kill |
| | 자원 선점 대상 선택 | 선점 자원 우선 순위 회복 |

---

## 예방 기법 상세

| 조건 | 예방 방법 | 문제점 |
|:-----|:---------|:-------|
| **상호배제** | 자원 공유 허용 | 현실적으로 불가능 |
| **점유와 대기** | 한 번에 모든 자원 요청 | 자원 낭비, 기아 발생 |
| **비선점** | 자원 강제 회수 허용 | 상태 손실 가능 |
| **환형 대기** | 자원 순서 지정 | 유연성 감소 |
