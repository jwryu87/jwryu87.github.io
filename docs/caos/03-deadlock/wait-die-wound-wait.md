# Wait-die, Wound-wait
{: .fs-8 }

3.1 개념/해결
{: .label .label-yellow }

---

## 핵심 키워드

`환형대기 부정` `비선점 부정`

---

## 정의/개념

- **Wait-Die**: 먼저 시작된 프로세스가 나중에 시작한 프로세스가 점유한 자원 요청 시 **Wait**, 나중에 시작한 프로세스가 먼저 시작된 프로세스의 자원 요청 시 **Die**
- **Wound-Wait**: 먼저 시작된 프로세스가 나중에 시작한 프로세스가 점유한 자원 요청 시 **Wound**, 나중에 시작한 프로세스가 먼저 시작된 프로세스의 자원 요청 시 **Wait**

---

## Wait-Die vs Wound-Wait 비교

| 구분 | Wait-Die | Wound-Wait |
|:-----|:---------|:-----------|
| **회피 과정** | 오래된 프로세스(T1)가 새 프로세스(T2)의 자원 **요청(Wait)** → 대기<br>새 프로세스(T2)가 오래된 프로세스(T1)의 자원 **요청(Die)** → 종료 | 오래된 프로세스(T1)가 새 프로세스(T2)의 자원 **요청(선점/Wound)** → 빼앗음<br>새 프로세스(T2)가 오래된 프로세스(T1)의 자원 **요청(Wait)** → 대기 |
| **특징** | 필요한 자원을 획득하기 위해 여러번 복귀<br>(다수의 roll back 발생) | 보유중인 자원을 요청하고 기다림<br>(소수의 roll back 발생) |
| **회피 방법** | **환형 대기 조건의 부정** | **비선점 조건의 부정** |

---

## 회피 과정 다이어그램

### Wait-Die

```
               요청
              (Wait)        점유
    ┌─────────────────▶ D1 ─────────────┐
    │                                   │
    │    T1          "환형대기      T2  │
    │  (고참) ─────────── 부정" ────(신참)──▶ 시간
    │                                   │
    │    점유         요청             │
    │      │         (Die)             │
    │      │    ╲     │                │
    │      └────▶ D2 ◀┘                │
    │                 ╳                │
    └─────────────────────────────────┘
```

### Wound-Wait

```
               요청           Wound
              (선점)   ╲
    ┌─────────────────▶ D1 ──"비선점"───┐
    │                        부정"      │
    │    T1                        T2  │
    │  (고참) ────────────────────(신참)──▶ 시간
    │                                   │
    │    점유         요청             │
    │      │        (Wait)             │
    │      │          │                │
    │      └────▶ D2 ◀┘                │
    └─────────────────────────────────┘
```

---

## 적용 시나리오

| 시나리오 | Wait-Die | Wound-Wait |
|:---------|:---------|:-----------|
| T1(오래됨)이 T2(새것)의 자원 요청 | T1 대기 (Wait) | T2 자원 선점 (Wound) |
| T2(새것)이 T1(오래됨)의 자원 요청 | T2 종료 (Die) | T2 대기 (Wait) |
