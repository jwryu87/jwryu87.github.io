# 데이터베이스 정규화 (Database Normalization)

## 개념

데이터베이스 정규화는 데이터의 중복을 최소화하고 데이터 무결성을 향상시키기 위해 테이블을 구조화하는 프로세스입니다.

<!-- more -->

## 정규화 단계

### 1차 정규형 (1NF)
- 모든 속성은 원자값(Atomic Value)을 가져야 함
- 반복 그룹이 제거되어야 함

**예시:**
```
❌ Before: 
학생(학번, 이름, 수강과목: "수학, 영어, 국어")

✅ After:
학생(학번, 이름)
수강(학번, 과목)
```

### 2차 정규형 (2NF)
- 1NF를 만족
- 부분 함수 종속 제거 (완전 함수 종속)

**조건:** 기본키가 복합키일 때, 기본키의 일부에만 종속되는 속성이 없어야 함

### 3차 정규형 (3NF)
- 2NF를 만족
- 이행적 함수 종속 제거

**조건:** 기본키가 아닌 속성이 다른 비기본키 속성에 종속되지 않아야 함

### BCNF (Boyce-Codd Normal Form)
- 3NF를 만족
- 모든 결정자가 후보키여야 함

## 정규화의 장점

1. **데이터 중복 최소화**
   - 저장 공간 절약
   - 갱신 이상(Update Anomaly) 방지

2. **데이터 무결성 향상**
   - 삽입 이상(Insert Anomaly) 방지
   - 삭제 이상(Delete Anomaly) 방지

3. **유지보수 용이**
   - 테이블 구조가 명확
   - 변경 시 영향 범위 최소화

## 정규화의 단점

1. **조인 연산 증가**
   - 테이블이 분리되어 조인 필요
   - 성능 저하 가능성

2. **복잡도 증가**
   - 테이블 수 증가
   - 쿼리 복잡도 상승

## 실무 고려사항

- **선택적 반정규화(Denormalization)**: 성능이 중요한 경우 의도적으로 정규화를 완화
- **읽기 vs 쓰기**: 읽기가 많은 시스템은 반정규화, 쓰기가 많은 시스템은 정규화 선호
- **캐싱 전략**: 정규화된 구조에서 조회 성능 개선

## 관련 개념

- 함수 종속성 (Functional Dependency)
- 후보키 (Candidate Key)
- 이상 현상 (Anomaly)
- 반정규화 (Denormalization)

## 참고자료

- 데이터베이스 설계론
- SQL 성능 최적화
